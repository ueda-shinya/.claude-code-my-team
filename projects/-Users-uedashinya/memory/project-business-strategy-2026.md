## 背景

2026-04-12のラウンドテーブル（ナギ・レン・タク・リナ参加）で事業戦略を策定。
シンヤさんの方針転換により「Web制作は縮小せず直受け化で成長＋AI事業を並行立ち上げ」の2本柱モデルに決定。

**Why:** 現状はWeb制作の下請けのみ（月3-5件、月商10-30万、案件単価6万）で売上の天井が構造的に決まっている。単価が低い・集客できてない・直クライアントがほぼないという3課題を解決する。

**How to apply:** 今後の事業相談・施策提案はすべてこの2本柱モデルを前提とする。

---

## ポジショニング

「地域中小企業のデジタル化を丸ごと引き受ける実務パートナー」
- Web＝デジタルの入口、AI＝デジタルの深化
- 肩書きは「Web制作会社」でも「AI会社」でもなく、ITで業務改善する人

## 収益モデル（24ヶ月計画）

### フェーズ1：基盤転換（1〜6ヶ月目）
- Web：下請け月3件維持（18万）＋ 直受け月1件（25-30万）
- AI：無料〜低額トライアル1件（実績づくり）
- 月商目標：25〜48万
- 稼働配分：Web 80% / AI 20%

### フェーズ2：外注化＋AI立ち上げ（7〜12ヶ月目）
- Web：直受け月2件×30万（うち1件外注→粗利15-18万）、下請け月1-2件に自然減
- AI：エントリー月3-5万×2-3社
- 月商目標：60〜80万
- 稼働配分：Web 60%（外注活用で実質40%）/ AI 40%

### フェーズ3：スケール（13〜24ヶ月目）
- Web：直受け月3-4件×30万（大半外注、ディレクション業化）
- AI：月5万×5-8社＋上位プラン1-2社
- 月商目標：100〜150万
- 月商100万の内訳イメージ：Web粗利60万＋AI月額40万
- ※ ITまるごとサポートの詳細シミュレーション（プラン別社数・工数）は `project-service-design.md` v4を参照

## 外注活用の粗利構造
- 直受け30万 → 外注費12-15万 → 粗利15-18万（粗利率50-60%）
- シンヤさんはディレクション＋顧客対応のみ（1件15-20時間に圧縮）

## AI事業の価格設計（段階制）

| 層 | 月額 | ターゲット規模 |
|---|---|---|
| エントリー | 3〜5万 | 従業員1〜5名 |
| スタンダード | 8〜12万 | 5〜20名 |
| プレミアム | 15〜20万 | 20名以上 |

最初の半年はエントリー層で実績を積む。

## 撤退ライン

**直受け＋AIの月商が50万を3ヶ月連続で超えるまで下請けは減らさない。**

## 営業方針

- 最初のチャネル：地域の交流会（月2回参加目標）
- 並行：商工会の専門家枠登録、GBP整備
- 営業の順番：Webで入ってAIを追加提案（逆にしない）
- 下請け元とはエリア・業種をずらして競合を避ける

## 2本柱の相乗効果

1. Web→AI：納品時にAIチャットボット等を月額サービスとして提案
2. AI→Web：AI相談から「サイトが古い」問題を拾ってWeb受注
3. 保守一体化：Web保守＋AI運用を月額パッケージ化（月5-8万）

## 集客導線（予算ゼロ前提）

優先順：
1. 地域の交流会・商工会（対面で信頼構築）
2. GBP（「地域名＋ホームページ制作」で表示）
3. 自社業務でのAI実験→コンテンツ化（note/LinkedIn/ブログ）
4. 事例が溜まったらLP整備→フェーズ3で広告（月3-5万）

## AI事業の種まき方針（2026-04-12決定）

- まず自分の業務でAIを使って効率化する（コストゼロ・説得力・コンテンツのネタ）
- その延長として「LP改善自動化プロジェクト」を自社で実証（下記参照）
- 実証で成果が出れば「LP＋広告のAI改善サービス」としてメニュー化
- さらに将来、Google Ads API等を接続して広告運用自体の自動化も視野（同一プロジェクトの発展形）

## 集客プロジェクト（2026-04-12決定、4-12改訂）

**目的：** SNS発信→LP→無料サイト診断→成約の導線を構築し、LP改善サイクルをAIチームで半自動〜全自動化する。自社LPで実証し、将来的に「LP＋AI自動改善サービス」としてメニュー化を目指す。

**ラウンドテーブル決定事項（2026-04-12）：**
- 広告で売るサービス：Web制作の直受け（AI事業は広告段階ではない）
- 広告媒体：Google検索広告に集中（月3-5万）※ただし実績が揃うまでSNS集客を先行
- CTA：「無料サイト診断」（具体的成果物がありハードルが低い）
- LP：サブドメインに新規作成（SNS流入用）
- 価格：25-30万（投資対効果で納得感を作る）
- 差別化：「作って終わりじゃない」＝制作後3ヶ月のアクセスレポート付き
- 実績づくり：まず知人2-3社を格安で受けてビフォーアフター事例を作る

**導線：**
```
SNS発信（事例・tips・診断告知）
  → Web制作LP（サブドメイン・無料サイト診断CTA）
    → 診断レポート送付＝初回商談
      → 提案・成約
```

**チーム体制（集客チーム＝SNS運用＋LP改善を統合）：**
```
レン（戦略統括・SNS×LP一貫性担保）
├── SNS運用ライン：ミナト → コト/ハル/ルナ（制作）
└── LP改善ライン：カイ（LP設計）→ シュウ（実装）→ サクラ（レビュー）
    └── リナ（改善提案の論理検証）
```
- レンがSNS訴求軸とLPメッセージの一貫性を統括
- ミナトはSNS運用に専念しつつ、LP改善サイクルに流入データを提供
- 週1レビューでレンがSNS×LP横断評価

**LP改善サイクル：**
```
①データ収集（GA4/GSC） → ②分析・KPI評価（レン） → ③改善提案（コト：コピー / カイ：構成・デザイン） → ④実装（シュウ）→ ⑤効果検証 → ①に戻る
```

**運用設定：**
- 対象：自社LP（オフィスウエダ・サブドメイン）
- サイクル頻度：週1回（初期は手動レビュー → 月間LPセッション100以上で自動化移行）
- 成功基準：4サイクル（1ヶ月）でCVR or 問い合わせ数に改善
- ステータス：LP制作から着手

**構築が必要なもの：**
- Web制作直受けLP（サブドメイン・無料サイト診断CTA）← 今ここ
- LP評価スキル（GA4/GSCデータ取得＋KPI自動評価＋課題レポート）
- 改善サイクルスキル（評価→提案→実装→検証の一連フロー）
- ※スキル構築はカナタ（agent-builder）に依頼

**サービス化の道筋：**
自社LPでサイクルを回す → 実証データが溜まる → 「LP＋AI自動改善サービス」としてメニュー化（AI事業のエントリー商品候補）
