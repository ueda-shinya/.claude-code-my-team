`session-browser.py --no-interactive` を Bash 実行した結果が Claude Code の出力折りたたみ（`+N lines (ctrl+o to expand)`）で1件目しか見えない状態になることがある。その場合はアシスタント本文にテキストとして貼り直して、全件がそのまま見えるようにする。

**対応ルール**:
- **3件以上表示する場合は常に貼り直す**（折りたたみ発生を待たず機械的に判定。1〜2件なら貼り直し不要）
- 貼り直し時は生出力をそのままアシスタント本文にコピーする。生出力の形式（1エントリ3行：日時・往復数・ID＋パス／最初の発言／最後の発言）は**維持する**。1行圧縮や再整形は行わない
- `--limit N` で **20件を超える**指定の場合は、貼り直しせずシンヤさんに「折りたたみ解除（ctrl+o）をお願いします」と依頼する（本文が極端に長くなるのを防ぐため）
- 本ルールの対象は `session-browser.py --no-interactive` のリスト表示のみ。`--resume N` 実行時は対象外

**Why:** 2026-04-24、シンヤさんから「直近のセッション一覧見せて」と依頼されたとき、Bash 出力が折りたたまれて1件目しか表示されず番号選択ができなかった。私が1行形式に圧縮して対処したが、シンヤさんの真意は「生出力の3行/件のままで10件全部見えればいい」だった。表示形式を変える必要はなく、折りたたみ対策だけ行えばよい。

**How to apply:**
- 「直近のセッション」「過去のセッション」「最近のセッション」「セッション一覧」等のトリガーで `session-browser.py` を実行した直後、Bash 結果が折りたたまれて表示されている場合は、続くアシスタント本文に生出力をそのまま貼り直す
- 折りたたまれていない（全件最初から見える）なら貼り直し不要
- 整形・圧縮・並び替えは一切しない（`--resume N` の番号対応を崩さないため）
