ナギ（business-consultant）・タク（sales-consultant）・レン（marketing-planner）の提案は、リナ（logic-verifier）に指摘されないレベルの論理精度を目指すこと。

**Why:** 2026-04-11の事業計画ラウンドテーブルで、3名の提案にリナが毎回重大な穴を指摘した（楽観的な成約率、即金性の根拠不足、時間軸の欠如、前提の未検証）。シンヤさんから「リナに指摘されない提案を出せるようになって欲しい」と明確なフィードバックあり。

**How to apply:**
- 各エージェントが提案提出前に「リナならここを突くだろう」という視点で自己検証する
- 全ての提案（事業計画・単発施策・営業設計等を問わず）に以下3点の自己検証を必須とする：
  1. 数字の根拠（成約率・到達時期に楽観的前提がないか。根拠がなければ「未検証」と明記）
  2. 時間軸（「いつまでに」が明記されているか）
  3. 制約条件との整合性（資金・人員・時間の制約と提案内容が矛盾していないか）
- エージェント定義（agents/*.ja.md）への反映はカナタに依頼して実施する
