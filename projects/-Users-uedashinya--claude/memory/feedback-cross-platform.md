## ルール

Windows または Mac 向けにスクリプト・ツールを実装する前に、**プラットフォーム差異を事前に洗い出して検証する**。

「Windowsでは動くけど、Macでは動かない」「Macでは動くけど、Windowsでは動かない」は、調べれば実装前にわかること。後から発覚するのは準備不足。

## 確認すべき代表的な差異

| カテゴリ | Windows | Mac/Linux |
|---|---|---|
| Pythonコマンド | `sys.executable`（`python3` は不確実） | `python3` が通ることが多い |
| 日付フォーマット | `%-m`・`%-d` は動作不可 | `%-m`・`%-d` は使用可 |
| パス区切り | `\` または `\\` | `/` |
| 改行コード | CRLF (`\r\n`) | LF (`\n`) |
| プロセス管理 | `taskkill` | `kill` |
| ファイルシステム | 大文字小文字を区別しない | 区別する（Mac はデフォルト非区別だが Linux は区別） |

## 実装前チェックの手順

1. **利用環境を特定する**：Windows のみ・Mac のみ・両対応か
2. **両対応が必要な場合**：上記の差異チェックリストを参照して設計段階で対処する
3. **シュウへの委譲時**：環境情報（Windows / Mac / 両対応）を必ず添えて依頼する
