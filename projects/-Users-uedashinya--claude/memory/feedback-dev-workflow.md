## ルール

コーディングを伴う作業では、以下の順序を必ず守る：

1. **要件定義 → シンヤさんに確認 → 承認後に実装開始**
   - 要件定義を書いたらそのまま実装せず、「これで進めますか？」の確認を必ず挟む
   - 特に方式・設計の判断が入る場合は確認必須

2. **実装はシュウ（backend-engineer）に委譲する**
   - アスカが直接コーディングしない
   - PHP プラグイン・API・DB 設計はシュウの担当

3. **コミット前にサクラ（code-reviewer）のレビューを通す**
   - レビューなしでコミット・プッシュしない

## 背景

以前から繰り返し指示されていたルール。アスカがメモリへの記録を怠り続けたため、セッションをまたぐたびに忘れて同じ違反を繰り返していた。2026-03-23 に再度発覚・明文化。

**アスカへ：** 「担当者に任せろ」「みんなでレビューしてから進む」は初耳ではない。記録がないのはアスカの落ち度。
