---
name: 新事業・新ドメイン検討時の自動ディレクトリ対応
description: クライアントの新事業・新サービス・新ドメイン・新サイトの話が出たら、アスカが確認してディレクトリ構成変更まで自動でやる
type: feedback
---

クライアントの会話で「新しいサービス」「新事業」「新ドメイン」「新サイト」「〜を検討」などの言葉が出たとき、アスカが自発的に確認して、該当する場合はディレクトリ構成変更まで完結させる。

**Why:** 毎回シンヤさんが「ディレクトリ構成変えて」と指示するのが手間。アスカが気づいて動いてくれると助かる。

**How to apply:**
1. 会話中に以下のいずれかが出たら、CLAUDE.md「クライアントディレクトリの構成ルール」（4パターン: A/B/C/D）に照らして適切な構成を提案する：
   - 新事業・新サービスのにおい（「新しい〇〇を検討」「〜事業を始めたい」等）→ パターン C/D 方向
   - 新ドメイン・新サイトのにおい（「別ドメインで〇〇」「新しいサイトを作りたい」等）→ パターン B/D 方向
2. 事業数とドメイン数で機械的に判定し、「これは [パターンX] として `<該当パス>/` ディレクトリを作りますか？」と確認する
3. YES なら CLAUDE.md の該当パターンルールに従ってディレクトリ構成を変更する（既存 reports/ や images/ の移動・各階層の README.md 作成・親 README.md の事業/ドメイン一覧更新まで）
4. 事業名（`biz-xxx` の xxx 部分）は内容から推測して提案し、別の名前を希望なら聞く
5. ドメイン名はそのまま実ドメイン名（ドット含む・例: `company.bakuchis.com`）を使用する。ハイフン化や省略形は使わない
