# GMC（Google Merchant Center）登録・反映マニュアル

作成日：2026-03-19
対象：WordPress で商品紹介ページを作り、GMC に掲載するまでの手順

---

## 全体の流れ

```
① サイトに JSON-LD（Product schema）を入れる
        ↓
② Google Search Console にサイトを登録・認証する
        ↓
③ GMC アカウントを作成する
        ↓
④ GMC にサイトを認証する（GSC と連携）
        ↓
⑤ GMC でフィードを登録する
        ↓
⑥ 商品審査を通過する
        ↓
⑦ Google ショッピングに掲載される
```

---

## ① JSON-LD の準備（コーディング側の作業）

商品ページに `sample-product-section.php` の JSON-LD を実装する。

### GMC が必須とするフィールド（これが欠けると審査落ち）

| フィールド | schema.org | 説明 |
|---|---|---|
| 商品名 | `name` | 正確な商品名 |
| 説明 | `description` | 50文字以上推奨 |
| 画像URL | `image` | 最小 100×100px・推奨 800×800px以上 |
| 価格 | `offers.price` | 税込価格 |
| 通貨 | `offers.priceCurrency` | `JPY` |
| 在庫状況 | `offers.availability` | `InStock` / `OutOfStock` |
| SKU | `sku` | 自社管理コード |
| 商品URL | `offers.url` | 商品詳細ページの絶対URL |

### あると審査が通りやすいフィールド（任意だが推奨）

| フィールド | schema.org | 説明 |
|---|---|---|
| GTIN / JANコード | `gtin13` | 流通商品は必須に近い |
| ブランド | `brand.name` | |
| 商品状態 | `offers.itemCondition` | `NewCondition` |
| 価格有効期限 | `offers.priceValidUntil` | 年末日など |

## ② Google Search Console（GSC）の設定

> GSC が設定済みなら③へ

1. [Google Search Console](https://search.google.com/search-console/) にアクセス
2. 「プロパティを追加」→ ドメイン型でサイトURLを入力
3. DNS TXT レコードまたは HTML ファイルでサイト認証を完了

---

## ③ Google Merchant Center アカウント作成

1. [merchants.google.com](https://merchants.google.com/) にアクセス
2. Google アカウントでログイン
3. ビジネス情報を入力：
   - ビジネス名（例：岩本商店 神の里）
   - 国：日本
   - タイムゾーン：日本標準時
4. 利用規約に同意して作成完了

---

## ④ GMC でサイトを認証する

### 方法A：GSC と連携（推奨・最も簡単）

1. GMC 管理画面 → 「ビジネス情報」→「ウェブサイト」
2. サイトのURLを入力
3. 「Search Console で確認」ボタンをクリック
4. GSC で認証済みなら自動で認証される

### 方法B：HTML タグで認証

1. GMC が発行するメタタグをサイトの `<head>` に追加
   ```html
   <meta name="google-site-verification" content="XXXXXXXXXX">
   ```
2. GMC 管理画面で「確認」をクリック

## ⑤ フィードの登録

GMC に「どこから商品情報を読み取るか」を教える設定。

### 方法A：ページフィード（URL リスト）＝ 推奨

商品ページのURLを列挙した CSV/TSV を登録する方法。
JSON-LD が正確に書かれていれば、GMC がページを読んで自動で商品情報を取得する。

**手順：**

1. GMC 管理画面 → 「商品」→「フィード」→「＋」
2. 国：日本 / 言語：日本語 / チャンネル：オンライン
3. フィード名を入力（例：商品ページフィード）
4. 入力方法：「ウェブサイトのクロール」を選択
5. サイトURLを入力して保存
6. GMC がサイトをクロールして商品を自動検出（数時間〜1日かかる）

### 方法B：スプレッドシートフィード

Google スプレッドシートに商品情報を入力してGMCに連携する方法。
商品数が少ない（〜100点程度）場合に手軽。

**手順：**

1. GMC 管理画面 → 「商品」→「フィード」→「＋」
2. 入力方法：「Google スプレッドシート」を選択
3. GMC が提供するテンプレートをコピー
4. 必須列に商品情報を入力：

   | 列名 | 内容 |
   |---|---|
   | id | SKUコード |
   | title | 商品名 |
   | description | 説明文 |
   | link | 商品ページURL |
   | image_link | 画像URL |
   | condition | `new` |
   | availability | `in stock` |
   | price | `3500 JPY` |
   | brand | ブランド名 |
   | gtin | JANコード（あれば） |

5. GMC と連携して「今すぐ取得」をクリック

## ⑥ 商品審査の流れ

フィード登録後、Google が商品を自動審査する（通常 1〜3 営業日）。

### 審査結果の確認場所
GMC 管理画面 → 「商品」→ 各商品のステータスを確認

| ステータス | 意味 |
|---|---|
| 承認済み | ショッピング広告・無料リスティングに掲載可能 |
| 未承認 | 問題あり → 理由を確認して修正が必要 |
| 保留中 | 審査中 |

### よくある審査落ちの原因と対策

| 原因 | 対策 |
|---|---|
| 価格がページと JSON-LD で一致しない | ページ表示価格と `offers.price` を必ず一致させる |
| 画像が小さすぎる（100px未満） | 最小 800×800px 以上の画像を使う |
| 商品説明が短すぎる | 50文字以上・具体的な説明を書く |
| ランディングページが購入できない状態 | 商品ページで実際に購入・カート追加できることを確認 |
| プライバシーポリシーがない | サイトにプライバシーポリシーページを作成 |
| 特定商取引法の表記がない | 特定商取引法に基づく表記ページを作成（ECサイト必須） |
| 返品・返金ポリシーが不明確 | 返品ポリシーページを作成し GMC 管理画面に URL を登録 |

---

## ⑦ 無料リスティング vs ショッピング広告

| | 無料リスティング | ショッピング広告 |
|---|---|---|
| 費用 | 無料 | クリック課金（Google Ads 連携必要） |
| 掲載場所 | Google ショッピングタブ | 検索結果上部・ショッピングタブ |
| 設定 | GMC のみ | GMC ＋ Google Ads |
| 推奨順序 | **まずこちらから始める** | 無料リスティング後に検討 |

```

☐ GSC でサイト認証済み

☐ GMC でサイト認証済み

☐ 商品が「承認済み」になった
```

## officeueda クライアント向け 提供フロー

```
1. コーディング（アスカ）
   └ 商品セクション HTML ＋ JSON-LD を実装

2. クライアントに確認依頼
   ├ プライバシーポリシー・特商法ページの有無
   ├ 返品ポリシーの有無
   └ GMC アカウントの有無（なければ作成を依頼 or 代行）

3. GMC 設定（officeueda が代行）
   ├ GSC 認証確認
   ├ GMC アカウント作成・サイト認証
   └ フィード登録

4. 審査通過確認
   └ 承認済みになったらクライアントに報告
```
