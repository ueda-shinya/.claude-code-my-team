## 0. このロードマップの位置付け

- **2026-04-22 に v4（ITまるごとサポート月額モデル）を棚上げ**した教訓を踏まえ、**v5 は需要検証を最優先**にする設計。
- v4 棚上げの真因「需要検証不足」を**再発させない**ため、CLAUDE.md「Competitive Absence Audit Rule」準拠で **仮説B（需要なし）が×と判定されるまで新事業の本格投資を凍結**する。
- 戦略確定の前段に**Stage 0（検証フェーズ）を必ず置く**ことが本ロードマップの最大の特徴。

### 全体構造

```
[Stage 0: 検証] → [Stage 1: 基盤] → [Stage 2: 拡張] → [Stage 3: シフト] → [Stage 4: 法人化判定]
   0〜2ヶ月         3〜5ヶ月          6〜8ヶ月         9〜14ヶ月        15〜18ヶ月
   月40〜50万      月60万            月90万           月120万          月150万
```

### 2-4. ヒアリング設計（T0-1）の中身

**対象**：直エンド5社（US-saijo / 松原工業 / 巻き爪じゅん / marin / ランドプランニング）

**確認項目**（リナの確認事項★★★を埋めるため）：
1. 現在の集客課題（具体に何で困っているか）
2. 現在の集客投資額（月額・年額）と内訳
3. 「Web制作 × SNS × 広告」を統合してくれる外注パートナーがいたら使うか
4. 払える月額の上限感（5万 / 10万 / 15万 / 20万 / それ以上）
5. 「AIエージェントが裏側で動く」ことへの抵抗感の有無
6. 成果定義のすり合わせ（流入 / CV / 売上 のどれを成果と見るか）
7. 既存パートナー（オフィスウエダ含む）への満足度・不満点

**判定基準（リナ修正反映・支払意思の証明を必須化）**：

以下の **(a) AND (b) AND (c)** を全て満たした場合のみ、仮説B = ×（需要確認）と判定する：

- (a) 5社中**3社以上**が「月10万円以上で使う意向あり」と回答
- (b) (a)の3社全員が「現状の集客手段に不満あり」と回答
- (c) **そのうち1社以上から書面の発注意向書（LOI = Letter of Intent）または条件付き発注合意を取得**

**v4再発防止の論理**：意向確認だけでは「言ってみただけ」と区別がつかない（v4棚上げの真因と同じ）。少なくとも1社が書面で意思表示することで、「実際に金を払う」前提が一段階強化される。LOI が取れない場合は仮説B = △ にとどめ、Stage 0 を延長する。

**LOI のサンプル文言**：
> 「貴社の AI集客代行受託サービスを月額○○円で○ヶ月利用する意向があります。サービス内容（成果KPI・運用範囲・契約期間）が合意できた段階で正式契約に進むことを表明します。」

### 2-5. Stage 0 の収益目標

- **既存事業を絶対に縮小しない**
- 月売上 40〜50万円を維持
  - 下請けWeb：35万円
  - 直請けWeb：5〜10万円
  - 保守ストック：2,200円（現状維持）
- 新規投資は **Stage 0 内では実施しない**（広告費・外注追加・エージェント開発投資は凍結）

### 2-5b. シンヤさん稼働の実測ルール（リナ修正反映）

3軸並走（既存事業維持＋検証タスク＋仕様要件定義）の物理破綻を防ぐため、稼働実測の仕組みを必須化：

- **週次セルフ報告**：シンヤさんが毎週日曜の morning-briefing-weekly に合わせて、当週の実稼働時間を3カテゴリ（既存事業 / 検証タスク / その他）で申告
- **集計主体**：アスカが Notion 案件管理DB の作業ログから集計し、申告値とクロスチェック
- **アラート閾値**：
  - 週稼働 > 25h（月100h相当）が**2週連続** → アスカが警告報告
  - 週稼働 > 30h が**1週でも発生** → 即時 Stage 0 タスクの優先順位再設定
- **アラート発火時の対応**：T0系タスクのうち優先度低いものを延期、または下請けWeb受注を一時抑制
- **対応が打てない場合**：Stage 0 のタイムラインを延長（2ヶ月→3ヶ月）して、無理に圧縮しない

## 6. Stage 4：法人化判定フェーズ（15〜18ヶ月目）

### 6-1. 目的

- 法人化発火条件3つのAND判定。
- 達成なら登記準備、未達なら個人事業継続。

### 6-2. 月次目標（18ヶ月目時点）— **リナ重大欠陥修正反映**

| 要素 | 月売上目標 | 構成比 |
|---|---|---|
| A. 下請けWeb | 30万円 | 16% |
| B. 直請けWeb | 35万円 | 19% |
| **C. AI集客代行** | **100万円** | **53%** |
| D. 保守ストック | 20万円 | 11% |
| **合計** | **185万円** | 100% |

**修正の論理（リナ指摘 観点3 への対処）**：

- 旧版では §6-3 法人化条件①「C事業 月100万円以上が3ヶ月連続」と §6-2 「C=65万円」が論理矛盾しており、**18ヶ月時点で法人化判定が不能な設計**だった。
- 法人化条件のほうを下げる（100万→70万）案も検討したが、**リナ慎重ラインを崩すとペイライン未達リスクが顕在化**するため、目標売上を引き上げる方針で整合化。
- C事業 100万円達成には **8〜10社契約が必要**（月額10〜15万円想定）。Stage 3 での6〜7社からの積み上げで論理的に到達可能なライン。
- ただし1人運営の物理キャパ上限（5〜10社）の境界に位置するため、**Stage 4 突入時点で必ず外注パートナー2〜3社の安定運用体制確保**を前提とする（→ §6-3 ② に追記）。

### 6-3. 法人化発火条件（AND判定・3条件すべて満たす場合のみ登記）

| # | 条件 | 閾値 | 検証方法 |
|---|---|---|---|
| ① | C事業の月次売上 | **月100万円以上が3ヶ月連続**（リナ慎重案採用、ナギ案80万から引き上げ） | 月次売上集計 |
| ② | C事業の粗利率 | **60%以上**＋**外注パートナー2〜3社の安定運用体制確保** | 原価（自時間×単価＋API費＋外注費）控除後・パートナー稼働実績 |
| ③ | 継続契約クライアント数 | **5社以上**（**最低契約期間3ヶ月以上**かつ**うち6ヶ月以上の実継続が3社**） | 契約管理 |

**③の最低期間定義（リナ修正反映）**：
- 5社全員が **最低契約期間3ヶ月以上**の契約を締結していること
- そのうち**3社は実際に6ヶ月以上の継続稼働実績**があること
- 残り2社は契約締結時点で3ヶ月以上の最低期間が設定されていればOK（実継続期間は問わない）

### 6-4. 主要アクション

|---|---|---|

### 7-2. KPIモニタリング項目（月次必須レポート）

1. **売上系**：A/B/C/D別売上・前月比・目標比、新規受注／失注／継続契約の純増減、案件あたり平均単価、月間総工数
2. **顧客系**：C事業の継続率（チャーン率）、契約期間平均、β顧客のNPS、解約理由カテゴリ集計
3. **オペレーション系**：エージェント稼働率、1顧客あたり平均工数の推移、配下エージェント別タスク成功率
4. **アラート系**：解約予兆検知、月次継続率<90%・粗利率<50%で即アラート
