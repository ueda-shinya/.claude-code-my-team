## 2. 管理番号フォーマット

### 形式

```
YYMMDD + 識別記号 + - + 版数
```

**例**：`220824A-1`

| 要素 | 説明 |
|---|---|
| `YYMMDD` | 発行日（西暦下2桁） |
| 識別記号 | 顧客を一意に特定する記号。CRM DBの`識別記号`フィールドに定義 |
| `-` | 区切り文字 |
| 版数 | 同日・同顧客の既存レコード数 + 1 |

### 採番ルール

- 識別記号はCRM（顧客リスト）側で顧客ごとに管理する
- 採番時は「発行日 + 識別記号 + 版数」を組み合わせて生成する
- 版数は同日・同顧客の既存レコード数をカウントして自動決定する

### 経緯

- 元のExcel台帳がYYMMDD形式を採用していたため踏襲
- 当初YYYYMMDDへの変換を検討したが、シンヤさんの指示でYYMMDDに戻した

## 4. CLIスクリプト（notion-ledger.py）

**ステータス：完了（2026-03-26）**

`~/.claude/scripts/notion-ledger.py` として実装済み。

| コマンド | 内容 |
|---|---|
| `list` | 台帳一覧（発行日降順） |
| `add` | 新規追加（管理番号自動採番） |
| `search <キーワード>` | 管理番号・件名・メモで検索 |
| `show <管理番号>` | 詳細表示（顧客名はCRMから取得） |
| `update <管理番号>` | ステータス・金額等の更新 |

## 5. CRMとの連携設計

- 台帳の「顧客」フィールドはCRM DBへのリレーション
- 識別記号はCRM側に持たせることで、台帳からも顧客を一意に特定できる
- CRM側の構造変更ルール（フィールド名変更等はアスカに依頼）は引き続き有効
