## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| **プロジェクト名** | Notion 顧客リスト管理システム（Notion CRM） |
| **目的** | 営業・受注・既存顧客の情報を一元管理し、フォロー漏れ・商談ロストを防ぐ |
| **対象** | officeueda の全顧客（Web制作・AI事業共通） |
| **担当** | シンヤさん（運用） / アスカ（スクリプト実装・保守） |

---

## 1-2. 管理No.の運用ルール

| ルール | 内容 |
|---|---|
| **発行タイミング** | 見積書・請求書・領収書の発行が必要になったとき |
| **発行方法** | `python3 notion-crm.py issue-no` を実行 |
| **採番方式** | A→Z→AA→ZZ→AAA→ZZZ（重複チェック済みで自動採番） |
| **顧客追加時** | 管理No.は発行しない（空欄のまま登録） |

> **理由**：顧客登録≠取引開始。請求が発生した段階で番号を確定させることで、番号の空き・重複・請求書との紐づけ管理がしやすくなる。

## 2. スコープ（やること）

### Phase 1：Notion DB 構築
- Notionワークスペースにデータベースを作成する
- 以下のフィールド構成でプロパティを設定する

### Phase 2：Python スクリプト実装
- `~/.claude/scripts/notion-crm.py` を作成する
- Notion API を使って、CLIから顧客情報をCRUDできるようにする

### Phase 3：運用開始
- 既存の顧客情報を手動または一括でインポートする
- 日常的な顧客管理フローに組み込む

---

## 3. フィールド構成（承認待ち）

| フィールド名 | 型 | 選択肢・備考 |
|---|---|---|
| 会社名 / 屋号 | タイトル（必須） | |
| 担当者名 | テキスト | |
| 電話番号 | 電話番号 | |
| メールアドレス | メール | |
| ステータス | セレクト | 見込み・商談中・成約・失注・既存 |
| 事業種別 | セレクト | Web制作・AI・その他 |
| 担当 | テキスト | 複数担当がいる場合を想定 |
| 最終連絡日 | 日付 | |
| 流入元 | セレクト | Instagram・紹介・HP問い合わせ・その他 |
| メモ | テキスト（長文） | 商談メモ・特記事項 |

> **シンヤさんへの確認事項**：上記フィールド構成に追加・変更があれば教えてください。確定後にDB作成に進みます。

## 4. スコープ外（やらないこと）

- メール自動送信・自動フォロー（別プロジェクト：メール自動化 Phase 2 で対応）
- Notionのビュー設計・フィルター設定（シンヤさんが手動で設定）
- 既存ツール（スプレッドシート等）からの自動マイグレーション
