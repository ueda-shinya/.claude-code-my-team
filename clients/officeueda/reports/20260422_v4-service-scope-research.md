# オフィスウエダv4 サービス提供範囲ブラッシュアップのためのファクトリサーチ

**作成日：** 2026-04-22
**作成：** ミオ（researcher）
**目的：** v4「ITまるごとサポート」の「やる/やらない」判断材料として、7カテゴリの中小企業需要・業界慣行・提供リスクを調査
**ステータス：** リク（fact-checker）によるファクトチェック前

---

## 1. エグゼクティブサマリー

7カテゴリを調査した結果、**「v4採用推奨」は4カテゴリ、「条件付き採用」は2カテゴリ、「対象外推奨」は1カテゴリ（セキュリティ系）** という判定となる。シンヤさんの「保証が難しければ対象外にする」方針は、業界慣行・法的リスクの観点から合理的であり、特にセキュリティカテゴリは初期段階での対象外が推奨される。一方、データ・アカウント管理・調達支援・導入支援・監視系は、**免責明記を前提とした条件付き採用が現実的**。カテゴリ7（業種特有の支援）は、税理士・社労士業務との厳密な線引きが法律上必須であり、違反は刑事罰に至る可能性がある。

---

## 2. 各カテゴリ詳細

### カテゴリ1：セキュリティ系【最重要】

**推奨判定：対象外推奨（初期フェーズ）**

#### 中小企業の需要実在性
- IPA 2024年度調査（全国4,191社）：情報セキュリティ担当者を任命していない中小企業が**69.7%**、セキュリティ投資ゼロが**62.6%**
- 「必要性は感じているが何をすべきかわからない」層が多数存在
- 需要は明確に実在するが、「サポートを有償で買う」認識の有無は未確認

#### 類似サービス事業者の提供範囲
**サイバーセキュリティお助け隊サービス（IPA認定制度）**が正式に存在する：
- 内容：システム監視・異常検知・事案発生時の初動対応・簡易サイバー保険のワンパッケージ
- 価格上限（ネットワーク監視型）：**月額1万円**（補助金対象）
- 2024年から「2類（中規模向け）」が新設
- この制度に認定された事業者が「責任持って提供する」業界構造であり、無認定の個人事業者が同等のサービスを提供することは業界慣行として確立していない

#### 業界慣行
- 保守契約書の標準条項として「**第三者による不正アクセス・不正アタックに起因する損害は責任を負わない**」旨を明記するのが通例
- 「関与した時点で善管注意義務が発生しうる」という構造的リスク

#### 提供リスク（裁判例）
- 東京地判2014年：クレジットカード情報7,000件流出事案でシステムベンダーに約2,000万円の損害賠償命令（被告30%過失相殺）
- 個人情報漏洩：1件あたり平均想定損害賠償額**28,308円**（2016-18年データ）。100件で約280万円、1,000件で約2,800万円
- 免責条項を設けても「重過失があった場合は免責無効」と裁判所が判断する事例複数

#### 「対象外運用」検討時の論点整理

| 論点 | 内容 |
|---|---|
| 法的根拠 | 保守契約書にセキュリティ対応を対象外業務として列挙することで一定の免責は可能。ただし善管注意義務の範囲は争いの余地あり |
| 顧客への説明義務 | 対象外としても「誰が守るのか」の案内責任は残る。IPAお助け隊・サイバー保険紹介にとどめる構成 |
| マーケティング上の問題 | 「ITまるごとサポート」という名称との齟齬。除外項目を正直に説明できる文脈設計が必要 |
| 現実的な落とし所 | ウイルス対策ソフトの「推奨」「案内」は可。「設定代行」「責任持った運用」は対象外。バックアップは「設定支援」として含め、「復旧の保証」は対象外が現実的 |
| 代替策 | サイバー保険の加入を顧客に勧める文脈を契約書・サービス説明書に含める |

### カテゴリ2：データ・アカウント管理

**推奨判定：条件付き採用（免責明記必須）**

#### 業界慣行・提供リスク
- **個人情報保護法上の委託先監督義務**：IT事業者が第三者として個人情報を操作する場合「委託先」に該当、漏洩時は委託元の管理責任
- **データ消失の裁判例**：「IT事業者が専門業者として相当な注意を怠ってデータを消失させた場合は不法行為が成立する」
- **免責条項の限界**：「一切の責任を負わない」は消費者契約法・裁判所解釈により無効リスク。「損害賠償上限を月額料金の範囲内に限定する」が業界標準

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| 退職者アカウント削除（手順案内） | 採用 |
| 退職者アカウント削除（代行実施） | 条件付き（免責明記・クライアント承認フロー必須） |
| データ復旧の「試行」 | 条件付き（「復旧保証はしない」を明記） |
| データ復旧の「保証」 | 対象外 |
| メールアドレス管理（設定変更） | 採用 |

### カテゴリ3：調達・選定代行

**推奨判定：条件付き採用（リベート開示・利益相反開示が前提）**

#### 業界慣行
- IT業界の代理店マージンは**10〜20%**が相場（PC調達代行特化データは未確認）
- SaaS選定代行は利益相反リスクが業界共通の問題として認識されている

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| PC・機器選定（複数案提示） | 採用（「最終決定はクライアント」と明確化） |
| PC代行購入（クライアント名義） | 採用（クライアント承認後） |
| SaaS選定（複数案提示・評価） | 採用（中立的な立場を明示） |
| リベート・マージンの受取 | 条件付き（クライアントへの透明な開示前提） |

---

### カテゴリ4：導入支援・移行

**推奨判定：採用（アップセルとして適切）**

#### 相場
- Google Workspace導入支援：**1社あたり5万円〜（税抜）**が主流
- 中小企業のDX取組：「取り組んでいる・検討中」42.0%（中小機構2024年）

#### v4との棲み分け
- 月額サポート内：設定変更・アカウント作成など1〜2時間で完結するものは作業枠内
- アップセル②（業務デジタル化サポート33〜66万）：全社展開・データ移行・研修を伴うプロジェクト規模

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| Google Workspace/M365 アカウント設定（数名規模） | 月額作業枠内 |
| クラウドストレージの初期設定・共有設定 | 月額作業枠内 |
| 全社展開・既存データ移行・運用定着支援 | アップセル②（別途見積） |
| kintone・Notion導入 | v4対象外（既設定） |

### カテゴリ5：研修・教育

**推奨判定：条件付き採用（月額外・スポット単発が現実的）**

#### 相場
- カスタマイズ研修：**1日あたり3万〜10万円**（大手教育機関）
- 中小企業向け個人事業者提供の相場は未確認（1〜3万円程度と推定）

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| 電話・リモートでの操作説明（30分以内） | 月額相談枠内 |
| 新入社員向けPC設定講習（1〜2時間） | スポット追加（11,000円/回既設定と整合） |
| 複数人向けITリテラシー研修（半日〜） | スポット別途見積（33,000円〜） |

### カテゴリ6：監視・予防

**推奨判定：採用（ただし「監視」は通知止まり・責任制限を明記）**

#### 相場
- 稼働監視：月額5,000円〜10万円（規模・ツールにより差）
- 定期点検の中小向け相場は未確認

#### 「障害を見逃した場合」の責任
- 「予見可能性があったか」が問われる（IT専門弁護士・2024年8月改訂版）
- SLAに「100%稼働を保証しない」「検知不能な障害は対象外」と明記が標準慣行

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| ドメイン・SSL有効期限のカレンダー管理・アラート通知 | 採用（低リスク・高ニーズ） |
| サブスク自動更新の確認アラート | 採用 |
| 四半期IT点検（チェックリスト実施・報告） | 採用（専属・まるっと対象） |
| サーバー・ルーター常時稼働監視 | 条件付き（SLA明記で責任範囲限定） |
| 障害発生時の「復旧対応」保証 | 対象外（「一次連絡の実施」までが上限） |

### カテゴリ7：業種特有の支援

**推奨判定：条件付き採用（法律上の線引きを厳格に守る前提）**

#### 税理士法との線引き

**税理士の独占業務（資格なしでは有償・無償問わず禁止・違反は刑事罰）：**
1. 税務代理（税務署への申告代理等）
2. 税務書類の作成
3. 税務相談（税額計算・節税策の提案等）

**IT事業者が行っても違反にならない業務（国税庁Q&A明記）：**
- インボイス・電子帳簿対応のためのシステム構築・設定
- 会計ソフトの導入・設定・操作方法の説明
- 電子帳簿保存法の要件を満たす保存方法の案内

**グレーゾーン（要注意）：**
- 「この取引はインボイスが必要ですか」→ 税務相談該当の可能性
- 「消費税の計算方法を教えてほしい」→ 同上

#### 社労士法との線引き

**社労士の独占業務（違反：1年以下の懲役または100万円以下の罰金）：**
- 労働社会保険関係の申請書・届出書の作成および代理提出
- 給与計算に付随した労働社会保険諸法令上の申請書・帳簿書類
- マイナンバーを含む社会保険申請書類の作成

**IT事業者が行っても問題ない業務：**
- マイナンバー収集・保管システムの構築・設定（委託先としての技術支援）
- 電子帳簿保存法対応のファイル保存ルール整備（技術支援）

#### 「やる/やらない」線引き

| 対応内容 | 推奨判定 |
|---|---|
| インボイス対応の会計ソフト設定・操作説明 | 採用 |
| 電子帳簿保存法の対応ファイル設定 | 採用 |
| 税額計算・節税相談 | **絶対対象外（税理士法違反）** |
| 社会保険申請書類の作成代行 | **絶対対象外（社労士法違反）** |
| マイナンバー収集・保管システムの設定支援 | 採用（技術支援の範囲） |
| マイナンバーを使った申請書類の作成 | **絶対対象外（社労士法違反）** |

## 4. セキュリティ「対象外運用」検討時の論点整理

シンヤさんの「保証が難しければ対象外」方針は業界慣行・法的リスクの観点から**合理的**。

### 採用する理由
1. 賠償リスクの上限が見えない（漏洩件数×28,308円、100件で約280万円）
2. 免責条項の限界（重過失・故意は免責不可、善管注意義務は裁判で問われる）
3. IPA認定の専門制度（お助け隊サービス）が既に存在

### 対象外としながら顧客を守る現実的方法
1. 推奨・案内にとどめる（ウイルス対策ソフト紹介・サイバー保険案内）
2. IPA認定サービスへの橋渡し
3. バックアップは「設定支援」まで、「復旧保証」は除外

### サービス名との整合性問題
- 「ITまるごとサポート」で「セキュリティは対象外」は矛盾として受け取られるリスク
- **対策**：契約書・LP・サービス説明書に明記：「当社はIT操作・設定・相談のサポートを提供します。セキュリティ事故への対応保証・ウイルス感染後の復旧は対象外です。セキュリティ対策の方向性についてはご案内いたします」
