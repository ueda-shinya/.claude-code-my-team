---

## 何が起きたか

シンヤが「おはよ」と送信したが、モーニングブリーフィングが実行されなかった。
カレンダー確認・git pull・引き継ぎ確認がすべて省略され、単なる挨拶返答に終わった。

---

## 原因分析

### 原因1（主因）：アスカが Skill ツールを呼ばなかった

`SKILL.md` のトリガー条件には以下が明記されている：

> ユーザーが「おはよ」「おはよう」「おはようございます」と発言した

にもかかわらず、`Skill` ツールで `morning-briefing` を呼び出さなかった。
これがブリーフィング未実行の直接原因。

### 原因2（副因）：hook の JSON 取得ロジックに潜在的な問題（サクラ指摘）

`morning-trigger.sh` の `prompt` フィールド取得ロジック：

```python
data.get('prompt', '')
```

Claude Code が hook に渡す JSON の構造が想定と異なる場合、`MESSAGE` が空になり無音で終了する。
デバッグログが存在しないため、現状では実際の動作を確認できない。
→ **デバッグログを追加して次回「おはよ」送信時に確認する**（対応済み）

### 構造的問題：`additionalContext` は「ヒント」であり「強制」ではない（ソウ指摘）

仮に hook が正常動作して `additionalContext` が注入されたとしても、Claude がスキルを呼ぶかどうかは保証されない。hook はあくまでコンテキストへのヒント注入であり、ツール呼び出しを強制するものではない。

### その他（サクラ指摘）：hook と SKILL.md の「おはよ + 追加指示」矛盾

- hook の正規表現は「おはよ」のみの発言にしかマッチしない設計（追加テキストがあると不発）
- SKILL.md 85-87行目には「おはよ + 追加指示」のケースを処理すると記載されている
- この2つが矛盾している → 今後の対応課題として残す
