CORE MECHANISM / 02

「司令塔」って何をしているの？

🏢
Flask （フラスク）
Webサーバーの機能を担う部品。
「LINEからメッセージが来たら受け取って、返事を返す」という受付窓口の役割。
※ Webサーバー＝インターネット経由でやりとりする仕組み
🌐
ngrok （エングロック）
自宅PCへ外の世界からアクセスできるようにする出口トンネル。
LINEのサーバーからシンヤさんのPCに届けるために必須。
※ 通常、自宅PCは外からアクセスできない。ngrokがその壁を突破する
たとえ話で理解する
📮

(ngrok)
👤
受付係
(Flask)
⚙️
処理を実行
(server.py)
📤
返事を送信
(LINE WORKS)

AUTO SCHEDULER / 03

APScheduler ― タイマーで自動仕事をこなす係

🕐
「毎時0分になったら、自動で仕事を始める」
人が操作しなくても、設定した時間が来ると自動で動き出す仕組み。
定期的なチェックや通知を全部自動でこなしてくれます。
実際に何をやっているか
1時間ごと
Chatworkの未読メッセージをチェック → 重要な連絡があれば LINE WORKSに通知
定期実行
GA4レポートの定期取得、Notionタスクの同期など
💡
たとえ話：「毎時0分になったら、郵便受けを自動で確認しに行くロボット」。
シンヤさんが気づく前に、もう仕事を終わらせてくれています。

LINE WORKS INTEGRATION / 04

LINE WORKSで使えるコマンド一覧

LINE WORKSのチャットに入力するだけで、サーバーが自動で処理して返答してくれます
/ga4
GA4アクセス解析レポートを取得して返信（キャッシュ優先・当日初回のみ取得）
/tasks
Notionの未完了タスク一覧を取得して返信
/clients
クライアント一覧を取得して返信
/memo <テキスト>
テキストをメモとして保存
/notion <タイトル>
Notion議事録DBに新規追加
自然文
「今日の予定は？」 → Google Calendarから今日の予定を取得して返信
自然文
「明日14時にMTG追加して」 → Google Calendarに予定を自動登録（1時間）
💬
コマンドは覚えなくてもOK。普通の言葉で話しかけるだけで、サーバーが意図を読み取って動いてくれます。

Chatworkとの連携 ― 自動チェック＆通知

自動処理の流れ

🔔
重要な連絡を発見したら
優先度が高いと判断したメッセージは、すぐにLINE WORKSへ通知。
Chatworkを開かなくても気づける。

INFRASTRUCTURE / 06

なぜシンヤさんのPCで動かしているの？

🖥️
現在
シンヤさんのPC
• PCが起動している間だけ動く
• PC起動時に自動起動する設定済み
• コスト0円で運用できる
▲ PCを切ると止まる
移行予定
☁️
将来
VPS（クラウドサーバー）
✓ 24時間365日稼働
✓ PCを切っても動き続ける
✓ 月数百円〜数千円の費用が発生
💡 VPS = Virtual Private Server。インターネット上に借りられる「常時起動のPC」。サーバーを外部に移すことで信頼性が大幅に向上します。

SUMMARY / 07

このサーバーがやっていること、3行まとめ

1
💬
LINEやChatworkからの連絡を受け取る
ngrok + Flask がコンビを組んで、外からのメッセージをPCの中に届ける。コマンドや自然文に反応して返答する。
2
🔄
自動でNotion・Google Calendarと連携する
「タスクを見せて」「予定を追加して」といった指示を受け取ると、サーバーが各サービスのAPIを叩いてデータを取得・更新する。
3
⏰
タイマーで定期チェック・自動通知する
APSchedulerが1時間ごとにChatworkを監視。重要な連絡があれば即座にLINE WORKSへ転送し、必要なら一次返信まで自動化。
