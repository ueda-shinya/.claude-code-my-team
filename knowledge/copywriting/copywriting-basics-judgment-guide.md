# コピーライティングの基礎と判断軸

出典：実務版：コピーライティングの基礎と判断軸ガイドライン.pdf
目的：コトへの依頼精度向上・コピーの良し悪しを論理で判断できるようにする

---

## 1. コピーライティングの3大原則

| 原則 | 内容 | NG例 → OK例 |
|---|---|---|
| **読者目線** | 企業目線より読者目線を優先 | 「弊社は提供します」→「あなたは得られます」 |
| **具体性** | 数字・固有名詞・シーンで語る | 「効率化できます」→「月20時間の工数を削減」 |
| **1メッセージ1コピー** | 1文に1訴求のみ | あれもこれも詰め込むと誰にも刺さらない |

## 2. ヘッドライン（見出し）の判断軸

**なぜ最重要か**：79%のユーザーが新しいページをまずスキャン行動でとらえる（NN/g調査）。本文を読む前に見出しで「読む価値があるか」を判断する。

### 良いヘッドラインの3要素（Copyblogger準拠）
1. **Deliverable（成果物）**：何が得られるかが明確か
2. **Goal（ゴール）**：誰のどんな悩みを解決するか
3. **Intrigue（好奇心）**：続きを読ませる「仕掛け」があるか

→ **clarity over cleverness（明快さを、うまさより優先する）**

### ヘッドライン NG vs OK

| NG | OK | 理由 |
|---|---|---|
| 「弊社のサービスのご紹介」 | 「3ヶ月で商談数が1.5倍になった、中小企業向け営業メール術」 | ベネフィットと数字が明確 |
| 「新機能リリースのお知らせ」 | 「もう報告書に3時間かけなくていい。自動化で月20時間を取り戻す新機能」 | 苦痛からの解放を提示 |
| 「ありがとうございます」 | 「◯◯様だけに。今週末で締め切る特別プランのご案内」 | 特別感と緊急性 |
| 「詳しくはこちら」 | 「無料で試す・14日間」 | リスクゼロの提案 |
| 「キャンペーンのご案内」 | 「今だけ30%オフ｜◯月◯日23:59まで」 | 明確な締め切り効果 |

## 3. PAS・AIDAフレームワーク

### PAS（感情重視）
**Problem → Agitation → Solution**
- 使いどころ：SNS広告・コールドメール・潜在層向けコンテンツ
- 例：(P)月末の領収書地獄で終電帰りになっていませんか？ → (A)営業時間を事務作業に奪われるのは機会損失 → (S)スマホ撮影だけで経費精算を月5分で終わらせます

### AIDA（構成重視）
**Attention → Interest → Desire → Action**
- 使いどころ：LP・サービス紹介資料・クロージングトーク
- 発展形：AIDCA（A,I,**Conviction**,D,A）← Convictionは顧客の声・実績
- 例クロージング：(A)御社の課題解決にはプランBが最適と確信 → (I)他社でも導入1ヶ月で成果 → (D)懸念の◯◯もこの機能でカバー → (A)今なら来月からスタート可能です。このプランで進めてよろしいでしょうか？

## 4. CTA（行動喚起）の判断軸

### 悪いCTAの3パターン
- **曖昧**：「詳細」「こちら」
- **コスト強調**：「登録する」「支払う」（痛みを連想）
- **主語が企業**：「送信する」（システム目線）

### 良いCTAの3原則
- **動詞で始める**：具体的アクションを示す
- **ベネフィット**：得られる価値を書く
- **緊急性/具体性**：今やる理由を作る

### CTA NG vs OK

| NG | OK | ポイント |
|---|---|---|
| 「詳しくはこちら」 | 「◯分で読める導入ガイドを受け取る」 | 所要時間と報酬が明確 |
| 「お問い合わせください」 | 「まず無料相談を予約する（30分・ゼロ円）」 | ハードルを極限まで下げる |
| 「今すぐ申し込む」 | 「今すぐ14日間の無料トライアルを始める」 | お試しできる安心感 |
| 「資料請求」 | 「成功事例3社の資料を無料でもらう」 | 欲しい情報であることを強調 |
| 「Get Started」 | 「今月の残り3席を確保する」 | 希少性で背中を押す |

## 5. コピー自己診断チェックリスト

### ヘッドライン・訴求チェック
- [ ] 読者が得られるものが具体的に書かれているか
- [ ] 企業主語ではなく読者目線で書かれているか
- [ ] 数字・固有名詞・固有のシーンが含まれているか
- [ ] 1文に複数のメッセージを詰め込んでいないか
- [ ] 「いい感じ」「洗練された」などの曖昧な形容詞に頼っていないか
- [ ] ターゲットが読んだとき「自分ごと」と感じられるか
- [ ] 競合が同じ言葉を使っていないか（独自性）

### CTA・クロージングチェック
- [ ] CTAは動詞で始まっているか
- [ ] 「詳しくはこちら」「お問い合わせ」など曖昧な言葉を使っていないか
- [ ] クリック後に何が起きるか読者に伝えているか
- [ ] 読者にとってのベネフィットが明示されているか
- [ ] 緊急性・限定性の根拠が本物か（偽の締め切りになっていないか）
- [ ] クロージングで「どうぞよろしくお願いします」だけで終わっていないか

## 6. 「良い・悪い」判断フローチャート

```
START: このコピーを読んだとき…
  ↓
読者が得るものがわかるか？ → NO: NG（書き直し）
  ↓ YES
企業目線ではなく読者目線か？ → NO: NG（視点変更）
  ↓ YES
次のアクションが明確か？ → NO: NG（CTA改善）
  ↓ YES
「自分ごと」と感じる具体性があるか？ → NO: 要改善（数字/シーン追加）
  ↓ ALL YES
合格！リリースしましょう
```

## 7. コトへの依頼テンプレート（「なんかいい感じで」をやめる）

```
■ 成果物の種類：（メール件名 / LP見出し / CTA文言 など）
■ ターゲット読者：
■ 読者の最大の悩み/ペイン：
■ この文章を読んだ後に取ってほしいアクション：
■ そのアクションで読者が得られるベネフィット：
■ 使えるデータ・数字・実績：
■ トーン（親しみやすい／信頼感重視／緊急感あり）：
■ 文字数制限・フォーマット制約：
■ 参考にしてほしい競合や良い例（URL可）：
```

### 記入例：休眠顧客掘り起こしメールの件名
```
■ 成果物の種類：休眠顧客掘り起こしメールの件名
■ ターゲット読者：過去に資料請求したが商談に至らなかった人事担当者
■ 読者の最大の悩み/ペイン：採用業務が多忙で新しいツールの検討が後回しになっている
■ この文章を読んだ後に取ってほしいアクション：開封し、本文中の事例URLをクリックする
■ そのアクションで読者が得られるベネフィット：他社がどうやって採用工数を半減させたか知れる
■ 使えるデータ・数字・実績：導入企業平均で工数50%削減、採用単価30%改善（架空例）
■ トーン：売り込み感を消し、有益な情報提供というスタンス
■ 文字数制限：スマホで切れないよう25文字以内
■ 参考：問いかけ形式のメルマガ
```
