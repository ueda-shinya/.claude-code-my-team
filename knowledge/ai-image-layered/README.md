# AI画像生成 レイヤー分割ワークフロー

> 初版：2026-03-21（実案件での検証結果に基づく）

## 概要

バナー・LP FV・広告KV等の複合的なビジュアルをAIで生成する際、「一発で完成品を出す」のではなく **レイヤーごとに分割して生成し、Figmaで合成する** ワークフロー。

---

## なぜレイヤー分割か

Gemini Imagen 4・Ideogram等の現行モデルは、背景・人物・テキスト・エフェクトを1回で生成すると品質が下がる。

| 一発生成の問題 | レイヤー分割での解決 |
|---|---|
| テキストが崩れる・混入する | L4をFigmaで後乗せするため発生しない |
| 構図の細かい制御が難しい | 各層のプロンプトがシンプルになり制御しやすい |
| やり直しで全体を再生成 | 問題の層だけ再生成できる |
| エフェクトと背景が干渉する | ブレンドモードで独立合成 |

---

## レイヤー構成（4層）

| レイヤー | 内容 | 生成方法 | 背景色指定 | Figmaブレンドモード |
|---|---|---|---|---|
| L1 | 背景（グラデーション・テクスチャ等） | AI生成 | -- | 通常 |
| L2 | メインビジュアル（人物・商品等） | AI生成 | **白背景** | **乗算** |
| L3 | アクセント・エフェクト（光・パーティクル等） | AI生成 | **純黒背景** | **スクリーン** |
| L4 | テキスト・ロゴ・CTA | **Figmaで手作業** | -- | 通常 |

### ブレンドモード解説

- **L2（乗算）**：白背景で生成した画像をFigmaの「乗算」で重ねると、白部分が透明化しメインビジュアルだけが残る
- **L3（スクリーン）**：純黒背景で生成した画像をFigmaの「スクリーン」で重ねると、黒部分が透明化し光・エフェクトだけが浮く

---

## ワークフロー

```
1. ルナがレイヤーごとにプロンプトを設計
   - L1用・L2用・L3用の3つのプロンプトを返す
   - L2は「white background」、L3は「pure black background」を明示

2. アスカがGemini Imagen 4 APIで各レイヤーを生成
   - 1層ずつ生成・確認

3. Figmaで合成
   - L1（通常）→ L2（乗算）→ L3（スクリーン）の順に重ねる
   - L4（テキスト・ロゴ）をFigmaで作成して最上部に配置
```

---

## 重要ルール

- **テキスト（日本語含む）はAIに生成させない**。Figmaで後乗せに統一する
- L2は必ず白背景、L3は必ず純黒背景で生成する（ブレンドモードの前提）
- 全レイヤーが必要とは限らない。案件に応じてL3を省略する等、柔軟に構成する

---

## 適用対象

- バナー画像
- LP ファーストビュー
- 広告キービジュアル
- SNS投稿画像（複合的なもの）

シンプルな写真1枚・イラスト1枚の生成には不要。
