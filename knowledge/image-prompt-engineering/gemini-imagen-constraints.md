# Gemini Imagen API 制約メモ

## アスペクト比の問題（2026-04-16確認）

- `imagen-4.0-generate-001` の `predict` エンドポイントで `3:4` を指定しても、実際の出力は **896x1280（7:10）** になる
- 実体はPNGだが拡張子を `.webp` にしても画像としては機能する
- `3:4` は本来 960x1280 のはず → API側が内部で丸めている可能性
- 対応策: テンプレート側の `lp-ratio` を実際の出力比率に合わせるか、生成後にリサイズする

## 出力フォーマット

- Gemini Imagen の出力は実質PNG形式（ファイルヘッダーが `\x89PNG`）
- `.webp` 拡張子で保存してもブラウザは表示できる（コンテンツスニッフィング）
- 正確な WebP が必要な場合は ImageMagick 等で変換が必要

## banana-claude が使う別モデル

- `gemini-3.1-flash-image-preview` は `generateContent` エンドポイントを使用
- こちらは `4:5` を含むより多くのアスペクト比に対応（VALID_RATIOS に 4:5, 5:4, 21:9 等あり）
- ただし当環境での動作は未検証
