# 03. スライドデッキ

## ピッチデック・営業資料（英語）

```
Using the messaging and competitive research I've uploaded, create a [N]-slide
[deck type] for our [team/audience]. Structure:
- Slide 1: Title
- Slide 2: Market context
- Slides 3–4: Our positioning
- Slides 5–6: Head-to-head feature comparison
- Slides 7–9: Top 3 objection handling
- Slides 10–11: Customer proof points
- Slide [N]: Next steps CTA
Design: clean, minimal. White background, dark navy text, green accent (#00C97A).
Font: Bold display for headlines, lighter weight for body.
```

## 提案書（日本語・実例）

```
チーム向けのAIリスキリングプログラムの提案書を作成してください。
構成: 表紙、課題提起、提案概要、カリキュラム詳細、料金プラン、次のステップ
対象: 製造業の人事部長
デザイン: 白背景、濃紺テキスト、アクセントカラーは企業ブランドカラー #0A2E5E
フォント: 見出しは太字のNoto Sans JP、本文は通常ウェイト
1枚ずつ大きめのビジュアルと最小限のテキストで構成。
「読むスライド」ではなく「見て理解できるスライド」を作成してください。
```

## スライド品質ガードレール（追加で貼る）

```
Design rules for this deck:
- Max 2 background colors across all slides
- Section headers use a distinct layout (full-bleed color band)
- Font minimum: 24px. Never smaller.
- Use visual variety: alternate full-bleed image slides with text-only slides
- No more than 4 bullet points per slide
- Each slide must have one and only one hierarchy level that is clearly dominant
```

## アニメーション指定

```
On the [slide name] slide, show [chart/visual element].
Animate [element] [animation behavior] on slide entry.
Add a subtle fade transition between all slides.
Each slide: headline first, supporting text second, visuals third.
```

## アニメーション実例

```
On the Traction slide, show a bar chart of MRR growth over 12 months.
Animate the bars growing up on slide entry.
Add a subtle fade transition between all slides.
Each slide: headline first, supporting text second, visuals third.
```

- スライドはHTMLエクスポートを推奨（PPTXはホバー・アニメ消失）
- 1スライド = 1メッセージの原則を保つ
