## セクション8：パフォーマンス設計方針

### 8-1. 画像フォーマット指定

| フォーマット | 使用箇所 |
|---|---|
| **WebP** | 写真・イラスト（基本はすべてWebP） |
| **SVG** | アイコン・ロゴ・単純な図形 |
| **PNG** | 透過が必要でWebP非対応環境向けフォールバックのみ |
| **JPEG** | 原則不使用（WebPで代替） |

**FV ヒーロービジュアルのファイルサイズ目標：** 300KB 以内

### 8-2. lazy loading 適用方針

| 対象 | loading 属性 | 理由 |
|---|---|---|
| FV内ヒーロービジュアル | `eager`（または属性なし） | LCPに直結するため lazyにしない |
| FV より下の全画像 | `lazy` | 初期表示を軽くする |

### 8-3. 禁止エフェクト

- [ ] パーティクルアニメーション — 禁止
- [ ] 視差スクロール（Parallax）— 禁止
- [ ] `transform` / `opacity` 以外への CSS アニメーション — 禁止
- [ ] 背景への自動再生動画 — 禁止

**使用するアニメーション・エフェクト（あれば）：** ___
