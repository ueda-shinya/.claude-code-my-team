# seo-content-strategy

オーガニック検索からの流入を体系的に増やすSEOコンテンツ戦略スキル。SEO3対策（コンテンツ／内部／外部）の全体像を整理し、コンテンツSEOの4STEPと検索意図4分類で記事戦略を立て、EEAT原則で品質を担保する。

## 適用条件

- 使う場面：
  - 新規メディア立ち上げのSEO戦略を一から設計
  - 既存メディアの記事戦略をリニューアル（KW選定・コンテンツ計画）
  - 検索意図に合わない記事が多くインデックスはされるが上位表示されない
  - EEATの観点で自社コンテンツの信頼性を高めたい
  - 内部対策・外部対策の実装漏れを点検したい
- 使わない場面：
  - 既存サイトの個別SEO監査・診断 → `seo-audit`（**英語で SEO audit / SEO health check / technical SEO / on-page SEO / why am I not ranking と呼ばれた場合は必ずこちら**）
  - AI検索エンジン対策（ChatGPT/Perplexity等の引用獲得） → `llmo-audit` ／ `llmo-compliant-coding`
  - 流入減・CV減の1次診断 → `ga-gsc-diagnosis`
  - ブログ記事個別の執筆 → `blog-post`
  - LP制作（短期CV特化） → `lp-create`

**`seo-audit` との住み分け（重要）**：
- `seo-audit`：**既存サイトの問題特定・診断**専用（既存記事や既存LPを対象にした「現状の何が悪いか」を出すスキル）
- `seo-content-strategy`：**新規メディア戦略立案・既存メディアの戦略リニューアル**専用（「これから何を作るか・どう設計するか」を出すスキル）
- 英語起動の場合は `seo-audit` を優先。「SEO戦略を立てる」「コンテンツSEO設計」「KW選定」と日本語で言われた場合は `seo-content-strategy`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象サイト/メディア | 必須 | URL・テーマ | 「BtoB SaaS製品オウンドメディア」 |
| 主要事業領域 | 必須 | サービス／商品ジャンル | 「マーケティングDX支援」 |
| 競合上位サイト | 推奨 | 主要KWの検索1ページ目に出る競合 | 「サイトA・B・C」 |
| 既存KPI | 推奨 | 現状の検索流入数／表示回数／CTR／順位 | 「月間PV 5万・平均順位18.9位」 |
| 目標 | 必須 | 流入数／指名検索／CV／ブランド認知 | 「主要KW 3つで上位3位」 |

## 実行ステップ

### ステップ1：SEOの定義と3つの特徴を確認する

- 入力：対象サイト
- 処理：SEOの基本を共有
  - **SEO（Search Engine Optimization）**：検索結果の上位に表示されるためのマーケティング手法
  - **SEO記事**：特定のキーワードをターゲットにし、検索結果での上位表示を狙う記事
  - **3つの特徴**：
    1. **検索意図の理解**：ユーザーが求める情報を提供し、疑問や悩みを解決する内容
    2. **キーワードの最適化**：ターゲットKWをタイトル・見出しに自然に組み込む
    3. **内部リンクの活用**：他の関連ページとリンクし、サイト全体の価値を高める
  - **SEOの4メリット**：
    1. 検索意図の理解（上位3位で全クリックの約68.1%を獲得）
    2. 長期的な集客効果（一度上位表示されれば継続的にトラフィック獲得）
    3. ブランドの信頼性向上（専門性アピール）
    4. コスト効率の良さ（広告と違い、出稿停止しても流入は止まらない）
- 出力：SEO戦略の前提共有

### ステップ2：SEO対策3種類の役割を整理する

- 入力：対象サイトの現状
- 処理：以下3種類の役割を理解し、優先度をつける
  - **対策①：コンテンツSEO**：記事を構成するキーワードやコンテンツの質による対策（記事品質＝最重要）
  - **対策②：内部対策**：ユーザーによりよいサイト体験を届けるための技術的な対策（サイト構造・速度）
  - **対策③：外部対策**：第三者評価を得るための対策（被リンク・引用・SNS）
- 出力：3対策の優先順位（通常はコンテンツSEO最優先）

### ステップ3：コンテンツSEO 4STEPを実行する

#### STEP1：キーワードの調査・選定（5つの方法）

- **方法①：今検索されているキーワードから考える**：Search Consoleで現在の流入KWを調査
- **方法②：ユーザーが調べそうなキーワードを書き出してみる**：自社サービスを利用しそうなユーザーの検索ワードを書き出す（例：「Webマーケティング」「通販 売上改善」「広告運用 方法」）
- **方法③：サジェスト機能を活用する**：書き出したKWをもとに検索窓予測KWを取得
- **方法④：ラッコキーワードを活用する**：https://related-keywords.com/ で関連KW・サジェスト・Q&Aを一括取得
- **方法⑤：Google Keyword Plannerを活用する**：検索ボリューム・競合性を確認し、優先順位付け
  - 検索ボリューム大すぎ → 上位表示まで時間がかかる
  - 検索ボリューム小すぎ → 誰にも検索されない
  - 適度なボリューム×低～中競合性 を狙う

#### STEP2：検索意図の分析・評価（4分類）

| 分類 | 意図 | 例KW |
|---|---|---|
| **Know** | 情報収集 | 「Webマーケティング とは」「Webマーケ 会社 比較」 |
| **Go** | 目的地（特定サイト・店舗） | 「株式会社koujitsu」「和食 日本橋」 |
| **Do** | 行動に結びつく | 「Web広告 運用方法」 |
| **Buy** | 購買に結びつく | 「Webコンサル 料金」 |

- **戦略**：
  - 基本的にBuyに近いKWから優先対策（CV直結）
  - 多くのユーザーは情報収集（Know）から始まるため、Knowで信頼を獲得→Buyで適切な導線設計
  - 検索ボリュームの大きさはKnow > Go > Do > Buy、購入意欲は逆順

#### STEP3：競合他社の調査

- リストアップしたKWで現在検索結果上位に表示される他社サイトを調査
- 「広告」表示は調査対象外
- 検索結果1ページ目記事の「タイトル」と「見出し」を調査

#### STEP4：タイトルと見出しの作成

- **hタグの仕組み**：
  - h1：記事のタイトル
  - h2/h3/h4：見出し（h2以下は階層構造）
  - 検索エンジンはhタグの情報を優先取得
- **作成方法**：
  - 検索結果1位の記事のタイトル・見出し構成を参考にする
  - hタグ内（タイトル・見出し）にKWを盛り込む
  - 上位記事よりも「情報が網羅されて、構造が分かりやすく、KWに対するアンサーとしてより適切な情報」を提供できれば上位を獲れる

- 出力：KW候補リスト＋検索意図分類＋競合調査結果＋タイトル/見出し構成

### ステップ4：EEAT原則でコンテンツ品質を担保する

- 入力：ステップ3のコンテンツ計画
- 処理：Googleが評価する4要素で点検
  - **E：Experience（経験）**：実体験に基づく内容
  - **E：Expertise（専門性）**：誰にでも書けない専門性
  - **A：Authoritativeness（権威性）**：誰が書いたか明確
  - **T：Trustworthiness（信頼性）**：信頼できる情報源
- **EATを高めるためのライティング**：
  - 経験/専門性：一次情報を伝える／特化したジャンルをつくる
  - 権威性：著者名・運営者情報・メディア情報を明記
  - 信頼性：信頼できるサイトからの引用を記載
- **重要事実**：SEOを目的としたリライトに近い記事ばかり量産してもユーザー価値が低い。インタビュー、運営者本人が取材した記事、独自ノウハウ等の **一次情報を発信することが重要**
- 出力：EEAT点検結果＋ライティング方針

### ステップ5：内部対策3種類を点検・実装する

#### ①検索エンジンがサイトを巡回しやすくする（クローラー最適化）

- **XMLサイトマップの作成**：検索エンジンにサイト構造を早く理解させる
- **内部リンクの最適化**：パンくずリスト／グローバルメニュー／フッターメニューの適切な設置
- **SSLの適用**：HTTPS化（SSL化していないサイトは検索上位に表示されにくい）

#### ②検索エンジンに情報を伝達しやすくする（インデックス最適化）

- **タイトル・見出しタグで構造を分かりやすくする**：h1〜h4の階層を正しく使用
- **タイトル・見出しにKWを含める**：マスト
- **meta descriptionを適切に設置する**：検索結果に表示されるサイトタイトルと説明（title・description）

#### ③ユーザー体験を向上する

- **ポイント①**：「ユーザーが見やすいサイトにする」ことが重要
- **ポイント②**：スマホユーザーを想定してデザインする（Mobile First Index）
- **対策**：
  - サイトの表示スピードを改善する
  - レスポンシブデザインにする
  - 押したいリンクがタップできるようリンクとリンクの間が適切に離れている
  - 文字が小さすぎない
  - 画面サイズにあっていて、拡大や横スクロールをしなくてよい
- **チェックツール**：Google Search Console（モバイルユーザビリティ）／PageSpeed Insights（速度）

- 出力：内部対策チェックリスト＋実装タスク

### ステップ6：外部対策3種類を実行する

#### ①被リンクを獲得する

- 被リンクとは別のサイトで自社のサイトリンクを紹介されること
- **被リンクSEO効果のポイント**：
  1. 被リンクされたサイトの数
  2. 被リンクされたサイトのドメインパワー
  3. 被リンクされたサイトと自社サイトとの関連性

#### ②引用を獲得する

- 引用とはGoogle/Yahoo!検索、SNS、YouTube、ブログなど、あらゆるテキスト情報で自社情報を引用されること
- **引用の対象となる情報**：
  1. 名前（サイト名、企業名、サービス名）
  2. テキストURL
  3. 電話番号・住所

#### ③SNSシェア対策をする

- SNSのシェア・拡散の数もSEOに貢献
- 記事をシェアしやすいようSNSシェアボタンをつける

- 出力：外部対策の実行計画

### ステップ7：成功の鍵を確認する

- 入力：ステップ1〜6の計画
- 処理：以下3つを最終確認
  1. **検索意図の理解**：ユーザーが何を求めているかを把握する
  2. **質の高いコンテンツ**：情報の正確性、読みやすさを確保
  3. **継続的な分析と改善**：トラフィックや順位を分析し、内容を最適化
- **SEOで一番大切なこと**：ユーザーに役立つ情報を提供すること！
- 出力：実行優先度確認＋PDCA計画

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 新規メディア立ち上げ | コンテンツSEO最優先（記事の質）→ 内部対策併用 | 記事がないと外部対策も効かない |
| 既存メディアで上位表示されない | EEAT点検＋検索意図再分析 | 量産記事は評価されない |
| 検索ボリュームが大きすぎるKW狙い | 関連ロングテールKWへ分散 | 上位表示まで時間がかかりすぎる |
| ターゲットがCV直結のKW中心 | Buy/Do KWを優先 | 短期CV直結 |
| 認知拡大期のメディア | Know KWで信頼獲得→Buyへ導線 | ユーザー多数派は情報収集から |
| サイトが古くSSL未対応 | 内部対策①SSL化を最優先 | Google評価減 |
| モバイル比率高い | UX向上のレスポンシブ＋速度を優先 | Mobile First Index |
| 被リンクが少ない | 引用獲得＋SNSシェアから始める | 被リンクは時間がかかる |
| 競合が圧倒的に強いKW領域 | ニッチKW＋一次情報で差別化 | 勝てる土俵を作る |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ボリューム大すぎKWばかり狙う | 関連ロングテール＋ニッチで土台作り | 大KWは数年かかる |
| KW選定だけして検索意図無視 | STEP2で4分類に必ず分けて意図確認 | 意図と内容のズレで上位不可 |
| リライト量産でEEATを軽視 | ステップ4で一次情報を必須化 | 量産記事は評価されない |
| 内部対策を「コーディング担当任せ」 | ステップ5で運用者がチェックリスト管理 | SSL未対応・hタグ崩れの放置事故 |
| 被リンク購入で短期成果狙い | 引用獲得＋SNSシェアで自然な評価 | Googleペナルティリスク |
| Mobile First Indexを後回し | ステップ5-③でモバイル最優先 | toC事業はモバイル比率多数派 |
| 上位3位の重要性を軽視 | 上位3位で68.1%のクリック獲得を意識 | 1ページ目以下は事実上見られない |

## 連携スキル

- 監査：`seo-audit`（既存サイトのSEO監査・診断）
- 個別執筆：`blog-post`（記事執筆）
- AI検索：`llmo-audit`（AI検索引用獲得）／`llmo-compliant-coding`（新規LP/サイトのLLMO遵守）
- 流入診断：`ga-gsc-diagnosis`（流入減・CV減1次診断）
- LP連動：`lp-create`（短期CV LPと長期SEOコンテンツの併用設計）
