# credo-language-design スキル

文化（クレド・約束）の言語化 = 企業や組織のミッション・ビジョン・価値観を「行動規範」として言語化し、「価値観を体現しているか？」を解像度高く問いかけられるようにする「約束事」。事業戦略スクール「知足」のフレームワーク準拠。

## クレドの5特徴（5つの定義）

01. クレドは、つよい会社をつくるための指針である（憲法と法律のような関係性）
02. クレドは経営陣・社長よりも上位概念である（経営暴走の抑止力でもあるため、経営への反論もクレドを元にであれば受け付け咀嚼する）
03. クレドとは中心線である（多様性を生むための、真ん中の根であり、軸）
04. クレドはチェーンである（1つだけでなく、つながって価値となる。1つだけできていれば良いわけではない）
05. クレドは問題文である（迷った時、判断するとき、これを見てほしい）

## クレドの2目的

| 目的 | 内容 |
|---|---|
| **目的01：組織の価値観をメンバー間で統一する** | 「価値観」は人それぞれのため、解釈がぶれてしまう。クレドは具体的に表現して日常的に体現しやすくするための装置 |
| **目的02：企業ブランドを強化し、外部にも文化を伝える** | クレドを公開することで「こういう行動をする企業だ」という法人格を対外的に表現するコンテンツ。法人の人格を表す言語 |

→ クレドが、日常業務や意思決定の基準として機能し、効率を高める。

## 適用条件

- このスキルを使う場面：
  - クレド（行動規範・約束事）を新規策定したい
  - 既存のミッション・バリューを「日常で体現できる」レベルまで具体化したい
  - 全員が納得できる共通認識として組織文化を言語化したい
  - 採用ブランディング素材としてカルチャーを公開したい
  - 意思決定が人によってブレるのを防ぎたい
- このスキルを使わない場面：
  - MVV（ミッション・ビジョン・バリュー）の上位概念設計 → `mvv-design`（クレドはMVVの下位、行動レベル）
  - 評価制度の策定 → `evaluation-system-design`（クレド体現を評価項目に組み込むのは別工程）
  - ブランドコンセプト・トーン＆マナー → `concept-message-tonemanner`
  - ブランディング全体設計 → `branding`
  - オンボーディング設計 → `onboarding-design`（クレド浸透施策の一部として連携）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 会社情報 | 必須 | 業種・規模・既存MVV有無 | 「Web制作・20名・MVVあり・クレドなし」 |
| 既存の価値観文書 | 任意 | MVV・行動指針・社訓等 | 「ミッション：身近な人を幸せに」 |
| ワーク参加メンバー | 必須 | 全員 or 代表者 | 「全社員10名で実施」 |
| ワーク実施期間 | 必須 | 半日 / 1日 / 数週間 | 「半日ワーク+1週間投票期間」 |
| 公開範囲 | 必須 | 社内のみ／対外公開 | 「対外公開（採用ブランディング含む）」 |
| 想定アウトプット | 必須 | クレド項目数（5〜20項目の範囲。指定なき場合のデフォルトは8項目。20項目超は禁止） | 「8項目（デフォルト）」 |

## 実行ステップ：完成までの8ステップ

### STEP 01：クレドの種作り（全員でアウトプット）

- 入力：会社情報、既存の価値観文書、ワーク参加メンバー
- 処理：
  1. **理想状態を共有**：クレドを作る上で目指す5つの理想状態を最初に全員で確認
     - ① カルチャーに適したクレド（約束）が言語化されている
     - ② 上下関係なく会社全員が納得した共通認識となっている
     - ③ 浸透・体現を継続するためにクレドへの意識が日常的に盛り込まれている
     - ④ クレドに沿った行動・意識ができている
     - ⑤ 意思決定や決断時にクレドを指針にする
  2. **企業文化（カルチャー）を洗い出す**：例：「身近な人を幸せにする」「101%を出す姿勢」「人を大切にする利他の心」等
  3. **企業文化に対するアクション例を全員でアウトプット**：付箋／ホワイトボード／Miro等を使い、各文化を体現している具体的な行動例を出し合う
- 出力：企業文化×アクション例の付箋集合（数十〜数百枚）

### STEP 02：グルーピング（クレドワーク）

- 入力：STEP 01のアクション例
- 処理：
  1. **「誰に対して」の軸でグループ分け**を意識すると書き出しやすい
     - 例：「自分・家族」「クライアント・メンバー（社員）」など対象別にグルーピング
  2. **テーマ別グルーピング**：似た意味合いの行動例を寄せ集める（例：「幸せ」「関係の質」「マインド・人格」「チームビルディング」「コミュニケーションスキル」「101%」）
  3. グループに仮タイトルを付ける（例：「身近な人を大切にしたアクション」「Sayで終わらない姿勢」「常に前向きな思考」）
- 出力：グルーピング済みアクション例（5〜10グループ）

### STEP 03：クレド総選挙シート（投票）

- 入力：グルーピング済みアクション例
- 処理：
  1. **クレド投票シートを作成**：以下のフォーマット
     ```
     スコアリングの解釈：
     2：自社のカルチャーとして大切にしたい
     1：どちらかというと自社のカルチャーに適合している
     0：どちらとも言えない or 自社のカルチャーに適合していない

     | 順位 | クレドを体現するアクションの種 | 概要・解釈 | 集計 | A | B | C | ... |
     |---|---|---|---|---|---|---|---|
     | 1 | 挑戦することを否定せず積極的に応援する | 誰かの挑戦は否定するのではなく応援し、フ... | 54 | 2 | 2 | 2 | ... |
     ```
  2. 全メンバーがそれぞれの行動例に0/1/2点をつける（自分の名前列に入力）
  3. 集計列で全員の合計点を自動計算
  4. **降順ソート**で自社が大切にしたいアクションが上位に並ぶ
- 出力：投票結果ランキング（集計点付き）
- ダウンロード可能なテンプレート：https://docs.google.com/spreadsheets/d/1fcilRkEFSZbExw0-tY6ioNfMTILMlNjy1nhBuqEwWDE/edit?usp=sharing

### STEP 04：取捨選択

- 入力：投票結果ランキング
- 処理：
  1. **想定アウトプット数（例：8項目）に絞る**：上位8項目を採用
  2. **チェーン（つながり）を確認**：クレドはチェーンなので、選んだ項目同士が補完関係にあるか確認
  3. **「中心線」を確認**：選んだ項目群が組織の真ん中の軸になっているか
  4. **満場一致のものは確実に残す**：全員が2点をつけた項目は最優先
  5. **意見が割れた項目は議論**：なぜ大切と思うか／思わないかを共有してから決定
- 出力：採用クレドリスト（取捨選択済み）

### STEP 05：クレドの再言語化（ワーディング）

- 入力：採用クレドリスト
- 処理：
  1. **アクションのたねを「しよう」「もとう」など意志表現に変換**
  2. **エモくてわかりやすく**を意識：抽象語ではなく具体的かつ感情を動かす表現
  3. **クレド案①／クレド案②を複数案出す**：以下のテンプレートに沿う

| 分類 | アクションのたね | 概要・解釈 | クレド案① | クレド案② |
|---|---|---|---|---|
| 挑戦思考 | クライアントから常に頼られる存在を目指す | 単なる業務代行ではなくパートナーとして常に担当者に頼られる関係性を目指す | 徹底的に寄り添おう | 「成果」に集中しよう |
| 幸せ | 仕事を通じて家族を喜ばせる | 家族に誇れる、自慢できる、話して喜ばせられるような仕事をする | 透明に、誠実に、誇りを持とう | 家族に誇れる仕事を |
| 関係の質 | 関係の質を履き違えないこと | 仲が良く、甘やかすことではなく、リスペクトをもって間違いやmottoを言い合える関係 | 全力で助け、本気で向き合う | リスペクトファースト |
| マインド・人格 | 信頼されリスペクトされる器を持つこと | 人に優しいだけではなく、信じて頼られる存在になること | 理解され、信頼され、任される人であろう | 信頼と尊敬を勝ち取れ |
| マインド・人格 | 思考停止せず、考え尽くす | 難しい、わからないで終わらず、他人に聞く前にまず自分で徹底的に思考する | 徹底的に思考 | 「考えた」では足りない、「考え尽くす」 |
| マインド・人格 | 否定から入らずに先ずは肯定する心を持つ | 頭ごなしに否定せず相手の言動を肯定した上で自身の意見やフィードバックを伝える | 否定より肯定／アサーティブコミュニケーション | 傾聴姿勢 |
| チームビルディング | 個人よりもチームの利を尊重する | 個人よりもチームにとってプラスになること、良くなることを優先する | チームファースト | フォロードリブン |
| チームビルディング | 決められていないことでも誰かのために行動する | 仲間のタメになるのであれば、決められていなくても行動に移す | 人に、チームに、柔軟に | 圧倒的当事者（マインド）思考 |
| コミュニケーションスキル | 相手の目線に立った言動を常に意識する | 必ず一度は相手目線で物事を考えたうえで意見やアクションを行う | 仲間が誇り | 相手をインストールする |
| 101% | 顧客へ期待値以上の価値提供をする | 定量と定性で価値提供できているか、常に意識する | 101%がスタート | 「101%」にこだわろう（毎日101%） |

  4. **POINT**：このワーディングは、組織のあり方によって異なるため、自社の状況にあわせてしっくりくるものを考える
  5. メンバー間で再度投票し、案①／案②から最終ワーディング決定

- 出力：最終ワーディング済みクレドリスト

### STEP 06：クリエイティブ化

- 入力：最終ワーディング済みクレドリスト
- 処理：
  1. **デザイン化**：クレドカード／ポスター／Webページとして視覚化
  2. **参考事例**を確認しトーンを揃える
     - **事例1：SHE CREDO**：「Heartful professional 心あるプロフェッショナルになろう」「New era hero 誰も成し遂げたことのない大きなチャレンジをし続けよう」「Respect every life 一人一人の人生を尊重し、寄り添い合う存在になろう」「Chill and Play いつも少しの遊びゴコロを持って仕事に取り組もう」
     - **事例2：オンリーストーリー18項目**：「民意を高く」「船に乗るな、漕げ、創れ」「選択を肯定する」「点で見るな、線で見ろ」「悩むな、考えろ」「スピードイズキング」「手を挙げろ」「神は細部に宿る」「ラスト1%」「数字から逃げない」「顧客を勝たせる」「競争と協力」「陰口は日向口に、不満は指摘」「成果からの要求」「主張と尊重」「属性の尊重と、レベルの向上」「プラスルール」「はたらくを、たのしむ」
     - **事例3：ユーザーベース DO/DON'T**：「DON'T：周りのせいにする」「DO：原因をまず自分に求める」のように対比型で表現
  3. **イメージ図**：クレドを中心に「コアバリュー（常に守る指針・軸）／OSメンバー（円）／個人の価値観（属性の尊重と多様性の範囲）」が配置される構造を視覚化
- 出力：クレドのクリエイティブ（カード／ポスター／Webページ）

### STEP 07：完成・発表

- 入力：クレドのクリエイティブ
- 処理：
  1. **全社発表会の開催**：策定プロセスを共有しながら最終クレドをお披露目
  2. **発表時のメッセージ**：「価値観はそれぞれでいい。でも、青い円（クレド）を通らないと組織はバラバラになる」を強調
  3. **ストーリーで語る**：なぜこのクレドになったか、各クレドに込めた想いを言語化
  4. **公開判断**：採用ページ／コーポレートサイトに掲載するか決める
- 出力：発表完了、社内外向け公開コンテンツ

### STEP 08：浸透・運用

- 入力：完成したクレド
- 処理：
  1. **日常への浸透施策**を選定：
     - クレドカードの携帯（オンボーディングで配布）
     - 朝会／全体会で1日1クレド読み合わせ
     - 1on1の冒頭で「今期体現したクレド」を共有
     - 360度評価／社内表彰で「クレド体現賞」を設置
     - Slackでクレド体現エピソードを共有チャネル
  2. **継続的改善**：年1回のクレド見直し会を実施（言葉が古くなっていないか／追加すべき項目はないか）
  3. **新メンバーへの伝承**：オンボーディング初日にクレド説明セッションを必ず設置
  4. **意思決定での活用**：会議での判断に迷った時、「これはクレドのどれに沿うか／反するか」を必ず議論
- 出力：浸透施策一覧と運用カレンダー

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| MVVが未策定 | クレド策定の前にMVV策定（`mvv-design`）を先に実施 | クレドはMVVの下位概念で、上位がないと中心線がブレる |
| メンバー数が3名以下 | クレド策定はまだ早い（暗黙知で十分） | 5名超で価値観のブレが顕在化してくる |
| メンバーが30名超 | 全員参加のワークは難しいため代表者選出 | 全員参加だと収束しない |
| 「価値観に正解がない」と議論が紛糾 | クレド5特徴の②「経営陣・社長よりも上位概念」を持ち出し、経営者も一歩引く | 経営者の独断で決めるとクレドが機能しない |
| 投票結果がほぼ全員2点で差がつかない | 「2点の中でも特に大切にしたい3つ」と再投票 | 取捨選択ができない |
| ワーディングが「すごく真面目で硬い」 | エモくてわかりやすく再構成（事例参考） | 浸透しないクレドになる |
| 既存の社訓・行動指針がある | クレド策定時に必ず参照／補完関係を明示 | 既存のものと重複・矛盾すると現場が混乱 |
| 公開判断で迷う | 採用競争力強化を狙うなら公開／秘伝にしたいなら社内のみ | 採用ブランディング目的ならほぼ公開推奨 |
| 浸透が進まない | 評価制度との連動／表彰制度の追加を検討 | 評価と切り離すと「飾り物」になる |
| 1年経って言葉が古くなった | 定期的なリフレッシュ会で言い換えを検討（軸は変えない） | 言葉だけ更新で軸は不変が原則 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 経営者が独断でクレドを決める | 全員参加ワーク（最低半日）で策定 | 経営者独断のクレドは現場に浸透しない |
| 抽象的なきれいごとで終わる（「お客様第一」等） | 具体的アクションに落とす（「お客様の期待値を120%上回るアクションを毎日3つ実行」等） | 抽象では行動変容しない |
| 30項目以上作って結局誰も覚えていない | 8〜18項目に絞る（記憶可能な数） | 多すぎは無いに等しい |
| ワーディングが堅すぎて浸透しない | エモくわかりやすく（事例参考） | 共感できないと浸透しない |
| 1度作って終わり／読み返さない | 朝会唱和／評価制度連動／表彰制度で日常に組み込む | 飾り物になると意味がない |
| 既存MVVと矛盾するクレドを作る | MVVを先に確認し、整合性チェック | 矛盾すると現場が混乱 |
| 公開せず社内秘 | 採用ブランディングで活用するなら公開推奨 | 採用競争力の機会損失 |
| 投票時に経営者と若手で同じ重みづけしない | 全員1票で対等に投票（クレドは上下関係なく決める） | 「上下関係なく全員納得」の前提が崩れる |
| 「これは個人の価値観だから」と多様性のない項目をクレドに入れる | クレドは「中心線」であり、属性の尊重・多様性の範囲は別途認める | クレドは多様性を生むための真ん中の根 |
| クレド完成後に評価制度と切り離す | 評価制度設計時にクレド体現項目を組み込む | 評価と連動しないクレドは形骸化 |

## 連携スキル

- `mvv-design`：MVV（上位概念）を先に策定／クレドはMVVの下位
- `evaluation-system-design`：クレド体現を評価項目に組み込む（「らしさ評価」軸）
- `concept-message-tonemanner`：クレドのワーディングをトーン＆マナーと整合させる
- `branding`：対外発信時のブランド戦略との接続
- `onboarding-design`：新メンバーへのクレド浸透施策（初日説明会／カード配布）
- `meeting-cadence-design`：朝会唱和／クレドレビュー会の運営設計
- `success-cycle-relationship-quality`：クレドはPOINT 03「ビジョン・目的を共有」の具体ツール
- `recruitment-strategy`：クレドを採用ブランディング素材として活用

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。本契約は適用しないが、運用上の**任意**ベストプラクティスとして以下のマーカー使用を**推奨**する：

- `[OK:0件]`：投票で2点獲得項目が0件のとき
- `[PARTIAL:N成功/M失敗]`：複数チームで並行ワークして一部の投票が締切までに揃わなかったとき
- `[FAIL]`：投票結果が集計不能（重複入力等）のとき。エラー要約を必ず添える
- `[SKIP]`：投票未参加メンバーがいる場合（試みなかったケースのみ）
