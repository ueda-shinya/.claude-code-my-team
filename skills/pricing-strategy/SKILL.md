# pricing-strategy

価格決定の3要素と2大戦略（スキミング／ペネトレーション）、PSM分析（価格感度分析）を統合した値付け設計スキル。マーケティングミックス4PのうちPrice領域に特化。

## 適用条件

- 使う場面：
  - 新製品・サービスの値付けを根拠付きで決定したい
  - 既存製品の価格改定を検討
  - 競合との価格戦略の差別化判断
  - PSM分析でユーザー調査の質問設計から実施
- 使わない場面：
  - 4P全体を俯瞰したい → `marketing-mix-4p4c`
  - 製品の構成要素 → `product-strategy-design`
  - プロモーション → `promotion-strategy`
  - 市場規模推定 → `market-size-tam-sam-som`
  - PMFの検証 → `pmf-journey`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 製品・サービス名 | 必須 | 値付け対象 | 「BtoB会議室予約SaaS」 |
| 自社の製品コスト | 推奨 | 原価＋人件費＋オーバーヘッド | 「1ユーザーあたり月額10,000円」 |
| 競合価格レンジ | 推奨 | 直接競合・代替品の価格 | 「競合A 30,000円、競合B 50,000円」 |
| 顧客のカスタマーバリュー（CV）想定 | 任意 | 顧客が払ってもいいと感じる上限 | 「会議室管理を効率化できれば月10万円までOK」 |
| ターゲット層 | 任意 | イノベーター層／マジョリティ層等 | 「アーリーアダプター」 |

## 実行ステップ

### ステップ1：価格決定の3要素（下限／上限／プライシング可能帯）を算出

- 入力：自社の製品コスト、競合価格、顧客CV
- 処理：以下3要素を整理
  - **下限（自社の製品コスト）**：これより下は赤字。原価＋人件費＋オーバーヘッド
  - **上限（顧客のカスタマーバリュー）**：これ以上は誰も買わない。顧客が認識する価値
  - **プライシング可能帯（競合／代替品の価格）**：下限と上限の間、競合価格を参照しながら自社の値付け可能ゾーンを決定
- 出力：3要素の数値＋プライシング可能帯（〜円〜〜円）

### ステップ2：価格戦略の選択（スキミング／ペネトレーション）

- 入力：ターゲット層、ステップ1のプライシング可能帯
- 処理：以下2戦略のどちらが適切か判定
  - **スキミング・プライシング（高価格戦略）**：
    - **特徴**：「高くても品質が良ければ買う」ユーザーを対象
    - **ターゲット**：イノベーター層／高所得層／専門家
    - **適用場面**：高品質訴求が可能な新製品、特許・独自技術あり、ブランド価値高い
    - **狙い**：早期に開発投資を回収、高利益率
  - **ペネトレーション・プライシング（低価格戦略）**：
    - **特徴**：早期の市場浸透、普及を狙う／品質の差では勝負しない
    - **ターゲット**：マジョリティ層／広いマーケット
    - **適用場面**：類似品が多い市場、ネットワーク効果が効く（プラットフォーム）、シェア重視
    - **狙い**：規模の経済、シェア獲得、競合参入の抑止
- 出力：選択した戦略＋根拠

### ステップ3：PSM分析（価格感度分析）の質問を設計する

- 入力：製品・サービス、想定価格レンジ
- 処理：以下4つの質問を設計
  1. **「これ以上高いと買わない」と思う値段はいくらですか？** （最高価格を測る）
  2. **「安すぎて品質が不安だ」と思う値段はいくらですか？** （最低価格を測る）
  3. **「高い」と思い始める値段はいくらですか？** （妥協価格を測る）
  4. **「安い」と思い始める値段はいくらですか？** （理想価格を測る）
- **質問形式の注意**：
  - **悪い例**：「いくらなら買いたい？」（自由回答）→ 新概念のサービスは値段感覚なくバラける／「1,000円で買いたい？」（4段階選択）→ 価格1点だけでは情報不足
  - **良い例**：「いくらなら買いたい？」＋ 単一選択で「5,000円／7,500円／10,000円」のように複数価格選択肢を提示
- 出力：4質問＋選択肢価格レンジ（5〜10段階推奨）

### ステップ4：PSM分析の4交点を算出する

- 入力：ステップ3のユーザー調査回答（実施後）
- 処理：4つのカーブ（高い／安い／高すぎて買えない／安すぎて品質疑わしい）の交点から4価格を読み取る
  - **最高価格**：「これ以上高いと誰も買わなくなる」価格＝「高い」と「高すぎて買えない」の交点
  - **最低価格（最低品質保証価格）**：「これ以上安くすると品質が疑わしいと思われる」価格＝「安い」と「安すぎて品質が疑わしい」の交点
  - **妥協価格**：「少し高いが、このくらいなら許容範囲」価格＝「高い」と「安い」の交点
  - **理想価格**：「購買に否定的な意見を持つ人が最も少ない」価格＝「高すぎて買えない」と「安すぎて品質疑わしい」の交点
- 出力：4価格の数値＋意味解釈

### ステップ5：最終価格決定

- 入力：ステップ1のプライシング可能帯、ステップ2の戦略、ステップ4のPSM結果
- 処理：以下を統合して最終価格を決定
  - **理想価格が利益確保可能** → 理想価格採用（販売数量と利益のバランスがベスト）
  - **理想価格では原価コストで利益が出ない** → 妥協価格を採用（許容範囲内の価格上限）
  - **スキミング戦略の場合** → 妥協価格〜最高価格の間で利益最大化
  - **ペネトレーション戦略の場合** → 最低価格〜理想価格の間で市場浸透優先
- 出力：最終価格＋値付けの根拠

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 競合より明確に高品質・差別化済み | スキミング戦略 | 高価格を正当化する価値があり、ブランド毀損も防げる |
| 類似サービスが多くシェア争い | ペネトレーション戦略 | 価格優位で先行シェア取り |
| ネットワーク効果が効くSaaS／プラットフォーム | ペネトレーション戦略 | 規模が成立しないと価値が出ない |
| BtoB・LTV重視のSaaS | スキミング寄り | 単価×継続率で利益を最大化 |
| 新概念サービスでユーザーに値段感覚なし | PSM分析を必ず実施 | フリーコメント自由回答だとバラけて使えない |
| 理想価格で原価が回らない | 妥協価格を採用 | 利益確保が優先 |
| BtoCコモディティ商材 | ペネトレーション＋競合価格を強く参照 | 顧客は競合と比較して購買判断 |
