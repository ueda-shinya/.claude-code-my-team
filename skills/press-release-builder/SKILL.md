# press-release-builder

メディア掲載を狙うプレスリリースを、TOPPINGフレームワークでネタを磨き込み、TODAY/YESTERDAY/TOMORROW構造で執筆するスキル。

## 適用条件

- このスキルを使う場面：
  - 新商品/新サービスのプレスリリース執筆
  - イベント開催・キャンペーン告知のメディア発信
  - 業務提携・人事・調査結果などのプレスリリース
  - 既存のプレスリリース企画にニュース性を加えたいとき
- このスキルを使わない場面：
  - 売上直結の広告コピー作成 → `ad-copy-7-angles`／`ad-mix-design`
  - SNS投稿 → `sns-post`
  - ブログ記事 → `/blog-post`
  - LP・営業資料 → `lp-create`／`proposal-builder`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 発表内容 | 必須 | 何を発表するか | 「新エコバッグ発売」 |
| 業種 | 必須 | カテゴリ | 「アパレル／環境配慮製品」 |
| ターゲットメディア | 必須 | 狙うメディア層 | 「環境系メディア／ライフスタイル誌」 |
| 発信日時 | 必須 | リリース日 | 「2026-05-15」 |
| 自社の背景／実績 | 任意 | これまでの取り組み | 「5年間の研究開発」 |
| 今後の方針 | 任意 | 将来の構想 | 「2030年までに全製品80%リサイクル素材化」 |
| 数字データ | 任意 | 具体的な実績/規模 | 「100%リサイクル素材」 |
| 配信方法 | 任意 | 配信チャネル | 「PR TIMES＋自社サイト＋SNS」 |

## 実行ステップ

### ステップ1：PRと広告の違いを確認しスタンスを決める

- 入力：発表内容、目的
- 処理：以下の対比でスタンスを明確化
  | 観点 | PR（広報）| 広告 |
  |---|---|---|
  | 目的 | 社会的な信用の獲得 | プロモーション・売上増加 |
  | 主体 | メディアが主体 | 自社が主体 |
  | 掲載決定 | メディアが決定／干渉不可 | 自社で決定 |
  | コスト | 配信費のみ（基本無料）| 広告費発生 |
- スタンス：「人が見たい・知りたい・面白いと感じる**ニュース性の高い情報**」を発信
- 売り込み色を排除し、客観的事実とニュース性で勝負する
- 出力：PRスタンスの確認

### ステップ2：ニュース性3要素で素材を点検する

- 入力：発表内容
- 処理：以下3要素のうち何を持っているか確認
  | 要素 | 内容 |
  |---|---|
  | 社会性 | 社会的なインパクトや意義（環境配慮／地域貢献／時流対応など）|
  | 新規性 | 「初めて知る」驚き（業界初／日本初／独自手法など）|
  | 共感性 | 読者が「自分事」として感じる視点（創業者の想い／ユーザーストーリー）|
- 1要素のみだと弱い → 2-3要素を組み合わせる
- 例：新商品（新規性）＋環境配慮（社会性）＋創業者の想い（共感性）
- 出力：保有する3要素＋不足要素の補強案

### ステップ3：TOPPINGフレームワークで切り口を磨く

- 入力：発表内容、ニュース性3要素
- 処理：以下7切り口で表現を強化する
  | 文字 | 要素 | 意味 | 表現例 |
  |---|---|---|---|
  | T | Trend | トレンド・時流・季節性 | 「歳末」「在宅需要」「ChatGPT」 |
  | O | Original | 最上級・初・独自 | 「業界初」「日本最大級」 |
  | P | Public | 社会性・地域性 | 「脱プラ」「地方創生」 |
  | P | People | 役者・語り手の影響力 | 「高校生社長」「元〇〇」 |
  | I | Inverse | 逆説・対立構造（意外性）| 「アクション映画監督が子ども向けアニメを手がける」 |
  | N | Number | 数字 | 「100万人突破」「70周年」 |
  | G | Going forward | 今後の方針（構想）| 「2025年1月にユーザー1,000万人を目指す」 |
- 各切り口で1つ以上の要素を素材から見つける
- 該当要素0の切り口は無理に作らない（嘘になる）
- タイトルとリード文に上位3要素を盛り込む
- 出力：TOPPING該当表＋強化版タイトル候補3-5案

### ステップ4：プレスリリース構成5パートで骨子を作る

- 入力：発表内容、TOPPING強化素材
- 処理：以下5パートで骨子作成
  | # | パート | 内容 |
  |---|---|---|
  | 1 | 発信日時 | 発信者名・発信日（年月日）|
  | 2 | タイトル | 簡潔でインパクトある見出し（30-40文字推奨／SEO観点も）|
  | 3 | リード文 | 発表内容のポイントを3-5行で簡潔に（5W1H含める）|
  | 4 | 本文・画像 | 詳細情報（背景・目的・具体的な内容）／画像添付推奨 |
  | 5 | 会社情報・問い合わせ先 | 広報担当者名／メール／電話番号 |
- タイトル作成ルール：
  - キーワードを前半に配置（SEO）
  - TOPPINGの上位2-3要素を含む
  - 数字を入れる（インパクト＋具体性）
- 出力：5パート骨子

### ステップ5：本文をTODAY/YESTERDAY/TOMORROWで執筆する

- 入力：5パート骨子
- 処理：本文を以下3パート構造で執筆
  - **TODAY（今起きていること）**：
    - 新商品/サービス発表
    - イベント/キャンペーン開始
    - 新プロジェクト立ち上げ
    - 記念日・節目の取り組み
    - 例：「当社は本日、新素材を活用したエコバッグを発表しました」
  - **YESTERDAY（これまでやってきたこと・背景）**：
    - 過去の取り組み・実績
    - 社会的課題やトレンド
    - 開発・実現に向けた苦労やエピソード
    - 例：「5年間にわたって環境配慮型製品の研究を行ってきました。創業者が目の当たりにした環境破壊への危機感がきっかけです」
  - **TOMORROW（将来やっていくこと・展望）**：
    - 今後のビジョン・目標
    - 今後の展開・計画
    - 社会・顧客への貢献の約束
    - 例：「2030年までに全製品の80%をリサイクル素材へ移行することを目指します」
- 出力：本文（TODAY→YESTERDAY→TOMORROW構造）

### ステップ6：報道されにくい3パターンで点検する

- 入力：執筆済みプレスリリース
- 処理：以下3パターンに該当しないか点検
  | NG パターン | 具体例 | 理由 |
  |---|---|---|
  | 広告・宣伝色が強い | 「PRを前面に」「単なる告知」 | 報道は客観性・中立性が必要／宣伝目的は敬遠 |
  | 曖昧・未確定の内容 | 「来春予定」「世界中で注目される予定」 | 信頼性低い／確定情報が必要 |
  | 既出情報・鮮度低い | 過去報道の繰り返し／長期間放置情報の再掲 | 新規性なく関心を引かない |
- NG該当 → 該当箇所を書き直す or 別のニュース素材を探す
- 出力：点検結果＋修正版

### ステップ7：配信方法を4種から選定する

- 入力：プレスリリース、ターゲットメディア
- 処理：以下から1〜複数選択（併用推奨）
  | # | 方法 | メリット | 適する場面 |
  |---|---|---|---|
  | 1 | PR配信サービス（PR TIMES／共同通信PRワイヤー）| 多数メディアに一括配信 | 認知最大化 |
  | 2 | メディア直接配信（記者リスト個別送付）| 関係性構築しやすい | 業界特化メディア・継続関係 |
  | 3 | 自サイト公開（ニュースリリース欄）| 既存顧客／ステークホルダー直接訴求 | SEO効果・自社資産化 |
  | 4 | SNSシェア（X／Threads等）| 拡散力 | 話題化狙い |
- 推奨：**併用が基本**。PR TIMES＋自サイト＋SNSが最低ライン
- 出力：配信プラン

### ステップ8：プレスリリース以外のPR施策6種から追加検討する

- 入力：PR目的、予算、期間
- 処理：以下から追加施策を検討
  | 施策 | 概要 |
  |---|---|
  | プレスリリース | 標準（このスキルで作成）|
  | ニュース配信 | トレンドニュースを企画書としてメディア向けに配信 |
  | メディアキャラバン | 狙ったメディア・記者にダイレクトアプローチ |
  | プレス向け勉強会 | 業界勉強会＋懇親（複数企業共催も可）|
  | PRイベント | 専門家・ユーザー・影響力者を招き取材獲得 |
  | オウンドメディア | 自社メディア／Wantedly／note／Spotify等のプラットフォーム活用 |
- 出力：補完PR施策リスト

### ステップ9：プレスリリース完成版＋配信プランをまとめる

- 入力：ステップ1-8
- 処理：出力フォーマットに沿って整形
- 出力：プレスリリース完成版＋配信プラン

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ニュース性3要素のうち0-1個しか該当しない | プレスリリース化を見送り or 別企画として再検討 | 報道されない確率が高い |
| TOPPINGで該当0要素 | 切り口を無理に当てはめず、本質的なニュース価値を再考 | 嘘や誇張は信頼毀損 |
| 「業界初」「日本初」を使いたい | 出典・根拠を必ず確認（事実検証）| 景表法違反リスク |
| 数字を入れたい | 具体的・検証可能な数字のみ採用 | 曖昧な数字は信頼低下 |
| 売り込み色が強い | 客観的事実ベースに書き直し | 報道採用率が下がる |
| 過去類似発表あり | 新規性を再構成（角度を変える）| 既出情報は採用されない |
| BtoB商材 | 業界専門メディア中心の配信 | 一般メディアでは関心低い |
| 地方企業 | 地方紙・地方TV局へのメディアキャラバン併用 | 地域性を強みに |
| トップの人物像が立つ | People要素を全面に出す | 影響力ある人の発信は採用率UP |
| 季節性のある商品 | Trend要素で時流に乗せる | 季節は強力な切り口 |
| 配信日が金曜午後 | 月曜午前に変更検討 | メディアの稼働時間に合わせる |
| 画像なし | 必ず1枚以上添付 | 採用率と訴求力が大きく違う |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 「素晴らしい」「画期的」など主観表現多用 | 具体的な数字／第三者評価で表現 | 主観表現は信頼度低い |
| タイトルに数字が1つもない | 必ず1-2個入れる | インパクト＋具体性 |
| TOPPING7要素を全部無理に詰め込む | 該当する3-4要素に絞る | 嘘や誇張は信頼毀損 |
| 「業界初」を出典なしで使う | 出典・根拠を脚注で明示 | 景表法違反リスク |
| 売り込み色満載 | 客観的事実とストーリーで構成 | 報道採用率が下がる |
| YESTERDAY（背景）省略 | 必ず背景を書く | 「なぜ今これを発表するか」が伝わらない |
| TOMORROW（展望）省略 | 必ず将来構想を書く | 単発発表に見える |
| 画像なしで配信 | 画像1枚以上添付 | 採用率と拡散力が違う |
| 金曜夕方や深夜に配信 | 月-木の午前中に配信 | メディアが見ない時間帯 |
| 配信して終わり | 配信後にメディアフォロー（電話確認等）| 取材獲得率UP |
| 1社に絞ってメディア訪問 | 複数併用（PR TIMES＋直接配信＋SNS）| 単一チャネルは弱い |
