# listing-ad-campaign-design

Google検索広告（リスティング広告）を、アカウント構造・キャンペーン構成・キーワード戦略・配信ロジックまで包括設計するスキル。

## 適用条件

- このスキルを使う場面：
  - Google検索広告（リスティング広告）の新規キャンペーン設計
  - キーワード選定とマッチタイプの決定
  - 指名キャンペーンと一般キャンペーンの分離設計
  - 広告グループの分け方
  - 配信開始後の運用設計
- このスキルを使わない場面：
  - ディスプレイ広告 → `display-ad-design`
  - Meta広告（Facebook/Instagram）→ `meta-ad-campaign-design`
  - 4分類10種類から媒体を選定する段階 → `ad-mix-design`
  - 広告改善診断 → `ad-performance-diagnosis`
  - 広告コピー作成 → `ad-copy-7-angles`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 商品/サービス | 必須 | 広告対象 | 「ネットスーパー」 |
| 業種 | 必須 | カテゴリ | 「EC／小売」 |
| ターゲット | 必須 | 属性＋エリア | 「東京23区在住・30-50代」 |
| 月予算 | 必須 | 媒体費 | 「月50万円」 |
| 目標CPA | 必須 | 1CV獲得コスト目標 | 「3,000円」 |
| 主要LP URL | 必須 | 遷移先 | 「example.com/lp」 |
| 自社ブランド名 | 必須 | 指名キャンペーン用 | 「イオン／AEON」 |
| 主要競合名 | 任意 | 競合KW候補 | 「西友／イトーヨーカドー」 |

## 実行ステップ

### ステップ1：アカウント5階層構造を設計する

- 入力：商品/サービス情報
- 処理：以下の階層を設計
  ```
  アカウント
   └─ キャンペーン（複数）
       └─ 広告グループ（複数）
           ├─ 広告文（複数：15パターンまで）
           ├─ 説明文（複数：4パターンまで）
           └─ キーワード（複数）
  ```
- 各階層の役割：
  - アカウント：1社につき原則1アカウント
  - キャンペーン：予算・地域・配信時間帯を分けたい単位
  - 広告グループ：訴求テーマ／LPごとに分ける
  - 広告：1広告グループに広告文15・説明文4を登録（機械学習でAIが最適表示）
  - キーワード：広告グループに紐づくKW群
- 出力：5階層構造図

### ステップ2：キャンペーンを「指名」と「一般」に分割する

- 入力：自社ブランド名、商品名
- 処理：以下の2キャンペーンに分割（基本構造）
  - **指名キャンペーン**：
    - 自社ブランド名・商品名のKW
    - 例：「イオン」「AEON」「イオンネットスーパー」
    - 特徴：CPCが安く、CVR高い
  - **一般キャンペーン**：
    - 商品カテゴリ・課題系のKW
    - 例：「ネットスーパー おすすめ」「ネットスーパー 比較」「宅配 食品」
    - 特徴：CPCが高く、新規顧客獲得用
- 分離理由：
  - 予算配分を独立管理（指名は獲得しやすいので予算少なめ）
  - CPA目標を別々に設定（指名は1,000円、一般は5,000円等）
  - 効果測定が明確化
- 出力：2キャンペーン構成

### ステップ3：軸KW＋サブKWでキーワードを洗い出す

- 入力：商品カテゴリ、ターゲット、競合
- 処理：以下のフレームでKWを生成
  - **軸キーワード**：商品カテゴリ名（例：「ネットスーパー」）
  - **サブキーワード4軸**：
    | 軸 | 例 | 用途 |
    |---|---|---|
    | エリア | 「ネットスーパー 横浜」「ネットスーパー 23区」 | 地域指定で表示／広告カスタマイザで自動表示可 |
    | おすすめ／比較 | 「ネットスーパー おすすめ」「ネットスーパー 比較」 | 比較検討層 |
    | 競合 | 「ネットスーパー 西友」「ネットスーパー イトーヨーカドー」 | 競合KW（出稿可否はポリシー確認）|
    | 価格／クーポン | 「ネットスーパー 送料」「ネットスーパー クーポン」 | 価格訴求層 |
  - 各軸で5-10KWを生成 → 計20-40KW
- ツール活用：
  - Googleキーワードプランナー（検索ボリューム・CPC相場確認）
  - ラッコキーワード（関連KW洗い出し）
  - Googleトレンド（季節性・トレンド把握）
- 出力：KW候補リスト（軸×サブで分類済み）

### ステップ4：マッチタイプを3種から選定する

- 入力：KW候補リスト
- 処理：各KWに以下3種のマッチタイプを設定
  | マッチタイプ | 表記 | 拡張範囲 | 用途 |
  |---|---|---|---|
  | インテントマッチ | `ネットスーパー 横浜` | 広い（関連語まで） | 認知拡大／広く拾いたい時 |
  | フレーズ一致 | `"ネットスーパー 横浜"` | 中（語順含むフレーズ） | バランス重視 |
  | 完全一致 | `[ネットスーパー 横浜]` | 狭い（完全一致のみ） | CV重視・無駄打ち回避 |
- 戦略ルール：
  - 指名KWは完全一致中心（無駄打ち防止）
  - 一般KWはフレーズ＋インテントを組み合わせ
  - 新規KW発見はインテントマッチ＋検索クエリレポートで実態把握
  - 不要なKWは「除外キーワード」として登録
- 出力：マッチタイプ別KW一覧＋除外KWリスト

### ステップ5：広告文と説明文を作成する（機械学習最適化）

- 入力：選定KW、訴求ポイント、LP内容
- 処理：以下を準備
  - **広告文**：1セットにつき**最大15パターン**登録（機械学習でAIが最適選択）
  - **説明文**：1セットにつき**最大4パターン**登録
  - 必須要素：
    - タイトルにキーワードを含める
    - 行動を促すCTA（「今すぐ確認」「無料お試し」「資料請求」）
    - 簡潔・ユーザー視点・興味を引く文章
- 文字数制限：
  - 広告文タイトル：30文字×3
  - 説明文：90文字×2
- コピー作成は `ad-copy-7-angles` スキルを併用
- 出力：広告文15パターン×説明文4パターン

### ステップ6：配信ロジック3要素を最適化する

- 入力：キャンペーン設定、LP
- 処理：以下3要素を整える
  - **入札額**：
    - 自動入札（目標CPA設定）が推奨
    - 手動入札はキーワード単位で細かく管理する場合のみ
    - 1日予算・予算上限を設定
  - **広告の品質（品質スコア1-10点）**：以下3要素で決定
    - ①推定クリック率（CTR）
    - ②広告の関連性（KW vs 広告文）
    - ③ランディングページの利便性（広告文 vs LP内容）
    - 一貫性が重要（KW→広告文→LPで同じテーマを訴求）
  - **推定アクション数**：
    - 週50CV以上が機械学習の理想
    - 50CV未達の場合は **マイクロCV（mCV）** を設定（問い合わせボタンクリック等）
- 出力：3要素の設定値・改善ポイント

### ステップ7：監視KPIを4指標で設定する

- 入力：キャンペーン目標
- 処理：以下4指標で監視
  | 指標 | 内容 | 目安 |
  |---|---|---|
  | CV / CPA | CV件数／1CV獲得コスト | 業種・目標による |
  | CTR | クリック率 | 業種で異なるが2-5%が目安 |
  | CPC / CPM | クリック単価／1000表示単価 | リスティングはクリック課金 |
  | インプレッションシェア | 競合と比較した広告表示シェア率 | 低い場合は入札額改善検討 |
- インプレッションシェアが低い時の対処：
  - 入札額が低すぎ → 入札UP
  - 予算が少なすぎ → 予算UP
  - 品質スコア低い → 広告文・LP改善
- 出力：KPIダッシュボード設計

### ステップ8：3コツでLPまで含めて最適化する

- 入力：キャンペーン全体
- 処理：以下3コツで点検
  - **コツ1：適切なキーワード選定**
    - ユーザーがそのKWで何を求めているか分析
    - 競争の少ない具体的KW（ロングテール）でCV狙う
  - **コツ2：魅力的な広告文**
    - タイトルにキーワードを含める
    - 簡潔かつユーザーの興味を引く
    - 行動を促すCTA（「今すぐ確認」「無料お試し」）
  - **コツ3：LPの最適化**
    - 広告文とLPでメッセージに一貫性
    - ユーザーが目的アクションをすぐ取れる設計
    - LPと広告のテーマ統一は品質スコアにも影響
- 出力：3コツチェック結果

### ステップ9：3注意点でリスク点検する

- 入力：稼働中キャンペーン
- 処理：以下3チェック
  | # | 注意点 | 対策 |
  |---|---|---|
  | 1 | 過剰な広告費のリスク | 予算上限設定／効果の悪いKWを除外 |
  | 2 | 広告疲れ | 定期的に広告文を変更／15パターン活用 |
  | 3 | 運用の専門性 | 広告運用代行検討／管理ツール活用 |
- 出力：リスク点検結果

### ステップ10：リスティング広告計画書としてまとめる

- 入力：ステップ1-9
- 処理：出力フォーマットに沿って整形
- 出力：リスティング広告計画書

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 自社ブランド名で検索される | 指名キャンペーンを必ず作る | 競合に指名KWを取られないため |
| 月予算30万円未満 | 指名＋一般1キャンペーン×広告グループ2-3に絞る | 細分化しすぎると最適化が効かない |
| 月予算100万円以上 | 訴求軸ごとに広告グループ細分化 | データ量が十分で個別最適可能 |
| 競合KWで出稿したい | Google広告ポリシー確認（商標申請ある場合は出稿不可）| 法的リスク |
| 週50CV未達 | mCV（問い合わせボタンクリック等）を中間CVに設定 | 機械学習の学習データ確保 |
| 品質スコア低（1-5点） | KW vs 広告文 vs LPの一貫性を見直す | 入札効率が悪化／表示順位低下 |
| インプレッションシェア低 | 入札額UPまたは予算UP | 配信機会が物理的に足りない |
| CTR 2%未満 | 広告文を全面刷新／KWとの整合性確認 | リスティング目安2-5% |
| CPC高騰（業界相場の2倍超） | 完全一致比率UP／除外KW追加 | 無駄打ちの排除 |
| 同じ広告文で3ヶ月配信 | 広告文を更新／15パターン活用 | 広告疲れでCTR低下 |
| Auto bidding初日に予算消化 | 入札上限を仮設定（学習中の暴走防止） | 学習期間の予算保護 |
| LPと広告の訴求がズレ | LP改修または広告文を合わせる | 品質スコア低下＋CVR低下 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 指名と一般を1キャンペーンで運用 | 必ず指名/一般を分離 | 予算配分／CPA目標が混ざる |
| 広告文1パターンで配信 | 必ず15パターン登録 | 機械学習が機能しない |
| 全KWをインテントマッチで配信 | 完全/フレーズ/インテントを使い分け | 無駄打ち多発でCPA高騰 |
| 除外KW未設定 | 「無料」「違法」「無関係カテゴリ」等を除外登録 | 無駄クリック発生 |
| KW・広告文・LPがバラバラ | 一貫性を持たせる | 品質スコア低下／CVR悪化 |
| 週50CV未達でも本CV最適化を続ける | mCVを設定 | 学習データ不足で最適化不能 |
| 競合の商標KWで出稿 | Google広告ポリシー確認 | 法的リスク／配信停止 |
| 入札手動でキーワード単位管理 | 自動入札（目標CPA最適化） | 機械学習活用が現代の主流 |
| LPの内容と広告文の訴求がズレ | 統一感ある訴求／LP改修 | 品質スコア・CVR両方悪化 |
| 月1回も広告文を変えない | 月1回更新／A/Bテスト | 広告疲れでCTR低下 |
| インプレッションシェア無視 | 50%以上を目標／低い場合は入札・予算改善 | 機会損失大 |
| 週次レビューなし | 週1回／月1回大規模見直し | 異常検知遅れる |
