# goal-hierarchy-design

事業目標→年間→四半期→月間→個人 の4階層に目標を分解し、長期ビジョンを具体的な行動に接続する。各階層で持つべき要素を明確化し、上下の整合性を担保する。

## 適用条件

- 使う場面：
  - 年間目標が決まったが、四半期・月間に落とせていない
  - 事業目標と個人タスクが接続していない感じがする
  - 中期計画を立てたが、現場の動きとズレている
  - 期初・四半期初に目標を再設計したい
- 使わない場面：
  - 単一目標をSMART化したい → `smart-goal-setting`
  - 進捗運用のサイクル設計 → `pdca-cycle`
  - モニタリングの仕組み化 → `goal-execution-system`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 大目標 | 必須 | 年間または中期の事業目標 | 「年間売上1億円」「ARR 5,000万円」 |
| 期間 | 必須 | 階層化する全体期間 | 「2026年度」「今期12ヶ月」 |
| 組織構成 | 任意 | 関与するチーム・人数 | 「営業3名/CS2名」 |
| 既存の組織目標／個人目標 | 任意 | 整合性チェック用 | 「営業部目標：商談数200件」 |

## 実行ステップ

### ステップ1：年間目標の3要素を整理する

- 入力：大目標、期間
- 処理：年間目標を以下3つに分解する
  - **事業目標**：会社・事業全体としての達成数値（例：売上1億円、ARR 5,000万円）
  - **組織目標**：その達成に必要な部門・チーム別の数値（例：営業部 新規受注6,000万円、CS部 解約率3%以下）
  - **個人目標**：各メンバーが担う数値・役割（例：営業A 1,500万円、営業B 2,000万円）
- 各層で「数値」「期限」「責任者」を明記する
- 整合性チェック：個人目標の合計 ≥ 組織目標 ≥ 事業目標 になっているか確認
- 出力：年間目標 3層シート

### ステップ2：四半期目標に分解する（4つの要素）

- 入力：年間目標 3層シート
- 処理：年間目標を四半期（Q1〜Q4）に分割し、各四半期で以下を決める
  - **優先順位付け**：その四半期で最重要のテーマを1〜2つに絞る（例：Q1=新規受注獲得、Q2=既存LTV向上）
  - **リソース配分**：人員・予算・時間の配分を四半期ごとに変える（例：Q1は営業に予算70%）
  - **進捗確認ポイント**：四半期内のチェックタイミング（月次／隔週レビュー等）
  - **数値配分**：年間数値を四半期ごとにブレイクダウン（均等割／前傾／後傾を選択）
- ブレイクダウンルール：
  - **均等割**：四半期ごとに1/4ずつ（季節性なし／新規事業）
  - **前傾**：Q1〜Q2で60〜70%（早期成果が必要）
  - **後傾**：Q3〜Q4で60〜70%（仕込み期間が必要）
- 出力：四半期目標シート（4枚）

### ステップ3：月間目標に落とす（3つの要素）

- 入力：四半期目標シート
- 処理：各四半期目標を3ヶ月に分け、各月で以下を決める
  - **具体的なアクション**：その月に実行する施策・活動を3〜5個（例：「展示会出展」「メルマガ4回配信」）
  - **数値目標**：その月の到達点（例：「商談数20件」「受注額500万円」）
  - **実行計画**：誰が・いつ・何をやるかの担当割り当て
- 整合性チェック：月間数値の合計 = 四半期数値 になっているか確認
- 出力：月間目標シート（3枚×4=12枚）

### ステップ4：個人目標に接続する

- 入力：月間目標シート、組織構成
- 処理：月間目標から各個人のアクション・KPIを切り出す
  - 各メンバーが「今月、自分が達成すべき数値」を1〜3個持つ状態にする
  - 個人目標は SMART化されていることを確認（必要なら `smart-goal-setting` スキルを併用）
- 出力：個人別月間目標一覧

### ステップ5：階層全体の整合性を最終確認する

- 入力：年間/四半期/月間/個人 の全シート
- 処理：以下のチェックリストで整合性を確認する
  - □ 年間 = Q1+Q2+Q3+Q4 の数値が合っている
  - □ 四半期 = 月1+月2+月3 の数値が合っている
  - □ 組織目標の合計が事業目標を満たす
  - □ 個人目標の合計が組織目標を満たす
  - □ 各階層で「期限」「責任者」が明記されている
  - □ 上位目標と下位目標の関連性が言語化されている（「なぜこの個人目標がこの月間目標に繋がるか」）
- 不整合があれば該当ステップに戻って修正する
- 出力：整合性チェック結果＋階層化マトリクス

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 年間目標がSMART化されていない | 先に `smart-goal-setting` スキルでSMART化してから本スキルを実行 | SMART化されていない目標は分解しても無意味 |
| 季節性のある事業 | 四半期配分は「前傾／後傾」を選ぶ | 均等割では現実と乖離する |
| 仕込み期間が必要な事業（PR/SaaS立ち上げ等） | 四半期配分は「後傾」 | 初期に成果を求めると無理が出る |
| 組織が小さく個人=組織になる | ステップ4を簡略化して個人目標を組織目標と一致させる | 階層を作ること自体が目的化しない |
| 個人目標の合計が組織目標を下回る | 達成不可能。組織目標を下げるか、人員追加を別タスク化する | 数字が合わない目標は形骸化する |
| 個人目標の合計が組織目標を大幅に上回る | バッファとして許容（10〜20%以内）。それ以上はリソース過剰投下 | 適度な余裕は必要だが、過剰は非効率 |
| 既存の組織目標がある | ステップ1の組織目標と既存の整合性をチェックし、ズレがあれば調整 | 並行する目標体系を作らない |

### 記入例（要約）

年間売上1億円 →
- Q1（前傾）2,800万円・展示会＋既存深堀り
- Q2 2,500万円・新規LP立ち上げ
- Q3 2,400万円・パートナー経由
- Q4 2,300万円・契約更新

→ Q1月別：1月800万・2月900万・3月1,100万 → 個人別：営業A月300万・営業B月400万…

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 年間目標を均等割で4等分する | 季節性・仕込み期間を考慮して前傾／後傾を選ぶ | 現実の事業は均等に進まない |
| 階層ごとに数値を作り直す | 上位目標を機械的に分解して整合性を保つ | 数字がずれた目標体系は形骸化する |
| 個人目標を「がんばる」レベルで終わらせる | 各個人にSMART化されたKPIを持たせる | 個人レベルで具体化されていないと行動に落ちない |
| 月間アクションを10個以上並べる | 月3〜5個に絞る | 多すぎると優先順位が消える |
| 整合性チェックをスキップする | 4つの整合性（年=Q／Q=月／組織≥事業／個人≥組織）を必ず確認 | 数字が合わない目標体系は破綻する |
| 上位目標との関連性を言語化しない | 「なぜこの個人目標がこの月間目標に繋がるか」を1文で書かせる | 関連性が見えないと現場のモチベーションが落ちる |
