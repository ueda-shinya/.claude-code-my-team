# onboarding-design スキル

オンボーディング（新入社員が職場環境や業務にスムーズに適応できるようサポートするプロセス）を3段階×3要素で設計する。

## 適用条件

- このスキルを使う場面：
  - 新入社員（新卒／中途）の受け入れプログラムをゼロから設計したい
  - 既存のオンボーディングが形骸化しており、再設計したい
  - 早期離職が多く、定着率を上げたい
  - 企業文化の浸透が弱く、新メンバーが組織に馴染まない
- このスキルを使わない場面：
  - 採用戦略・採用ガイドライン策定（採用前のフェーズ） → `recruitment-strategy`
  - 評価制度の設計 → `evaluation-system-design`
  - キャリアロードマップと中長期育成計画 → `career-roadmap-development`
  - 給与レンジの決定 → `salary-range-design`
  - 役割と責任範囲（恒久的職位定義） → `roles-responsibilities`
  - 指導アプローチの使い分け（ティーチング/コーチング/リーディング） → `teaching-coaching-leading`
  - クライアントワークの標準化（社員向けではなく顧客向け） → `delivery-build`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象者の属性 | 必須 | 新卒／中途／第二新卒／業界未経験 | 「中途・マーケ未経験」 |
| 採用ポジション | 必須 | 職種・役職 | 「マーケ事業部メンバー」 |
| 受け入れ規模 | 必須 | 同時入社人数／年間入社人数 | 「四半期ごと2〜3名」 |
| 既存制度の有無 | 必須 | バディ制度／メンター／オリエン等の有無 | 「メンター制度なし／オリエンは初日のみ」 |
| 解決したい課題 | 必須 | 早期離職／戦力化遅延／文化浸透不足 等 | 「3ヶ月以内の離職率20%」 |

## 実行ステップ

### ステップ1：オンボーディングの定義と目的を確認する

#### オンボーディングの定義
オンボーディングは、新入社員が職場環境や業務にスムーズに適応できるようサポートするプロセス。

#### 目的（2軸）
1. 新入社員が早期に業務を遂行できる状態を目指す
2. 企業文化や価値観への理解を深め、一体感を醸成する

#### 期間
- 期間の目安：**3ヶ月〜1年**
- 段階的進行：**プレオンボーディング → 初期トレーニング → フォローアップ**

### ステップ2：オンボーディングの重要性3観点で目的を明確化する

| 観点 | 内容 |
|---|---|
| 離職率の低下 | 新入社員は慣れない環境にストレスを感じやすく、適応に失敗すると早期離職に繋がる可能性が高い。段階的に適応するためのサポートを受けることで、不安を減少させ、新入社員の初期離職を防ぎ、長期定着を促進させる |
| 業務効率の向上 | 新入社員が実際に成果を出せるようになるまでには時間がかかる。オリエンテーションやOJTを通じて業務内容を体系的に学び、スムーズに業務に取り組めるようになる。e-learningを活用することで自分のペースで学習し早期に戦力化できる |
| 企業文化の浸透 | 新入社員が企業文化を理解しないまま働くと、組織と個人の価値観が乖離し、エンゲージメントが低下する。オンボーディングで組織の価値観・行動規範・ビジョンを明確に共有することで、新入社員はチームの一員としての一体感を強化することができる |

### ステップ3：オンボーディング設計の流れ3項目を決める

| 項目 | 内容 |
|---|---|
| 1. 目標設定 | 短期（3か月以内での業務遂行）と長期（1年以内の完全定着）を明確化。各タイミング（入社時・1ヶ月・2ヶ月・3ヶ月）でのプロセスKPI／成果KPIを設定 |
| 2. プログラム期間の決定 | 職種やスキルレベルに応じた期間設定（例：未経験中途は6ヶ月、即戦力中途は3ヶ月、新卒は12ヶ月） |
| 3. プログラム内容の設計 | プレオンボーディング → 初期トレーニング → 定期フォローアップの3段階構成 |

### ステップ4：オンボーディングに含めるべき要素3つを設計する

| 要素 | 具体内容 |
|---|---|
| 要素01：文化的要素 | 企業のビジョン、ミッション、行動指針を共有し、会社の方向性を理解してもらう／職場でのルールや期待される行動を具体的に伝える |
| 要素02：実務的要素 | 職務内容や業務フローを明確にし、初期トレーニングを実施／CRMや社内ITツールの使い方を教えるトレーニングを提供 |
| 要素03：社交的要素 | チームビルディング（社内交流の場づくり）／他部署との交流（横断的なネットワーク構築） |

3要素はバランスよく組み込むことが重要。文化偏重で実務が手薄／実務偏重で社交がないなど、欠けると課題（孤立／戦力化遅延／離職）に直結する。

### ステップ5：3段階のプログラム内容を具体化する

| 段階 | タイミング | 内容例 |
|---|---|---|
| プレオンボーディング | 内定〜入社前 | 入社前の事前情報共有（会社概要／チームメンバー紹介／読んでおく資料／PCセットアップ案内）、入社前面談 |
| 初期トレーニング | 入社初日〜1ヶ月 | オリエンテーション（企業文化／ビジョン／ルール）、職務内容説明、業務フロートレーニング、ITツールトレーニング、メンター紹介、初期タスク割り振り |
| 定期フォローアップ | 1ヶ月〜数ヶ月以降 | 週次1on1（メンターまたは上長）、月次振り返り、3ヶ月／6ヶ月／12ヶ月時点の評価ミーティング、キャリア相談 |

### ステップ6：成功事例3社の知見を参考にする

| 企業 | 取り組み |
|---|---|
| Google | バディ制度を活用し、スムーズな業務定着を支援／初日のオリエンテーションで企業文化を徹底的に共有 |
| Amazon | 実務を重視したプログラムを設計し、リーダーシップ行動基準（Leadership Principles）を基にトレーニングを提供 |
| 知足（chisoku） | 入社後3か月間のスキル・コミュニケーションオンボーディング設計／メンターによる週次の1on1で進捗をフィードバック |

※事例は参考であり、自社規模・文化・職種特性を踏まえて取捨選択する。Google／Amazonは大企業向けの大規模リソース投入型なので、SMBは知足（chisoku）型（メンター週次1on1中心）から始めるのが現実的。

### ステップ7：3課題と解決策をプログラムに組み込む

| 課題 | 解決策 |
|---|---|
| 課題01：定型化されていないプロセス | 標準的なオンボーディング計画書を作成し、全新入社員に適用する |
| 課題02：新入社員の孤立 | バディ制度やネットワーキングイベントを強化する |
| 課題03：トレーニングと実務の乖離 | トレーニング内容を現場の実務に即した内容に更新する（半期ごとに現場ヒアリングしてアップデート） |

### ステップ8：成功のポイント3つで自己点検する

最後に下記3ポイントで設計が完成しているか自己点検する。

1. プレオンボーディングからフォローアップまで、一貫性のあるプログラム設計になっているか
2. 新入社員の状況（経験／スキル／パーソナリティ）に応じた柔軟な対応ができるか
3. 継続的な改善を行い、オンボーディングの質を向上させる仕組みがあるか（半期ごとのプログラム見直し／離職データ分析）

### ステップ9：効果測定の4観点を設定する

オンボーディング終了後、以下の4観点で効果測定し、次回以降の改善につなげる。

1. **目標設定**：短期目標と長期目標が明確か？達成度は？
2. **プログラム内容の充実度**：文化的・実務的・社交的要素がバランスよく含まれているか？
3. **新入社員へのサポート体制**：メンター制度やフォローアップが効果的に機能しているか？
4. **効果測定**：オンボーディング終了後に成果を測定し、次回以降の改善につなげられているか？

## 判断基準

| 状況 | 判断 |
|---|---|
| 新卒・未経験中途中心 | 期間6〜12ヶ月、3段階すべてを手厚く設計 |
| 即戦力中途・経験者中心 | 期間3〜6ヶ月、初期トレーニング短縮しフォローアップ重視 |
| 同時入社が複数名 | 集合オリエン＋同期コミュニティ形成（社交的要素を強化） |
| 同時入社が単独 | バディ制度／メンター制度を必須化（孤立防止） |
| 早期離職率が高い | 課題02「新入社員の孤立」を最優先で対策／3ヶ月以内のフォローアップ頻度を上げる |
| 戦力化が遅い | 課題03「トレーニングと実務の乖離」を最優先で対策／実務OJT比率を上げる |
| 文化浸透が弱い | 文化的要素を初期トレーニングの前半に集中投下／経営層との対話セッション設定 |
| クライアントワーク標準化（顧客向け）と混同しそう | `delivery-build` に切り替え（顧客向けはこのスキルではない） |
| 採用フェーズの設計が必要 | `recruitment-strategy` を先に実施してから本スキルに戻る |
| 入社後のキャリアパス設計まで踏み込みたい | `career-roadmap-development` に引き継ぐ |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 初日のオリエンだけで以降は放置 | 3段階（プレ／初期／フォロー）を必ず設計し、フォローアップを継続 | 早期離職の主因は「初日後の放置」 |
| 文化的要素を口頭説明だけで済ます | ビジョン・行動指針をドキュメント化し繰り返し共有／経営層との対話セッションを組み込む | 文化は1回の説明では浸透しない |
| 実務的要素を「OJTで覚えろ」で片付ける | 業務フロー・ITツール使い方を体系化したドキュメント＋トレーニングで提供 | 属人化と教育格差の原因 |
| 社交的要素を「自然に馴染むだろう」と省略 | バディ制度／チームランチ／他部署交流イベントを意図的に組み込む | 孤立は早期離職に直結 |
| Google/Amazon事例をそのまま真似る | 自社規模・文化・職種特性に合わせて取捨選択（SMBは知足（chisoku）型から） | 大企業向けプログラムをSMBに適用するとリソース不足で形骸化 |
| 効果測定をせず毎回同じ内容で運用 | 3ヶ月／6ヶ月／12ヶ月の指標で効果測定し、半期ごとに改善 | 課題03「実務との乖離」が再発する |
| 採用戦略と混同する | 採用は `recruitment-strategy`、入社後は本スキルで切り分け | スコープが拡散する |

## 連携スキル

- **前段（採用フェーズ）**：`recruitment-strategy`（採用戦略・採用ガイドライン）
- **並走（指導法）**：`teaching-coaching-leading`（ティーチング／コーチング／リーディングの使い分け）
- **後段（中長期育成）**：`career-roadmap-development`（キャリアロードマップと育成計画）
- **後段（評価制度）**：`evaluation-system-design`
- **後段（給与決定）**：`salary-range-design`
- **関連（恒久的職位定義）**：`roles-responsibilities`
- **関連（会議体設計）**：`meeting-cadence-design`（1on1・評価ミーティングの会議体定義）

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存を持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし運用上のベストプラクティスとして、入力情報不足の場合は `[SKIP]` で「{{要素}}が未確定のため設計スキップ」と明示し、推測で埋めない。
