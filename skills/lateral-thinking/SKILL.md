# lateral-thinking スキル

ラテラルシンキング（Lateral Thinking）= 既存の枠にとらわれず、柔軟な発想で問題を解決する思考法（水平思考）。事業戦略スクール「知足」のフレームワーク準拠。

## ロジカルシンキングとの違い

| 観点 | ラテラルシンキング（水平思考） | ロジカルシンキング（垂直思考） |
|---|---|---|
| 進む方向 | 多面的に観察し、選択肢を発想する | まっすぐ掘り下げ、絞り込む |
| 特徴 | 飛躍的・非線形的 | 線形的・段階的 |
| 強み | 新しい視点でのアイディア創出／斬新な解決策や商品開発 | 論理的で一貫性のある結論を導く／データ分析やプロセス整理 |

両者は**バランスをとって併用**する。ラテラルで広げて、ロジカルで絞る。

## 適用条件

- このスキルを使う場面：
  - 論理的に詰めても突破口が見えない問題
  - 競合と差別化された新規アイデアが必要（新商品・新サービス・新キャンペーン）
  - 業務改善で常識を疑う必要がある
  - ブレストで「とりあえず」のアイデアを質と幅で広げたい
- このスキルを使わない場面：
  - データに基づき結論を絞り込みたい → `logical-thinking`
  - 既存仮説の妥当性検証 → `critical-thinking`
  - 競合不在を理由にした楽観判断のチェック → `competitive-absence-audit`
  - 複数アイデアからの意思決定 → `decision-making-framework`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象課題 | 必須 | 発想を広げたい問題・テーマ | 「売上を上げる方法を考えたい」 |
| 既存の前提 | 任意 | 現在の取り組み・暗黙のルール（疑う対象になる） | 「店舗で売る／対面で営業／顧客は若年層」 |
| 制約条件 | 任意 | 考慮すべき制約（予算・期間・法規制） | 「予算100万円以内」 |
| アウトプット数 | 任意 | 出したいアイデア数（無指定なら7〜10案） | 「20案出して」 |

## 3つの特徴と3つの基本アプローチ

### 3つの特徴
1. **非論理的アプローチ**：論理的でなくても、自由な発想を優先
2. **既存の枠組みを疑う**：常識や前提を意図的に打破
3. **複数の解決策を模索**：単一解決より、多様な選択肢を探る

### 3つの基本アプローチ

| アプローチ | やり方 | 例 |
|---|---|---|
| **問題の再定義** | 問題を新しい角度で捉え直す | 「売上を上げる方法」→「顧客満足度を高める方法」 |
| **前提を疑う** | 既存の考えを疑い、新たな可能性を探る | 「店舗で売るべき」を捨て、オンライン販売を検討 |
| **発想の飛躍** | 異なる分野のアイデアを適用 | IT技術をレストラン業に活用→スマートメニュー導入 |

## 3つの具体的手法

### 手法1：逆転発想
**問題を逆に考える**。
- 例：「顧客を増やす方法」→「顧客が減る理由を探す方法」
- 例：「売れる商品を作る」→「絶対に売れない商品の特徴を出して、それを反転する」

### 手法2：強制結合法
**無関係な要素を結びつける**。
- 例：「自動車」×「レストラン」＝ 移動型レストラン（フードトラック）
- 例：「保険」×「ゲーム」＝ 健康行動でポイントが貯まる保険
- 例：「銀行」×「カフェ」＝ 相談窓口併設型カフェ店舗

### 手法3：SCAMPER法（7視点チェックリスト）

| 視点 | 意味 | 問いかけ |
|---|---|---|
| **S**ubstitute（置換） | 何かに置き換える | 素材／人／場所を別のものに替えると？ |
| **C**ombine（結合） | 何かと組み合わせる | 別の商品・サービスと組み合わせると？ |
| **A**dapt（適応） | 既存のものを応用する | 他業界の成功事例を自社に適応すると？ |
| **M**odify（変更） | 大きさ・形・色を変える | サイズ／パッケージ／タイミングを変えると？ |
| **P**ut to another use（転用） | 別用途に使う | 別のターゲット・用途で売るとどうなる？ |
| **E**liminate（削除） | 何かを取り除く | 機能・工程・パーツを削るとシンプル化できる？ |
| **R**everse（逆転） | 順序や方向を逆にする | 提供順序・支払いタイミングを逆にすると？ |

## 実行ステップ

### ステップ1：既存の課題を再定義する
- 入力：対象課題＋既存の前提
- 処理：
  1. 元の課題文をそのまま受け取る
  2. **問題の再定義**：少なくとも3つの異なる角度で再定義する
     - 例：「売上を上げる方法」→「顧客満足度を高める方法」「離脱を減らす方法」「リピートを増やす方法」「新規層を開拓する方法」
  3. 最も発想が広がりそうな再定義を1つ選ぶ（複数並行も可）
- 出力：再定義された問い（複数）

### ステップ2：前提を疑い、解決策を考える
- 入力：再定義された問い＋既存の前提リスト
- 処理：
  1. 暗黙の前提を5つ以上書き出す（「店舗で売る」「対面で営業」「BtoB」「日本市場」「価格は競合と同等」等）
  2. それぞれの前提を**否定**して、新たな可能性をブレストする
     - 「店舗で売る」を否定 → ECオンライン／無人販売／サブスク／訪問販売／代理店
  3. 否定したときに浮かぶアイデアを各前提あたり2〜3案出す
- 出力：前提否定アイデアリスト（10〜15案）

### ステップ3：手法を活用して新しいアイデアを考える
- 入力：再定義された問い＋前提否定アイデア
- 処理（3つの手法を順に適用）：
  1. **逆転発想**：問題を逆に考えて2〜3案
  2. **強制結合法**：無関係な要素3つ（ランダム選定）×対象を組み合わせて2〜3案
  3. **SCAMPER法**：7視点を1つずつ適用し、各視点で1〜2案（合計7〜14案）
- 出力：手法別アイデアリスト（合計15〜20案）

### ステップ4：チームでブレーンストーミング（任意・1人実行時はスキップ）
- 入力：これまでのアイデアリスト
- 処理：
  1. 同僚・仲間に共有して追加アイデアを募る
  2. ブレストルール厳守：①批判禁止 ②質より量 ③突飛な案歓迎 ④便乗OK
  3. 出てきたアイデアをすべて記録する
- 出力：拡張アイデアリスト

### ステップ5：アイデアの整理と評価（出力前の最終整理）
- 入力：全アイデアリスト
- 処理：
  1. 重複・類似を統合する
  2. 4軸（**実現性／革新性／コスト／インパクト**）で各案を1〜5点評価
  3. 「すぐ試せる案（実現性高×コスト低）」「化ける可能性のある案（革新性高×インパクト高）」を分けてラベル付け
  4. **ロジカルシンキングへの引き継ぎ**：上位5案を `logical-thinking` または `decision-making-framework` で絞り込み可能な形に整理
- 出力：評価済みアイデアテーブル＋推奨次アクション

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| アイデアが10案未満で止まる | 強制結合法でランダム要素をさらに3つ追加して再ブレスト | ラテラルでは「量が質を生む」 |
| すべてのアイデアが現実的すぎる | SCAMPERの「Reverse」「Eliminate」を強めて再試行 | 現実的すぎるラテラルは「水平思考」になっていない |
| すべてのアイデアが非現実すぎる | 評価ステップで実現性で絞る／`logical-thinking` 連携 | ラテラルは「絞る」のは別工程 |
| 同じ前提を複数回否定している | 前提リストを5つ以上に増やす | 否定の対象が偏っている |
| 「無関係要素の選定」に困る | ランダムワード生成（業界／物体／動詞）を強制 | 関連が見える要素を選ぶと水平思考にならない |
| 顧客視点が抜けている | 「ターゲット顧客の代理発想」を1セッション挿入 | ペルソナ視点で再ブレスト |
| 競合不在を根拠にした楽観案 | `competitive-absence-audit` を併用 | CLAUDE.md「Competitive Absence Audit Rule」遵守 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 前提を疑わずに既存の延長案ばかり | 前提リストを5つ以上書き出して全否定する | 前提を温存すると水平思考にならない |
| SCAMPER 7視点のうち得意な3〜4個だけ使う | 必ず7視点すべて1案以上出す | 苦手視点こそ斬新な案が出る |
| 強制結合の要素が「関連業界」 | ランダムに無関係要素を選ぶ | 関連性ある組合せは予測できて陳腐 |
| 1人で完結させる | 可能ならチームに展開（ステップ4） | 集合知が量を増やす |
| 出した案をすぐ実現性で却下 | 評価はステップ5で行い、ステップ3では出し切る | 早期評価が発想を殺す |
| 「ロジカルシンキングと逆だから論理は無視」 | 出した案は最終的にロジカル／批判的検証で絞る | ラテラル単独では実行に進めない |
| ブレスト中に批判する | 批判禁止ルール厳守 | 批判は発散の天敵 |

## 連携スキル

- `logical-thinking`：出したアイデアを論理的に絞り込む（ステップ5の後段）
- `critical-thinking`：採用候補アイデアの前提・根拠を批判的に検証
- `decision-making-framework`：複数アイデアからの意思決定（重みづけスコアリング）
- `competitive-absence-audit`：「ブルーオーシャン」「競合いない」系のアイデアが出たときの強制チェック
- `pmf-journey`：化ける可能性のある案のフィット段階診断
- `pyramid-structure`：採用案を提案資料化する場合
- `business-model-canvas`：採用アイデアを9要素で事業モデル化

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：アイデアが1案も出なかった場合（再定義に戻る）
- `[PARTIAL:N成功/M失敗]`：手法別アイデア出しで一部の手法（例：強制結合法）でアイデアが出なかった場合
- `[SKIP]`：チームブレスト（ステップ4）を1人実行のため省略した場合
