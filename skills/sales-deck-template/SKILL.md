# 営業資料テンプレートスキル

商談創出を目的とした「概要資料」を中心に、24ページ標準構成で営業資料を設計する。提案資料・補足資料との使い分けも明示。商談化前の初回MTG用資料に特化。

## 適用条件

- 概要資料（サービス紹介資料）の構成を作りたい
- 既存資料が情報過多・自社目線・デザイン不統一で見直したい
- 営業資料の3種類（概要／提案／補足）を整理したい

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| 商材 | 必須 | サービス・製品名 |
| 営業フェーズ | 任意 | 初回MTG用 / 提案用 / 補足用 |

## 実行ステップ

### ステップ1: 営業資料3種類の使い分け

| 種類 | 目的 | 記載内容 | 営業フェーズ |
|---|---|---|---|
| **概要資料** | 商談の創出 | 自己紹介ページ／サービス概要／製品カタログ／会社概要／事業全体概要／FAB（特徴・利点・利益） | 初回MTG〜商談化 |
| **提案資料** | 商談の受注 | 要件（問題と課題）／解決策／経済的合理性訴求／スケジュール／価格／サポート体制／リスク | 商談化〜提案 |
| **補足資料** | 買い手の組織内合意形成 | 技術資料／ROI資料／業務フロー変更資料／プロダクト説明資料／事例 | 提案〜受注 |

### ステップ2: 面談の目的と構成（概要資料の使い方）

- **目的**: 商談を創出すること（商売の話に進むことを合意してもらう）
- **構成3パート**:
  1. 信用獲得・信頼構築 → 会社概要／自己紹介／事業背景と実績
  2. 課題整理・課題特定 → 問題の確認／原因および課題の仮説提示／原因整理と解決方法議論
  3. 商売の話への合意形成 → 自社プロダクトの"概要"説明／進め方の提示（提案前後の流れ）／次回打合せの合意

### ステップ4: その他Tipsチェックリスト

- ☐ 表紙は毎回相手に合わせて作る（社名・ロゴ反映）
- ☐ 意味のない画像は使わない
- ☐ 色を多用しない（基本2色＋アクセント1色）
- ☐ 1スライド1メッセージを守る
- ☐ 主従関係を意識（主＝会話、資料は従）

### ステップ5: よくある課題チェック

| 課題 | 原因 | 解決ヒント |
|---|---|---|
| 情報過多 | スペックや実績を詰め込みすぎ | 1スライド1メッセージ厳守 |
| 顧客視点欠如 | 自社目線 | 「よくある悩み」「事例」を顧客視点で記述 |
| デザイン一貫性なし | 雑多な見た目 | ロゴ・カラーを統一、テンプレ流用 |

### ステップ6: 自社事業の営業資料作成4ステップ

1. **目的とターゲット明確化** — 目標設定（新規獲得 or 既存追加）／対象顧客（業界・役職・課題）
2. **情報整理と構成設計** — 必要情報洗い出し／目次・ストーリーライン作成（課題→提案→解決策→実績）
3. **デザインとビジュアル化** — 図表・アイコン活用／ブランド要素統一／簡潔な表現
4. **レビューと改善** — 顧客目線確認／社内フィードバック／実営業場面でテスト

## 判断基準

| 状況 | 判断 |
|---|---|
| 商談がまだ確保できていない | 概要資料を作成 |
| 商談で相手の課題が明確 | 提案資料に進む（→`proposal-builder`） |
| 決裁者向けの追加説明が必要 | 補足資料（ROI資料／事例集）を準備 |
| 24ページが多すぎる | 必須12ページ（1〜9, 13, 16, 20）に絞る |

## 連携スキル

- スライド設計: `slide-create`
- 価値提案明確化: `value-proposition`
- 顧客課題分析: `loss-analysis-kbf-ksf`

## 5状態出力契約
- `[FAIL]`: 商材未指定
- `[OK:0件]`: 既存資料情報なし＋ヒアリング情報なしで構成不能
