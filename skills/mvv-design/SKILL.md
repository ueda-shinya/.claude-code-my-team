# mvv-design スキル

MVV設計 = Mission（ミッション・存在意義）／Vision（ビジョン・理想的な未来像）／Value（バリュー・価値観や行動基準）の3要素で企業が目指す方向性を明確にする。事業戦略スクール「知足」のフレームワーク準拠。

## MVVの定義

| 要素 | 定義 | 例 |
|---|---|---|
| **Mission（ミッション）** | 組織の存在意義や目的を示す | Google：「世界中の情報を整理し、誰もがアクセスできるようにする」 |
| **Vision（ビジョン）** | 組織が目指す理想的な未来像 | スタートアップ：「地域の課題を解決し、すべての人が安心して暮らせる社会を作る」 |
| **Value（バリュー）** | 組織が大切にする価値観や行動基準 | 「顧客中心主義」「挑戦を恐れない」「誠実さを持つ」 |

**3要素のピラミッド構造**：上から Mission（存在意義）→ Vision（未来像）→ Value（行動基準）。Mission が最上位の理由であり、Vision が方向性を示し、Value が日々の行動を規定する。

## なぜMVV設計が重要なのか？（3つの目的）

| 目的 | 内容 |
|---|---|
| **目的① 組織の目標達成を支援** | 中長期的な目標設定や戦略立案の土台となる |
| **目的② 従業員の一体感を醸成** | 共通の価値観と目標が、個人と組織を結びつける |
| **目的③ 企業文化の形成** | 一貫性のある行動基準が、企業文化を強化する |

## 適用条件

- このスキルを使う場面：
  - 創業時や事業フェーズ転換期に企業理念を言語化したい
  - MVVが古くなり時代に合わなくなったので刷新したい
  - 経営理念はあるが社内浸透していない／実践と乖離している
  - 採用や顧客獲得でブランド力を高めたい
- このスキルを使わない場面：
  - 中長期事業計画の策定（経営戦略・KPI） → `medium-term-business-plan`
  - 文化の言語化（既にあるカルチャーの文字化） → 別スキル
  - 目標を年/四半期/月に分解 → `goal-hierarchy-design`
  - ブランドアイデンティティ設計 → `branding`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 会社・組織情報 | 必須 | 会社名／業種／規模／創業年／フェーズ | 「Web制作会社／創業3年／20名」 |
| 既存MVV（あれば） | 任意 | 既存ミッション／ビジョン／バリュー | 「ミッション：未設定」「バリュー：誠実、スピード」 |
| 強み・特徴 | 任意 | 自社の強み・差別化要因 | 「マーケティングと制作の一気通貫」 |
| 想定顧客・社会的役割 | 任意 | 誰のどんな課題を解決しているか | 「中小企業のWeb集客課題」 |
| 経営層の関与度 | 任意 | 経営層が積極関与／部分関与／不在 | 「経営層フル関与」 |

## MVV設計の5ステッププロセス

### STEP 1：現状分析
**組織の強み・弱み、外部環境の変化を把握する**
- 手法例：
  - SWOT分析を実施（→ `swot-analysis` 連携）
  - 社員アンケートで「自社らしさ」を抽出（10〜20名のキーパーソン）
  - 顧客インタビューで「自社が選ばれる理由」を確認（→ `loss-analysis-kbf-ksf` 連携）
  - PEST分析で時代変化を加味（→ `pest-analysis` 連携）
- アウトプット：自社の現状像（強み・弱み・外部環境）

### STEP 2：ミッションの定義
**なぜ自社が存在するのかを明確にする**
- 質問例：
  - 私たちは何のために存在するのか？
  - 自社がなくなったら誰が困るか？
  - 創業者の原点（なぜこの事業を始めたのか）は何か？
- 言語化のコツ：
  - 主語は「自社」、動詞は「〜する」（行動を示す）
  - 抽象すぎず具体すぎず（業界全体の存在意義）
  - 「永続的に追い続ける目的」になっているか
- アウトプット：Mission文（1〜2文・短く）

### STEP 3：ビジョンの策定
**組織が目指す理想的な未来を描く**
- 質問例：
  - 5年後、10年後に私たちはどんな存在でありたいか？
  - 我々が事業で達成したら社会・業界はどう変わっているか？
  - 競合に対してどんなポジションでありたいか？
- 言語化のコツ：
  - 期限を明示（5年後・10年後など）または達成イメージを描く
  - ワクワクする・社員が共感できる
  - 数値を含めても良い（売上・シェア・規模）が、数値だけにしない
- アウトプット：Vision文（1〜2文）

### STEP 4：バリューの設定
**組織全体で共有する価値観や行動基準を策定**
- 質問例：
  - どのような考え方や行動を大切にするか？
  - 採用で見る／褒める／叱る基準は何か？
  - 社員が日常で意識すべき行動原則は？
- 設計のコツ：
  - 3〜5個（多すぎると守れない）
  - 各バリューは「行動レベルで具体化」できるか
  - 一般論（誠実・挑戦）でなく、自社らしい表現に
  - 「やらないこと」を含めても良い
- アウトプット：Value 3〜5項目（各項目に1行説明）

### STEP 5：社内外への共有
**社内ワークショップや外部への発信を通じて、MVVを浸透させる**
- 社内浸透：
  - 全社説明会（経営層が直接語る）
  - 部署ごとの自分ごと化ワークショップ
  - 採用要件・評価制度・昇進基準への組み込み
  - 入社オリエンテーション・社内報・壁面掲示
- 社外発信：
  - 公式サイト・採用ページ・会社概要資料への掲載
  - 顧客提案書・営業資料への組み込み
  - SNS・代表メッセージ・PR記事
- アウトプット：浸透計画＋媒体別アウトプット案

## 実行ステップ

### ステップ1：現状分析（STEP 1）
- 入力：会社・組織情報、既存MVV、強み、顧客像
- 処理：
  1. 入力情報を整理し、不足項目をユーザーに追加質問
  2. SWOT・PEST・社員/顧客の声 等から「自社らしさ」要素を10〜15個抽出
  3. 競合と比較して「うちならではの要素」3〜5個に絞る
- 出力：現状分析レポート＋「自社らしさ」キーワード集

### ステップ2：ミッション案を3案作成（STEP 2）
- 入力：「自社らしさ」キーワード
- 処理：
  1. 「私たちは [対象] に対して [価値] を提供する」の型で複数案
  2. 抽象度を変えて3案（広い社会的意義／業界での意義／顧客への意義）
  3. 各案について「10年後も追い続けられるか」「他社でも言えないか」をチェック
- 出力：Mission候補3案＋採用推奨1案＋根拠

### ステップ3：ビジョン案を3案作成（STEP 3）
- 入力：採用Mission案
- 処理：
  1. 期限を3パターンで設定（3年後／5年後／10年後）
  2. 各期限で「達成された世界」の具体像を描く
  3. 各案について「ワクワク感」「具体性」「実現可能性」を評価
- 出力：Vision候補3案＋採用推奨1案＋根拠

### ステップ4：バリュー案を5〜7項目から3〜5項目に絞る（STEP 4）
- 入力：採用Mission・Vision案＋自社らしさキーワード
- 処理：
  1. バリュー候補を5〜7項目挙げる（行動レベル）
  2. 重複・類似を統合し、3〜5項目に絞る
  3. 各バリューに「具体的な行動例」を1〜2個ずつ付ける
- 出力：Value 3〜5項目＋各項目の1行説明＋行動例

### ステップ5：成功ポイントと課題の事前チェック
- 入力：MVV案セット
- 処理：以下の3成功ポイント＋3課題で点検する

#### 3つの成功ポイント
| ポイント | チェック内容 |
|---|---|
| POINT 1：経営層のコミットメント | 経営陣がMVVを実践する宣言・行動計画があるか |
| POINT 2：シンプルでわかりやすい表現 | 短く具体的な言葉で記載されているか／専門用語に逃げていないか |
| POINT 3：従業員の意見を反映 | 社員の声が反映されているか／一方的に押し付けていないか |

#### 3つの課題と解決策
| 課題 | 解決策 |
|---|---|
| 抽象的すぎる内容 | 実例や具体的な行動基準を盛り込む |
| 社内浸透が不十分 | 社内説明会や日常業務での実践を促進 |
| 時代に合わなくなる | 1〜3年ごとにMVVを見直し、柔軟にアップデートする |

- 出力：チェック結果＋修正案

### ステップ6：浸透計画と発信媒体設計（STEP 5）
- 入力：確定MVV
- 処理：
  1. 社内浸透：説明会／ワークショップ／評価制度組込み の3軸
  2. 社外発信：公式サイト／採用ページ／営業資料／SNS の4軸
  3. 各アウトプットの担当者・期限を仮決め（実行はユーザー側で確定）
- 出力：浸透計画シート

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| Mission案が「業界全体の意義」レベルで自社特有性がない | キーワードを絞り込み再起案 | 他社でも言える Mission は機能しない |
| Vision案に期限・達成像がない | 期限と達成像を明示する | 抽象的すぎて社員がイメージできない |
| Value 6項目以上 | 3〜5項目に統合 | 多すぎると守れない・行動原則にならない |
| Value が一般論（「誠実」「挑戦」のみ） | 自社らしい言い回しに変換 | 一般論は競合と差別化できない |
| 経営層の関与が部分的 | 経営層フル関与の体制を提案 | コミットなき MVV は形骸化する（POINT 1 違反） |
| 既存MVVがあるが浸透していない | ステップ1〜4は軽め、ステップ6（浸透計画）を厚く | 文言問題ではなく運用問題のため |
| 創業3年未満 | Mission・Value 重視、Vision は3年後設定で軽めに | 創業期は Vision を未来固定すると変化に追従できない |
| 大企業（100名以上） | 部署別バリューや事業別ビジョンを併設可 | 規模により全社一律のMVVが機能しない場合がある |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 経営層不在で人事や広報が単独策定 | 経営層フル関与を必須化 | 経営層が実践しないMVVは形骸化（POINT 1） |
| 「誠実・挑戦・成長」等の一般論バリュー | 自社らしい具体表現に変換 | 一般論は他社と差別化できない |
| Vision に期限・達成像なし | 期限と達成像を明示 | 抽象的Visionは社員がイメージできない |
| Mission を「売上目標」と混同 | Mission は「存在意義」、目標は別途設計 | Mission は永続、目標は期限付き |
| Value を6項目以上 | 3〜5項目に絞る | 多すぎると守れず形骸化 |
| 策定後に発信せず社内告知のみ | 社外発信4軸（HP・採用・営業資料・SNS）も設計 | 採用力・ブランド力に直結 |
| 1度作ったら永久不変 | 1〜3年ごとに見直しサイクル設定 | 時代変化に追従しないと陳腐化 |
| 社員ヒアリングなし | 10〜20名のキーパーソンに必ずヒアリング | 押し付けMVVは浸透しない（POINT 3） |

## 連携スキル

- `swot-analysis`：現状分析（ステップ1）の自社強み・弱み・外部環境の整理
- `pest-analysis`：時代変化を加味した外部環境分析
- `loss-analysis-kbf-ksf`：顧客インタビューで「選ばれる理由」を抽出
- `medium-term-business-plan`：MVVを土台に中長期事業計画を策定
- `goal-hierarchy-design`：MVV → 年間目標 → 四半期 → 月間 への階層分解
- `kgi-kpi-kai-design`：MVV達成度を測るKPI設計
- `branding`：MVV発信のブランディング戦略接続
- `persona-design`：MVVの想定顧客（ペルソナ）設計
- `pyramid-structure`：Mission→Vision→Value のピラミッド構造で説明資料化

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：自社らしさキーワードが抽出できない場合（ステップ1に戻り情報追加）
- `[PARTIAL:N成功/M失敗]`：Mission・Vision・Value のうち一部しか確定できなかった場合
- `[SKIP]`：経営層関与が確認できず、ステップ6（浸透計画）を仮置きで進めた場合
