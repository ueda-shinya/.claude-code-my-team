# growth-phase-strategy スキル

企業の成長フェーズ（初期／成長／拡大／成熟）を年商規模・組織状態・経営課題で診断し、フェーズ別に求められる組織体制・権限移譲・役職の変化と、フェーズを乗り越えるための7ポイントを体系設計する。事業戦略スクール「知足」フレームワーク準拠。

## 適用条件

- このスキルを使う場面：
  - 自社が今どの成長フェーズにあるか診断したい
  - フェーズに応じた組織体制・権限移譲・役職を設計したい
  - 「年商◯億の壁」を越える戦略を立てたい
  - 次フェーズに進むために必要な変革を特定したい
- このスキルを使わない場面：
  - 組織計画・組織図設計の詳細 → `organization-planning` を使う
  - 役割と責任範囲の明確化 → `roles-responsibilities` を使う
  - 採用戦略 → `recruitment-strategy` を使う
  - MVV策定 → `mvv-design` を使う
  - キャッシュフロー診断 → `financial-statements-fundamentals` を使う
  - 中長期事業計画 → `medium-term-business-plan` を使う
  - 資金調達戦略 → `financing-strategy` を使う

## 入力

| 項目 | 必須/任意 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガーフレーズのいずれか |
| 自社の年商規模 | 必須 | 「1億」「3億」「12億」等 |
| 社員数 | 必須 | 「8名」「30名」「120名」等 |
| 業種・ビジネスモデル | 任意 | フェーズ判定の補正に使用 |
| 現状の経営課題 | 任意 | フェーズ診断の参考 |

## 実行ステップ

### ステップ1：4成長フェーズの全体像を理解する

**経営ストラテジーの変化**：
従業・売上の促進 → 組織の仕組み化 → 組織文化の定着 → コントロールの高度化

**2軸の進化**：
- 組織体制の進化（会社の格付け・権限移転の実施）
- 人材マネジメントの変化（人材ハイアリングや人事システムの進化）

### ステップ2：年商規模で自社のフェーズを判定する

| フェーズ | 年商規模 | 経営課題 | 組織の状態 |
|---|---|---|---|
| ①初期フェーズ | 1億円未満 | とにかく売上を作る | ほぼ社長1人 or 少数精鋭 |
| ②成長フェーズ前期 | 1億〜5億円 | 組織を作る・事業の安定化 | 小規模組織（5〜20名） |
| ②成長フェーズ後期 | 5億〜10億円 | 仕組み化と幹部育成 | 中規模組織（20〜50名） |
| ③拡大フェーズ前期 | 10億〜30億円 | 組織文化の確立 | 部門ができ始める（50〜100名） |
| ③拡大フェーズ後期 | 30億〜50億円 | 経営の高度化 | 100名以上、社内政治が発生 |
| ④成熟フェーズ | 50億〜100億円 | 大企業化・社会的影響力の強化 | 200名以上、グローバル展開 |

**注意**：フェーズに対する年商規模・課題・組織状態はあくまでも目安。ビジネスモデルによって各項目は大きく変わる。

### ステップ3：フェーズ別の組織図・権限移譲・役職を診断する

#### ①初期フェーズ（1億円未満）：非常にフラットで少人数
- **組織図**：CEO（全意思決定と業務）／スタッフ（営業・開発・マーケ全兼任）
- **権限の移譲**：組織全体の権限はCEOが一手に管理。スタッフは実務遂行に集中
- **役職の変化**：経営者中心、スタッフが幅広い業務、役職区分なし、ほとんどが同じ立場

#### ②成長フェーズ（1〜10億円）：部門制が導入され複数の役職が登場
- **組織図**：CEO（戦略的方向性）／COO・部門長（事業運営の全体管理）／部門長（営業・マーケ・開発）／スタッフ（各部門実務）
- **権限の移譲**：CEOは全体戦略に専念、部門長に各部門権限を委譲。部門長は業務運営・目標設定・リソース配分担当
- **役職の変化**：「部門長」「マネージャー」役職が新登場。経営者は戦略ビジョン維持、各部門長が実務進行担当

#### ③拡大フェーズ（10〜50億円）：階層が深くなりより細かく役職が分化
- **組織図**：CEO（最終意思決定・戦略）／C-suite（CFO・COO・CTO等、各部門最高責任者）／部門長（営業・マーケ・開発・財務等）／マネージャー（部門内細分化役職）／スタッフ（実務）
- **権限の移譲**：CEOの権限がC-suiteに移譲、各部門でマネージャーに権限分配。部門内意思決定もマネージャーが実施。経営者は全体戦略・ビジョン策定とコミュニケーション
- **役職の変化**：「C-suite」経営層役職が登場。各部門で責任が細分化・多層化。「マネージャー」「リーダー」が登場

#### ④成熟フェーズ（50〜100億円）：複雑な階層構造が完成
- **組織図**：CEO（戦略的ビジョンと最終意思決定）／C-suite（各部門トップ）／部門長／マネージャー（実務管理）／スタッフ（具体業務）
- **権限の移譲**：CEOは最終意思決定に集中。日々の運営はC-suite管理。各部門内でも権限が下位に分配、現場意思決定権が強化。マネージャー・リーダーが進行担当、経営層は大局的方向性決定に集中
- **役職の変化**：役職が細分化・階層構造が複雑化。「副部長」「シニアマネージャー」が増加。役職ごとに専門的な責任範囲が設定

### ステップ4：成長フェーズを乗り越える7ポイントを実行する

| # | ポイント | 内容 |
|---|---|---|
| 01 | 明確なビジョンと戦略 | 全員が同じビジョンに向かう方向性／短期・中期・長期戦略の設定（→ `mvv-design`） |
| 02 | 組織体制と業務フロー整備 | 役割の明確化（業務重複・無駄排除）／業務フロー最適化（→ `organization-planning`／`roles-responsibilities`） |
| 03 | 適切な人材育成 | 採用戦略強化（文化適合人材／スキル補完）／リーダーシップ強化（→ `recruitment-strategy`／`career-roadmap-development`） |
| 04 | コミュニケーション強化 | 透明性のあるコミュニケーション／チーム間協力の仕組み |
| 05 | キャッシュフロー管理 | 安定したCF確保／コスト管理と利益率維持（→ `financial-statements-fundamentals`） |
| 06 | 柔軟な対応とイノベーション | 市場変化への適応／変化を受け入れる文化の醸成 |
| 07 | ガバナンス強化 | 内部統制整備（透明性・監査・リスク管理）／法令遵守と倫理的行動 |

### ステップ5：成長フェーズ別6課題のステップアップを設計する

```
初期段階   組織化      仕組み化   文化形成      高度化         大企業化
（売上     （部門化   （業務     （企業文化と  （戦略的       （社会的責任と
従業員     基盤作り）  標準化）   価値観浸透）   意思決定と     ガバナンス強化）
拡大）                                          リスク管理）
```

各フェーズで取り組む課題：
- **初期段階**：売上・従業員拡大
- **組織化**：部門化・基盤作り
- **仕組み化**：業務標準化
- **文化形成**：企業文化と価値観浸透
- **高度化**：戦略的意思決定とリスク管理
- **大企業化**：社会的責任とガバナンス強化

### ステップ6：自社の現フェーズと次フェーズへの戦略を立案する

3つの問い：
1. 自社がどのフェーズに当てはまるか？
2. 組織図・権限移譲・役割の変化を確認し、自社での取り組みを振り返る
3. 次のフェーズに行くために必要な組織図・権限移譲・役割変化を確認し、具体的な事業戦略を立案（成長フェーズの場合は7ポイントを意識）

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 年商1億未満で社員10名超 | 売上拡大が最優先（人件費負担過大の可能性） | 初期フェーズの組織状態と乖離 |
| 年商5億超で社長一人で全意思決定 | 部門長への権限移譲が急務 | 成長フェーズで権限集中は組織硬直化の主因 |
| 年商10億超で部門長制度なし | C-suite or 部門長制度の導入が必須 | 拡大フェーズの組織体制不在は限界突破阻害 |
| 年商30億超で社内政治が発生 | ガバナンス・コミュニケーション強化（POINT04・07） | 拡大フェーズ後期の典型課題 |
| 急成長中だが利益率低下 | キャッシュフロー管理（POINT05） | 急成長期のコスト膨張対策 |
| 「うちは独自フェーズで一般論は当てはまらない」 | ビジネスモデル特有の補正は許容しつつフェーズ目安は参照 | 楽観判断による戦略遅延リスク |
| 次フェーズへの組織変革に抵抗が強い | POINT06（変化を受け入れる文化醸成）を先行 | 変革抵抗は成長停滞の主因 |
| フェーズ移行期に幹部不在 | POINT03（採用戦略・リーダーシップ強化） | 幹部不在は最大のボトルネック |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 年商規模だけでフェーズ判定 | 年商×社員数×組織状態の3軸判定 | ビジネスモデル次第で同年商でも組織状態が異なる |
| フェーズ移行を先送り（権限委譲しない） | 次フェーズの組織体制を半年〜1年前から準備 | 組織変革は急にはできない |
| 全7ポイントを同時並行で実行 | 次フェーズ準備に必要な3ポイントから優先実行 | リソース分散で全て中途半端になる |
| 急成長中にキャッシュフロー管理を後回し | 急成長期こそPOINT05を最優先 | 黒字倒産リスクが最大化 |
| 拡大フェーズで属人的経営を続ける | C-suite制度導入と権限移譲を実施 | 拡大フェーズ後期の経営限界の主因 |
| 「うちは特殊だから一般論は当てはまらない」 | フェーズ目安を参照しつつ自社特有の補正を加える | 楽観判断は戦略遅延の主因 |
| 幹部育成を怠ったまま拡大フェーズに突入 | 成長フェーズ後期からPOINT03（人材育成）を強化 | 幹部不在は最大ボトルネック |

## 連携スキル

- `organization-planning`：フェーズ別組織図の詳細設計
- `roles-responsibilities`：フェーズ別役職と責任範囲の明文化
- `recruitment-strategy`：フェーズに応じた採用戦略
- `career-roadmap-development`：フェーズに応じた育成計画
- `salary-range-design`：階層が増えた際の給与レンジ設計
- `evaluation-system-design`：フェーズ別評価制度
- `mvv-design`：明確なビジョンと戦略の策定（POINT01）
- `financial-statements-fundamentals`：キャッシュフロー管理（POINT05）
- `financing-strategy`：成長段階に応じた資金調達
- `medium-term-business-plan`：中長期事業計画と整合
- `ma-strategy-basics`：成熟フェーズでのM&A戦略
- `competitive-absence-audit`：「うちは独自フェーズ」等の楽観判断を検証

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし以下のケースで明示マーカーを使うこと：

- `[OK:0件]`：自社の年商規模・社員数等が不明で、フレームワーク説明のみで終了した場合
- `[SKIP]`：競合不在・独自フェーズ等の楽観判断が出た場合は `competitive-absence-audit` への誘導でスキップ

外部API・サブプロセスを持たないため、`[FAIL]`（試行後の失敗）は通常発生しない。
