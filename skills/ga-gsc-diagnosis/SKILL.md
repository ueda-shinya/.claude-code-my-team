# ga-gsc-diagnosis

GA（全体行動）と GSC（検索挙動）を 2段構えで突き合わせ、流入／CV異変の原因を仮説化して対策案まで落とし込む**1次診断**スキル。

## 実行主体

**レン（marketing-planner）**。アスカは発動判定のみ行い、実行はレンに委任する。CLAUDE.md「開発ワークフロー（専門領域はエージェントに委任）」に準拠。アスカ自身のGA4関連タスクは morning-briefing の数値読み上げ＋定型フラグ運用に限定され、マーケティング分析・判断は本スキル経由でレンが行う。

## 他スキルとの役割分担（選択順序ルール）

| 状況 | 使うスキル |
|---|---|
| 流入減・CV減の**1次診断**（原因がまだ分からない状態） | **`ga-gsc-diagnosis`（本スキル）** |
| 1次診断で技術的SEO／インデックス／Core Web Vitals／ペナルティが疑われた場合 | `seo-audit` に切替 |
| morning-briefing のGA4フラグを受けた「詳しく調べて」指示 | **`ga-gsc-diagnosis`（本スキル）** から着手 |
| 記事本文のリライト実行 | `/blog-post` |
| CVR改善そのもの（ファーストビュー・フォーム・オファー設計） | レンのCVR改善フレームワークに切替 |

## 推奨運用フロー（B→A→C の3段階）

初回の単発診断では仮説がデータに対して甘くなりやすい（実運用 2026-04-21 で検証済み）。**深掘り→検証→実装**の3段階で回すと品質が安定する。

**本スキルの責任範囲**: Step 1〜5（診断レポート出力）まで。Step 6（実装委任）と Step 7（効果検証）は**周辺フローの定義**であり、実行主体は各委任先。

### 段階B: 深掘り診断（本スキル本体・既存ステップ1〜6）

- 実行: レン
- 入力: ユーザー指示 + アスカが取得した MCP 基礎データ（両期間の combined_performance_report）
- 出力: 原因仮説＋対策案（1次版）
- **既存ステップとの対応**:
  - **初回診断** = 既存ステップ1〜5（ルート選択〜仮説立案）まで
  - **第2ラウンド** = Step 3 の追加データ取得後、既存ステップ4〜5を再実行（仮説を確度付きで再評価・棄却・新設）
  - **ステップ6（対策案出力）は第2ラウンド完了後に1回のみ実行**
- **重要**: 初回診断で「要追加調査」と判定した項目は、必ず第2ラウンドで MCP `page_analysis` / `gsc_search_analytics (query+page)` で深掘りする。初回だけで結論を出さない
- 第2ラウンドは初回の面子を守る修正ではなく、データに忠実な更新を行うこと

### 段階A: ファクトチェック
- 実行: リク（fact-checker）
- 入力: レンの最終診断レポート＋一次データ（MCP取得JSON、後述の保存パスを渡す）
- 検証観点:
  - 数値計算の整合性
  - 仮説の論理飛躍（特に「全体Organic×%がページXに波及」系の断定）
  - 用語の正確性（「圏外落ち／top10外／インデックス外れ」の混同、GSC position の表記等）
  - 絶対値の小ささを「トレンド」と呼んでいないか
- 出力: 数値判定＋論理判定＋用語判定＋総合所見
- **一次データの受け渡し**: ステップ2〜4で取得した MCP レスポンスJSONは `~/.claude/tmp/ga-gsc-<YYYYMMDD>-<site>.json` にアスカが保存し、リクに当該ファイルパスを引き渡す（レポート本文だけでは絶対値照合ができないため必須）

### 段階C: 実装委任（対策タイプ別）

段階Aを通過した対策案を、タイプに応じて以下のエージェント／スキルに委任する：

| 対策タイプ | 委任先 | 補足 |
|---|---|---|
| タイトルタグ／メタディスクリプション改訂（文言のみ） | コト（copywriter） | SERP CTR改善用。A/Bの候補3本出しが目安 |
| LP／記事のファーストビュー・本文コピー改訂 | コト（copywriter）→ カイ（lp-designer）または ユイ（web-designer） | コピー先行、ビジュアルは後段 |
| 既存記事のリライト（本文構成・見出し改修） | ミオ（researcher）競合調査 → ハル（writer）執筆 | `/blog-post` はハル執筆時のガイドとして参照（リサーチ内包なしのガイドスキル） |
| 新規記事作成 | ミオ → ハル（同上） | 実行順は段階B 側でGSC既存記事チェック済み前提（下記「重要」参照） |
| 技術的SEO／インデックス／CWV対応 | `seo-audit` に切替 | 本スキル範囲外 |
| CVR改善（フォーム・オファー・構造変更） | レンのCVR改善フレームワーク | 本スキル範囲外 |
| 広告運用変動（UTM移管・クリエイティブ変更等） | レン（marketing-planner）で別タスクとして切り出し | SEO診断から除外する |

**重要（段階B側での事前チェック義務）**: 対策案が「**新規記事作成**」を含む場合、段階B のステップ5〜6 内で `gsc_search_analytics (query+page)` により当該KWの既存ランディング有無を確認すること。既存があれば**リライト案に切替**（実運用で「対応記事なし」と誤判定して新規作成を提案したケースあり）。

### フロー図

```
Step 1: MCP基礎データ取得（アスカ、JSON を ~/.claude/tmp/ に保存）
   ↓
Step 2: 段階B 初回診断（レン、既存ステップ1〜5）
   ↓
Step 3: 追加MCPデータ取得（アスカ）← 「要追加調査」項目を潰す
   ↓
Step 4: 段階B 第2ラウンド診断（レン、既存ステップ4〜5を再実行）
   ↓
Step 5: 段階A ファクトチェック（リク）← 保存済みJSONパス添付必須
   ↓ （※本スキルの責任範囲はここまで）
Step 6: 段階C 実装委任（対策タイプ別にコト/カイ/ユイ/ミオ/ハル等）
   ↓
Step 7: 効果検証（各施策の検証方法に従う。ステップ6の検証タイミング表に準拠）
```

Step 6 実行時、**本スキル（既存ステップ6）の「対策案出力」で定めた「検証方法」を各委任先に引き継ぐ**こと。Step 7 のタイミング（2週後／4週後など）は施策ごとに異なるため、一律の日数は設けない。

### 運用上の例外

- **速度優先指示**: **シンヤさんが明示的に「速度優先／試しに／速報で」と指示した場合のみ**、段階Aをスキップし `[UNVERIFIED]` マークを付与して提示可。併せて Notion 案件メモにスキップした旨を記録（CLAUDE.md Deliverable Quality Gate L77・L85 準拠）。アスカ判断でのスキップは不可
- **緊急対応**: 段階Bを1ラウンドで終える代わりに、段階Aでの論理チェックを必ず行う（数値照合より論理飛躍の検出を優先）
- **小規模異変**: 対象ページの **Organic流入 sessions が比較期間同士で±20%以内、かつ判断基準表の「維持扱い（±10%以内）」範疇外の軽微変動**の場合、1ラウンド＋[UNVERIFIED]でも可。ただし**兆候アラート（流入維持＋KW入替）が検出されたら第2ラウンドに昇格**必須

## 適用条件

**このスキルを使う場面：**
- 特定ページの流入数が急増／急減した
- CV率の高かったLPの流入が減っている
- サイト全体または個別ページでアクセス異変を感じた
- 「なぜこのページの順位/流入が変わったのか」を特定したい
- morning-briefing のフラグ（直帰率超過・セッション半減など）を受けた調査依頼

**このスキルを使わない場面：**
- 新規サイト立ち上げ時の包括SEO監査 → `seo-audit`
- 技術的SEO／Core Web Vitals／インデックス問題の詳細診断 → `seo-audit`
- 記事のリライト作業そのもの → `/blog-post`
- モーニングブリーフィングでのGA数値読み上げ（数値フラグだけで済むケース）
- 有料広告の成果分析（GSCは検索経由のみ）

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象サイト | 必須 | 調査対象のドメイン/サイト名 | officeueda.com |
| 調査ルート | 必須 | ケース①アクセス上位 / ケース②CV上位 / 特定ページ指定 | ケース② |
| 対象ページ | 任意 | 特定ページを指定する場合のURL | /service/seo |

| 比較期間 | 必須 | 比較する2期間（推奨14日以上、最低7日。7日未満は警告のうえ続行可否をユーザー確認） | 2026-04-01〜04-14 vs 2026-03-18〜03-31 |
| GA4プロパティID | 任意 | search-analytics MCP を使う場合 | - |

## 実行前チェック（必ず最初に実施）

### データ取得経路の判定

| 状況 | 対応 |
|---|---|
| `mcp__search-analytics__*` が利用可能 | MCP経由で自動取得（推奨） |
| MCPが利用不可または未導入 | ユーザーにGA4／GSC画面のスクリーンショット、またはCSVエクスポートを依頼。取得できるまで停止 |
| 権限不足（プロパティID／サイトURL未共有） | ユーザーに権限付与を依頼。取得できるまで停止 |

### データ品質チェック（誤診防止）

以下を比較期間内に確認し、該当する場合は**影響を分離**して再分析する：

- [ ] URL変更・301リダイレクト設定があったか（旧URLと新URLが分離計測される）
- [ ] ページ統合・削除があったか（旧URLがステータス変化）
- [ ] サイトリニューアル・ドメイン移管があったか
- [ ] 広告キャンペーンのオン/オフ切替があったか
- [ ] GSCのディスカバー流入が急変動していないか（検索流入と分離）
- [ ] 比較期間が14日以上あるか（推奨）／7日未満の場合は警告し、続行可否をユーザーに確認

該当項目があれば、レポート冒頭に「構造変化あり：〇〇を分離分析」と明記する。

## 実行ステップ

### ステップ1：調査ルートを選択する
- 入力：異変の内容、ユーザーからの指示
- 処理：
  - 「流入が減った／急増した」など **アクセス数の変化** → ケース①（アクセス上位ページからの改善）
  - 「CVが減った／CV率が下がった」など **成果の変化** → ケース②（CV上位ページからの改善）
  - どちらか判断できない場合 → ユーザーに「流入の異変ですか？CVの異変ですか？」と確認する
- 出力：選択されたルート（①または②）

### ステップ2：GA4で対象ページの流入推移を取得する
- 入力：対象サイト、選択されたルート、比較期間
- 処理：
  - **ケース①の場合**：GA4「エンゲージメント > ページとスクリーン」で各ページの流入推移を取得
    - MCP利用可の場合 → `mcp__search-analytics__ga4_top_pages` 使用
    - MCP利用不可の場合 → ユーザー提供のスクリーンショット／CSVから抽出
  - **ケース②の場合**：GA4「エンゲージメント > ランディングページ」でキーイベント（CV）数で並び替え、CV上位ページの流入推移を取得
    - MCP利用可 → `mcp__search-analytics__ga4_top_pages` で metric=conversions を指定
    - MCP利用不可 → 同上
  - 比較期間（例：直近2週 vs その前2週、または今月 vs 先月）の両方を取得
  - 増減率が大きい上位5ページを抽出する
- 出力：ページURL別の流入推移データ（比較期間2つ分）

### ステップ3：ページ種別を判別し、流入元構成を確認する
- 入力：ステップ2で抽出した上位5ページ
- 処理：
  - 各ページのページ種別を判別する：
    - **メディア記事／ブログ**：Organic主体が想定（期待比率60%以上）
    - **サービスページ／事業案内**：Organic＋Direct混在（期待比率30〜60%）
    - **LP／広告ランディング**：広告主体が想定（Organic期待比率10〜40%）
    - **TOPページ**：Direct＋Organic混在（期待比率30〜60%、ブランド検索含む）
  - 各ページについて、GA4「集客 > トラフィック獲得」で流入元を確認
    - MCP利用可 → `mcp__search-analytics__ga4_acquisition_report` 使用
    - MCP利用不可 → ユーザー提供データから抽出
  - Organic比率が**ページ種別の期待値を大きく下回る**場合 → SEO以外の要因（広告停止・SNS変動）の可能性も併記するが、**SEO診断を打ち切らない**（優先度は下げる）
  - Direct流入が多い場合は「ブランド検索からの直訪の可能性」として注記する
- 出力：ページ種別×流入元構成の一覧、SEO診断対象の優先度付きリスト

### ステップ4：GSCで同一期間のクエリ・順位・CTRを取得する
- 入力：SEO診断対象のページリスト、比較期間（GAと同一）
- 処理：
  - 各ページについて、GSC「検索結果のパフォーマンス」で該当ページに絞り込む
    - MCP利用可 → `mcp__search-analytics__gsc_top_queries` にページURLを指定
    - MCP利用不可 → ユーザー提供データから抽出
  - 以下4指標を比較期間の両方で取得：クリック数、表示回数、平均CTR、平均掲載順位
  - 流入KW（クエリ）トップ10も取得し、新旧期間で並べる
- 出力：ページ×期間別の指標表＋クエリ比較表

### ステップ5：変化パターンを分類して原因仮説を立てる
- 入力：ステップ2〜4の全データ
- 処理：下記「判断基準」表のパターンに該当するか順に照合し、仮説を選定する
  - **兆候アラートの併出**：流入が維持されていてもKWが大きく入れ替わっている場合は「将来の流入減の予兆」として別項目で警告する
- 出力：仮説（1ページにつき1〜3個、確度を高/中/低で付記）＋兆候アラート（該当時のみ）

### ステップ6：仮説に応じた対策案と検証方法を出力する
- 入力：ステップ5の仮説
- 処理：
  - 各仮説に対し、「対策案」と「検証方法（何をどう測れば効果確認できるか）」をセットで出す
  - 実行容易性（高/中/低）× 期待効果（高/中/低）で優先順位をつける
  - **全体から優先度上位3施策**（複数ページを横断して選定）を提案書フォーマットで記載
  - 技術的SEO／インデックス／CWVが疑われた場合 → 「`seo-audit` への切替を推奨」と明記
  - **新規記事作成を提案しようとする場合は、出力前に必ず `gsc_search_analytics (query+page)` で当該KWの既存ランディングページ有無を確認**。既存があれば「新規作成」ではなく「既存記事リライト」案に切替（実運用で「対応記事なし」と誤判定したケースあり）
- 出力：「出力フォーマット」に沿った診断レポート

## 判断基準

### ルート選択の判断

| ユーザー発言のキーワード | ルート |
|---|---|
| 流入／PV／アクセス／セッション | ケース①（アクセス） |
| CV／コンバージョン／問い合わせ／購入 | ケース②（CV） |
| 「特定ページの異変」とURL指定 | 指定ページで①②両方を並行実施 |

### 原因仮説の判断（変化パターン別）

| パターン | 仮説 | 確度 | 次アクション |
|---|---|---|---|
| Organic流入減 + 主要KWの順位下落 | SEO対策に問題あり（コンテンツ鮮度／被リンク喪失／競合台頭） | 高 | コンテンツリライト or 競合調査 |
| Organic流入減 + 順位変わらず + CTR低下 | タイトル・ディスクリプションの訴求力低下、SERP上で競合に負けている | 高 | タイトル/ディスクリプション改修 |
| Organic流入減 + 順位も表示回数もCTRも変わらず | 季節的要因または検索需要自体の減少 | 中 | 経過観察（2〜4週後に再診断） |
| Organic流入が10%超減少 + ランディングKWが入れ替わっている | ページ主題とタイトル/ディスクリプションのズレ。新KW合算で流入維持なら問題なし、減少なら要対処 | 中 | タイトル改修 or コンテンツ追記 |
| **流入変動が±10%以内（維持扱い） + ランディングKWが大きく入れ替わっている** | **将来の流入減の予兆**。現状は無事でも新KWへの最適化が進んでいないため中期的にリスク | 中 | 兆候アラートとして報告、新KWへの最適化を推奨 |
| Organic流入急増 + 順位上昇 | SEO対策が効いた／競合順位下落／アルゴリズム変動で追い風 | 高 | 成功要因を他ページに横展開 |
| Organic流入急増 + 順位変わらず + 検索需要急増 | バズ／時事要因／季節要因による外部トレンド | 高 | 記事強化で波に乗る |
| **流入急増 + 新規KWが多数流入** | 機会発見。現ページが意図せずカバーできているKW群あり | 中 | 新規KWに最適化した関連コンテンツ展開 |
| 全体Organic流入減 + 個別ページに変化なし | サイト全体のインデックス問題／ペナルティ／技術的SEO問題の可能性 | 中 | **`seo-audit` に切替**を推奨 |
| CV減少 + 流入維持 | LPの離脱要因（ファーストビュー／フォーム／オファー）の可能性 | 高 | **レンのCVR改善フレームワーク**に切替 |

### データ不足・品質時の対応

| 状況 | 判断 |
|---|---|
| GA4／GSCのMCP利用不可 | ユーザーにスクリーンショット／CSV依頼、取得できるまで停止 |
| GA4プロパティID／GSCサイトURL未共有 | ユーザーに権限確認、取得できるまで停止 |
| 比較期間が7日未満 | 「14日以上の比較期間を推奨」と警告、続行可否をユーザーに確認 |
| URL変更／サイトリニューアルが期間内にあった | 「構造変化の影響を分離して再分析」とレポート冒頭に明記 |
| Organic比率がページ種別期待値を下回る | SEO診断は継続（優先度を下げる）、非SEO要因の可能性を併記 |

# GA×GSC 診断レポート

- 対象サイト: officeueda.com
- 調査ルート: ケース①アクセス上位
- 異変の内容: /service/seo の流入が先月比 -42%
- 比較期間: 2026-04-01〜04-14 vs 2026-03-18〜03-31
- データ取得経路: MCP（search-analytics）

## 検出した変化
/service/seo（ページ種別：サービスページ）
- 流入: -42%（Organic比率82% ＝ 期待値超、SEO診断対象：高優先度）
- 平均順位: 3.2 → 5.8（-2.6ランク）
- 平均CTR: 4.1% → 3.2%
- 表示回数: -18%

- 比較前TOP: 「SEO対策 費用」120クリック → 比較後30クリック
- 比較後TOP: 「SEO対策 低価格」新規流入85クリック

### 仮説1（確度：高）
ランディングKWが「費用」系から「低価格」系にシフト。タイトル/ディスクリプションに「低価格」訴求がなくCTRと順位が下落。

タイトルを「SEO対策 費用」→「SEO対策の低価格プラン｜費用比較も掲載」に改修。2週間後にGSCでCTRが3.2%→4.0%以上に回復するか検証。

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| GA4だけで「流入減った」と判断して対策を立てる | 必ずGSCで順位・CTR・クエリを突き合わせる | GA単独では原因（順位下落か？CTR低下か？需要減か？）を特定できない |
| 全流入（All Channels）で判断する | Organic Searchに絞り込んでから分析する | 広告停止・SNSバズなど非SEO要因が混入すると誤診する |
| Organic比率50%未満を一律「SEO対象外」と切り捨てる | ページ種別（LP／メディア記事／サービスページ）で期待値を変え、下回っても診断は継続する | LPは広告主体で Organic 低めが通常。一律閾値は誤診を招く |
| 単一KWの流入数だけ見る | ランディングKWの入れ替わりパターンを必ず確認する | KWが変わっただけで合算流入が維持されているケースを見落とす |
| 流入維持ならKW入れ替わりを無視する | 兆候アラートとして別途報告する | 将来の流入減の予兆を見落とす |
| 順位が下がっていない = 問題なし と結論づける | 順位据え置きでもCTR低下なら「タイトル・ディスクリプション劣化」を疑う | 競合のSERPスニペット改善で相対的にクリックされなくなるケースがある |
| 1週間の比較で結論を出す | 最低14日、できれば28日の比較期間を取る | 週次の曜日変動・キャンペーン影響で短期間は誤差が大きい |
| URL変更・リニューアルの影響を考慮せずそのまま比較 | 構造変化があれば必ず分離分析する | 旧URL/新URLの分離計測で「流入減」に見えるだけのケースがある |
| 仮説を1つしか出さない | 確度付きで複数仮説を並列提示する | データだけでは単一原因に特定できないケースが多く、仮説の検証コストまで含めて提案すべき |
| 対策案を出して終わり | 必ず「検証方法（何を2週後/4週後に測るか）」をセットで出す | 検証サイクルが無いと改善が属人化・精神論化する |
| 技術的SEO問題が疑われても本スキル内で無理に結論づける | `seo-audit` への切替を推奨する | 本スキルは1次診断。技術的深掘りは専用スキルの領分 |
