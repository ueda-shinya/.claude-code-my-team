# lead-nurturing

獲得した見込み顧客のうち**75%は直近購買検討にならない**が、**営業フォローを止めると2年以内に80%が競合から購入**する。継続的にコミュニケーションして商談機会を創出する仕組みを設計する。

## 適用条件

- 使う場面：
  - BtoBで獲得済みリードの活用が中途半端／フォローしきれていない
  - メルマガ運用の仕組み化（配信頻度・内容・営業連携）
  - 失注/保留リードのリサイクル設計
  - インサイドセールス×マーケの連携プロセス整備
- 使わない場面：
  - リードの定義（MQL/SQL）→ `lead-definition-mql-sql`
  - リード獲得施策（リードジェネレーション）→ `funnel-design` `promotion-strategy`
  - マーケ営業フロー全体 → `marketing-sales-workflow`
  - ホワイトペーパー制作 → 別途コンテンツ系スキル
  - クロージングトーク → `sales-script-builder`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 事業 | 必須 | BtoB SaaS／コンサル／パッケージ等 | 「BtoB SaaS（経理クラウド）」 |
| 現有リード数 | 必須 | ハウスリスト規模 | 「3,000件」 |
| 営業組織体制 | 推奨 | マーケ／IS／FS／CS の有無 | 「マーケ＋IS＋FS の3層」 |
| MAツール | 任意 | HubSpot／Marketo／Salesforce Pardot等 | 「HubSpot」 |
| ナーチャ目的 | 必須 | 商談創出／既存深耕／リサイクル | 「失注リードからの再商談獲得」 |

## 実行ステップ

### ステップ1：リードナーチャリングの位置付けを確認する

- 入力：営業組織体制、ナーチャ目的
- 処理：以下のプロセスマップで自社の位置を確認
  - **プロセス**：リードジェネレーション → リードクオリフィケーション → **リードナーチャリング** → 商談 → クロージング → リードナーチャリング（リサイクル）
  - **組織**：マーケティング → マーケ → **マーケ→IS** → IS→FS → FS → IS／マーケ
  - **実施内容**：見込み顧客の獲得 → 評価 → **育成** → 商談 → 受注／失注・保留
  - **ナーチャリング対象リード**：各段階で「離脱」したリード全体（成約しなかった/フォロー止まった顧客）
- **目的の確認**：
  - 見込み顧客の購買意欲を高める
  - 長期的な関係を築き、最適なタイミングで営業アプローチ
  - リードの質向上で成約率アップ
  - 早期離脱リードを減らす
- 出力：自社プロセス図＋ナーチャ対象リードの定義

### ステップ2：ナーチャリング5原則を理解する（前提）

- 処理：以下5原則を確認（現場でよく誤解されるポイント）
  1. **顧客は"育成"されない。メールはアプローチのきっかけ作り**
     - 「コンテンツで関心が温まる」のは幻想。実際は「メールをたまたま見たタイミングで思い出し、たまたま需要が高まっていたから問い合わせる」
     - メールは開封/クリックなど顧客アクションを可視化するツール。アクション起点での営業アプローチがナーチャリングの実体
  2. **セグメンテーションは不要、基本全体配信でOK**
     - BtoBメールの一般的な開封率30%・クリック率2〜3%
     - 5,000件リストでも1回配信のクリック2〜4人。セグメント切ると母数減で結果ゼロになりがち
     - リスト数がよっぽど多くない限り、セグメンテーション不要
  3. **メールに「送りすぎ」はない**
     - メール頻度とブロック率に相関なし（多くのビジネスパーソンは営業メールをそもそも見ていない）
     - 送る回数 = 自社名がメールボックスに出る回数 = 認知強化
     - **平日毎日1日3回送っても送りすぎではない**
  4. **全体配信メール＋個別アプローチを組み合わせる**
     - メールだけではナーチャリング完成しない
     - メールへのアクション起点で IS が架電 / 個別メール送信
     - ホットなリードには必ず個別アプローチを組み合わせる
  5. **5,000件未満ならリードジェネレーションに注力**
     - 5,000件未満のリストでメール配信しても、1通で検証できる幅が限られる
     - リード獲得を優先し、ハウスリストを5,000件まで増やしてからナーチャ強化
- 出力：自社が誤解している原則のチェック

### ステップ3：ナーチャリング5手法から選定

- 入力：MAツール有無、リソース
- 処理：以下5手法から組み合わせを選定
  1. **メールマーケティング**：定期ニュースレター／キャンペーンメール／パーソナライズドコンテンツ ※基本接点
  2. **コンテンツマーケティング**：ブログ／ホワイトペーパー／ウェビナー／動画／教育型コンテンツ
  3. **SNSマーケティング**：継続的な接点／フォローアップ／コメントやり取り
  4. **MAツール活用**：行動データ分析→適切タイミングのアプローチ／スコアリングで成約可能性高リード特定
  5. **ウェビナー・イベント**：直接コミュニケーションで関係強化
- 出力：選定手法＋運用方針

### ステップ4：ナーチャリング構築の4ステップを実行する

- 処理：以下4ステップで設計→実行→改善のサイクルを回す
  - **Step 1: 目標・コミュニケーション設計**
    - ナーチャリングの目標設定（例：失注リードから月5商談）
    - リード情報の管理（MAツール／CRMでのステータス管理）
    - コミュニケーション設計（配信頻度・コンテンツミックス）
    - 個別アプローチの設計（IS架電基準）
  - **Step 2: コンテンツの作成と送付**
    - コンテンツ企画（事例／お役立ち資料／ウェビナー告知／ニュースレター）
    - コンテンツ作成
    - メルマガ作成
    - メルマガ送付
  - **Step 3: 営業アプローチ**
    - 架電（メール開封・クリック・資料DLを起点に IS から）
    - 個別メール（営業担当者の個人メールから直接送信）
  - **Step 4: 成果測定と改善**
    - 数値管理（開封率／クリック率／商談化率／成約率）
    - ステータス管理（リードランク移動の追跡）
    - 会議体設計（マーケ×ISで週次／月次レビュー）
    - 改善提案（コンテンツ改善／配信頻度調整）
- 出力：4ステップの実行計画

### ステップ5：典型的なナーチャリング動線を設計する

- 処理：以下のフロー図で動線を可視化
  ```
  資料DL → セミナー案内メール → 事例記事紹介メール → お役立ち資料送付メール → 別資料DL → ISからの架電
       (顧客接点1) (メール接点1)        (メール接点2)        (メール接点3)        (顧客接点2)  (営業接点)
  ```
  - 各メールでセミナー案内／お役立ち資料／事例記事紹介などコンテンツを差し替え
  - メール開封・クリック・資料DL等のアクション起点で IS が個別アプローチ
- 出力：動線フロー＋月次配信カレンダー

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ハウスリスト 5,000件未満 | リードジェネ優先（メルマガは月1〜2回維持で可） | メール検証が成立しない |
| ハウスリスト 5,000件以上 | ナーチャ本格化（週2〜3回配信＋個別アプローチ） | 検証ループが回せる規模 |
| MAツールなし | まず手作業で月1〜2回メルマガ配信スタート | ツール導入は仕組み化後 |
| 失注リードが大量にある | リサイクルナーチャ優先 | 2年以内80%が他社購入する前にフォロー |
| メール開封率<20% | 件名・配信時間帯のA/Bテスト | コンテンツ前にまず開いてもらう |
| クリック率<1% | 内容の関連性／CTA文言の見直し | 開封されてもクリックされない |
| ホットリードが反応した | 即IS架電＋個別メール | 鉄は熱いうちに打つ |
| ISがマーケ施策に無関心 | 週次の合同レビュー会議を設置 | 連携不足はナーチャ最大の失敗原因 |
