# instagram-account-design

SNS戦略3段階フローの**第3段（媒体特化）**として、Instagramを「Instagram固有の役割→トレンド調査→5機能のジャーニー5区分（認知／興味／比較検討／行動／リピート）配置→KPI→施策」の流れで設計し、実行可能な運用計画書を生成する。

## SNS戦略3段階フロー上の位置づけ

| 段階 | スキル | 役割 |
|---|---|---|
| ① 第1段 | `sns-strategy-overview` | 媒体選定／KGI・KPI設計 |
| ② 第2段 | `sns-content-design` | **アカウントテーマ／ターゲット態度変容5区分／ポジショニング4象限／コンセプト一言／コンテンツ分類ツリー**（本スキルではここを再実施しない） |
| ③ 第3段（本スキル） | `instagram-account-design` | Instagram固有の機能配置・台本・施策・分析改善 |

**前段からの引き継ぎ前提**：本スキルは `sns-content-design` の出力（具体ペルソナ／ポジショニング4象限／コンセプト一言／コンテンツ分類ツリー）が**既に確定している前提**で起動する。媒体特化スキル間（IG/TikTok/YT）でこれらを重複実施しない。

### 実行停止プロトコル（必須遵守）

本スキル起動時、最初に以下を確認する。**いずれかの前段成果物が引き継がれていない場合は本スキルの実行を直ちに停止する**:

- [ ] **第1段成果物**: `sns-strategy-overview` のアウトプット（媒体選定／KGI/KPI／アカウントタイプ）が確定済みか
- [ ] **第2段成果物**: `sns-content-design` のアウトプット（具体ペルソナ／ポジショニング4象限／コンセプト一言／コンテンツ分類ツリー）が確定済みか

**未確定の場合の対応**:
1. 本スキルの実行を**停止**する（ステップ1以降に進まない）
2. ユーザー（または呼び出し元）に「前段（`sns-strategy-overview` / `sns-content-design`）が未実施です。先に起動してください」と通知する
3. 確定済みアウトプットが提供されたことを確認してから、本スキル ステップ1から実行を開始する

**例外**: シンヤさんが「前段スキップで直接Instagram設計だけ進める」と明示した場合のみ、警告ログを残した上で本スキルを実行可（Notion 案件管理に「前段スキップ例外」と記録必須）。

## 適用条件

- このスキルを使う場面：
  - 第2段の `sns-content-design` 完了後、Instagramを採用したケースの媒体特化設計
  - 既存Instagram運用が伸び悩んでおり、機能配置・施策からテコ入れしたいとき
  - 5機能（フィード／リール／ストーリーズ／ハイライト／ライブ）の優先度を整理したいとき
  - Instagramのトレンド調査・KPI・施策を設計したいとき
- このスキルを使わない場面：
  - SNS全体戦略（媒体選定）の設計 → `sns-strategy-overview`
  - アカウントテーマ・ポジショニング・コンセプト・ターゲット具体化 → `sns-content-design`
  - Instagram投稿1本のクリエイティブ制作 → コピー／デザイン担当へ
  - TikTok／YouTube単体の設計 → 各媒体特化スキルへ

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 業態タイプ | 必須 | BtoC／BtoB／採用 のいずれか | BtoC（D2Cコスメ） |
| 主要ターゲット | 必須 | デモグラ＋サイコグラ | 33歳女性・既婚・東京・育児中 |
| 事業の強み（USP） | 必須 | 競合にない自社固有の強み | 化学者監修・成分透明性 |
| 競合アカウント | 任意 | 同業界の競合3〜5アカウント | 未指定の場合はステップ2でリサーチ依頼 |
| 既存運用状況 | 任意 | フォロワー数・投稿頻度・現状KPI | フォロワー1,500・週2回投稿 |
| 主要KPI候補 | 任意 | 認知拡大／エンゲージメント強化／購買・問い合わせ | 認知拡大 |

## 実行ステップ

### ステップ1：Instagramの役割を業態別に確定する

- 入力：業態タイプ、事業の強み
- 処理：
  1. 役割3観点（ブランド認知拡大／信頼構築／購買導線）のうち、最重要を1つ選ぶ
  2. 業態別の活用ポイントから自社が訴求すべき軸を決める：
     - **BtoC企業**：商品魅力をビジュアル訴求／UGC生成／世界観訴求／購買促進
     - **BtoB企業**：企業文化・価値観の訴求／事例紹介／イベント情報掲載
     - **採用**：社内雰囲気・文化の可視化／社員ストーリー訴求／応募促進
  3. 役割3観点をそれぞれ「主目的／副目的／補助」の優先度に振り分ける
- 出力：Instagramアカウント役割定義書（主目的1＋副目的1＋補助1）

### ステップ2：Instagram固有の競合運用リサーチを実施する

- 入力：業態タイプ、`sns-content-design` で確定した主要ターゲット／ポジショニング4象限／コンセプト一言
- 処理：
  1. **前提確認**：`sns-content-design` の出力（ポジショニング4象限／差別化ポイント／コンセプト一言）が引き渡されていることを確認する。未実施なら本スキルを止めて `sns-content-design` の起動を案内する
  2. 同業界の **Instagram 競合** 3〜5アカウントを選定する（媒体特化リサーチ。`sns-content-design` の競合分析とは別観点）
  3. 各競合について以下4項目を記録する（Instagram固有の運用観点）：
     - フォロワー数
     - 投稿頻度（週N回）
     - **画像スタイル／投稿フォーマット**（Instagram特有：カルーセル/単発/リール比率）
     - エンゲージメント高い投稿の特徴（保存数／コメント数／いいね数）
  4. `sns-content-design` で決まったポジショニング4象限上の自社配置を、**Instagram上での運用挙動として再点検**する（その配置をInstagramで体現できるか。たとえばストーリー型を選んでいるのにフィード写真で世界観が出せていない等の整合性確認）
- 出力：Instagram競合運用リサーチ表＋ポジショニング配置のInstagram体現方針

### ステップ3：コンセプト一言から「Instagram用の投稿方向性」を導く

- 入力：`sns-content-design` で確定したコンセプト一言（キャッチコピー）、業態タイプ
- 処理：
  1. **前提確認**：`sns-content-design` の出力（コンセプト一言）が引き渡されていることを確認する。未実施なら本スキルを止めて `sns-content-design` の起動を案内する
  2. コンセプト一言を**そのまま流用**する（再生成しない。媒体特化スキル間でブレないようにする）
  3. Instagram運用上の方向性として、コンセプトに合わせて**取り扱うテーマ／取り扱わないテーマ**を箇条書きで決める（媒体特化での落とし込み）
  4. アカウント運用名・プロフィール文・固定ハイライト構成案など、Instagram上での表出を設計する
- 出力：投稿方向性（扱うテーマ／扱わないテーマ）＋プロフィール表出案

> **重要**：コンセプト一言・ポジショニング4象限・差別化ポイントは `sns-content-design` の成果を**そのまま流用**すること。媒体特化スキル側で再生成すると、TikTok/YouTubeとの整合性が崩れて出力統合不能になる（過去の重大指摘B）。

### ステップ4：トレンド調査を実施する

- 入力：コンセプト、業態タイプ
- 処理：
  1. Instagramで以下3項目をリサーチする：
     - **リールトレンド**：再生数の多いリールの形式・尺・冒頭演出を5本サンプリング
     - **人気音源**：トレンド音源を3つ特定し、使用されている投稿の傾向を確認
     - **ハッシュタグ分析**：人気ハッシュタグ（10万投稿以上）と業界特化ハッシュタグ（1万〜10万投稿）を各5個ピックアップ
  2. 自社の強み（USP）×トレンドの掛け合わせで使える組み合わせを3パターン以上抽出する
- 出力：トレンド調査レポート（リール形式／音源／ハッシュタグ）＋掛け合わせパターン

### ステップ5：5機能をカスタマージャーニーに配置する

- 入力：役割定義書、コンセプト
- 処理：
  1. 5機能の特性を以下表で確認する：

  | 機能 | 役割 | 特徴 | 成功ポイント |
  |---|---|---|---|
  | フィード投稿 | 世界観・信頼の蓄積／アカウントの名刺 | 資産性・接点維持・グリッド統一感 | ビジュアル統一／カルーセル活用／CTA設置 |
  | リール動画 | 新規顧客リーチ拡大 | 最大180秒・フォロワー外に拡散・発見タブ表示 | 最初の3秒勝負／視聴完了率重視／トレンド音源活用 |
  | ストーリーズ | 既存フォロワーとの接触頻度UP | 24時間限定・気軽な投稿・双方向 | 双方向コミュニケーション／アンケート/質問箱／リンク設置 |
  | ハイライト | ストーリーズの永続化／検討層への情報提供 | プロフィール上部固定・テーマ別整理 | カテゴリ別商品／FAQ／導入事例・口コミ |
  | ライブ配信 | リアルタイムでの信頼形成・販売促進 | 高い双方向性・購入障壁を下げる | 事前告知／参加者限定特典／台本準備 |

  2. カスタマージャーニー**5区分（認知／興味／比較検討／行動／リピート）**に機能を割り当てる（本3段階フロー全体で語彙統一）：

  | フェーズ | 主機能 | 補助機能 | 役割 |
  |---|---|---|---|
  | 認知 | リール／広告配信／インフルエンサー／キャンペーン運用 | （—） | 新規層への到達 |
  | 興味 | フィード／ストーリーズ（双方向） | リール（保存促進系） | 関係性構築・興味喚起 |
  | 比較検討 | ハイライト（FAQ/事例）／フィード（カルーセル詳細解説） | ライブ（Q&A配信） | 検討材料の提供 |
  | 行動（販売） | ライブ（販売） / ショッピング機能 / フィードCTA投稿 | ストーリーズ（CTA・URL添付） | 購買・問合せの一押し |
  | リピート | ストーリーズ（接触頻度UP）／ライブ（ファン交流） | コミュニティ機能（補助投票・予告） | 既存顧客との関係維持・ファン化 |

  ※ **ストーリーズの主役配置は「興味」と「リピート」**（既存フォロワー接触頻度UPという機能特性に合致）。「行動（販売）」では補助配置（CTA・URL添付の補助機能）として位置づける。「販売主機能はライブ／ショッピング機能／フィードCTA投稿」（過去の重大指摘C：ストーリーズ販売主役配置と機能特性「既存フォロワー接触頻度UP」の矛盾を解消）。

  3. 自社の現状課題（認知不足／興味弱／比較検討弱／行動弱／リピート弱）を1つ特定し、該当フェーズの機能を最優先で運用する
  4. 機能ごとの投稿頻度を週単位で決める
- 出力：機能配置マトリクス（フェーズ×機能×投稿頻度）

### ステップ6：目的別KPIを3タイプから選定する

- 入力：機能配置マトリクス、役割定義書
- 処理：
  1. 以下3タイプから自社の主目的に合うKPIを選ぶ：

  | 目的 | 概要 | KPI（中間指標） | 施策例 |
  |---|---|---|---|
  | 認知拡大 | 新規ユーザーへのリーチ | リーチ数／動画再生回数 | リール投稿頻度増加／トレンド音源活用 |
  | エンゲージメント強化 | 関係性構築と信頼醸成 | 保存率／コメント数／ストーリーズ反応率 | アンケート機能活用／教育系コンテンツ／UGCリポスト |
  | 購買・問い合わせ | CVに繋がるアクション促進 | プロフィールクリック率／リンククリック数／DM件数 | ライブ販売／ストーリーズURL添付／ハイライト活用 |

  2. 主KPIを1つ、副KPIを1〜2つ選ぶ
  3. 各KPIに数値目標・期限・測定方法を付与する
- 出力：KPI設計書（主KPI1＋副KPI1〜2＋数値目標・期限・測定方法）

### ステップ7：施策を4タイプから組み合わせる

- 入力：KPI設計書、コンセプト、トレンド調査レポート
- 処理：
  1. 以下4タイプから、KPI達成に必要な施策を最低2つ選ぶ：

  | 施策タイプ | 目的 | 具体例 |
  |---|---|---|
  | キャンペーン施策 | 短期的な認知・エンゲージメント増 | ハッシュタグチャレンジ／フォロー&コメントキャンペーン |
  | ライブ配信 | リアルタイム信頼構築・販売促進 | 新商品ライブ／限定特典付き販売ライブ／Q&Aライブ |
  | インフルエンサー活用 | 新規層への認知拡大 | 商品PR／動画作成／共同投稿 |
  | シリーズ企画 | 習慣的視聴の醸成 | 週1回商品レビュー連載／ビフォーアフター連載 |

  2. 各施策の実施頻度（月次／週次／四半期）と担当者を決める
  3. KPI連動を明示する：「この施策がどのKPIにいくつ寄与するか」
- 出力：施策プラン（施策名／頻度／担当者／連動KPI）

### ステップ8：運用計画書を1枚にまとめる

- 入力：ステップ1〜7の全出力
- 処理：以下のテンプレートで運用計画書を生成する
- 出力：Instagramアカウント運用計画書

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 業態がBtoC・商品ビジュアル訴求できる | 役割の主目的を「ブランド認知拡大」 | リールの拡散効果が活きる |
| 業態がBtoB | 役割の主目的を「信頼構築」 | 事例紹介・企業文化訴求に強い |
| 業態が採用 | 役割の主目的を「信頼構築」（社内文化可視化） | 応募者の安心感に繋がる |
| 競合が同じ4象限に集中 | 空白ゾーンに自社を配置 | 差別化が明確になる |
| 競合の空白ゾーンが自社の強みと無関係 | 既存ゾーンに入って強みで勝負 | 強みが活きないポジションは続かない |
| 認知不足が課題 | リール優先（週3〜5本）／キャンペーン施策 | 拡散性を最大化 |
| 興味弱が課題 | フィード＋ストーリーズ双方向投稿優先 | 関係性構築・専門性訴求 |
| 比較検討弱が課題 | ハイライト（FAQ/事例）＋フィード（カルーセル詳細）優先 | 検討層への判断材料提供 |
| 行動弱が課題 | ライブ販売＋ショッピング機能＋フィードCTA投稿優先（ストーリーズはCTA・URL添付の補助） | 購買・問合せの一押し |
| リピート弱が課題 | ストーリーズ（既存フォロワー接触頻度UP）＋ライブ（ファン交流）優先 | 既存顧客との関係性強化 |
| KPI候補が複数で迷う | 主KPI1つに絞る | 散漫運用を防ぐ |
| トレンド音源と自社世界観が合わない | 世界観優先・音源は採用しない | 世界観の一貫性が信頼の源泉 |
| ライブ施策で参加者が集まらない | 事前告知3回以上＋参加者限定特典を設ける | 集客動線が不足している典型パターン |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 競合分析なしでコンセプトを決める | 必ず競合3〜5アカウントを4象限マッピング | 差別化ポイントが言語化できない |
| 5機能を全て同じ頻度で運用しようとする | カスタマージャーニーの自社課題フェーズを最優先 | リソース分散で全機能の質が下がる |
| 1枚画像で情報を完結させようとする | カルーセル活用で複数枚に分ける | 情報過多なフィード画像は読まれない |
| リール冒頭で挨拶や前置きから始める | 最初の3秒で強いインパクトを与える | スクロールが速いため離脱される |
| ストーリーズを一方向的な情報発信に使う | アンケート/質問箱で双方向化 | 双方向コミュニケーションが信頼の源泉 |
| ライブを告知なしで実施 | 事前告知3回＋参加者限定特典＋台本準備 | 集客動線がないと視聴者ゼロになる |
| トレンド音源と世界観が合わなくても使う | 世界観の一貫性を優先 | 一貫性のないアカウントは信頼を失う |
| KPIを「フォロワー数」だけに設定 | 主KPI＋副KPIで段階的に設計 | フォロワー数=売上ではない |
| 全業態でBtoC的に運用する | BtoC／BtoB／採用で活用軸を変える | 業態によってユーザーが求める情報が異なる |
