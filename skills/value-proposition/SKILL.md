# value-proposition

バリュープロポジションキャンバスで「顧客が求める価値」と「自社が提供できる価値」のマッチを検証し、差別化された価値提案を1文で言語化する。「自社目線の押しつけ」を防ぐため、必ず顧客側から先に分析する。

## 適用条件

- 使う場面：
  - 新サービスの価値提案を1文で言語化したい
  - 既存サービスで「なぜ売れないか」を価値ズレから検証したい
  - LP・営業資料のメッセージ軸を決める前段階
  - ペルソナ別に価値提案を作り分けたい
- 使わない場面：
  - セグメンテーション・ターゲティング → STP分析（`stp-analysis`）
  - 顧客の根本的なジョブ分析 → Jobs to Be Done
  - 経営資源の評価 → VRIO分析（`vrio-analysis`）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 分析対象 | 必須 | 商品・サービス | 「自社の月額ITサポート」「新規SaaS」 |
| 顧客セグメント | 必須 | ペルソナ／顧客像 | 「世帯年収800万円以上の首都圏ペットオーナー」 |
| 既存の課題仮説 | 任意 | 顧客の悩み・ニーズの仮説 | 「いいペットフードを探す時間がない」 |

## 実行ステップ

### ステップ1：顧客セグメント側（円）を埋める ← 必ず先に

- 入力：顧客セグメント、既存の課題仮説
- 処理：以下3要素を順番に整理する（顧客理解を先に固める）
  - **顧客の解決したい課題**（中央）: 顧客が達成したい仕事・ジョブ・タスク
    - 例：「ペットの健康を維持しながら、毎月手間なくフードを買い続けたい」
  - **ゲイン**: 顧客が得たい価値・利益（時間節約／コスト削減／生活の質向上 等）
    - 例：「愛犬の食いつき」「ペットの健康」「続けられる価格」
  - **ペイン**: 顧客が抱えている課題・不満・リスク（手間／コスト／ストレス 等）
    - 例：「いいフードを探す時間がない」「合うフードが見つからない」「高くて続かない」
- ゲイン・ペインは各3〜5項目ずつ
- 顧客の声・ヒアリング・レビューを根拠として添える
- 出力：顧客セグメント側の3要素（解決したい課題＋ゲイン3〜5＋ペイン3〜5）

### ステップ2：商品・サービス側（四角）を埋める

- 入力：分析対象、ステップ1の顧客理解
- 処理：以下3要素を顧客側に対応する形で整理する
  - **サービス（解決策）**: 提供している商品・サービスそのもの
    - 例：「毎月届くパーソナライズ・プレミアムペットフード」
  - **ゲインクリエイター**: 顧客のゲインを生み出す要素・仕組み
    - 例：「使いやすいUIで作業時間を短縮」「AI機能で最適な提案」「特典・ポイント還元」
  - **ペインリリーバー**: 顧客のペインを和らげる・解決する仕組み
    - 例：「自動入力で手間削減」「返金保証で購入リスク軽減」「直感的UIで学習コスト低減」
- 各要素は2〜4項目ずつ
- 出力：商品・サービス側の3要素（サービス＋ゲインクリエイター2〜4＋ペインリリーバー2〜4）

### ステップ3：マッチ（フィット）を検証する

- 入力：ステップ1の顧客側＋ステップ2の自社側
- 処理：以下を1対1で照合する
  - **ゲイン → ゲインクリエイター**: 顧客のゲインを実際に生み出している要素があるか
  - **ペイン → ペインリリーバー**: 顧客のペインを実際に和らげる要素があるか
- 矢印を引いて「対応している」「対応が薄い」「対応なし」の3段階で判定
- マッチが取れていない箇所を特定する：
  - **対応なし**: 顧客の重要なペイン／ゲインに対応する自社要素がない → 機能追加・改善が必要
  - **過剰**: 顧客が求めていないゲインクリエイターを提供している → 訴求軸の見直しか機能削減
  - **理想**: 全項目で対応できている → このまま訴求
- 出力：マッチ検証結果（マッチ箇所＋ギャップ箇所）

### ステップ4：差別化要素を抽出する

- 入力：ステップ3のマッチ結果
- 処理：
  - **競合との比較**: 同じセグメントを狙う競合のゲインクリエイター／ペインリリーバーを並べる
  - **競合と被る要素**: コモディティ化している価値（差別化にならない）
  - **競合にない要素**: 自社独自の価値（差別化の源泉）
  - **競合より優れる要素**: 程度で勝っている価値
- 「自社が提供できて、競合他社が提供できない、顧客が求める独自の価値」を3つ以内に絞る
- 出力：差別化要素3つ以内

### ステップ5：バリュープロポジション文を作成する

- 入力：ステップ1〜4の全成果物
- 処理：以下のフォーマットで1〜2文に言語化する
  - 構文：「**[顧客セグメント]**に対して、**[ペインを和らげ／ゲインを生み出す方法]**を、**[競合と異なる差別化要素]**で提供する商品・サービス」
  - 例：「世帯年収800万円以上の首都圏ペットオーナーに対して、ペットの健康と続けやすい価格を、パーソナライズ＋獣医師相談＋サブスク自動配送で同時に実現するプレミアムペットフード」
- 短く（1〜2文以内）、誰でも理解できる平易な言葉で書く
- 出力：バリュープロポジション文（1〜2文）

### ステップ6：VPC完成シートとして統合する

- 入力：ステップ1〜5の全成果物
- 処理：以下のテンプレートに統合する
- 出力：バリュープロポジション完成シート

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 自社側から先に分析したくなる | 必ず顧客側（円）を先に埋める | 自社先行は「自社想いの押しつけ」になり顧客ニーズを無視する |
| ゲイン・ペインが「感覚」「予想」 | 顧客ヒアリング・レビュー・受注理由で必ず裏付ける | 主観のVPは形骸化 |
| 顧客のペインに対応するペインリリーバーがない | 機能追加・改善 or その顧客は対象外と判断 | ペインに対応しないサービスは選ばれない |
| ゲインクリエイターが「過剰」（顧客が求めていないゲイン） | 訴求軸を絞る／余計な機能を削減 | 過剰機能は価格を上げ、ターゲットを狭める |
| 全要素が競合と被る | 価値提案として弱い。新規差別化要素の追加が必須 | コモディティはどこかで負ける |
| 差別化要素が4つ以上 | 3つ以内に絞る | 多すぎるとメッセージが散る |
| バリュープロポジション文が3文以上 | 1〜2文に圧縮 | 長い価値提案は伝わらない |
| ペルソナごとに分けず1つで済ませる | 主要ペルソナ別に作成 | ペルソナで価値が違うことが多い |
| 1度作って放置 | 半年〜1年で再評価 | 顧客の課題は変わる |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 自社側（四角）から先に書く | 顧客側（円）から先に書く | 自社先行は押しつけになる |
| ゲイン・ペインを「予想」「感覚」で書く | ヒアリング・レビュー・受注理由で裏付ける | 主観VPは形骸化 |
| 全ペインに対応するペインリリーバーが揃わない | 機能追加 or 対象外と判断 | ペイン対応なしのサービスは選ばれない |
| ゲインクリエイターが「自社が提供したい機能」 | 顧客のゲインに対応した機能のみ残す | 過剰機能はターゲットを狭める |
| 競合比較を省略 | 競合3社以上と並べて差別化要素を抽出 | 比較なき差別化は楽観バイアス |
| 価値提案が3文以上の長文 | 1〜2文に圧縮 | 長いVPは伝わらない |
| 1ペルソナで全顧客カバー | 主要ペルソナ別に作成 | 価値はペルソナで変わる |
| 1度作ったら見直さない | 半年〜1年で再評価 | 顧客課題は動く |
