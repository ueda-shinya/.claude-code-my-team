# 受注失注分析・KBF/KSF特定スキル

営業活動の受注／失注パターンを分析し、顧客の購買決定要因（KBF）と業界での成功要因（KSF）を特定する。新規事業の場合は見込み顧客ヒアリングで代替。3C分析と統合して戦略立案まで一気通貫。

## 適用条件

- 営業活動データを蓄積した状態で勝ちパターン・負けパターンを抽出したい
- 提案資料の差別化軸を明確にしたい
- マーケ戦略のターゲットセグメント精緻化
- 解約防止施策（カスタマーサクセス）

## 入力

| 項目 | 必須/任意 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー語 |
| 商材 | 必須 | 製品・サービス |
| 受注データ | 任意 | 既存顧客の業種/規模/目的等 |
| 失注データ | 任意 | 失注理由・競合社名 |
| 解約データ | 任意 | 既存顧客の解約理由 |

## 実行ステップ

### ステップ1：受注分析（既存顧客がいる場合）

**5セグメント軸で顧客属性を整理**：

| セグメント | 例 |
|---|---|
| **1. 顧客属性（Who）** | 業種・業界／企業規模／役職／新規 or 既存 |
| **2. 商談属性（What/How）** | 商材カテゴリ／商談経路／商談件数／商談ステータス／価格帯／提案内容 |
| **3. 時間軸（When）** | 月別・四半期別／リードタイム／商談期間／営業稼働タイミング |
| **4. 担当者・組織（Who sells）** | 営業担当者／チーム・拠点／フォロー体制 |
| **5. 顧客課題・ニーズ（Why）** | 課題タイプ／導入目的／競合との比較軸 |

**訴求ニーズ3点に注目**：
- 導入の決め手（何を決め手として決定したか）
- 興味をもったきっかけ（何に興味を持って認知したか）
- アクションの方法（何でアクションしてもらったか）

ヒアリング・アンケート取得が望ましい。

### ステップ2：失注分析

**失注理由マトリクス**（提案後失注時）：

| 失注理由 | 詳細 | 回避策 |
|---|---|---|
| 価格が高い | 競合より高いと判断 | 価値訴求強化／分割・段階導入 |
| 決裁者との接点なし | 現場担当者だけで完結 | 早期に決裁者特定／複数ステークホルダーヒアリング |
| 提案がニーズとズレ | 顧客課題を正しく把握できず | ヒアリングフェーズ強化／仮説検証の場 |
| タイミングが合わない | 予算・優先順位で見送り | 定期フォローCRM活用／短期/中長期2段構え |
| 競合に負けた | 比較検討で他社選定 | 競合分析定期化／差別化ポイント明確化 |
| 社内調整が進まず | 顧客社内の稟議が滞る | 導入フロー可視化支援／導入事例提供 |
| 提案スピードが遅い | 提案タイミングを逃す | 案件管理強化／テンプレ活用 |
| 信頼感が足りない | 実績・印象が弱い | 事例紹介充実／個人信頼構築 |
| フォロー不十分 | 商談後連絡が遅い | アクションプラン提示／次の一手明確化 |

**解約分析マトリクス**（既存顧客解約時）：

| 解約理由 | 詳細 | 回避策 |
|---|---|---|
| 効果未実感 | 期待した成果出ず | KPI設定共有／オンボーディング |
| 利用頻度低下 | 社内で使われず | 利用モニタリング／トレーニング |
| 操作難・UI悪い | 定着せず | UI/UX改善／FAQ動画 |
| 類似サービス乗換 | 競合の方が魅力 | 競合調査／差別化発信 |
| 価格高と感じた | 費用対効果疑問 | 段階料金プラン／ROI報告 |
| 社内体制変更 | 担当異動で解約 | 複数担当者と関係構築／組織契約 |
| 導入目的曖昧 | 不明確で離脱 | 目的整理共有／CSの伴走 |
| フォロー不足 | サポート弱 | CS設置／定期フォロー |
| 「卒業」 | 成果出て不要判断 | 次ステージ提案／アップセル |

**解約防止5横断施策**：オンボーディング充実／CS定期接点／ヘルススコア可視化／NPSアラート／成功事例コミュニティ

**失注プロセス分析**：資料DL → 自動返信メール → IS事前架電 → 初回商談ニーズ把握 → 1週間後フォロー → 保留／失注の各段階で離脱要因特定

### ステップ3：ターゲットニーズ分析

#### 既存顧客がいる場合
1. セグメント切り口（業種／職種／事業フェーズ／従業員規模／顧客体制等）に傾向がないか検討、代表的セグメントを3〜4定義
2. 各セグメントに「問題事象／構造・状況／原因／課題／解決策／該当事例」を洗い出し

### ステップ4：KBF/KSF特定（3C統合）

**KBF（Key Buying Factor）**：顧客が購買決定で重視する要因（価格／品質／ブランド等）
**KSF（Key Success Factor）**：市場で成功するために必要不可欠な要因

#### 3Cからの導出
- 顧客分析（顧客ニーズ／ターゲット／購買プロセス）
- 競合分析（競合の強み・魅力／競合が見落としているニーズ）
- 自社分析（自社の強みと弱み／顧客要望への対応）
→ 上記3要素の交点 = **KBF**
→ KBF を踏まえた業界成功条件 = **KSF**

#### KBF分析の評価軸（4カテゴリ）
| カテゴリ | 評価項目例 |
|---|---|
| 企業の信頼性・権威性 | 会社規模／会社HP／企業理念・文化／事例／メンバー発信 |
| 顧客要因 | 時期／リソース／費用 |
| 第三者評価 | 実績／お客様口コミ／Web口コミ |
| サービス内容・コミュニケーション | 成果／投稿本数／企画力／費用／クリエイティブ／分析／手離れ／要望柔軟対応／コミット力・信頼／提案力 |

各項目を顧客視点で◎/○/△で自己評価し、重視されているKBFを抽出する。

#### KBF→KSF抽出の4手順
1. KBFの要因を洗い出す
2. 自社の顧客で要因に対する評価をマッピング
3. マッピング内容を元に「なぜか？」を深掘り
4. 深掘り内容を元にKSFを抽出

### ステップ5：戦略立案・実行

KBF/KSF特定の5ステップ：
1. **受注・失注データ収集** — 過去営業活動から案件データを集める
2. **要因分析** — 各案件の成功・失敗要因の共通点・パターン特定
3. **ターゲットニーズ特定** — 顧客の購買動機からKBFを明確化
4. **KSF抽出** — 業界・市場の成功要因から自社KSF特定
5. **戦略立案・実行** — 営業戦略・マーケ施策に反映

## 判断基準

| 状況 | 判断 |
|---|---|
| 受注データゼロの新規事業 | 見込み顧客ヒアリングで代替（ターゲットニーズ分析を主軸） |
| 失注理由が「価格」「競合」だけ | より深い背景まで掘り下げる（実績／決裁者接点／フォロー等の真因確認） |
| 解約予兆を察知したい | NPSアンケート／利用率モニタリング／ヘルススコア導入 |
| KBF項目が多すぎる | 自社評価で◎が3項目以上集中する領域に絞る |
| 競合と差別化できないと結論 | `competitive-absence-audit` 起動でバイアス確認 |

## よくある失敗パターン

| NG | OK |
|---|---|
| 失注理由を「価格」「競合」で終わる | 真因まで掘り下げる |
| KBFを自社視点で勝手に決める | 顧客アンケート・ヒアリングで取得 |
| 解約分析を商談失注と混ぜる | 解約は別マトリクスで分析 |
| KBFをKSFに直結 | 3C→KBF→KSFの順で導出 |
