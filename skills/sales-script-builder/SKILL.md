# sales-script-builder

即決営業メソッドの個別スキルを商談フェーズ順に呼び出し、1本の読み上げ可能な商談トークスクリプトに統合する。

**棲み分け：**
- 商談相談・フレームワーク選定・提案書設計 → `sales-playbook`
- 営業スライドの構成・骨子生成 → `sales-slide-builder`
- 読み上げ台本（トークスクリプト）の一気通貫生成 → **本スキル**

## 適用条件

- 使う場面：新規商談のトーク台本作成、新人研修用スクリプト、ロープレ台本、飛び込み〜クロージングまで通しで台本が欲しい場面
- 使わない場面：単一フェーズだけ欲しい（その場合は個別スキルを直接呼ぶ）、戦略相談（sales-playbook へ）、スライド作成（sales-slide-builder へ）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 商材/サービス名 | 必須 | 台本の対象 | 外壁塗装リフォーム |
| ターゲット | 必須 | 業種・役職・BtoC/BtoB | BtoC戸建オーナー50代 |
| 商談チャネル | 必須 | 対面 / 飛び込み / 電話（テレアポ含む） / オンライン | 飛び込み |
| 想定商談時間 | 必須 | 冒頭〜クロージング総尺 | 30分 |
| 主要ベネフィット | 必須 | 顧客が得る変化 | 塗装20年保証で再塗装コストゼロ |
| 決裁型 | 必須 | 即決型 / 高関与型 | 即決型 |
| 営業担当者情報 | 任意 | 自己開示用（趣味・強み・弱み） | - |
| 重点スキル指定 | 任意 | 特に強化したいパート | クロージング強化 |

## 実行ステップ

### ステップ1：商談フェーズマップを生成する
- 入力：想定商談時間
- 処理：以下のフェーズ配分を時間比率で割り付ける
  - 冒頭（10%）／アイスブレイク（15%）／ヒアリング（20%）／提案（30%）／クロージング（25%）
  - 時間を分単位で明示する
- 出力：フェーズタイムライン表

### ステップ2：呼び出すスキルを決定する
- 入力：商談チャネル、決裁型、重点スキル指定
- 処理：以下のマッピングで呼び出しスキルを選ぶ
  - 冒頭 → `first-impression-30sec`（必須）
  - アイスブレイク → `open-heart-praise` ＋ `two-second-self-disclosure`（必須）
  - ヒアリング → `sales-psychology-techniques` のバックトラッキング部分（必須）
  - 提案 → `conclusion-first-talk`（必須）＋（決裁型=高関与型なら `heaven-hell-rhetoric` を追加）
  - クロージング → 即決型なら `sokketsu-closing` / 高関与型なら `self-closing-4questions`
  - 全フェーズ横断 → `sales-psychology-techniques`（必須・FITD/DITF/聞き手の法則）
- 出力：呼び出しスキルリスト＋採用理由

### ステップ3：各スキルを順次読み込み・入力を渡して個別成果物を生成する
- 入力：呼び出しスキルリスト、ユーザー入力項目
- 処理：
  - **共通入力セットを固定する**：商材名／ターゲット／主要ベネフィット／決裁型／商談時間／チャネル を不変で全スキルに渡す
  - 各スキルの SKILL.md を Read で読み込む
  - 共通入力セットに加え、各スキル固有の項目（自己開示情報・証拠データ等）のみ追加投入する
  - 実行ステップに沿って個別成果物を生成する
- 出力：スキルごとの個別成果物セット

### ステップ4：個別成果物をフェーズ順に結合する＋心理テク重複正規化
- 入力：個別成果物セット、フェーズタイムライン表
- 処理：
  - 冒頭→アイスブレイク→ヒアリング→提案→クロージングの順に並べる
  - フェーズ間に橋渡しの1〜2文を挿入する（「それでは本題に入らせていただきます」等）
  - 重複フレーズ（ハロー効果チェック）を除去する
  - **心理テク重複の正規化ルール（必須）**：FITD / DITF / バックトラッキング / 聞き手の法則 は通し台本全体で各1回に正規化する。`sales-psychology-techniques` と他個別スキル（`conclusion-first-talk` `sokketsu-closing` 等）で同じテクが重複した場合、**後方フェーズ側の出現を優先採用し、前方側は削除する**
- 出力：結合済み通し台本

### ステップ5：総尺チェックと圧縮を行う
- 入力：結合済み通し台本、想定商談時間
- 処理：
  - 日本語読み上げ速度=1分300字で総尺を計算する
  - 想定時間±10%に収まらなければ、以下の削除優先順序で削る：
    1. 任意の追加心理テク（FITD/DITFのうち後回しに配置したもの）
    2. `conclusion-first-talk` の追加メリット文
    3. `conclusion-first-talk` の理由の補足説明
    4. アイスブレイクの `two-second-self-disclosure` レベル3（弱み）
    5. `open-heart-praise` の具体物褒めステップ
  - 必須スキル（first-impression / conclusion-first の結論・再結論 / クロージング）は削らない
  - ※ステップ1でチャネル=電話や時間15分以下の判断によりすでに省略済みの箇所は二重削除しない
- 出力：時間調整済み台本

### ステップ6：所作・間・NG行動の注記を統合する
- 入力：時間調整済み台本
- 処理：
  - 各スキルが指定した所作・沈黙指示・NG行動を台本内に【注記】として挿入する
  - 冒頭に「所作総則」セクションを追加する
- 出力：注記統合済み台本

### ステップ7：完成台本として出力する
- 入力：注記統合済み台本
- 処理：以下の固定フォーマットで整形する
- 出力：完成スクリプト

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 商談時間が15分以下 | アイスブレイクから `open-heart-praise` を省略、`two-second-self-disclosure` はレベル1のみ | 時間が足りない |
| 商談時間が60分以上 | ヒアリングフェーズを20%→30%に拡大 | 長時間商談はヒアリングで勝負 |
| BtoB法人・決裁者複数 | 決裁型を高関与型に強制変更し、出力冒頭に「決裁型を高関与型に自動変更しました」と明記する | 即決が構造上不可能 |
| チャネル=電話 | 聞き手の法則・お辞儀所作を省略、声トーン指示に差し替え | 視覚効果が無効 |
| 重点スキル指定あり | 該当フェーズの尺を＋5%増やす | 強化要望を反映 |
| 即決型かつ高単価 | sokketsu-closing を使いつつ、後半に self-closing-4questions の質問3・4を部分流用 | 即決×納得感の両立 |

### 記入例

記入例は各個別スキルの記入例を参照。本スキルは統合出力のため、個別例の結合版となる。

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 7スキル全部を毎回フル投入 | 決裁型・時間で取捨選択 | 総尺オーバーで棒読みになる |
| フェーズ間の橋渡し無し | 1〜2文の接続句を入れる | 台本が途切れて不自然になる |
| クロージング2スキルを同時投入 | 決裁型で必ず1つだけ選ぶ | 棲み分けが壊れる |
| 所作注記を削る | 必ず注記セクションに残す | 読み上げ時に動作が抜ける |
| 商材情報を各スキルに毎回入力し直さない | 初回入力を全スキルに引き継ぐ | 矛盾した台本が生まれる |
