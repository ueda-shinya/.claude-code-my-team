# evaluation-system-design スキル

評価制度 = 従業員の貢献度を評価し、モチベーション向上や人材育成、報酬決定に繋げるための重要な仕組み。適切な評価制度を策定することで、組織全体の活性化を図る。事業戦略スクール「知足」のフレームワーク準拠。

## 評価制度の5つの目的

| 目的 | 内容 |
|---|---|
| 従業員のモチベーション向上 | 個々の貢献を評価し、目標達成意欲を高める |
| 人材育成 | 強みと弱みを明確にし、個々の成長を支援 |
| 報酬決定 | 貢献度に応じた報酬を決定し、公平性を確保 |
| 組織目標の達成 | 個々の目標と組織目標を連動させ、組織全体の目標達成に貢献 |
| 人材の定着 | 従業員が組織に貢献していると感じ、定着率向上に繋がる |

## 適用条件

- このスキルを使う場面：
  - 評価制度を新規策定したい／既存制度を見直したい
  - 賞与・昇給・昇格の判定基準を明確化したい
  - 評価基準が曖昧／結果に不満／時間がかかる課題を解決したい
- このスキルを使わない場面：
  - 役割と責任範囲の明文化 → `roles-responsibilities`
  - 給与レンジ決定 → `salary-range-design`（別スキル）
  - 目標KPI設計 → `kgi-kpi-kai-design`
  - キャリアロードマップ → `career-roadmap-development`（別スキル）
  - 採用戦略 → `recruitment-strategy`（別スキル）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 会社情報 | 必須 | 業種・規模・既存制度有無 | 「Web制作・20名・評価制度なし」 |
| 評価対象 | 必須 | 全社員／管理職／営業／エンジニア | 「全社員（営業・非営業で分ける）」 |
| 組織目標 | 必須 | 評価と連動させる目標 | 「売上20%増・離職率5%以下」 |
| 既存の評価項目 | 任意 | あれば | 「目標達成度のみ・他なし」 |
| 評価頻度の希望 | 任意 | 年1回／半期／四半期 | 「半期（年2回）」 |

## 評価制度策定の7ステップ

| STEP | 内容 |
|---|---|
| 01. 目的の明確化 | 評価制度を導入する目的を明確にする（モチベーション向上、人材育成、報酬決定など） |
| 02. 評価基準の設定 | 評価項目を具体的に設定（目標達成度、能力、行動など）／各項目の重要度を数値化しウェイト付け |
| 03. 評価方法の選定 | 目標管理制度（MBO）、360度評価、バランススコアカードなどから選択 |
| 04. 評価者の選定 | 上司、同僚、部下など、誰が評価を行うのかを決定 |
| 05. 評価期間の設定 | 評価の頻度を決定（年1回、半期に1回など） |
| 06. 評価結果のフィードバック | 評価結果を個々にフィードバックし、改善点や今後の目標を設定 |
| 07. 評価制度の周知徹底 | 全員が評価制度を理解し、運用できるように周知徹底 |

## 評価制度設計の5ポイント

| ポイント | 内容 |
|---|---|
| 公平性 | 評価基準を明確にし、全員に公平に評価を実施 |
| 客観性 | 定量的なデータだけでなく、定性的な評価も組み合わせる |
| 透明性 | 評価基準やプロセスを透明化し、納得感を与える |
| 双方向性 | 評価者と被評価者の間で意見交換を行い、相互理解を深める |
| 継続的な改善 | 定期的に評価制度を見直し、改善していく |

## グランドデザイン（推奨3軸）

事業戦略スクール「知足」のkoujitsu原典より：

| 仕組み | 内容詳細 | 焦点 | 基本給/昇降給 | 賞与支給率 | 昇格/降格 |
|---|---|---|---|---|---|
| **目標達成評価** | 今期（個人）目標の達成度・進捗度 | 短期・一過性 | 50% | 50% | 50% |
| **らしさ評価** | 職務における自社らしさの発揮度 | 中長期・継続的 | 50% | 50% | 50% |
| **360評価** | 日頃の仕事ぶりについて、共に働く仲間がフィードバック | 日常の仕事ぶり | （参考） 加点・減点 | （参考） | （参考） |

→ 短期成果（目標達成）と中長期文化（らしさ）を50:50でバランス、360評価は参考値として加減点。

## 目標達成評価の具体例（ウェイト付き5段階）

| 評価項目 | ボリューム | 自己評価 | 上司評価 |
|---|---|---|---|
| 売上達成 | 40% | 100% | 100% |
| 新規顧客の開拓 | 20% | 90% | 70% |
| 新人の指導 | 20% | 100% | 100% |
| チームビルディング | 10% | 80% | 100% |
| アプローチマニュアル | 10% | 50% | 40% |

→ 上司が目標各々の進捗を総合的に評価し、5段階で総合評語をつける（A：期待レベルを上回っていた／B：期待レベル通り 等）

→ 結果は「賞与」に反映

## 年2回評価フロー（推奨）

| ステップ | 上期 | 下期 |
|---|---|---|
| 数字の確定→自己評価記入 | 2月第1週 | 8月第1週 |
| 一次評価＋フィードバック（部門長） | 2月第2週 | 8月第2週 |
| 二次評価＋評価調整会議（役員） | 2月第3週 | 8月第3週 |
| 最終フィードバック・賞与支給 | 2月第4週〜 | 8月第4週〜 |

**ルール**：
- 一次評価（および一次フィードバック）は部門長が実施
- 役員による「評価調整会議」で二次評価・最終評価が確定
- 最終決裁者は社長
- 一次評価と次期目標の設定は同時（同タイミング）で行う

## よくある4つの課題と解決策

| 課題 | 解決策 |
|---|---|
| 01. 評価基準が曖昧 | 評価基準を具体的に数値化し、客観的な評価を行う |
| 02. 評価結果への不満 | 評価結果に対するフィードバックを徹底し、改善点を共有する |
| 03. 評価に時間がかかる | 評価方法を簡素化しオンラインツールを活用する |
| 04. 評価結果が活かされない | 評価結果を人事評価や報酬に連動させる |

## 評価制度導入の3つの注意点

| 注意点 | 内容 |
|---|---|
| トップのコミットメント | トップが評価制度の重要性を理解し、率先して取り組むことが大切 |
| 全社員への周知徹底 | 評価制度の内容を全社員に周知徹底し、理解を深める必要がある |
| 柔軟な運用 | 組織の状況や変化に合わせて、評価制度を柔軟に運用する必要がある |

## 評価制度のルール（7原則）

| # | 原則 | 内容 |
|---|---|---|
| 01 | 定期評価の実施 | 3ヶ月に1度、自己評価と会社による評価を行う |
| 02 | 評価基準の透明性 | 定量的および定性的な評価基準を明確にし、適格な評価を実行 |
| 03 | 具体的な指標の設定 | 各評価項目に対して具体的な指標や目標を設定し、評価の公正性を確保 |
| 04 | 目標設定の見直し | 各評価サイクルの後に目標を見直し、個人の成長と組織のニーズに合わせて調整 |
| 05 | 結果の活用 | 評価結果を人材の配置、研修、報酬の決定などに活用 |
| 06 | 評価プロセスの継続的な改善 | 評価プロセス自体も定期的にレビューし、メンバーからのフィードバックを取り入れて改善 |
| 07 | 高エンゲージメントメンバーの底上げ | 社内・業務に対しての高エンゲージメントメンバーを底上げする評価制度とする |

## 実行ステップ

### ステップ1：目的の明確化（プロセス1）
- 入力：会社情報、組織目標
- 処理：
  1. 5目的（モチベーション／人材育成／報酬決定／組織目標達成／人材定着）から優先目的を決定
  2. 評価制度の存在意義を1〜3項目で言語化（自社らしい言葉で）
  3. 「評価結果が反映される処遇」を明示（基本給/賞与/昇格 等）
- 出力：評価制度の目的文＋反映処遇の明示

### ステップ2：評価基準の設定（プロセス2＋グランドデザイン）
- 入力：目的、組織目標
- 処理：
  1. **3軸グランドデザインを採用**（目標達成評価／らしさ評価／360評価）
  2. 各軸のウェイトを決める（推奨：目標達成50%＋らしさ50%、360は参考値）
  3. 各軸の中の評価項目を具体化＋ウェイト付け（業績30点・外見20点・内面25点・業務25点 等）
  4. 評価点基準を10段階等で数値化（売上達成率の例：0%/20%/30%/.../150%超）
- 出力：評価基準シート（軸×項目×ウェイト×評価点基準）

### ステップ3：評価方法の選定（プロセス3）
- 入力：評価基準
- 処理：
  1. **MBO（目標管理制度）**：目標達成評価軸で採用
  2. **360度評価**：360評価軸で採用（参考値）
  3. **バランススコアカード**：必要に応じて検討
  4. 5段階総合評語（A：期待レベルを上回っていた／B：期待レベル通り／…）の判定基準を設計
- 出力：評価方法と判定基準

### ステップ4：評価者の選定（プロセス4）
- 入力：組織体制
- 処理：
  1. **一次評価者**：直属の部門長
  2. **二次評価者**：役員（評価調整会議で確定）
  3. **最終決裁者**：社長
  4. **360評価者**：日常で関わる同僚・部下
- 出力：評価者役割表

### ステップ5：評価期間の設定（プロセス5）
- 入力：会社の事業サイクル
- 処理：
  1. 推奨：年2回（上期/下期、半期評価）
  2. 各回4ステップ：①数字確定→自己評価／②一次評価＋FB／③二次評価＋調整会議／④最終FB＋賞与支給
  3. タイミング：上期2月、下期8月（自社の決算期に合わせて調整）
  4. 一次評価と次期目標設定を同タイミングで実施
- 出力：年間評価カレンダー

### ステップ6：評価結果のフィードバック（プロセス6）
- 入力：評価結果
- 処理：
  1. 1on1形式で個別FB（30〜60分）
  2. 「良かった点」「改善点」「次期の目標」の3点を共有
  3. 双方向性を重視（被評価者の意見も聞く）
  4. 文書化して共有（後で見返せる形）
- 出力：FBテンプレ＋運用ルール

### ステップ7：周知徹底（プロセス7）
- 入力：完成した評価制度
- 処理：
  1. 全社説明会で評価制度の目的と運用を解説
  2. ハンドブック化（紙またはデジタル）
  3. オンボーディング資料に組み込み
  4. 「評価制度のルール7原則」を毎期共有
- 出力：周知計画＋ハンドブック

### ステップ8：自己点検（3観点＋4課題チェック）
- 入力：運用後のデータ
- 処理：
  1. **3観点点検**：①FBが適切に行われているか／②報酬や昇進と連動しているか／③従業員の納得感や透明性を確保しているか
  2. **4課題チェック**：基準曖昧／結果不満／時間かかる／結果未活用 の有無
  3. 改善計画を立て次回評価サイクルに反映
- 出力：自己点検結果＋改善計画

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 評価制度なしから新規策定 | 7ステップ全実施＋3軸グランドデザイン採用 | 一気通貫で公平性・透明性を担保 |
| 既存制度の見直し | 4課題チェックから着手 | 改修すべき部分を特定 |
| 部門ごとに業務が違う | セールス100点／非セールス70点 等で軸を分ける | 一律基準は不公平 |
| 評価頻度年1回のみ | 半期（年2回）に変更を提案 | 1年は長く、PDCAが遅い |
| トップが評価に関与しない | トップコミットメントを必ず確保 | 形骸化の最大原因 |
| 結果が報酬に反映されない | 反映処遇を明示し連動 | モチベーション低下 |
| 360評価を本評価扱いする | 360は参考値（加減点）に留める | 主観バイアスが強すぎる |
| 役割と責任範囲が未定 | `roles-responsibilities` を先行 | 評価指標の根拠が立たない |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 目標達成評価のみで「らしさ」を見ない | 3軸（目標達成・らしさ・360）採用 | 短期成果のみだと文化醸成が弱い |
| 評価基準を「がんばり」等の抽象表現 | 数値化＋ウェイト付け | 客観評価が不可能 |
| 年1回評価のみ | 半期（年2回）以上 | PDCAが遅すぎる |
| トップが関与せず人事任せ | トップコミットメント必須 | 形骸化の最大原因 |
| 結果が報酬・昇進に反映されない | 反映処遇を必ず明示・連動 | モチベーション低下 |
| 360評価を本評価扱い | 360は参考値（加減点）に留める | 主観バイアスが強い |
| FBなしで結果通知のみ | 1on1で双方向FB | 納得感が得られない |
| 制度を一度作って見直さない | 半期/年次で見直しサイクル | 環境変化に対応できない |
| 評価ルール7原則の周知不足 | 全社員にハンドブック配布＋説明会 | 運用フェーズで形骸化 |

## 連携スキル

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：評価対象が0件で設計できない場合（要再ヒアリング）
- `[PARTIAL:N成功/M失敗]`：7ステップのうち一部のみ完成した場合
- `[SKIP]`：組織目標・役割定義が未確定で評価制度設計を保留した場合（前提不足）

※外部API依存なしのため `[FAIL]` は通常発生しない。前提不足は `[SKIP]` で扱う。
