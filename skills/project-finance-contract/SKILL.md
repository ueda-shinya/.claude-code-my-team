# プロジェクト収支管理・契約締結スキル

プロジェクトの利益確保・コスト超過防止・キャッシュフロー安定化を実現する収支管理と、トラブルを未然に防ぐ契約締結を一体で設計。

## 適用条件

- 新規プロジェクトの予算策定・契約条件設計
- 既存PJの利益率改善
- 受注／発注時の契約形態選定（請負／準委任／成功報酬）
- リスク条項の整備

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| プロジェクト規模 | 必須 | 金額・期間・人員 |
| 自社立場 | 任意 | 受注側／発注側 |

## 実行ステップ

### ステップ1: 予算策定（コスト見積もり3手法）

| 手法 | 内容 | 適用場面 |
|---|---|---|
| **ボトムアップ見積もり** | タスク単位で積み上げて計算 | 詳細仕様が決まっている場合 |
| **トップダウン見積もり** | 過去のプロジェクトデータを参考に算出 | 大枠のみ決まっている初期検討 |
| **類推見積もり** | 似たプロジェクトの実績から概算 | 経験則での粗い見積もり |

予算策定 = 必要なリソース（人件費・設備・外注費）を洗い出し適切な予算設定。

### ステップ2: 収支モニタリングとコントロール（3要素）

| 要素 | 内容 |
|---|---|
| **KPI設定** | 収益性を測る指標（利益率・ROI等） |
| **進捗管理とコスト管理の統合** | 進捗に応じたコスト把握、過剰支出を未然防止 |
| **EVM（アーンド・バリュー・マネジメント）** | 進捗とコストを可視化する手法 |

### ステップ3: 収益最大化の3ポイント

| ポイント | 内容 |
|---|---|
| **適切な価格設定** | 原価＋利益率を考慮した価格戦略 |
| **コスト削減施策** | 工程の効率化／無駄の排除／リソース最適活用 |
| **キャッシュフロー最適化** | 前払い契約・分割支払い交渉で資金繰り安定化 |

### ステップ4: 契約の4基本要素

| 要素 | 内容 |
|---|---|
| **スコープ（範囲）** | 提供するサービス・成果物の明確化 |
| **納期・スケジュール** | 遅延時の対応策も含める |
| **コスト・支払い条件** | 金額／支払い時期／追加費用の条件 |
| **責任分担** | リスク発生時の対応策を規定 |

### ステップ5: 契約3形態とリスク管理

| 契約形態 | 特徴 | 受注側リスク |
|---|---|---|
| **請負契約（固定報酬）** | 事前に報酬決定 | コスト超過リスクは受注側が負う |
| **準委任契約（時間単価制）** | 実作業時間に応じた支払い | プロジェクト柔軟性高、工数管理が重要 |
| **成功報酬型契約** | 成果に応じた支払い | リスク高だが利益率上がる可能性あり |

**リスク対策必須条項**：
- 契約解除条項
- 損害賠償条項
- 不可抗力条項

### ステップ6: 交渉の3ポイント

| ポイント | 実践 |
|---|---|
| **Win-Winの関係構築** | お互いに納得できる条件で合意、長期信頼関係を築く |
| **バッファ（余裕）を持たせた交渉** | スケジュール・コスト・成果物に余裕を持たせて突発変更に対応 |
| **リスク回避策の明文化** | トラブル時の責任範囲・対応を事前に明確化、紛争を未然に防ぐ |

## 判断基準

| 状況 | 判断 |
|---|---|
| 仕様が明確 + コスト超過リスク取れる | 請負契約（固定報酬） |
| 仕様変更が多そう | 準委任契約（時間単価） |
| 自社の自信が高い + 大きな利益狙う | 成功報酬型契約 |
| 自社受注で資金繰りが厳しい | 前払い・分割支払い交渉でキャッシュフロー優先 |
| 法的リスクを完全排除したい | ケン（legal-advisor）にレビュー依頼 |
