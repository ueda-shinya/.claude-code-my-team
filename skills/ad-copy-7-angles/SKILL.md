# ad-copy-7-angles

広告コピーを7つの切り口で作成するスキル。1つの商品/サービスから複数のコピー案を生成し、A/Bテスト用の複数バリエーションをワンセット出す。

## 適用条件

- このスキルを使う場面：
  - SNS広告（Meta／X／TikTok等）の本文/キャッチコピー作成
  - リスティング広告／ディスプレイ広告のテキスト
  - バナー広告の見出しテキスト
  - LPファーストビューのキャッチコピー
  - A/Bテスト用の複数コピー候補の一括生成
- このスキルを使わない場面：
  - LP本文・記事・メルマガなど長文コピー → コト（copywriter）の通常フロー
  - SEO記事タイトル → 別途SEO系スキル
  - プレスリリース → 別途PRスキル

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 商品/サービス名 | 必須 | コピー対象 | 「テンピュール」 |
| 主要ベネフィット | 必須 | 顧客が得られる利益 | 「最高の睡眠」 |
| ターゲット層 | 必須 | 顕在層／潜在層／その他層別 | 「睡眠の質を上げたい30-50代」 |
| ターゲットの課題/痛み | 任意 | コンプレックスや悩み | 「夜中に何度も目が覚める」 |
| 比較対象 | 任意 | 類似商品やわかりやすい比喩 | 「他社マットレス」 |
| 数字/データ | 任意 | 価格・成果・所要時間など定量情報 | 「1日56円」「2か月で-10kg」 |
| 権威性要素 | 任意 | 受賞歴・専門家推薦・実績 | 「コスメアワード受賞」 |
| 文字数制限 | 任意 | 媒体上限 | 「90文字以内」 |

## 実行ステップ

### ステップ1：入力情報を整理する

- 入力：商品/サービス情報
- 処理：以下8項目を整理する
  - 商品名・主要ベネフィット
  - ターゲット層と層別状態（顕在層／潜在層／離反層）
  - ターゲットの課題・痛み
  - 比較対象（あれば）
  - 数字・データ（あれば）
  - 権威性要素（あれば）
  - ターゲットの「心の声」（一人称で書く）
  - 媒体の文字数制限
- 出力：整理済み情報シート

### ステップ2：7つの切り口でコピー案を生成する

- 入力：整理済み情報シート
- 処理：以下7切り口で各1〜3案を作成する

  **① ベネフィットを伝える**
  - 対象層：顕在層（既に商品/サービスを知り、検討中の層）
  - 公式：5W1Hでリアルに具体化したベネフィット
  - 例：「最高の睡眠は1日たった56円」（テンピュール）
  - チェック：抽象表現を避け、数字・具体物・時間を入れる

  **② ニーズを伝える**
  - 対象層：潜在層（課題には気づいているが商品を知らない層）
  - 公式：「〇〇が気になるあなたに」「〇〇したいなら」（自分事化を促す）
  - 例：「野菜不足が気になるあなたに！」（ファンケル青汁）
  - チェック：「あなた」「自分」を入れて自分事化

  **③ 課題をつく**
  - 対象層：コンプレックスや痛みを持つ層
  - 公式：疑問形でコンプレックスを直撃／放置すると続く未来を示唆
  - 例：「一生剃り続けるの？」（TCB）
  - チェック：直接的すぎて反感を買わないか確認

  **④ 比較する**
  - 対象層：他社商品と比べている層
  - 公式：類似商品 or 誰もが知る対象との対比で優位性を示す
  - 例：「クリーニング級の消臭パワー」（ファブリーズ）
  - チェック：景表法違反（比較対象明示なしの優良誤認）に注意

  **⑤ 変化を伝える**
  - 対象層：結果を出したい層
  - 公式：ビフォーアフター／時間×変化量を明示
  - 例：「たった2か月で、このカラダ。」（ライザップ）
  - チェック：個人差や条件を併記（薬機法対象は特に注意）

  **⑥ 共感する**
  - 対象層：広告を嫌う層・他人の声を信じる層
  - 公式：ユーザー体験談・口コミ風に書く
  - 例：「もうこれしか使えないかも」（ファンケルクレンジング）
  - チェック：実際の声をベースに書く（捏造禁止）

  **⑦ 感情移入する**
  - 対象層：強い感情・衝動を持つ層
  - 公式：ターゲットの心の声をそのまま発する（一人称・口語）
  - 例：「ちくしょう。転職だ！」（Green）
  - チェック：ターゲットインサイトと一致しているか

- 出力：7切り口×各1〜3案 = 計7〜21案

### ステップ3：追加要素（数字・権威性）を組み込む

- 入力：ステップ2の各コピー案
- 処理：以下2要素のうち該当するものを各案に追加する
  - **数字**：具体的な金額／日数／量（例：「1日たったの30円」「1杯で300g分の野菜」）
  - **権威性**：受賞歴／専門家推薦／導入実績（例：「コスメアワード多数受賞」「XX氏絶賛」）
- 数字を入れるルール：
  - 単位が小さいほどインパクト（年5,000円より日13円）
  - 切りの良い数字より具体数字（10万円より98,000円）
- 権威性のルール：
  - 第三者の推薦／メディア掲載／受賞／導入社数のいずれか
  - 嘘や誇張は厳禁（景表法・優良誤認）
- 出力：強化版コピー案

### ステップ4：媒体文字数制限でフィルタする

- 入力：強化版コピー案、媒体文字数制限
- 処理：
  - 媒体ごとの上限例：
    - Meta広告本文：90文字以内推奨
    - Google検索広告タイトル：30文字×3
    - Google検索広告説明文：90文字×2
    - X広告：140-280文字（リンクを除く）
    - LPファーストビュー：30文字以内推奨
  - 制限超過の案は短縮または除外する
- 出力：媒体適合済みコピー案

### ステップ5：A/Bテスト用に整理して出力する

- 入力：媒体適合済みコピー案
- 処理：以下フォーマットで整理する
  - 切り口番号と名称を明記
  - 想定ターゲット層を併記
  - 文字数を併記
  - A/Bテスト推奨ペア（顕在層向けと潜在層向け、感情訴求と論理訴求など）を提案
- 出力：コピー案リスト＋A/Bテスト推奨ペア

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ターゲットが顕在層 | ①ベネフィット／④比較／⑤変化を中心に作る | 既知の課題に対してメリットを訴求するのが効く |
| ターゲットが潜在層 | ②ニーズ／③課題／⑦感情移入を中心に作る | 自分事化を促す必要がある |
| ターゲットが反広告層 | ⑥共感を中心に作る | ユーザー目線・体験談ベースが信頼を獲得 |
| 強い差別化要素がある | ④比較を活用 | 比較で優位性が際立つ |
| ビフォーアフターが明確 | ⑤変化を活用 | 数字＋時間で説得力UP |
| 数字データがある | 必ず数字を入れる | 抽象表現より圧倒的にCTRが高い |
| 権威性がある | 必ず権威性を入れる | 信頼性で意思決定を後押し |
| 薬機法対象（化粧品/医療/サプリ） | ③課題／⑤変化は表現を抑える | 個人の感想＋効果効能の禁止表現を避ける |
| 金融商品 | ④比較／⑤変化は注意 | 元本保証等の優良誤認禁止 |
| 文字数制限が厳しい（30文字以内）| ①ベネフィット／⑦感情移入が短く決まりやすい | 長い説明は不要 |

### 記入例

```markdown
# 広告コピー案 — 月額ITサポート（広島県中小企業向け）

## 入力サマリー
- ターゲット層：潜在層中心（ITトラブルに悩むが既存サービスを知らない経営者）
- 主要ベネフィット：月額固定でIT全般を任せられる安心
- 媒体・文字数制限：Meta広告本文 90文字以内

## 7切り口×コピー案

### ①ベネフィットを伝える
- 案A：月額3万円で「IT全部おまかせ」。広島の中小企業100社が選んだ理由。 [40文字]

### ②ニーズを伝える
- 案A：「ITトラブルで業務が止まる」が気になる経営者へ。月額固定で全部解決します。 [42文字]

### ③課題をつく
- 案A：いつまで「IT担当不在」のまま走り続けますか？広島の中小企業の8割が抱える悩み。 [44文字]

### ④比較する
- 案A：人を雇うより安い、月額3万円。専属IT部門があなたの会社にやってきます。 [40文字]

### ⑤変化を伝える
- 案A：導入1ヶ月で「IT問い合わせゼロ」。3ヶ月で「業務効率20%アップ」。 [38文字]

### ⑥共感する
- 案A：「もう自分で対応しなくていい」— 導入企業の経営者が口を揃える一言。 [37文字]

### ⑦感情移入する
- 案A：もう、ITで悩むのは終わりにしよう。 [19文字]

## A/Bテスト推奨ペア
- ペア1：②ニーズ案A vs ③課題案A（潜在層への訴求方法の比較）
- ペア2：⑤変化案A vs ⑦感情移入案A（論理 vs 感情の効きを見る）

## 制作上の注意
- 「100社」「8割」等の数字は実績裏付けがあるもののみ採用（景表法）
- 「業務効率20%アップ」は導入企業の平均値であることを併記または除外検討
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 7切り口のうち1〜2しか作らない | 必ず全7切り口で生成する | A/Bテストの母数が確保できない |
| 数字を入れずに「お得」「安い」だけ | 「月3万円」「1日100円」と具体化 | 抽象は伝わらない |
| 「業界No.1」のような根拠なき優良誤認 | 出典付き数字／実績裏付けあるもののみ | 景表法違反リスク |
| 薬機法対象で「治る」「効く」表現 | 「サポート」「整える」等の婉曲表現 | 行政指導・配信停止リスク |
| ターゲット心の声を勝手に作る | 実際のヒアリング/口コミから抽出する | インサイトと乖離するとスベる |
| 文字数制限を無視して書く | 媒体上限内に収める | 表示で省略され伝わらない |
| 7切り口の使い分けを意識せず混ぜる | 切り口別に整理して提示する | A/Bテスト設計ができない |
| 比較対象を明示せず「最高」「No.1」 | 比較対象を明示するか、抽象表現を避ける | 景表法違反 |
| 文字数だけ揃えて意味が薄いコピー | 短くても主張が立つコピー | 短縮のために本質を削らない |
