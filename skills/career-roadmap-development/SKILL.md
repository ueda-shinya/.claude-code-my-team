# career-roadmap-development スキル

キャリアロードマップ（従業員が企業内でどのように成長しキャリアを築いていくかを示した計画）と育成計画・アサインメント基準を一体的に設計し、人材定着・育成・モチベーション向上・組織生産性向上を実現する。事業戦略スクール「知足」が定義する3役割／3キャリアパス／5階層ロードマップ／5フェーズ育成計画／アサインメント設計を統合的に運用する。

## 適用条件

- このスキルを使う場面：
  - 自社のキャリアロードマップ（昇進ルート・キャリアパス）を設計したい
  - 1〜3年の育成計画を立てたい
  - アサインメント基準（誰がどのプロジェクトを担当するか）を明文化したい
  - 人材定着率を上げたい・離職リスクを下げたい
  - 従業員のモチベーション向上の仕組みを作りたい
- このスキルを使わない場面：
  - 採用戦略・採用ガイドライン → `recruitment-strategy` スキルを使う
  - 給与レンジの決定 → `salary-range-design` スキルを使う
  - 評価制度の策定（評価軸・評価期間）→ `evaluation-system-design` スキルを使う
  - 役割と責任範囲の明確化 → `roles-responsibilities` スキルを使う
  - 組織計画・組織図設計 → `organization-planning` スキルを使う

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガーフレーズのいずれか | 「キャリアロードマップ作って」 |
| 会社・事業の前提 | 必須 | 業種・規模・成長フェーズ | 「広島のWeb制作会社、社員8名、急成長期」 |
| 主要職種 | 必須 | ロードマップを設計したい職種 | 「マーケター／エンジニア／ディレクター」 |
| 既存の階層・等級 | 任意 | 現状の役職・等級（あれば） | 「アシスタント／メンバー／マネージャー」 |
| 育成期間 | 任意 | 何年計画にするか | 「3年計画」「5年計画」 |
| キャリア志向の選択肢 | 任意 | 管理職／専門職／起業の3コース提供有無 | 「3コース全部用意したい」「専門職のみ」 |

## 実行ステップ

### ステップ1：キャリアロードマップの3つの役割を確認する

**自社にとってキャリアロードマップを設計する目的を以下3観点で言語化する：**

| 役割 | 内容 |
|---|---|
| ROLE01 従業員の成長を促進 | 従業員が自身のキャリアパスを明確にイメージできることで、目標を持ってスキル向上や経験の蓄積に取り組みやすくなる |
| ROLE02 人材の定着率向上 | キャリアの見通しが立たないと、優秀な人材ほど他社へ転職するリスクが高まる。明確なロードマップがあることで「この会社で成長できる」と感じ、定着率が向上する |
| ROLE03 モチベーションの向上 | 成長のステップや評価基準が明確になることで、従業員はやる気を持って仕事に取り組むことができる。「どのスキルを伸ばせば次のステージに進めるか」が明確になることで、適切な努力がしやすくなる |

ヒアリング項目：
- 自社で最も重視する役割はどれか？
- 過去に「キャリアの見通しが立たない」「成長機会が見えない」と離職された事例はあるか？

### ステップ2：3つのキャリアパスの選択肢を設計する

**企業の職種や状況に合わせて、複数パターンのキャリアパスを設計する：**

| キャリアパス | 内容 |
|---|---|
| 管理職コース（マネジメント職） | リーダー・マネージャー・役員への昇進ルート |
| 専門職コース（スペシャリスト職） | 特定分野のエキスパートとして成長するルート |
| 新規事業・起業コース | 社内起業や事業開発に挑戦するルート |

**設計のポイント**：
- 1コースのみだと「管理職にならないと出世できない」という閉塞感が生まれる
- 3コース全てを提供する必要はないが、最低2コース（管理職＋専門職）は用意することを推奨

### ステップ3：5階層のキャリアロードマップを設計する

**3コース×5階層のマトリクスでロードマップを作成する（マーケティング会社の例）：**

| 階層 | マネジメント | スペシャリスト | 新規事業 |
|---|---|---|---|
| アシスタント | 基礎スキル（広告運用、SEO、SNS、データ分析）を習得 | 同左 | 同左 |
| マーケター | クライアント案件を多数経験し、マーケティングスキルを習得 | 同左 | 同左 |
| リーダー | プロジェクトリーダーとしてチームを率いる | 会社の専門領域をリードするポジション | 社内の新規サービス企画・テスト運用 |
| マネージャー（部長クラス） | チームマネジメント（評価・採用・予算管理）、経営層との連携 | 専門領域における登壇、社内研修の責任者 | 社内新規事業の事業責任者 |
| ディレクター（役員クラス） | 全社のマーケティング戦略を統括 | 研究活動、新技術の調査等。IT技術・広告・AI・UX等特定領域の専門家として会社全体の品質のトップラインを引き上げる | 新規事業のスピンアウト、子会社の代表、新規事業部のマネジメント |

**設計のポイント**：
- アシスタント／マーケターまでは3コース共通（基礎習得期）
- リーダー以降で3コースが分岐する
- 自社の職種に応じて、上記マトリクスをカスタマイズする（エンジニア／営業／デザイナー等）

### ステップ4：1〜3年の育成計画を設計する

**5フェーズの育成計画（デジタルマーケティング会社の例）：**

| 期間 | 育成フェーズ | 目標 | 具体的な育成施策 |
|---|---|---|---|
| 入社〜6ヶ月 | 基礎習得 | 基礎知識を学び、実務に慣れる | ・座学研修（広告運用、SEO、SNS、データ分析）<br>・OJT（先輩社員の業務サポート）<br>・週次1on1で進捗確認 |
| 6ヶ月〜1年 | 実務対応 | 小規模案件を担当し、成果を出せるようになる | ・Google広告やMeta広告の運用を担当<br>・クライアントとのコミュニケーション訓練<br>・月次のフィードバックと改善 |
| 1年〜2年 | 応用スキル習得 | 大規模案件の運用や戦略提案ができるようになる | ・成果分析のPDCAサイクルを回す<br>・KPI設計・レポーティングの精度向上<br>・外部セミナーや資格取得支援 |
| 2年〜3年 | リーダー育成 | プロジェクトを率い、チームマネジメントを経験 | ・若手の教育・メンター業務<br>・クライアントのマーケティング戦略立案<br>・チームマネジメント研修 |
| 3年以降 | スペシャリスト or マネージャー | 組織の中心として活躍 | ・キャリアパスに応じた育成（専門職 or 管理職）<br>・経営視点の戦略策定・事業開発 |

**重要ポイント**：
- 育成計画により社員の早期戦力化や定着が進み、業務成果の向上や離職率の低下につながる
- 結果として組織力や生産性が高まり、企業の成長を支える基盤となる
- 業種・職種に応じて施策を調整（IT技術／医療／飲食／製造業など）

### ステップ5：アサインメント設計（人のレベル×プロジェクトのレベル）

**アサインメントとは**：一人一人の到達レベルと、アサインするプロジェクトの難易度を見極め、社員にとってもチャレンジ（成長機会）でありつつ、会社としてもクオリティ担保できるラインを保つこと。

#### 5-1. 人のレベルの定義
**プロジェクトの種類×メンバーのレベルのマトリクスを作成し、レベルに名前をつける（マーケティング会社の例）：**

| 等級 | アシスタント | コンサルタント | シニアコンサルタント | プロジェクトマネジャー | ディレクター |
|---|---|---|---|---|---|
| インサイドセールス | IS1 | IS2 | IS3 | レベル4 | レベル5 |
| アカウントプランナー（広告） | AP1 | AP2 | AP3 | レベル4 | レベル5 |
| SEOコンサルタント | SEO1 | SEO2 | SEO3 | レベル4 | レベル5 |
| マーケティング戦略 | ST1 | ST2 | ST3 | レベル4 | レベル5 |
| マーケティングBPO | BPO1 | BPO2 | BPO3 | レベル4 | レベル5 |

**重要ポイント**：
- 各レベルに対して第三者が判別できる定義を設ける
- 社員一人一人が、どのレベルの人なのかを分かるようにしておく

#### 5-2. プロジェクトのレベルの定義
**5指標×5レベルのマトリクスでプロジェクト難易度を定義する：**

| 指標 | レベル1（スポット施策） | レベル2（小規模マーケ戦略） | レベル3（事業成長戦略） | レベル4（新規事業マーケ支援） | レベル5（経営戦略マーケ支援） |
|---|---|---|---|---|---|
| スコープの広さ | 単一施策のみ（広告運用・SEO対策など） | 複数施策の組み合わせ（広告＋LP最適化） | 広範囲のマーケ施策（広告・SEO・CRMなど統合） | マーケ全般＋顧客開発＋プロダクト改善提案も含む | マーケ×経営戦略×組織設計（M&A・資金調達にも影響） |
| 経営レベル | 施策レベルのみ（経営戦略への影響なし） | 部分的なマーケ戦略（KPI設定・運用改善提案） | 事業戦略と密接に関わる（中期戦略・組織への影響） | 事業戦略と連携し、組織・財務への影響も考慮 | 経営陣と議論し、会社全体の方向性に関与 |
| 事業の特性 | 既存事業の運用改善フェーズ | 成長フェーズの支援（売上拡大施策） | 既存事業のグロースフェーズ | 新規事業の立ち上げフェーズ | 企業の成長戦略フェーズ（IPO・M&A準備など） |
| クライアントのコミュニケーション | 指示型・受け身（要望を伝え、対応を求める） | 質問や要望を投げるが、最終判断は自社 | 意見を求めるが、決定権はクライアント | 共同作業型（週次MTGで戦略相談） | フルパートナー型（意思決定の一部を担う） |
| クライアントの期待値 | 工数期待が高い（「◯時間で何をしてくれるか」） | 成果期待あり、中期視点の期待 | 成果期待が高く、専門性も求められる | 成果期待が大きく、柔軟な対応を求められる | 高度な専門性と経営視点を求める |

**アサインメントの判断基準**：
- レベル1のプロジェクトには レベル1〜2の人材を配置（成長機会も兼ねて少しストレッチ）
- レベル5のプロジェクトはディレクター（役員クラス）レベルのみ
- 「成長機会＝1段階上のプロジェクトに挑戦」が原則

### ステップ6：3つのまとめポイントを確認する

| ポイント | 内容 |
|---|---|
| POINT01 明確なキャリアロードマップが成長を加速する | 従業員が自身の成長ステップを理解し、適切なスキルを習得することで、組織全体のパフォーマンス向上につながる |
| POINT02 育成計画と継続的なフィードバックが成長の鍵 | 計画的な研修やOJT、定期的な評価・フィードバックを通じて、実践的なスキル習得とキャリアアップを支援することが重要 |
| POINT03 適切なアサインメントで能力を最大限に発揮 | 個々のスキル・経験・キャリア志向に基づいたアサインメントを行うことで、モチベーションを維持しながら組織の成長を促進できる |

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 1コースのみ（管理職コースだけ等） | 最低2コース（管理職＋専門職）に拡張 | 「管理職にならないと出世できない」は専門人材の離職要因 |
| 5階層が多すぎる（小規模組織） | 最低3階層（アシスタント／メンバー／リーダー）から開始 | 階層の細かさより「次のステップが見える」ことが重要 |
| 育成計画が「とりあえず3年」で具体施策なし | 各フェーズの目標＋具体施策（座学／OJT／1on1／資格支援）を必ず明文化 | 抽象的計画は形骸化する |
| アサインメントが「なんとなくこの人」で決まる | 人のレベル×プロジェクトのレベルの2軸マトリクスで明文化 | アサインの納得感がモチベーションを左右する |
| 「成長機会のため難しすぎるプロジェクトに挑戦」 | 1段階上までに留める（2段階以上はクオリティリスク） | 過度なストレッチは品質低下と離職リスク |
| プロジェクトレベルが定義されていない | 5指標（スコープ／経営レベル／事業特性／コミュ／期待値）×5レベルで定義 | レベル定義なしのアサインは判断ブレの温床 |
| 専門職コースに進む人材の処遇 | 管理職コースと同等以上のレンジを `salary-range-design` で確保 | 専門職冷遇は人材流出の主要因 |
| 新規事業コース志向者が増えた | 社内起業制度・スピンアウト機会を制度化 | 起業志向人材の社内起業誘導 |
| 「採用市場でこのキャリアパスは独自・競合がない」と楽観判断 | `competitive-absence-audit` スキルで6仮説検証 | キャリアパス設計の独自性主張は競合不在の楽観判断と同質、検証必須 |
| 1on1や評価フィードバックを実施していない | 週次1on1（基礎習得期）／月次FB（実務対応期）／半期評価（応用以降）を制度化 | 育成計画はFBサイクルなしには機能しない |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 管理職コース1本のみ | 管理職＋専門職の2コース最低提供 | 専門人材の離職を防ぐ |
| ロードマップを作って従業員に共有しない | 全社員に開示し、定期面談で各人の現在地を確認 | 開示なしのロードマップは存在しないのと同じ |
| 育成計画が「とりあえず研修受けてOJT」 | 期間×フェーズ×目標×具体施策の4列で明文化 | 抽象的計画は機能しない |
| 1on1や評価FBを実施しない | 週次1on1（基礎期）／月次FB（実務期）／半期評価（応用以降） | FBサイクルなしの育成は形骸化 |
| アサインメントが「なんとなくこの人」 | 人のレベル×プロジェクトのレベルの2軸で判断 | 判断ブレが不公平感とモチベ低下の原因 |
| 成長機会のため2段階以上難しいプロジェクトに挑戦 | 1段階上までに留め、メンター配置 | 過度なストレッチは品質低下＋メンタル不調＋離職リスク |
| 専門職コースの処遇を管理職より下に置く | `salary-range-design` で同等以上のレンジを確保 | 専門職冷遇は人材流出の主因 |
| キャリア志向のヒアリングをせず一律に管理職を推す | 個別面談で管理職／専門職／新規事業の志向を確認 | 志向と乖離したキャリアパスは離職リスク |
| 「うちのキャリアパスは独自で競合がない」と楽観判断 | `competitive-absence-audit` で需要・競合を検証 | 独自性主張は競合不在の楽観判断と同質 |

## 連携スキル

- `recruitment-strategy`：採用ターゲットとキャリアロードマップの入口（アシスタント／マーケター層）を連動させる
- `salary-range-design`：5階層の各レベルに対応する給与レンジを設計
- `evaluation-system-design`：育成計画の各フェーズの評価基準・FBサイクルを設計
- `roles-responsibilities`：各階層の役割と責任範囲を明確化
- `organization-planning`：組織図・組織計画とキャリアロードマップを整合させる
- `mvv-design`：キャリアロードマップの「成長像」をMVVと整合させる
- `teaching-coaching-leading`：育成施策（OJT・1on1・メンタリング）の指導アプローチ選択
- `katz-three-skill-approach`：階層ごとに必要な3スキル（テクニカル／ヒューマン／コンセプチュアル）の比重設計
- `competitive-absence-audit`：キャリアパスの独自性主張を楽観判断しないための仮説検証
- `goal-hierarchy-design`：年間目標→四半期→月間→個人の階層化と育成計画を整合
- `ms-matrix-talent-grid`：4象限ごとのキャリアパス設計（特に④専門家タイプの専門職パス）
- `goal-cascade-kpt-1on1`：育成KPIを目標カスケードに組み込み、KPT 1on1で進捗管理
- `success-cycle-relationship-quality`：育成期の関係の質を高めることで成長加速
- `credo-language-design`：キャリアステージごとのクレド体現度を育成基準に組み込む

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし以下のケースで明示マーカーを使うこと：

- `[OK:0件]`：対象組織の職種・階層情報が不足し、ロードマップを設計するための前提が揃わない場合
- `[SKIP]`：競合不在の楽観判断が出た場合は `competitive-absence-audit` スキルへの誘導でスキップ

外部API・サブプロセスを持たないため、`[FAIL]`（試行後の失敗）は通常発生しない。
