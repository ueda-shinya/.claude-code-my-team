# heaven-hell-rhetoric

顧客の課題に対し「地獄（放置した場合の最悪シナリオ）→ 天国（解決後の理想状態）」の対比構造で営業トーク・提案スクリプトを生成する。

## 適用条件

- このスキルを使う場面：
  - 顧客が「現状維持でいい」と思っている場面で行動を促したいとき
  - 初回提案・プレゼンのニーズ喚起パートを作るとき
  - クロージング前に顧客の意思決定を後押ししたいとき
  - 競合比較で差別化を伝えたいとき
  - 将来リスク商材（保険・教育・不動産・ITインフラ等）の営業トークを作るとき

- このスキルを使わない場面：
  - 顧客との信頼関係がまだない初対面の冒頭（先にラポール構築が必要）
  - 顧客の現状を十分にヒアリングできていない段階（的外れな地獄になる）
  - 製品・サービスに実力が伴っていないとき（天国を実現できないなら使わない）
  - 同じ顧客に3回以上使っている場合（嫌悪感を生む）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 顧客情報 | 必須 | 業種・規模・役職・現在の課題 | 「地域の工務店、社長、Webからの集客ゼロ」 |
| 提案サービス | 必須 | 天国の実現手段として提案するもの | 「Web制作＋SEO/MEO対策」 |
| 商談フェーズ | 任意 | 現在の営業ステージ | 「初回提案」「クロージング前」 |
| 既知の事例 | 任意 | 地獄・天国に使える実際の事例 | 「同業A社が放置して問い合わせ年3件→導入後月10件」 |
| 出力形式 | 任意 | トークスクリプト / 提案書パート / メール文面 | デフォルト：トークスクリプト |

## 実行ステップ

### ステップ0：前提条件を確認する

- 入力：顧客情報、商談フェーズ
- 処理：
  - 顧客との信頼関係（ラポール）が構築済みかを確認する
    - 初対面 → このスキルは使わない。「先にラポール構築をしてから使ってください」と報告して終了する
    - 2回目以降 or ヒアリング済み → 続行する
  - 顧客の現状課題が具体的にわかっているかを確認する
    - 課題が曖昧（「なんとなく集客に困っている」レベル） → ステップ1のヒアリングを丁寧に行う
    - 課題が具体的（「Web経由の問い合わせが月0件」レベル） → ステップ1は簡略化してステップ2に進む
- 出力：続行可否の判断結果

### ステップ1：顧客の課題を特定する

- 入力：顧客情報
- 処理：
  - 以下のヒアリング質問を生成する（顧客の業種に合わせて文言を調整する）
    - 「現在、〇〇についてはどのようにされていますか？」（現状確認）
    - 「その中で、一番お困りのことは何ですか？」（課題の焦点化）
    - 「それによって、具体的にどんな影響が出ていますか？」（損失の数値化）
  - 顧客情報から課題を1つに絞る。複数ある場合は最もインパクトが大きいものを選ぶ
    - 選択基準：金額的損失が大きい > 時間的損失が大きい > 精神的負担が大きい
- 出力：特定された課題（1文）、ヒアリング質問リスト

### ステップ2：地獄シナリオを構築する

- 入力：特定された課題、顧客情報、既知の事例（あれば）
- 処理：
  - 課題を放置した場合のシナリオを3つの時間軸で描写する：
    - **3ヶ月後**：直近で起きる具体的な問題
    - **1年後**：問題が拡大した状態
    - **3年後**：取り返しがつかなくなった状態
  - 各時間軸に必ず数字を入れる（コスト・損失額・時間・件数のいずれか）
    - 数字がない場合は業界平均・一般的な統計から推定する
  - 「他のお客様のケース」として語る形にする（顧客を直接攻撃しない）
    - NG：「御社はこのままだと失敗します」
    - OK：「同じような状況だったお客様で、そのままにされた結果…」
  - 恐怖の強度を調整する：「このままでは少し怖いな」程度に抑える
    - NG：「倒産します」「取り返しがつきません」（強すぎ → 嫌悪感）
    - OK：「機会損失が年間〇万円に」「競合に差をつけられる可能性が」
- 出力：時系列の地獄シナリオ（3段階）、トーク用セリフ

### ステップ3：天国シナリオを構築する

- 入力：特定された課題、地獄シナリオ、提案サービス、既知の事例（あれば）
- 処理：
  - 地獄シナリオと**同じ時間軸・同じ指標**で理想状態を描写する：
    - **3ヶ月後**：最初の変化が見える状態
    - **1年後**：成果が安定している状態
    - **3年後**：大きなリターンが出ている状態
  - 地獄で使った数字に対応する改善値を入れる
    - 地獄：「年間120万円の機会損失」→ 天国：「年間300万円の売上増」
  - 提案サービスが天国の実現手段であることを自然に織り込む
  - 実際の導入事例がある場合は「うまくいったお客様」として具体的に語る
    - 事例がない場合は「一般的にこのくらいの改善が見込めます」と正直に伝える
- 出力：時系列の天国シナリオ（3段階）、トーク用セリフ

### ステップ4：対比構造を組み立てる

- 入力：地獄シナリオ、天国シナリオ
- 処理：
  - 地獄と天国を表形式で並べる：

    | 時間軸 | 地獄（放置した場合） | 天国（解決した場合） |
    |---|---|---|
    | 3ヶ月後 | 〇〇 | 〇〇 |
    | 1年後 | 〇〇 | 〇〇 |
    | 3年後 | 〇〇 | 〇〇 |

  - ギャップ（差分）を算出して強調ポイントとする
    - 金額差、時間差、件数差のうち最もインパクトが大きいものを選ぶ
  - ギャップを言語化するブリッジセリフを生成する：
    - 「先ほどのお話と比べると、大きな違いですよね。この差がそのまま1年、2年と続くとしたら…」
- 出力：対比表、ギャップの数値、ブリッジセリフ

### ステップ5：行動喚起（CTA）を生成する

- 入力：対比表、提案サービス、商談フェーズ
- 処理：
  - 商談フェーズに応じたCTAを1つだけ生成する：
    - 初回提案 → 「まずは無料で〇〇を診断してみませんか？」（低リスクの次ステップ）
    - クロージング前 → 「〇月から始められるよう、今週中に契約書をお送りしますね」（具体的な日程）
    - 競合比較中 → 「他社さんとの違いを具体的にお見せできますが、30分ほどお時間いただけますか？」
  - CTAは「今日できる最初の一歩」にする。大きな決断を求めない
- 出力：CTA文（1文）

### ステップ6：トークスクリプトとして組み立てる

- 入力：ヒアリング質問、地獄セリフ、天国セリフ、ブリッジセリフ、CTA、出力形式
- 処理：
  - 出力形式に応じて全パーツを1つのスクリプトに組み立てる：
    - トークスクリプト → 話し言葉で、セリフとト書きを交互に
    - 提案書パート → 見出し＋本文の構成で
    - メール文面 → 件名＋本文の構成で
  - 全体の流れが「現状確認 → 地獄 → 深掘り → 天国 → 対比 → CTA」の順番になっていることを確認する
  - 地獄パートが全体の30〜40%、天国パートが30〜40%、残りがヒアリングとCTAになるバランスにする
- 出力：完成したトークスクリプト/提案書パート/メール文面

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 顧客との面識が全くない | このスキルを使わない | 信頼なしに地獄を語ると「脅し」に聞こえる |
| 顧客の課題がヒアリングできていない | ステップ1を丁寧に実施 | 的外れな地獄は刺さらない |
| 既知の事例がない | 業界統計・一般データで代替する | 捏造は信頼を失う。「一般的には」と前置きする |
| 地獄シナリオが顧客の現実と乖離している | シナリオの規模を下げる | 非現実的な恐怖は「関係ない」と思われる |
| 顧客が既に危機感を持っている | 地獄パートを短縮し天国パートを強調する | 既に痛みを感じている人にさらに地獄を見せると嫌悪感を生む |
| 同じ顧客に2回以上使用済み | このスキルを使わない。別の話法に切り替える | 繰り返しの恐怖訴求は逆効果 |
| 高額・複雑商材で即決が難しい | 恐怖の強度を「弱」にする | 過剰な恐怖は後日キャンセルにつながる |

### 記入例

```
## 天国と地獄トークスクリプト

**対象顧客**: 山田工務店 / 地域工務店 / Webからの問い合わせゼロ
**提案サービス**: Web制作＋SEO/MEO対策
**商談フェーズ**: 初回提案

---

### 1. 現状確認

「現在、新規のお客様はどういう経路で来られていますか？」
「Webサイトからのお問い合わせは、月にどのくらいありますか？」

### 2. 地獄の提示

「実は、同じ地域の工務店さんで、Webを後回しにされていたケースがあるんですが、
気づいたときには検索結果の3ページ目以降になっていて、
年間で約150万円分の見込み客を競合に取られていた、という試算が出たんです。」

### 3. 深掘り

「山田さんのエリアでも、最近Web経由で問い合わせを取っている同業さんが増えていませんか？
もし1年後もこのままだったら、どうなると思いますか？」

### 4. 天国の提示

「一方で、Webを整備された工務店さんでは、
3ヶ月目から月2〜3件の問い合わせが入り始めて、
1年後には月10件ペースで安定されています。
年間の売上増が300万円を超えたというケースもあります。」

### 5. 対比

| 時間軸 | このままの場合 | Web整備した場合 |
|---|---|---|
| 3ヶ月後 | 問い合わせ月0件のまま | 月2〜3件の問い合わせ開始 |
| 1年後 | 競合に年150万円分の見込み客を取られる | 月10件ペース、年間売上+300万円 |
| 3年後 | エリア内の存在感が薄れ、紹介頼みに | 「〇〇エリアの工務店といえば山田さん」のポジション確立 |

「先ほどの年間150万円の機会損失と、年間300万円の売上増。
この差が毎年続くとしたら、3年で1,350万円の差になります。」

### 6. 行動喚起

「まずは、山田さんのエリアで今どんなキーワードが検索されているか、
無料で調べてみましょうか？30分もあればお出しできます。」
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 「御社はこのままだと失敗します」と直接言う | 「同じ状況だったお客様で、こんなことが起きまして…」と他社事例で語る | 直接的な否定は防御反応を生む |
| 「倒産するかもしれません」と過度に脅す | 「年間〇万円の機会損失になっている計算です」と数字で冷静に伝える | 強すぎる恐怖は嫌悪感を生み、記憶から消されやすい |
| 天国を先に語ってから地獄を語る | 必ず地獄→天国の順番を守る | 天国が先だと「また営業か」と警戒される。地獄で課題を認識させてから天国を見せる |
| 地獄だけ語って天国を忘れる | 地獄→天国→ギャップ確認を必ず3点セットで | 恐怖だけでは行動に繋がらない。希望で締める |
| 事例を捏造する | 実際の事例がなければ「一般的には」と前置きして統計データを使う | 捏造が発覚すると信頼を一瞬で失う |
| 1回の商談で3回以上使う | 1商談につき1〜2回に絞る | 繰り返しの恐怖訴求は嫌悪感を生む |
| 天国の数字が曖昧（「きっと良くなります」） | 具体的な数字で語る（「月10件」「年300万円」） | 曖昧な天国はギャップが見えず、行動喚起に繋がらない |
