# 中長期事業計画策定スキル

5〜10年の中長期事業計画を「6ステップ作成プロセス＋2つの可視化フォーマット（中長期ロードマップ／事業の計画）」で一気通貫策定する。ビジョン設定からリスク管理まで網羅し、組織構築と事業戦略の双方を時系列に並べる。

## 適用条件

- このスキルを使う場面：
  - 5〜10年スパンの中長期事業計画を策定したい
  - 事業全体のロードマップを2〜3年単位で時系列に描きたい
  - 複数事業を抱える企業で、事業ごとの計画を時間軸で整理したい
  - ビジョン達成に向けた戦略的リソース配分を検討したい
- このスキルを使わない場面：
  - 1年以内の短期計画 → 別スキル（goal-execution-system / smart-goal-setting）
  - 既に策定済み計画の進捗管理 → pdca-cycle
  - 単発フレームワーク分析（PEST／SWOT等）のみ → 各分析スキル

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー語のいずれか | 「中長期計画作って」 |
| 計画期間 | 必須 | 計画対象の年数 | 「5年」「2024-2028」「3年スパン」 |
| ビジョン | 必須 | 計画期間の終着点となる目標 | 「2030年までに業界トップ」「売上10億円達成」 |
| 事業構成 | 必須 | 計画対象の事業（単一／複数） | 「BtoB映像事業」「映像事業＋税理士事業＋不動産事業」 |
| 現状の組織体制 | 任意 | 現時点の人員・体制（あれば差分が描きやすい） | 「社員10名・社長依存度高」 |
| 既知の市場環境情報 | 任意 | 競合・市場動向・規制等（あればステップ②で活用） | 「競合A社が2025年新規参入予定」 |

## 実行ステップ

### ステップ1：ビジョンと目標の設定

- 入力：計画期間、終着点となるビジョン
- 処理：
  1. 計画期間の最終年に到達するビジョンを1文で言語化する（定性的）
  2. ビジョンを定量的に測れる指標に分解する（例：売上◯億円／顧客数◯社／シェア◯%）
  3. 中間マイルストーン（例：3年目時点／半分到達時点）を1〜2個設定する
  4. ビジョン共有方法（社内浸透の手段）を1〜2個提案する
- 出力：
  - 最終ビジョン文 + 定量指標（KGI候補）
  - 中間マイルストーン
  - ビジョン共有施策

### ステップ2：環境分析

- 入力：ステップ1のビジョン、既知の市場環境情報
- 処理：
  1. **外部環境**：市場動向・競合・法規制・技術革新を整理する
     - 詳細分析が必要な場合 → `pest-analysis` スキルへ誘導
  2. **内部環境**：自社の強み／弱みを整理する
     - 詳細分析が必要な場合 → `swot-analysis` スキルへ誘導
  3. **業界競争環境**：参入障壁・代替品の脅威等を確認する
     - 詳細分析が必要な場合 → `five-forces-analysis` スキルへ誘導
  4. 各分析の主要発見を3〜5項目に集約する
- 出力：
  - 外部環境のリスク／機会（各3項目以内）
  - 内部環境の強み／弱み（各3項目以内）
  - 関連スキルへの誘導コメント（必要な場合のみ）

### ステップ3：戦略の選定

- 入力：ステップ1のビジョン、ステップ2の環境分析結果
- 処理：
  1. 成長戦略の主要選択肢から取捨選択する（複数併用可）：
     - **新市場開拓**（既存商品×新顧客層）
     - **新製品開発**（新商品×既存顧客）
     - **多角化**（新商品×新顧客）
     - **市場浸透**（既存商品×既存顧客でシェア拡大）
     - **コスト削減**（既存事業の効率化）
  2. 各戦略について「なぜこの戦略を選ぶか」を1〜2文で言語化する
  3. 戦略間の優先順位を3段階（最優先／並行実施／後回し）で振る
- 出力：
  - 採用する成長戦略リスト（優先順位付き）
  - 各戦略の選定理由

### ステップ4：実行計画の策定（中長期ロードマップ＋事業の計画の2フォーマット出力）

- 入力：ステップ1〜3の全結果、事業構成、現状の組織体制
- 処理：

#### 4-1. 中長期ロードマップ（フォーマットA）の作成

5フェーズ×階層図で、組織構築と事業戦略の双方を時系列に並べる。

- **時系列軸**：計画期間を5フェーズに分割する（例：2024下期／2025上期／2025下期／2026上期／2026下期）
- **上段（フェーズ別マイルストーン）**：各フェーズの主要テーマを1単語で記載（例：「組織体制の構築」「売上拡大戦略」「売上10億円に向けた事業戦略」）
- **中段（時系列レイヤー）**：複数の取組ストリームを階段状に積み上げる
  - 下層から「現状維持の戦略」「拡大戦略」「次フェーズに向けた戦略」「ビジョン達成戦略」「組織構築」のように積む
  - 各ストリームが開始するフェーズを明示する
- **下段（フェーズ別アクション）**：各フェーズで実施する具体アクションを箇条書き

#### 4-2. 事業の計画（フォーマットB）の作成

複数事業×時間軸で、各事業の進捗段階を並べる。

- **縦軸（事業）**：計画対象の事業を縦に並べる（例：BtoB映像事業／プラットフォーム開発／プラットフォーム配信／BtoB他事業）
- **横軸（時間）**：年次（5年なら2024〜2028）
- 各事業について「PJ開始」「MVP実装」「収益化の安定」「内製化開始」「属人的な内製化の完了」「平準的な内製化の完了」など、進捗段階のマーカーを時間軸上に配置する
- 複数事業を並行展開する場合、事業立ち上げの順序を可視化する

- 出力：
  - 中長期ロードマップ（5フェーズ×階層図）の表形式
  - 事業の計画（複数事業×時間軸）の表形式

### ステップ5：リスク管理

- 入力：ステップ1〜4の全結果
- 処理：
  1. ステップ2の外部環境リスク・ステップ3の戦略実行リスクを統合し、主要リスクを3〜5項目に絞る
  2. 各リスクについて発生確率（高／中／低）と影響度（大／中／小）を評価する
  3. 「高×大」「高×中」「中×大」のリスクに対し、回避策／低減策／受容判断を提案する
- 出力：
  - リスク一覧表（リスク／確率／影響度／対応策）

### ステップ6：定期レビュー設計

- 入力：ステップ1〜5の全結果
- 処理：
  1. レビュー頻度を設定する（推奨：四半期ごと）
  2. レビュー指標（KGI／KPI／戦略進捗）を3〜5項目に絞る
  3. 計画修正の判断基準（KPI未達◯%以上で戦略見直し等）を明示する
  4. 関連スキル `pdca-cycle` への引き継ぎを案内する
- 出力：
  - レビュー頻度・指標・修正判断基準
  - pdca-cycle スキルへの引き継ぎ案内

### ステップ7：完了報告

ステップ1〜6の全成果物を統合し、後述の出力フォーマットでまとめる。

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ビジョンが「業界トップ」のように曖昧 | 必ず定量指標に分解する | 進捗測定不能を防ぐ |
| 計画期間が3年未満 | スキル適用外、`smart-goal-setting`＋`pdca-cycle` を案内 | 中長期＝5年以上が前提 |
| 環境分析の入力情報が乏しい | 関連スキル（PEST／SWOT／5フォース）に一旦戻す | 仮の前提で計画を進めない |
| 戦略が5つ以上採択される | 優先順位を強制的に3段階に振る | 全方位戦略は実行困難 |
| 事業が単一の場合 | フォーマットB（事業の計画）は省略可、フォーマットA（ロードマップ）のみで十分 | 単一事業に複数事業フォーマットは冗長 |
| リスクが「高×大」で対応策がない | 戦略選定（ステップ3）に戻し戦略を見直す | リスク許容できない戦略は採用不可 |
| ビジョンが3年で達成可能と判明 | 計画期間を見直す | 簡単すぎる目標は中長期計画として不適切 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ビジョンを「業界トップ」のままにする | 必ず定量指標（売上・シェア・顧客数）に分解する | 進捗測定不能 |
| 環境分析を省略して戦略選定に進む | リスク・機会を最低3項目ずつ整理する | 楽観バイアスを防ぐ |
| 戦略を5つ以上全て「最優先」とする | 3段階の優先順位を強制する | 実行不能 |
| 中長期ロードマップだけ作って事業の計画を省く（複数事業時） | 複数事業がある場合は両方作成する | 事業間の立ち上げ順序が不明になる |
| リスク管理を「精神論で頑張る」と書く | 回避策／低減策／受容を明示する | 計画破綻時の判断基準なし |
| 定期レビュー頻度を設けない | 必ず四半期ごとを推奨する | 計画が形骸化する |
| 「2024〜2028」を5フェーズに分けず年次のみで書く | 5フェーズ（半期単位）に分けるとアクション粒度が上がる | 年単位は粗すぎる |
| 競合不在を前提に戦略を組む | `competitive-absence-audit` スキルを必ず経由する | CLAUDE.md「Competitive Absence Audit Rule」遵守 |

## 連携スキル

- **前段**：`pest-analysis` `swot-analysis` `five-forces-analysis` `3c-analysis` `vrio-analysis` `stp-analysis` `value-proposition`（環境・自社分析）
- **同時併用**：`smart-goal-setting`（KGI/KPI設計）、`goal-hierarchy-design`（ビジョン→年間→四半期→月間に分解）、`innovator-theory`（事業フェーズ判定）、`pmf-journey`（PMF段階診断）
- **後段**：`pdca-cycle`（運用フェーズ）、`goal-execution-system`（モニタリング設計）

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存を持たないため対象判定基準には該当しないが、運用上は以下マーカーを使う：

- `[OK:0件]`：入力情報が不足し計画策定不可と判定された場合
- `[FAIL]`：必須入力（ビジョン／計画期間／事業構成）が欠落している場合（処理を中断し再入力依頼）
