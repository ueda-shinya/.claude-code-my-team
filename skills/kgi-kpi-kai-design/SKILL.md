# KGI・KPI・KAI設計スキル

最終目標（KGI）→中間指標（KPI）→行動指標（KAI）の3階層で目標達成の道筋を構造化。KPIは「結果指標」、KAIは「行動指標（数値に直結する具体行動）」。月次目標vs実績トラッキング表で組織全体の方向性を統一する。

## 適用条件

- 売上目標等を分解して具体行動まで落としたい
- 部門間でKPIの定義が揃っていない
- 「目標は決まったが何をすべきか分からない」状態を解消したい
- PDCA運用の基準指標を設計したい

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| KGI（最終目標） | 必須 | 例：年間売上3,440万円 |
| 期間 | 必須 | 例：2025年1月〜12月 |
| ビジネスモデル | 任意 | サブスク／単発販売／代理店等 |

## 実行ステップ

### ステップ1: 3階層の定義整理

| 指標 | 略称 | 役割 | 例 |
|---|---|---|---|
| **KGI** (Key Goal Indicator) | 最終目標 | 数値化した最終ゴール | 年間売上1億円／市場シェア20% |
| **KPI** (Key Performance Indicator) | 中間指標 | KGI達成のための進捗指標 | 月間売上1,000万円／新規顧客50件/月／フォロワー1万人増 |
| **KAI** (Key Action Indicator) | 行動指標 | KPI達成のための具体行動・活動量 | 毎日SNS1投稿／毎週10商談／毎月1広告キャンペーン |

### ステップ2: KGIの設定

- 組織やプロジェクトの最終目標を明確化・数値化
- POINT: 具体的かつ明確（何を達成すれば成功か）／測定可能（数値で評価できる）
- 関連: `smart-goal-setting` でKGIをSMART化、`goal-hierarchy-design` で年→月に分解

### ステップ3: KPIの階層設計（KPIツリー）

KGIを売上方程式で分解する典型例：

```
KGI: 年間売上目標
  ├─ 月額顧客単価 × 顧客数
       └─ 顧客数 = 既存顧客 − 解約顧客 + 新規顧客
            └─ 新規顧客 = 商談数 × 成約率
                 └─ 商談数 = リード数 × 商談化率
                      └─ リード数 = 代理店 + 紹介 + 問合せ
                           └─ 問合せ = アクセス数 × CVR
```

各ノードがKPI候補。重要度の高いものを「中間指標」として採用する。

### ステップ4: KAI（行動指標）の設計

**KAI設計の3ポイント**：
- KPI = 結果指標、KAI = 行動指標（数値に直結する行動）
- KAIは「いつ／誰が／どの頻度で／どんな行動をすれば良いか」が具体的になっているのが理想
- KAIの達成が積み重なると、KPIが改善される設計

**KPI→KAI例（顧客系・営業系）**：

| KPI | KAI（行動目標） |
|---|---|
| 月額顧客単価 | 定期的な利用状況ヒアリング回数／顧客フォロー回数（月次） |
| 顧客数 | 顧客フォロー回数（月次） |
| 既存顧客 | 満足度調査の実施件数／アカウント管理ミーティング実施件数／契約更新前リマインド実施率 |
| 解約顧客 | 解約前アラート顧客への事前接触数／解約理由ヒアリング実施率 |
| 新規顧客 | 初回オンボーディング完了率／初月定着サポート実施件数 |
| 商談数 | 初回提案実施数／週次訪問・オンライン商談件数 |
| 成約率 | クロージング面談の同席件数／失注理由レビュー実施頻度 |
| リード数 | 週次のリード獲得数目標／新規リードの営業対応初速（◯時間以内対応率） |
| 商談化率 | インサイドセールスのヒアリング完了率／提案資料送付率 |

**KPI→KAI例（マーケ系）**：

| KPI | KAI（行動目標） |
|---|---|
| 代理店 | 代理店勉強会の開催頻度／代理店からの案件紹介数 |
| 紹介 | 既存顧客への紹介依頼回数／紹介特典キャンペーン実施回数 |
| 問合せ | フォーム改善テストの実施数／問合せメールへの即日対応率 |
| アクセス数 | ブログ・LP更新頻度／SNS投稿回数・広告出稿回数 |
| CVR | CTA配置改善実施件数／フォーム離脱率改善のABテスト回数 |

### ステップ6: 実行とモニタリング・フィードバック

- 設定したKGI/KPI/KAIに基づき行動
- 定期的に進捗確認・評価（推奨：月次）
- モニタリング結果を基に指標・行動計画を見直し → `pdca-cycle` 連携

## 判断基準

| 状況 | 判断 |
|---|---|
| KGI達成不能と途中で判明 | KPI再分配 or KGI見直し（リスケ） |
| KAIが定性表現（「頑張る」） | 必ず数値化（◯件・◯回・◯%） |
| KPIが多すぎる（10個超） | 上位3〜5個に絞る |
| KAIとKPIが連動しない（KAI達成してもKPI改善せず） | KAI設計をやり直す |
| 1か月放置 | 必ず月次レビュー |
