# smart-goal-setting

SMARTの5要素（具体性／測定可能性／達成可能性／関連性／期限）で目標を設計し、合格基準で点検する。曖昧な目標を行動指針に変える。

## 適用条件

- 使う場面：
  - 目標を新規に設計したい（事業・組織・個人いずれも）
  - 既存の目標がSMARTかを点検したい
  - 「売上を上げる」「成長する」等の曖昧目標を具体化したい
  - レビュー前後にチェックリストとして使いたい
- 使わない場面：
  - 大目標を年間→四半期→月間に分解したい → `goal-hierarchy-design`
  - PDCAでの運用 → `pdca-cycle`
  - 仕組み化（モニタリング/フィードバック） → `goal-execution-system`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 目標案 | 必須 | 粗い形でよい。1文程度 | 「売上を上げる」「新規顧客を増やす」 |
| 文脈 | 任意 | 上位目標、組織状況、リソース | 「年間売上目標1億円のうち、新規分」 |
| 期間 | 任意 | 想定する達成期限 | 「今期（3ヶ月）」「2026年内」 |

## 実行ステップ

### ステップ1：目標案を受け取り、SMART5要素で初期診断する

- 入力：目標案（1文）
- 処理：
  - 目標案を読み、各要素について「現状そろっているか」を3段階で判定する
    - ○：明示されている
    - △：暗示されているが具体性が足りない
    - ×：欠落
  - 5要素の判定表を作る
- 出力：初期診断表（5行 × 判定）

### ステップ2：欠落・不足要素を質問で補う

- 入力：初期診断表
- 処理：×と△の要素について、以下の質問を提示して回答を得る
  - **S（具体性）が△/×**: 「具体的な数値・対象・行動を1つに絞ると何になりますか？（例：『売上を上げる』→『新規法人顧客を月10件獲得する』）」
  - **M（測定可能性）が△/×**: 「達成したかを判定する指標と数値は何ですか？（KPI候補を3つ挙げてください）」
  - **A（達成可能性）が△/×**: 「今あるリソース（人/時間/予算/スキル）で実現可能ですか？不足するものがあれば何ですか？」
  - **R（関連性）が△/×**: 「この目標は上位の事業目標／組織目標／個人成長と、どう繋がっていますか？」
  - **T（期限）が△/×**: 「いつまでに達成しますか？マイルストーンの時期を3つ挙げてください」
- 各質問について、回答が曖昧な場合はもう一段深掘りする
- 出力：補完された5要素の情報

### ステップ3：SMART文に変換する

- 入力：補完された5要素の情報
- 処理：
  - 元の目標案を「数値（M）+ 対象（S）+ 行動（S）+ 期限（T）」の構造で1文に再構成する
  - 例：「売上を上げる」→「2026年12月末までに（T）、関東エリアの中堅製造業（S）から、新規受注額3,000万円を獲得する（S+M）。営業1名と既存パートナーで実行可能（A）、年間売上目標1億円の30%を占める重要施策（R）」
  - 1文が長くなる場合は、メイン文 + 補足文（A・R）の2文構造にする
- 出力：SMART化された目標文

### ステップ4：合格基準でチェックする

- 入力：SMART化された目標文
- 処理：以下の合格基準（10項目）でチェックする
  - S-1: 行動の対象（誰／どの市場／どの商品）が1つに絞れているか
  - S-2: 具体的な数値が含まれているか
  - M-1: 達成判定の指標（KPI）が定義されているか
  - M-2: 進捗を測る頻度（週次／月次）が決まっているか
  - A-1: 必要なリソースが特定されているか
  - A-2: 過去実績や他社事例から実現可能性が裏付けられているか
  - R-1: 上位目標（事業／組織）との接続が明記されているか
  - R-2: この目標を達成すると、何が次に進むかが見えるか
  - T-1: 達成期限が日付で明記されているか
  - T-2: マイルストーン（中間チェック点）が2つ以上あるか
- 1項目につき○/△/×で評価し、△・×がある場合はステップ2に戻る
- 全項目○になるまで繰り返す
- 出力：合格判定結果

### ステップ5：SMART目標シートを出力する

- 入力：SMART化された目標文 + 合格判定結果
- 処理：以下のテンプレートに従って整形する
- 出力：完成したSMART目標シート（コピペで運用に使える形）

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 目標案が抽象的（例：「成長する」） | ステップ2の S と M の質問を最優先で投げる | 数値と対象が決まらないと他要素が決まらない |
| リソースが明らかに不足（A-1が×） | 目標値を下げる or リソース確保を別タスク化 | 達成不能な目標は意味がない |
| 上位目標が見えない（R-1が×） | 「これを達成して何に繋げたいですか？」を最初に確認 | 上位目標との接続なしの目標は孤立する |
| 期限が「いつか」になる（T-1が×） | デフォルトで「3ヶ月後／半年後／1年後」の3択を提示 | 期限なしは目標ではない |
| 数値が複数候補ある（M-1で迷う） | 「3つ挙げて、最も意思決定に影響する1つを選ぶ」と促す | KPIが多すぎると形骸化する |
| 全項目○で合格 | ステップ5に進み、シートを出力する | これ以上の精緻化は過剰 |

### 記入例

```
# SMART目標シート

## 目標文（SMART化済み）
2026年12月末までに、関東エリアの中堅製造業（従業員50〜300名）から、月次新規受注額500万円・累計3,000万円を獲得する。営業1名＋既存パートナー2社で実行可能、年間売上目標1億円の30%を占める。

## SMART 5要素の内訳
| 要素 | 内容 |
|---|---|
| S | 関東エリア中堅製造業（従業員50〜300名）／新規受注獲得 |
| M | 月次新規受注額500万円、累計3,000万円。月次レビューで進捗測定 |
| A | 営業1名＋既存パートナー2社、過去パートナー経由3,000万円実績あり |
| R | 年間売上目標1億円の30%、達成後は中堅製造業向け専用プランの拡販に進む |
| T | 2026年12月末。マイルストーン：6月末1,000万円、9月末2,000万円、12月末3,000万円 |

## 合格判定（10項目）
全項目○。合格。

## 次のアクション
- 営業1名にこのSMART目標を共有し、月次KPIを5月から運用開始
- 次の進捗確認：2026年5月31日（5月実績レビュー）
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 「売上を伸ばす」のような目標を許容する | 数値・対象・期限を必ず追記して具体化する | 行動に落ちない目標は目標ではない |
| KPIを5個以上設定する | KPIは1〜3個に絞る | 多すぎると進捗管理が形骸化する |
| 「がんばる」「強化する」等の動詞を使う | 「獲得する」「削減する」等の達成判定可能な動詞を使う | 達成条件が曖昧になる |
| 期限を「年内」「今期中」とぼかす | 「YYYY-MM-DD」で日付を明記する | 締切効果が働かない |
| マイルストーンを置かずに最終期限だけ設定する | 中間チェック点を2〜3個置く | 最終直前に間に合わないと気付くリスク |
| 上位目標との接続（R）を省略する | 「これを達成すると、何が次に進むか」を必ず明記する | 単発目標が孤立し、戦略から切り離される |
