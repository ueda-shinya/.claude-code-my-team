# teaching-coaching-leading スキル

ティーチング・コーチング・リーディングの3つの指導アプローチを状況に応じて使い分けるスキル。事業戦略スクール「知足」のフレームワーク準拠。

## 3つのアプローチの違い

| 観点 | ティーチング | コーチング | リーディングスキル |
|---|---|---|---|
| **目的** | 知識やスキルを伝える | 潜在能力を引き出し、自己解決を促す | チームや組織を導き、成果を最大化する |
| **指導の主体** | 教える側 | 指導される側 | リーダーとメンバー双方 |
| **方法** | 知識伝達や指示 | 質問や対話 | ビジョン提示と意思決定 |
| **適用場面** | 新人研修・特定スキルの習得 | キャリア支援・問題解決 | プロジェクトリーダー・組織運営 |
| **必要なスキル** | 分かりやすい説明力／フィードバック能力 | 傾聴力／質問力 | ビジョン提示力／モチベーション管理能力 |

## 各アプローチの詳細

### 1. ティーチング（Teaching）
**知識やスキルを直接伝え、指導対象者が特定のスキルを習得することを支援する指導方法**

特徴：
1. 教える側が主導権を持つ
2. 明確な目標と手順を提示
3. 初心者や特定のスキルが必要な場面で有効

必要なスキル：
- **分かりやすい説明力**：簡潔で具体的な説明
- **フィードバック能力**：理解度を確認し、改善点を伝える

適用例：新入社員研修で業務フローやシステムの使い方を指導

### 2. コーチング（Coaching）
**相手の潜在能力を引き出し、自己解決や目標達成を支援する対話型の指導方法**

特徴：
1. 相手が主体的に考え、行動することを促す
2. 傾聴や質問を通じて、相手の気づきを引き出す
3. 長期的な成長を支援

必要なスキル：
- **傾聴力**：相手の話に耳を傾け、理解する
- **質問力**：自分で答えを見つけられるような質問を行う

適用例：中堅社員のキャリアプランニング支援

### 3. リーディングスキル（Leading）
**チームや組織を率いて目標達成に導く能力**

特徴：
1. チームの目標を設定し、方向性を示す
2. メンバーの強みを活かしながら、チーム全体の成果を最大化
3. 長期的なビジョンで戦略を示す

必要なスキル：
- **ビジョン提示力**：チーム全員が理解できる目標を提示する
- **モチベーション管理能力**：メンバーのやる気を引き出し維持する

適用例：プロジェクトリーダーとして進行管理とチームのモチベーション維持を行う

## 適用条件

- このスキルを使う場面：
  - 部下や後輩への指導アプローチを選びたい
  - 同じ相談に対して複数のアプローチで答え分けしたい
  - 自分の指導スタイルを点検したい
  - チームマネジメントで状況に応じた使い分けを設計したい
- このスキルを使わない場面：
  - 質問・ヒアリング技法そのもの → `hearing-questioning-skills`
  - チーム体制・役割分担表の設計 → `project-team-structure`
  - キャリアロードマップの設計 → 別スキル（キャリアロードマップ）
  - MVV策定 → `mvv-design`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 場面 | 必須 | 指導の場面 | 「新人研修／中堅キャリア支援／PJ運営」 |
| 相手の状態 | 必須 | 相手の経験・スキル・モチベーション | 「入社1年目・基礎習得中」 |
| 目的 | 必須 | 指導で達成したいこと | 「営業フロー習得」「自走できるリーダーへ成長」 |
| 制約 | 任意 | 時間・回数・関係性 | 「週1回1時間×3ヶ月」 |
| 相談内容（あれば） | 任意 | 相手から受けた相談 | 「キャンペーン施策のアイデアが固まらない」 |

## 実行ステップ

### ステップ1：場面・相手の状態を診断
- 入力：場面、相手の状態、目的
- 処理：以下3観点で診断
  1. **相手の習熟度**：初心者／中堅／ベテラン
  2. **相手の主体性**：受け身／自走できる／主体的
  3. **必要な成果**：知識習得／問題解決・成長／組織成果
- 出力：診断結果（3観点のスコア）

### ステップ2：アプローチを選定
- 入力：ステップ1の診断
- 処理：以下マトリクスで主アプローチを選定
  | 習熟度 | 主体性 | 推奨アプローチ |
  |---|---|---|
  | 初心者 | 受け身 | ティーチング |
  | 中堅 | 自走できる | コーチング |
  | ベテラン | 主体的 | リーディング（双方向） |
  | 中堅 | 受け身 | コーチング＋部分的にティーチング |
  | 初心者 | 主体的 | ティーチング＋方向性のリーディング |
- 出力：主アプローチ＋補完アプローチ

### ステップ3：相談内容に対する3アプローチでの答え分け（任意）
- 入力：相談内容（あれば）
- 処理：同じ相談に対し、3つのアプローチでそれぞれの答えを生成（事業戦略スクール「知足」のサンプル形式）

#### 例：「新商品キャンペーンを成功させるためのアイデアを考えているのですが、どこから手を付けていいか分かりません。ターゲット層は20代～30代の女性を想定していますが、具体的な訴求ポイントやチャネルが決まりません」

##### ティーチング（知識やスキルを直接伝える）の答え
> まず、キャンペーン設計では以下のプロセスを踏むと良いです。
> - ターゲット分析：ターゲット層が普段どのような課題を抱えているかリサーチしましょう
> - 価値提案の明確化：商品がどのようにターゲットの課題を解決するのかを整理します
> - チャネル選定：ターゲットが主に利用するSNSプラットフォームやメディアを調べ、優先度をつけます
> - テスト＆PDCA：まず小規模でキャンペーンを試し、効果測定を基に改善を加えましょう

##### コーチング（質問を通じて自発的な解決を促す）の答え
> どんな視点から考えを進めるか、一緒に整理してみましょう。
> - この商品が、ターゲットである20代～30代の女性にとってどんな価値を提供すると考えていますか？
> - ターゲット層のライフスタイルや関心事について、どれくらい具体的なイメージを持っていますか？
> - 今まで見た中で、あなたが「これは素晴らしい」と思ったキャンペーンはありますか？それはどんな点が良かったですか？
> - もし私がターゲット顧客だとしたら、この商品を知るためにどんな場所や方法でアプローチされたいと思いますか？

##### リーディング（方向性や指針を示しつつ、主体性を尊重）の答え
> いい視点を持っているね。まずはターゲット層を具体化するところから始めるのが良いと思います。
> 例えば、20代～30代の女性でも、学生、働く女性、育児中のママなど、ニーズや課題が大きく異なりますよね。どのサブセグメントを特に狙いたいか、一度リスト化してみませんか？
> その後、彼女たちの主な関心事や、SNSでの行動パターンを調べると、どんなチャネルやコンテンツが響きそうか見えてくると思います。次回のミーティングまでに、このリサーチ結果を簡単にまとめておいてくれる？そこからさらに一緒に方向性を固めていきましょう。

- 出力：3アプローチでの答え（同じ相談に対する3パターン）

### ステップ4：効果的な活用方法の3ポイントを設計
- 入力：選定したアプローチ
- 処理：選定アプローチに応じて以下を設計
  1. **ティーチング活用ポイント**：明確な目標を設定し具体的な手順を伝える／理解度を確認し適切なタイミングでフィードバック
  2. **コーチング活用ポイント**：質問を活用して相手の考えを引き出す／共感的なコミュニケーションで信頼関係を築く
  3. **リーディング活用ポイント**：チーム全体に方向性を示しメンバーの役割を明確化／定期的に進捗を確認し適切な指導
- 出力：選定アプローチ別の活用設計

### ステップ5：自己評価とフィードバック計画
- 入力：実施記録
- 処理：以下3軸で点検
  - **ティーチング**：自分が得意なスキルや知識を誰かに伝える練習をしたか／質問を受けた場合に分かりやすく説明する練習をしたか
  - **コーチング**：相手の考えを引き出す質問を3つ考えたか／日常の対話で傾聴力を意識したか
  - **リーディング**：チームの目標を設定し分かりやすく伝える練習をしたか／進捗確認のタイミングと方法を計画したか
- 出力：自己評価＋次回改善ポイント

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 新入社員に業務手順を教える | ティーチング主体 | 初心者には知識伝達が必要 |
| 中堅社員のキャリア相談 | コーチング主体 | 自分で答えを見つけられる成長段階 |
| プロジェクトリーダーがチーム運営 | リーディング主体 | 方向性提示と意思決定が必要 |
| ベテラン社員の指導 | コーチング＋リーディング | ティーチングは押し付けになる |
| 相手が答えを持っているが言語化できない | コーチング | 質問で引き出す |
| 緊急対応で時間がない | ティーチング | 即座に伝える必要 |
| チームのモチベーションが下がっている | リーディング（ビジョン再提示＋モチベーション管理） | 目標を再共有 |
| 一律のアプローチに固執している | 状況に応じて柔軟に組み合わせる | 単一アプローチでは多様な相手に対応できない |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 全員にティーチングで対応 | 相手の習熟度・主体性で使い分け | 中堅以降には押し付けになる |
| ベテランにティーチング | コーチング・リーディングで対応 | プライドを傷つけ反発を招く |
| 初心者にコーチング | まずティーチングで基礎習得 | 答えを持たない相手に質問しても進まない |
| 全部自分で決める（過剰リーディング） | コーチングで主体性を引き出す | メンバーが受け身になる |
| 質問だけして方向性を示さない | リーディングで方向性も示す | 迷子になる |
| FBなしで放置（ティーチング後） | 理解度確認＋FBで定着させる | 学習効果が出ない |
| 状況に応じた使い分けをしない | 3アプローチを柔軟に組み合わせ | 単一アプローチでは限界がある |

## 連携スキル

- `hearing-questioning-skills`：コーチングの質問力・傾聴力の基礎技術
- `project-team-structure`：リーディングのチーム体制設計
- `mvv-design`：リーディングで提示するビジョン・ミッションの言語化
- `goal-hierarchy-design`：目標の階層分解（ティーチング・リーディングで活用）
- `pdca-cycle`：継続的な指導改善サイクル
- `decision-making-framework`：リーディングの意思決定フレーム
- `goal-execution-system`：モニタリングとフィードバックループ

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：3アプローチのいずれも適用条件を満たさない場合（要再診断）
- `[PARTIAL:N成功/M失敗]`：複数アプローチを組み合わせる際、一部しか設計できなかった場合
- `[SKIP]`：相手の状態が不明確で診断を保留した場合（前提不足）

※外部API依存なしのため `[FAIL]` は通常発生しない。前提不足は `[SKIP]` で扱う。
