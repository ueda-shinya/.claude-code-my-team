# proposal-builder

商談・ヒアリングで得た情報から、9セクション構成の営業提案書を一貫生成する。ストーリー設計→セクション別コピー→最終成果物の順で進行し、各段階でレビュー・修正を挟む。

## 適用条件

- **使う場面：**
  - 新規見込み客への営業提案書・サービス提案書
  - コンサルティング案件の提案書
  - パートナーシップ提案書
  - 社内プロジェクト提案書

- **使わない場面：**
  - 技術仕様書中心のRFP対応（技術ドキュメントのスキルを使う）
  - 単純な見積書（1-2ページで済む場合はクイック版を使う）
  - 法的契約書（法務レビューが必要）
  - 価格カスタマイズなしの定型見積り

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 見込み客情報 | 必須 | 会社名、業種、規模、担当者 | 「広島の製造業、従業員30名、社長の田中さん」 |
| ヒアリング内容 | 必須 | 商談・会話のメモ、課題、要望 | 「ホームページが古い、問い合わせが来ない」 |
| 提案するサービス | 必須 | 提案内容、含まれるもの | 「ITまるごとサポート まるっとおまかせプラン」 |
| 価格 | 必須 | 価格構成、支払条件 | 「月額27,500円、初月半額、最低6ヶ月」 |
| 競合情報 | 任意 | 比較されている他社、差別化ポイント | 「全国対応のリモート保守業者と比較中」 |
| 過去の提案書 | 任意 | 類似案件の提案書（テンプレート参照用） | ファイルパス |
| 提案書タイプ | 任意 | フル提案書/エグゼクティブサマリー/クイック見積 | デフォルト: フル提案書 |

## 実行ステップ

### ステップ1：入力情報を収集・整理する

- 入力：ユーザーが提供した商談情報（テキスト、ファイル、または口頭情報）
- 処理：
  1. 以下の6カテゴリに分類する
     - 見込み客情報（会社名・業種・規模・担当者・現状）
     - 課題・ペインポイント（見込み客が言った言葉をそのまま記録）
     - 成功基準（見込み客が望む状態）
     - 提案するサービスの内容
     - 価格・条件
     - 競合・差別化
  2. 不足している項目を特定する
  3. 不足が2項目以上ある場合 → ユーザーに追加情報を求める
  4. 不足が1項目以下の場合 → 推測可能なら補完して進行、不可能ならユーザーに確認
- 出力：6カテゴリに分類された情報一覧

### ステップ2：提案書タイプを判定する

- 入力：整理済み情報、ユーザーの指定（あれば）
- 処理：
  - ユーザーが明示的にタイプを指定している → その通りにする
  - 指定がない場合、以下の基準で判定する

| 条件 | タイプ | セクション数 |
|---|---|---|
| 価格50万円以上 or 長期契約（6ヶ月以上）or 複数意思決定者 | フル提案書 | 9セクション |
| 価格10〜50万円 or 月額サービス | エグゼクティブサマリー | 5セクション |
| 価格10万円未満 or 単発スポット | クイック見積 | 3セクション |

- 出力：選択された提案書タイプ

### ステップ3：ストーリーの骨格を設計する

- 入力：分類済み情報、提案書タイプ
- 処理：
  1. **見込み客の課題を1文で要約する**（提案書全体を貫くメインメッセージ）
  2. **フル提案書（9セクション）の場合**、各セクションの核メッセージを1文ずつ決める：

| # | セクション | 核メッセージの設計基準 |
|---|---|---|
| 1 | 表紙 | 提案タイトルは「見込み客の課題＋解決の方向性」で書く |
| 2 | エグゼクティブサマリー | 見込み客の言葉で課題→解決→投資→期待効果の順で書く。自社の話は最小限 |
| 3 | 課題の理解 | 見込み客が発言した課題をそのまま引用し、ビジネスインパクトを数字で示す |
| 4 | 提案する解決策 | 課題→解決の対応を1対1でマッピングする。機能ではなく効果で書く |
| 5 | なぜ当社か | 実績数字、具体事例、差別化3点に絞る |
| 6 | 実施アプローチ | フェーズ・マイルストーン・役割分担を可視化する |
| 7 | 投資 | ROI→価格の順で提示する。先に価値を示してから金額を出す |
| 8 | 次のステップ | 具体的な日程・アクションを3つ以内で明示する |
| 9 | 付録 | 判断に必要だが本文を長くするものを格納する |

  3. **エグゼクティブサマリー（5セクション）の場合**：セクション1,2,4,7,8を使用
  4. **クイック見積（3セクション）の場合**：課題理解（1段落）、提案内容＋価格、次のステップ
- 出力：セクション別の核メッセージ一覧

### ステップ4：セクション別コピーを生成する

- 入力：セクション別の核メッセージ一覧、分類済み情報
- 処理：
  1. 各セクションのコピーを順番に生成する
  2. 生成時のルール：
     - **見込み客の言葉を最優先で使う**（「パソコンの調子が悪い」は「IT環境の最適化」に言い換えない）
     - **数字を入れられるところは全て数字にする**（「効率化」→「月4時間の削減」）
     - **「自社の話」は全体の30%以下に抑える**（70%は見込み客の課題と解決）
     - **各セクションは1ページ以内**（エグゼクティブサマリーは厳密に1ページ）
  3. セクション2（エグゼクティブサマリー）は最後に書く（全体を書いた後の方が精度が上がるため）
- 出力：全セクションのコピー（Markdown形式）

### ステップ5：品質チェックを実行する

- 入力：生成された全セクションのコピー
- 処理：以下のチェックリストを1つずつ確認する

| チェック項目 | 合格基準 | 不合格時の対応 |
|---|---|---|
| 見込み客の課題が最初に来ているか | セクション1-3が見込み客の話 | 順序を入れ替える |
| 数字が入っているか | 各セクションに最低1つの数字 | 推定値でも入れる（「推定」と注記） |
| 見込み客の言葉を使っているか | ヒアリングメモの表現が3箇所以上引用 | 置き換える |
| ROIが価格の前に来ているか | セクション7の構成確認 | 順序を入れ替える |
| 次のステップが具体的か | 日付・アクション・担当が明記 | 追記する |
| NGワードが含まれていないか | ブランディング方針のNGワードチェック | 置き換える |

- 出力：チェック済み・修正済みの全セクションコピー

### ステップ6：最終成果物を出力する

- 入力：品質チェック済みの全セクションコピー
- 処理：
  1. **提案書本体**（Markdown形式）を1ファイルに結合して出力する
     - 保存先：`~/.claude/clients/<クライアント名>/proposals/<提案名>.md`
     - クライアントディレクトリが存在しない場合 → `~/.claude/reports/proposals/<提案名>.md`
  2. **エグゼクティブサマリー版**（1-2ページ抜粋）を別ファイルで出力する
  3. **送付メール文面**を生成する
     - 件名、本文（3行以内で要点）、次のステップを含む
  4. ユーザーに以下を確認する：
     - 「スライド版も作りますか？」→ はいの場合、sales-slide-builder スキルまたは pptx-from-layouts スキルに引き継ぐ
     - 「Web版も作りますか？」→ はいの場合、create-an-asset スキルに引き継ぐ
- 出力：提案書本体、エグゼクティブサマリー、送付メール文面

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ヒアリング情報が少ない（課題が1つしかない） | クイック見積に切り替える | 無理にフル提案書を作ると内容が薄くなる |
| 価格が未定 | 「価格は別途ご提示」セクションにして進める | 提案書の構成は価格がなくても作れる |
| 競合情報がない | セクション5は「実績と差別化」に絞る | 競合不明で比較を書くと的外れになる |
| 見込み客が技術者 | 付録に技術詳細を充実させる | 意思決定者向けの本文は変えない |
| 過去の提案書がある | 構成・トーンを参考にする | ただしコピペはしない。毎回カスタマイズ |

## エグゼクティブサマリー

サンプル製作所様は現在、「パソコンやネットのことで困っても、聞ける人がいない」

社長の山田様ご自身が月4時間以上をIT対応に費やしており、

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 自社紹介から始める | 見込み客の課題から始める | 読み手は自分の問題に関心がある |
| 「効率化を実現します」 | 「月4時間のIT対応時間がゼロになります」 | 数字がないと信じてもらえない |
| 全機能を羅列する | 課題に対応する機能だけ書く | 関係ない機能は読み飛ばされる |
| 価格を最初に出す | ROI→価格の順で出す | 先に価値を理解してから金額を見る |
| 「ご検討ください」で終わる | 「来週月曜に30分、お電話させてください」 | 具体的でないと後回しにされる |
| 専門用語で書く | 見込み客の言葉で書く | 相手が使わない言葉は伝わらない |
