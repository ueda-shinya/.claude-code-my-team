# ga4-analysis-fundamentals

Googleアナリティクス4（GA4）の基本構造を理解し、初期設定から恒常運用までを設計するスキル。「誰が」「どこから」「どのように」「目標達成したか」を4構成（集客／エンゲージメント／ユーザー属性／リアルタイム）で把握し、ウェブマーケの大枠の考え方（訪問数×目標完了率=目標数）で改善打ち手を導出する。

## 適用条件

- 使う場面：
  - GA4を導入したばかりで何を見ればいいか分からない
  - 既存GA4の初期設定（ユーザー属性／Search Console連携／キーイベント）が不十分
  - GA4の用語の意味と「どう使うか」を整理したい
  - 月次/週次でGA4を見るときの定常チェック観点を作りたい
  - GA分析を活用した改善計画を立てたい
- 使わない場面：
  - 流入減・CV減の1次診断 → `ga-gsc-diagnosis`
  - 既存記事のSEO監査 → `seo-audit`
  - 新規メディアのSEO戦略設計 → `seo-content-strategy`
  - LPの改善 → `lpo-improvement-design`
  - 広告のCPA分析 → `ad-performance-diagnosis`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象サイトURL | 必須 | GA4を設置済のサイト | 「koujitsu.co.jp」 |
| GA4設置状況 | 必須 | 設置済／未設置／設置済だが不安 | 「設置済・基本設定のみ」 |
| 既存KPI | 推奨 | 現在の月間PV／セッション／CV数 | 「月間PV5万・CV月15件」 |
| 改善目的 | 推奨 | 流入増／CV増／離脱改善／チャネル別最適化 | 「CV月25件達成」 |

## 実行ステップ

### ステップ1：GA4の4構成を理解する

- 入力：対象サイトURL
- 処理：GA4で「誰が」「どこから」「どのように」「目標達成したか」を把握する4つの観点を確認
  - **集客**：どこからサイトに来たか
  - **エンゲージメント**：どのように目標達成したか
  - **ユーザー属性**：誰がサイトを見ているか
  - **リアルタイム**：今何人サイトに来ているか
- メニュー構成：レポートのスナップショット／リアルタイム／Life cycle（集客・エンゲージメント・収益化・維持率）／Search Console／User（ユーザー属性・テクノロジー）
- 出力：GA4 4構成の全体像

### ステップ2：5チャネルの定義を確認する

- 入力：GA4の集客レポート
- 処理：以下5つのチャネルでサイトへの流入を分類
  - **Organic Search**：検索エンジンからの流入
  - **Referral**：他サイトからの流入（ブログなど）
  - **Direct**：ブックマークなどからの流入
  - **Social**：SNSからの流入
  - **Paid Search**：リスティング広告
- 補足チャネル（主要なもの）：Cross-network／Email／Organic Social／Organic Shopping／Paid Other／Unassigned 等。GA4のデフォルトチャネルグループは合計17種類あるため、自社サイトに該当するチャネルは管理画面で全件確認すること
- 出力：自社サイトの主要流入チャネル特定

### ステップ3：6主要用語を理解する

- 入力：GA4のレポート画面
- 処理：以下6用語の定義を共有

| 用語 | 英語 | 定義 |
|---|---|---|
| セッション | Sessions | ユーザーがサイトを訪問してから離脱するまでの一連の行動 |
| ユーザー | Users | サイトを訪問した個別のユーザー数（再訪問者も含む） |
| ページビュー数 | Pageviews | サイト内で閲覧されたページの総数 |
| 直帰率 | Bounce Rate | 訪問者が最初のページだけを閲覧して離脱した割合 |
| コンバージョン率 | Conversion Rate | 訪問者のうち、目標（購入、登録など）を達成した割合 |
| 平均セッション時間 | Average Sessions Duration | 訪問者がサイト内で過ごした平均時間 |

- 出力：用語定義表（チームでの共通言語化）

### ステップ4：GA4 最初にすること（3つの初期設定）

#### ①ユーザー属性の有効化

- ユーザー属性 > 概要 をクリックして表示されない場合 → ユーザー属性が取れていない
- 設定方法：管理メニュー > データの収集と修正 > データの収集 > 「Googleシグナルのデータ収集」右のチェックボックスを有効化
- 効果：年齢・性別・地域・興味関心が取得可能に

#### ②サーチコンソール設定と紐づけ

- 左サイドメニューに「Search Console」が表示されていなければ紐づけ未完了
- 設定方法：管理メニュー > サービス間のリンク設定 > Search Console連携
- メニュー表示は「ライブラリ」から選択
- 効果：オーガニック検索クエリ・順位データが GA4 内で確認可能に
- 参考：https://ayudante.jp/column/2023-01-18/15-00/

#### ③目標設定（キーイベント）

- 重要なイベント（お問い合わせ等）を「キーイベント」として登録
- 設定方法：管理メニュー > 「キーイベント」から設定
- 前提：キーイベント登録対象のイベントが GA上で計測されている必要がある
- 効果：チャネル別CV数・CV率が比較可能に

- 出力：3つの初期設定の完了確認チェック

### ステップ5：見るポイント4軸を運用ルール化する

#### POINT 01：比較と比較時期

- **必ず比較する数値を用いて見ること**
- 比較の基本：「昨年、同月」
- 閑散期と繁忙期を比較しても良い悪いが分からない
- 例外：できたてのサイト等で昨年データが参考にならない場合は昨月などと比較

#### POINT 02：トラフィック獲得

- 集客 > トラフィック獲得 から、流入経路別アクセス数を確認
- 流入経路別にキーイベント発生数が確認できる
- **全体のアクセス状況を見るバロメーターとして最もよく参照するレポート**

#### POINT 03：サーチコンソールの検索クエリ

- 集客 > サーチコンソール > 検索クエリ から、どんな検索でユーザーが流入したかを確認
- 対象KWはGoogleで検索したときのサイト順位、表示回数、クリック数、クリック率を一覧で出してくれる
- 効果のあるワードの抽出ができ、強めたいワードの選定でSEO対策にもつながる

#### POINT 04：ウェブマーケ大枠の考え方

- **基本式：訪問数（セッション）× 目標完了率（CV率）= 目標数（CV数）**
- どの企業も共通の課題：「売上を上げたい」
- そのために重要視されるのは「認知度を上げる」「問い合わせを増やす」
- 「問い合わせを増やす」=③目標数アップ
- ③を増やすために：仮説を立てて実行し、分析して改善する
- 前提：As-Is To-Beを立てた上の話

**大枠の考え方の例**：
- **①訪問数を上げるため**：SEO強化／広告出稿／SNS運用／リファーラル追加／メルマガ／アプリなど、チャネル増加 or 既存チャネルからの流入アップで仮説立て→効果計測→改善
- **②目標完了率を上げるため**：既存サイトの問題点を分析して改修／場合によってはサイト全体をリニューアル→計測→改善
- **③を伸ばすために**：①の増加 or ②の割合増加 or 両方

- 出力：見るポイント4軸の運用ルール

### ステップ6：GA分析を活用した改善計画を立てる

- 入力：ステップ5の見るポイント結果
- 処理：成功事例から学びを取り入れた改善計画を立案
- **参考成功事例**：
  - **ドミノ・ピザ**：GAでクロスチャネル・クロスデバイス行動分析→マーケ施策統合・オンオフデータ結合→月間収益6%増加・広告支出80%削減
  - **パナソニック**：GA360で全マーケデータ一元管理→特定ライフイベント基づくターゲティング→広告ROAS 30%向上
  - **Fairmont Hotels**：GAでX（Twitter）からのトラフィック品質測定→URLビルダー活用→ソーシャルメディア戦略改善
- データに基づいた意思決定を行うことで、収益向上・顧客ニーズ対応・競争力強化・持続的成長を実現
- 出力：自社GA改善計画

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| GA4導入直後 | ステップ4の3つの初期設定を最優先 | 設定欠如のままだと比較・分析不可 |
| ユーザー属性が取れていない | Googleシグナル有効化（ステップ4-①） | 属性データなしではターゲティング不可 |
| サーチコンソール未連携 | ステップ4-②で連携必須 | SEO効果の可視化に必須 |
| キーイベント未設定 | ステップ4-③でCV計測の基盤を作る | CV数・CV率なしでは改善判断不可 |
| 月次レビュー時 | POINT 01「昨年同月比」を厳守 | 季節性・閑散繁忙の影響排除 |
| サイト立ち上げ初期 | 昨月比など可能な比較を運用 | 過去データ不足は補完的に対応 |
| 全体のアクセス傾向把握 | POINT 02 トラフィック獲得を最初に見る | バロメーター指標 |
| SEO戦略立案前 | POINT 03 サーチコンソール検索クエリ精査 | 流入KW根拠化 |
| CV数を増やしたい | POINT 04 訪問数×CVR=CVで分解 | 改善対象の特定 |
| 訪問数が頭打ち | チャネル増加（SEO/広告/SNS/メール）優先 | 流入の蛇口を増やす |
| CVRが低い | サイト改修（LPO/UI/UX）優先 | 通過率改善 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 用語の意味を曖昧なままレビュー | ステップ3で用語定義表を共通言語化 | チーム議論が成立しない |
| 初期設定（属性/SC/キーイベント）スキップ | ステップ4で3点必ず確認 | 設定欠如だと分析データが片手落ち |
| 数値を絶対値だけで評価 | POINT 01「昨年同月比」を厳守 | 季節性で誤判断 |
| トラフィック獲得を見ずチャネル別分析だけ | POINT 02を最初に見る | 全体感→詳細の順 |
| 検索クエリ確認なしでSEO戦略立案 | POINT 03で流入KWを根拠化 | 推測ベースのSEO戦略は外す |
| CV数だけ追って訪問数/CVRを分解しない | POINT 04の式で原因切り分け | 改善対象が特定できない |
| 全部同時に改善（訪問数もCVRも） | 優先順位を付けて1〜2施策に集中 | リソース分散で効果薄 |
| キーイベントの命名がバラバラ | チーム内で命名規則を統一 | 集計・比較ができなくなる |
