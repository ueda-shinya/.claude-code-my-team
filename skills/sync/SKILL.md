# /sync スキル

GitHub リポジトリと `.claude` 設定を同期します。

## 使い方

- `/sync` → 状態確認 + 変更があれば push
- `/sync pull` → GitHub から最新を取得（別PCで変更した後に使う）
- `/sync push` → 現在の変更を GitHub へ push

## 手順

### ステップ0：セッション振り返り（引数なしの場合のみ・必須）

git 操作の前に、アスカが今日の会話を振り返り、以下のシーケンスを**必ず**実行する。

#### 0-1. 振り返り（内部）

今日の会話全体を振り返り、以下を洗い出す：
- やったこと・完了したこと
- 決まった方針・ルール・暫定対策
- 解決した問題・判明した原因
- 発覚した制約・前提・仕様
- エージェントの動き方の修正
- フィードバック・指摘
- シンヤさんについて新たに分かったこと（好み・思考スタイル等）

#### 0-2. 内容をシンヤさんに報告

振り返りの内容を整理してシンヤさんに報告する。形式は以下を目安に：

```
【今日のセッション振り返り】
■ やったこと
- ...
■ 決まったこと・学んだこと
- ...
■ 持ち越し（あれば）
- ...
```

#### 0-3. 記録

すでにナレッジ化・メモリ保存済みのものを除いて、残っているものを記録する：
- フィードバック・ルール → `memory/` の該当ファイルに追記または新規作成
- 技術知見・ワークフロー → `knowledge/` の該当ファイルに追記または新規作成
- シンヤさんの情報 → `memory/user-shinya.md` または分割ファイル

#### 0-4. 記録したことをシンヤさんに報告

記録したファイルと内容を一言報告する。記録するものがなかった場合も「記録するものはありませんでした」と報告する。

```
【記録完了】
- memory/feedback-xxx.md：〇〇を追記
- knowledge/xxx/yyy.md：〇〇を追加
```

#### 0-5. ファイル更新した場合はその場でコミット

ファイルを更新した場合は `git add -A` → `git commit` → `git push` をここで完結させる（状態確認ステップに委ねない）。

### 状態確認（引数なしの場合）

```bash
cd ~/.claude
git status --short
git log origin/main..HEAD --oneline  # pushされていないコミット
git log HEAD..origin/main --oneline  # pullが必要なコミット
```

※ ステップ0でメモリ・ナレッジファイルを更新済みの場合、ここでは既にコミット・push済みのはず。万一残っていれば漏れなので必ず拾うこと。

状態を確認して以下を報告する：
- ローカルに未コミットの変更があるか
- GitHub より先行しているか（push が必要）
- GitHub に新しいコミットがあるか（pull が必要）

変更がある場合は、コミット内容を確認して `git add` → `git commit` → `git push` を実行する。

### pull の場合

```bash
cd ~/.claude
git pull origin main
```

完了後「最新の状態に更新しました」と報告する。

### push の場合

```bash
cd ~/.claude
git add -A
git status --short
```

変更ファイルを確認後、コミットメッセージを生成して `git commit` → `git push` を実行する。

## 注意事項

- `.credentials.json` など認証情報はコミットしない（.gitignore で管理済み）
- push 前に必ず変更内容をシンヤさんに確認する
- コンフリクトが発生した場合は、内容をシンヤさんに報告して判断を仰ぐ
