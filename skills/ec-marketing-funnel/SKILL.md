# ec-marketing-funnel スキル

ECマーケティング（電子商取引を通じた商品の認知拡大・売上向上を図るマーケティング活動）の戦略設計を、売上方程式（アクセス数×CVR×客単価）と顧客5層×ファネル4ボトルネック構造で実施する。

## 適用条件

- このスキルを使う場面：
  - ECサイト（Amazon／楽天／Yahoo!ショッピング／STORES／Shopify／ecforce／makeshop／BASE 等）の売上拡大戦略を立案したい
  - 既存ECサイトの売上停滞要因を「アクセス／CVR／客単価」のどこに問題があるか特定したい
  - 顧客層別（ロイヤル／一般／離反／認知未購買／未認知）の施策を体系化したい
  - F2転換（初回→2回目購入）や休眠復活、ロイヤル化のボトルネック対策を設計したい
- このスキルを使わない場面：
  - BtoB SaaS のリード獲得・MQL/SQL設計 → `lead-definition-mql-sql` ／ `inside-sales-sdr-bdr`
  - 一般的なファネル3タイプ（購入／検討／行動）設計 → `funnel-design`
  - LP単体のCVR改善 → `lpo-improvement-design`
  - サイト全体（複数ページ）のUI/UX改善 → `ui-ux-improvement-fundamentals`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 商材カテゴリ | 必須 | 食品／コスメ／アパレル／日用品 等 | 「日本酒EC」「化粧品EC」 |
| プラットフォーム | 必須 | 自社EC／モール（Amazon/楽天等）／併用 | 「Shopify自社EC」「楽天モール出店」 |
| 現状のKPI | 任意 | 月間アクセス数／CVR／客単価／月商 | 「月10,000UU・CVR1.2%・客単価4,500円」 |
| 顧客データの有無 | 任意 | ロイヤル/一般/休眠の人数把握状況 | 「ハウスリスト3,000名・休眠1,200名」 |
| 解決したい課題 | 必須 | 売上拡大／F2転換改善／休眠復活 等 | 「F2転換率が5%以下で改善したい」 |

## 実行ステップ

### ステップ1：売上方程式で現状を分解する

ECサイトの売上は **アクセス数 × CVR × 客単価** の3要素で構成される。まず現状の数値を分解し、ボトルネックを特定する。

| 要素 | 定義 | 主な改善対象 |
|---|---|---|
| アクセス数（訪問者数） | サイトを訪れるユーザー数 | 集客5方法 |
| CVR（コンバージョン率） | 訪問者のうち購入や問い合わせに至る割合 | CVR改善3ポイント |
| 客単価 | 1回の購入でユーザーが使う平均金額 | 客単価向上3施策 |

数値が把握できない場合は、GA4／Google Search Console／プラットフォーム管理画面（Shopify Analytics／楽天RMS／Amazon Seller Central等）から取得を推奨。

### ステップ2：集客方法を5つから選定する

| # | 方法 | 概要 | 向く商材／フェーズ |
|---|---|---|---|
| 01 | 自然検索（SEO） | サイトの検索エンジン順位を上げて流入を増やす | 中長期視点・競合度低い商品 |
| 02 | SNS投稿 | Instagram、TikTok、X（旧Twitter）などを活用 | 視覚訴求しやすい商材／若年層ターゲット |
| 03 | WEB広告 | リスティング広告やディスプレイ広告で特定ユーザーにリーチ | 即効性が必要・予算ある場合 |
| 04 | メールマーケティング | メルマガで顧客と継続的な接点を確保 | 既存顧客リストあり／F2転換施策 |
| 05 | 被リンク（Referral） | 他サイトからのリンクで流入を促進 | プレスリリース／紹介プログラム |

詳細な集客チャネル設計は `promotion-strategy` ／ `seo-content-strategy` ／ `display-ad-design` ／ `listing-ad-campaign-design` ／ `meta-ad-campaign-design` に引き継ぐ。

### ステップ3：CVR改善3ポイントで購入率を上げる

| ポイント | 具体施策 | Before/After判定 |
|---|---|---|
| ① 商品画像と説明の最適化 | 高品質な画像・分かりやすい説明文を提供。シズル感／使用シーン／成分・原産地などを明示 | 商品ページ離脱率が改善するか |
| ② UI/UX の向上 | ユーザーが直感的に操作できる導線設計（カテゴリ整理／検索性／レコメンド配置） | カート投入率・遷移率が改善するか |
| ③ カートと決済機能の改善 | 購入プロセスをシンプル化（ステップ削減）／安心感のある決済手段（Amazon Pay／PayPay／コンビニ決済等）を導入 | カート離脱率が改善するか |

### ステップ4：客単価向上3施策を実施する

| 施策 | 具体内容 | 期待効果 |
|---|---|---|
| セット商品の販売 | 関連商品を組み合わせたセット販売で1回の購入額を増やす | 客単価UP／在庫回転改善 |
| 送料無料の条件設定 | 一定額以上で送料無料（例：5,000円以上）にすることで購入額の底上げ | 客単価UP／単品買いから複数買いへ誘導 |
| レコメンド機能 | 顧客に関連商品を提案し追加購入を促す（カート画面／購入完了画面／メールに配置） | 客単価UP／クロスセル |

### ステップ5：顧客5層で施策を分類する

EC顧客は購買経験と頻度で5層に分類できる。各層に対して適切な施策を実施する。

| セグメント | 定義 | 抽出条件例 | 適用施策の方向 |
|---|---|---|---|
| ① ロイヤル顧客 | 認知あり／購買頻度・高 | 直近1年に3回以上購入 | ロイヤル化促進・LTV最大化 |
| ② 一般顧客 | 認知あり／購買頻度・中〜低 | 直近1年に1回以上購入 | F2転換／継続購入促進 |
| ③ 離反顧客 | 認知あり／購買経験あり／現在購買なし | 過去に利用したが直近1年に購入なし | 休眠復活施策 |
| ④ 認知・未購買顧客 | 認知あり／購買経験なし | 直近1年のサイト訪問者UU | 初回購入促進 |
| ⑤ 未認知顧客 | 認知なし | ターゲット人口 - ④ | 認知拡大／初回購入促進 |

**運用方針（5層別の対応施策）**：

- ① ロイヤル顧客 → ロイヤル化維持・LTV最大化施策（ステップ6④へ）
- ② 一般顧客 → F2/F3転換・継続購入促進施策（ステップ6②と④へ）
- ③ 離反顧客 → 休眠復活（再購入）施策（ステップ6③へ）
- ④ 認知・未購買顧客 → 初回購入促進施策（ステップ6①へ）
- ⑤ 未認知顧客 → 認知拡大＋初回購入促進施策（ステップ2集客5方法 → ステップ6①へ）

層ごとに必要な施策が異なるため、ハウスリストの構成比（ロイヤル何%／離反何%／未購買何%）を把握してから優先順位を決める。

### ステップ6：ファネル4ボトルネック対策を講じる

一般的に、ECサイトのボトルネックは **初回購入・2回目購入（F2）・休眠・ロイヤル化** の4つにある。それぞれに対応施策を設計する。

#### ① 初回購入者の獲得（運用開始間もないショップ向け）

| 施策 | 概要 |
|---|---|
| 初回お試しセット（入口商品） | 「初回限定お試しセット」等、1回だけ購入できるエントリー商品。通常価格より安く設定。複数お試しできるセット販売が多い |
| 初回購入特典 | 初回購入時だけの特典。初回のみの割引やポイントアップ、次回使える割引率の高いクーポン、サンプルプレゼントなど |
| 目玉商品（ヒーローアイテム） | 楽天でいうとランキングが狙えるようなヒーローアイテム。ECサイトは特定の商品の売上が全体の売上を引っ張ることが多い |

#### ② 2回目購入（F2転換）の促進

新規顧客が継続顧客になる第1ステップが「2回目の購入」。「買った瞬間」「商品が届いた瞬間」が最もホットで、F2促進の最大のチャンス。

| 施策 | 概要 |
|---|---|
| 初回購入直後のオファー | 購入直後が最も関心度が高いタイミング。「次回使える1か月限定クーポン」等をサンクスメールで配布 |
| 公式LINE登録のオファー | プッシュで情報を届けられるよう、LINE登録のオファーをサンクスメール等で送付。登録率が低い場合はクーポンを特典に |
| 同梱物の工夫 | 商品の同梱物に他商品の案内・各種SNSのQRコード・クーポンなど次につなげる要素を盛り込む。コスメなら使い方/サンプル、食品ならレシピブック等 |

#### ③ 休眠復活の促進

一定期間以上の購入がない顧客を「休眠顧客」と定義。新規顧客獲得より、一度買ったことがある人に思い出してもらうほうがコストがかからない。ハウスリストに休眠が溜まっている場合は対策を打つ。

| 施策 | 概要 |
|---|---|
| 誕生日クーポン | メルマガやLINEで配布する誕生日クーポンをきっかけに再購入してもらう |
| 起死回生アイテム | 休眠から復活した顧客のデータが複数ある場合、購入されているアイテムに共通項が見られないかを調査し、その共通アイテムでの呼び戻し施策を打つ |

#### ④ ロイヤル顧客化の促進

一般的に既存顧客のほうが新規顧客より客単価が高く、購入回数が多い顧客のほうが1回の購入単価が高い傾向がある。ECサイトの理想は「2割のロイヤル顧客が8割の売上を作り出している」状態。複数回の購入がみられたらロイヤル化対策を打つ。

| 施策 | 概要 |
|---|---|
| ゴールデンルート分析 | ロイヤル化しやすい商品の購入傾向を調査（例：①初回お試しセット → ②定番商品のセット → ③頒布会 など） |
| 顧客調査 | ロイヤル顧客にアンケートまたはインタビュー調査。商品の何を魅力に思っているか、普段どのメディアやSNSを見ているか、再購入のきっかけは何か等 |

### ステップ7：成功のポイントと次のアクションをまとめる

#### ECマーケティング成功の3ポイント

1. 売上方程式を基に目標を明確化（KGI＝月商×構成3要素＝アクセス・CVR・客単価のKPI設定）
2. 顧客行動データを分析して施策を実施（GA4／プラットフォーム管理画面／CRMデータの統合分析）
3. ファネルごとの課題を特定し、適切な対策を行う（4ボトルネックそれぞれに対する施策の優先順位付け）

#### 次のステップ

- KPIの設定とモニタリング（→ `kgi-kpi-kai-design` ／ `ga4-analysis-fundamentals`）
- 小さな改善を積み重ね、売上拡大を目指す（→ `pdca-cycle`）

## 判断基準

| 状況 | 判断 |
|---|---|
| 月商伸び悩みで原因が不明 | ステップ1の売上方程式分解から開始（アクセス／CVR／客単価のどれがボトルネックか特定） |
| アクセスは多いが買われない | CVR改善3ポイント（ステップ3）を最優先 |
| 初回購入は取れるが2回目に繋がらない | F2転換施策（ステップ6②）を最優先 |
| ハウスリストに休眠が溜まっている | 休眠復活施策（ステップ6③）を実施 |
| 客単価が低く新規獲得コストを回収できない | 客単価向上3施策（ステップ4）＋ロイヤル化（ステップ6④）を併走 |
| 自社EC vs モールどちらに注力するか不明 | プラットフォーム特性別の集客5方法（ステップ2）を比較し、商材特性と相性を判定 |
| BtoBで購買サイクルが長い案件 | ECスキルではなく `lead-definition-mql-sql` ／ `inside-sales-sdr-bdr` に切り替え |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| アクセス施策（広告／SEO）だけに偏る | 売上方程式3要素を分解し、CVR・客単価のボトルネックも同時診断 | アクセスだけ伸ばしてもCVR/客単価が低いと利益が出ない |
| 顧客5層をひとくくりに「顧客」として施策を打つ | 5層別に施策を分類し、層ごとの最適アプローチを設計 | ロイヤルと未認知では必要な施策が真逆 |
| F2転換施策を「初回購入直後」ではなく後追いで実施 | 「買った瞬間」「商品が届いた瞬間」のホットなタイミングで施策を打つ | 関心度が下がるとF2転換コストが急上昇 |
| 休眠顧客を放置して新規獲得ばかりに投資 | ハウスリストの休眠を起死回生アイテム／誕生日クーポンで復活させる | 新規獲得コスト > 休眠復活コスト |
| 「ECサイトのファネル」を `funnel-design` で代用 | ECは売上方程式×4ボトルネックで設計し、本スキルで対応 | 一般ファネル3タイプ（購入/検討/行動）とEC特化のボトルネック構造は別物 |
| LP単体のCVR改善を本スキルで対応 | LP単体は `lpo-improvement-design` に引き継ぐ | スコープがズレて改善施策が拡散する |

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存を持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし運用上のベストプラクティスとして、入力情報不足の場合は `[SKIP]` で「{{要素}}の現状値が未確定のため診断スキップ」と明示し、推測で埋めない。
