# LP Design System

コピーファースト設計でLP/スワイプLPのビジュアルデザイン仕様書を作成する。コピーが主役、ビジュアルは補強。

## 適用条件と起動経路

**呼び出し経路は2系統:**

1. **`lp-create` 経由（推奨）**: 本スキルは `lp-create` の **Step 5（Design Specification）の実行スキル**として呼び出される。`lp-create` 経由で起動された場合、Step 1〜4 の成果物（strategy.md / copy.md / ヒアリングシート Phase 1, 4, 7 等）を入力として受け取る。
2. **単独起動（Route D 相当）**: シンヤさんが「LPデザインして」「LPのビジュアル設計して」等で直接呼び出した場合。この場合、入力（コピー一覧・CVゴール・ターゲット・LP形式・ブランドカラー・ビジュアル制作手段・CTA仕様）はシンヤさんから直接受け取る。`lp-create` Route D の Step 0 ゲート（コピー確定）を通過していることが前提のため、**ステップ-1（コピー確定チェック）を最初に必ず実行する**。

**既存のStep 3.5（design-spec-template.md）との関係:** このスキルは旧 Step 3.5（`design-spec-template.md`）の**完全な代替**であり、現在の `lp-create` スキルでは **Step 5（Design Specification）の実行スキル**として位置付けられる。`lp-create` 経由で起動された場合、Step 1〜4 の成果物を入力として受け取る。**旧 `design-spec-template.md` は `~/.claude/knowledge/lp-design-legacy/` に退避済みで、本スキル・lp-designer エージェント・lp-create スキルからの参照は禁止**。

## 適用条件

- 使う場面: LP構成が確定した後のビジュアルデザイン設計、スワイプLPのスライドデザイン設計、既存LPのデザイン改善
- 使わない場面: LP構成そのものの設計（`lp-create` Step 3 = Route S の範囲）、コピーの作成（`lp-create` Step 4 = Route C のコト担当）、HTML/CSS実装（`lp-create` Step 6 = Route I のシュウ担当）
- コピーが仮の場合: 仮コピーでも実行可能。ただし出力の冒頭に「※仮コピーに基づく設計。コピー確定後に再設計が必要」と明記する

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| コピー一覧 | 必須 | 各セクション/スライドのメインコピーとサブコピー | 「コーディング、誰かに頼んでいいんです。」 |
| CVゴール | 必須 | コンバージョンの定義 | 無料相談の問い合わせ |
| ターゲット | 必須 | 誰に向けたLPか | コーディングが苦手なWebデザイナー |
| LP形式 | 必須 | スクロール型 / スワイプ型 | スワイプ型（8スライド） |
| アスペクト比 | 任意 | スワイプ型の場合 | 3:4 |
| ブランドカラー | 任意 | アクセントカラー等。未指定の場合はコーラルオレンジ系（#ff6b4a）をデフォルト採用 | コーラルオレンジ #ff6b4a |
| ビジュアル制作手段 | 任意 | AI画像生成 / Canva / 撮影 / 素材サイト。未指定の場合はAI画像生成の制約を適用 | AI画像生成（Gemini Imagen） |
| CTA仕様 | 任意 | 常時固定 / セクション末 / 両方。未指定の場合は「FV内＋ページ末尾の2点配置」をデフォルト | 常時固定フッター（高さ80px） |

## 実行ステップ

### ステップ-1: コピー確定チェック（単独起動時のみ実行）

`lp-create` 経由で起動された場合は、Step 0 で既にコピー確定マーカーがチェック済みなので **本ステップはスキップ**。`lp-create` を経由せずシンヤさん（または他エージェント）から直接「LPデザインして」等で起動された場合のみ実行する。

- **入力**: copy.md パス（シンヤさんから受け取る）
- **処理**:
  - copy.md を Read で読み込む
  - ファイル末尾近傍に `<!-- LP-CREATE-GATE: COPY-CONFIRMED YYYY-MM-DD -->` マーカーが存在するか正規表現 `<!-- LP-CREATE-GATE: COPY-CONFIRMED [0-9]{4}-[0-9]{2}-[0-9]{2} -->` で確認
  - **マーカーがない**: 「コピーが未確定です。先に `lp-create` の Route C を実行してコピー確定マーカー（`<!-- LP-CREATE-GATE: COPY-CONFIRMED YYYY-MM-DD -->`）を付与してから、本スキルを再起動してください」とシンヤさんに報告して中止
  - **マーカーがある**: ステップ0（制約の確認）に進む
  - **copy.md パスが不明・指定されていない**: 「`lp-design-system` 単独起動には copy.md パスの指定が必要です。`lp-create` Route C で生成した copy.md のパスをご指定ください」とシンヤさんに確認
- **出力**: コピー確定確認済みフラグ（後続ステップに引き継ぐ）

**起動経路の判定基準**:
- 呼び出し元が `lp-create` の Step 5（アスカからの委任メッセージに「lp-create Step 5」「Route F」「Route D（lp-create 経由）」等の文言が含まれる） → 単独起動ではない（本ステップスキップ）
- 上記以外（シンヤさんからの直接依頼、他のスキル/エージェントからの直接呼び出し） → 単独起動（本ステップ実行）

### ステップ0: 制約の確認

- 入力: ビジュアル制作手段
- 処理:
  - ビジュアル制作手段が「AI画像生成」の場合、以下の制約を全ステップで適用する:
    - 得意: フォトリアル写真、テクスチャ・素材感、照明・雰囲気、抽象アート、空間表現
    - 苦手: 正確な幾何学図形、ベクター線画、UI要素、正確なレイアウト制御、テキスト描画
    - 判断基準: モチーフを思いついたら「これはAIが高品質に生成できるか？」を自問する。幾何学的精度が必要なら不可
  - ビジュアル制作手段が「Canva」「Figma」等のデザインツールの場合、制約なし
  - ビジュアル制作手段が未指定の場合、「AI画像生成」の制約を適用する（安全側に倒す）
- 出力: 適用する制約リスト

### ステップ1: コピーの感情マッピング

- 入力: コピー一覧、CVゴール
- 処理:
  - 各セクション/スライドのコピーを読み、以下を判定する:
    - **感情フェーズ**: 共感 / 問題提起 / 転換 / 信頼 / 安心 / 行動促進 のいずれか
    - **感情温度**: 冷（不安・問題） / 中（転換・発見） / 暖（安心・信頼・行動）
    - **コピーの重さ**: 重い（メインコピーが長い・情報量が多い） / 軽い（短い・一言）
  - 全セクションを通しての感情曲線を描く（冷→暖への遷移、転換点の特定）
  - 転換点スライドを特定する（感情フェーズが「転換」のスライド。ない場合は「問題提起」から「信頼」への切り替わり点）
- 出力: セクションごとの感情フェーズ・感情温度・コピーの重さ + 全体の感情曲線

### ステップ2: コピー配置設計（最重要ステップ）

- 入力: コピー一覧、感情マッピング結果、LP形式
- 処理:
  - **各セクション/スライドについて、以下を全て決定する:**
  - **2-1. メインコピーの位置**
    - 感情温度が「冷」→ 画面下部配置（ビジュアルで引き込んでからコピーへ）
    - 感情温度が「暖」→ 画面上部配置（コピーを先に読ませて安心させる）
    - 転換点スライド → 画面中央配置（最も目立つ位置、特大サイズ）
    - コピーが重い → テキスト専用エリアを広く取る（ビジュアルは控えめに）
    - コピーが軽い → ビジュアル面積を広く取れる
  - **2-2. メインコピーのサイズ感**
    - 通常: 大（24-28px相当）
    - 転換点: 特大（28-32px相当、LP内で最大）
    - CTA誘導: やや抑え（22-24px。叫ぶのではなく手を差し伸べる温度）
  - **2-3. メインコピーの文字色**
    - 背景がダーク → 白 (#ffffff)
    - 背景がライト → チャコール (#1a1a1a〜#2a2a2a)
    - 転換点の文字色はステップ3（アクセントカラー登場計画）で決定する。ここでは仮で白 or チャコールを設定
    - コントラスト比4.5:1以上を必ず確保
  - **2-4. サブコピーの配置**
    - メインコピーの直下、間隔16-24px
    - サイズ: 小（13-14px）
    - 文字色: メインコピーの文字色を70-75%の不透明度で
  - **2-5. テキスト揃え方向**
    - 情報提供セクション（サービス紹介、FV等）→ 左揃え
    - 感情訴求セクション（転換点、CTA誘導等）→ 中央揃え
    - スワイプLPで統一感を出す場合 → 全スライド左揃えまたは全スライド中央揃え
  - **2-6. テキスト可読性の背景処理**
    - ビジュアルが全面の場合 → グラデーションオーバーレイ必須（黒40-60%→透明、またはその逆）
    - ビジュアルとテキストが上下分割の場合 → テキストエリアは白/ダーク背景で分離
    - 転換点でカード処理を使う場合 → 白カード（角丸16px）にテキスト、背景はビジュアル。※カード処理はCanva側で実施。AI画像生成には背景のみ依頼し、カードはCanvaで重ねる
- 出力: セクションごとのコピー配置仕様（位置/サイズ/色/揃え/背景処理）

### ステップ3: アクセントカラー登場計画

- 入力: ブランドカラー、感情マッピング結果、セクション数
- 処理:
  - アクセントカラーの登場タイミングを計画する。原則:
    - **序盤（共感・問題提起フェーズ）: 使わない**。共感の時間に色でアピールしない
    - **転換点: 初登場**。メインコピーの文字色 or 背景の一部として。「変化の合図」として機能させる
    - **中盤〜終盤: 要所のみ**。重要ワードの下線、アイキャッチの縦線、区切り線などで節制して使用
    - **最終CTA: CTAボタンの色と一致させる**。LP全体が「CTAへの伏線」になる設計
  - 各セクションのアクセントカラー使用を「なし / テキスト色 / 装飾要素 / 背景」で決定する
  - 1スライドにつきアクセントカラーは1-2箇所まで（節制が効果を生む）
- 出力: セクションごとのアクセントカラー使用計画

### ステップ4: 視線誘導設計

- 入力: コピー配置仕様、LP形式
- 処理:
  - **各セクション/スライドについて、以下の3点を設計する:**
  - **4-1. 最初に目が行く場所（Entry Point）**
    - 全面ビジュアルの場合 → ビジュアルの輝点・コントラストが最も高い箇所
    - テキスト上部配置の場合 → メインコピーの1行目
    - 中央配置の場合 → メインコピー
  - **4-2. 視線の移動経路（Flow）**
    - Entry Point → メインコピー → サブコピー → ビジュアル → CTA方向
    - テキストが上の場合: 上→下のF字型
    - テキストが下の場合: ビジュアル→テキストの下降誘導
    - テキストが中央の場合: 中央集中→周辺へ放射
  - **4-3. 最終着地点（Terminal Point）**
    - 常時固定CTAがある場合 → 視線の着地が自然にCTA方向へ流れるか確認
    - 着地点がCTAと離れている場合 → 視線の橋（アクセントカラーの線、矢印的要素）を検討
- 出力: セクションごとの「Entry → Flow → Terminal」設計

### ステップ5: ビジュアル役割指定

- 入力: コピー配置仕様、感情マッピング、視線誘導設計、制約リスト（ステップ0）
- 処理:
  - **各セクション/スライドのビジュアルについて、以下を決定する:**
  - **5-1. ビジュアルの役割を1つ選ぶ**（必ず1つだけ。複数兼任させない）
    - 感情補強: コピーの感情を視覚的に増幅する（例: 安心感のコピー → 温かみのあるシーン）
    - 具体化: コピーの抽象的な内容を具体的に見せる（例: 「渡すだけ」→ デスクの受け渡しシーン）
    - 雰囲気づくり: テキストが読みやすい背景を提供する（例: テクスチャ、ぼかし）
    - 視線誘導: Entry Pointとして視線を引き込む（例: 光の方向で視線を導く）
  - **5-2. ビジュアルの面積比率**
    - テキストが重い → ビジュアル30-40%、テキスト領域60-70%
    - テキストが軽い → ビジュアル60-70%、テキスト領域30-40%
    - 全面ビジュアル＋オーバーレイ → ビジュアル100%、テキストは重ねて配置
  - **5-3. ビジュアルモチーフ**
    - ステップ0の制約を確認し、制作手段で実現可能なモチーフを選ぶ
    - AI画像生成の場合: 写真風シーン、テクスチャ、空間、抽象アートから選ぶ
    - 「何を描くか」「大まかな色味」「照明の雰囲気」を自然言語で記述する
    - 正確な構図・配置は指定しない（AIが解釈できる大まかな方向性に留める）
  - **5-4. テキストとの重なり確認**
    - テキスト配置領域にビジュアルの主要被写体が重ならないか確認
    - 重なる場合 → オーバーレイ処理の追加、またはビジュアルの被写体位置を調整
- 出力: セクションごとのビジュアル仕様（役割/面積比率/モチーフ/テキストとの関係）

### ステップ6: 次セクション接続設計（スワイプLP専用）

- 入力: 感情マッピング、コピー一覧、LP形式、アクセントカラー登場計画（ステップ3）
- 処理:
  - LP形式が「スワイプ型」の場合のみ実行する。スクロール型の場合はスキップ
  - **各スライドについて、次スライドへスワイプしたくなる仕掛けを1つ設計する:**
    - オープンループ: 問いかけで終わる → 答えが次にある予感
    - 心理的ツッコミ: 「でも本当に？」「費用は？」という読者の疑問を次で解消
    - 結論先出し: 結論を言って「なぜ？」を次で説明
    - 未完結の感情: 共感だけして解決策はまだ → 続きが気になる
    - 期待の提示: 「次に具体的な方法を見せます」的な暗示
  - 最終スライドのみ「次」はCTAボタンへの誘導。コーラルオレンジ等のアクセントカラーでCTAボタンと視覚的に接続する
- 出力: スライドごとの接続手法

### ステップ7: デザイン仕様書の統合出力

- 入力: ステップ1〜6の全出力
- 処理:
  - 全ステップの結果を統合し、出力フォーマットに従って仕様書を生成する
  - 生成前に以下の最終チェックを実施:
    - [ ] 全セクションのコピー配置（位置/サイズ/色/揃え/背景処理）が指定されているか
    - [ ] アクセントカラーが序盤で使われすぎていないか
    - [ ] 視線誘導の着地がCTA方向を向いているか
    - [ ] ビジュアルの役割が「コピーの補強」になっているか（ビジュアルが主役になっていないか）
    - [ ] AI画像生成制約に違反するモチーフがないか（該当する場合）
    - [ ] スワイプLPの場合、各スライドに接続手法が設計されているか
    - [ ] 「コピーが載った完成形」を想像して、美しく機能するか
- 出力: 統合デザイン仕様書

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ビジュアル案が先に思いつく | コピーを先に配置設計してから、ビジュアルを考える | コピーファースト原則。ビジュアルありきの設計はLPデザイナーの仕事ではない |
| コピーが長くてビジュアル領域が狭い | ビジュアルを控えめにし、テキスト領域を優先 | コピーが主役。読めないテキストは存在しないのと同じ |
| 全スライド同じレイアウトになりそう | テキスト位置（上/中/下）を感情フェーズに応じて変化させる | 単調さは離脱を招く。ただし変化のための変化はしない |
| 転換点のスライドをどう差別化するか | テキストサイズ最大＋アクセントカラー初登場＋背景色の明暗切替 | 3つの変化を同時に起こすことで「何かが変わった」感を確実に伝える |
| AI生成で正確な図形が必要なモチーフを使いたい | 写真風 or 抽象アートに変更する | AI画像生成の制約。幾何学的精度が必要ならCanva/Figmaを推奨 |
| ビジュアルのモチーフが思いつかない | 「雰囲気づくり」役割に設定し、テクスチャ/背景色グラデーションで処理 | 無理にモチーフを入れるよりも、余白とテキストで語らせる方がLPとして強い |

### 記入例（スワイプLPのFVスライド）

```
### スライド 01 — FV

**コピー配置:**
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ビジュアルから先に考え、コピーを後から載せる場所を探す | コピーの位置・サイズ・色を先に決め、残りの空間にビジュアルを配置する | コピーファースト原則。ビジュアルありきだとコピーが読みにくくなる |
| 全スライドでアクセントカラーを均等に使う | 序盤は温存し、転換点で初登場。以降は要所のみ | 色の節制が効果を生む。均等に使うと「特別感」がなくなる |
| ビジュアルの説明に「ダークテーマのLP風画像」と書く | 「夜のデスク、モニターの青白い光だけが部屋を照らす。静物。」と書く | AIはコンセプト（LP風）を理解できない。カメラが見るシーンを描写する |
| 「おしゃれな画像」「クオリティの高いビジュアル」で済ませる | ビジュアルの役割を「感情補強」「具体化」「雰囲気づくり」「視線誘導」から選び、何のためのビジュアルかを明示する | 役割なきビジュアルはデザインではなく装飾。装飾はCVRに貢献しない |
| AI生成で正確な幾何学図形（グリッド、フラットアイコン等）を指定する | 写真風のシーンか抽象アートに変更する | AI画像生成の制約。正確な図形はCanva/Figmaの領域 |
| 視線誘導を考えず「見た目が良い」配置にする | Entry→Flow→Terminalを設計し、TerminalがCTA方向を向いているか確認する | 見た目の良さとCVRは別物。視線がCTAに到達しないLPはコンバージョンしない |
