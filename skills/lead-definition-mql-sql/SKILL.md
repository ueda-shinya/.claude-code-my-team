# lead-definition-mql-sql

BtoBビジネスのリード定義（MQL・SQL）と中間KPI設計、CPA逆算、評価ポイント、リード質×量を上げる施策を一気通貫で出力する。マーケと営業の認識ズレを解消し、リードの質を高めて成約率を向上させる。

## 適用条件

- 使う場面：
  - BtoB SaaS／サブスクのKPIツリー設計
  - マーケと営業のリード認識合わせ（MQL/SQLの基準合意）
  - インサイドセールス（IS）導入時のリード受け渡しルール策定
  - リードスコアリング基準の作成
  - チャネル別リード獲得評価（CPAだけで評価しない）
- 使わない場面：
  - マーケと営業の役割分担全般 → `marketing-sales-workflow`
  - ファネル全体構造 → `funnel-design`
  - ナーチャリング施策 → `lead-nurturing`
  - KGI/KPI/KAI 3階層全体設計 → `kgi-kpi-kai-design`
  - 営業プロセス設計 → `marketing-sales-workflow`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| ビジネスモデル | 必須 | SaaS／サブスク／単発／コンサル | 「BtoB SaaS（SMB向け会計）」 |
| KGI | 必須 | 最終目標指標 | 「新規MRR 月500万円」 |
| 平均成約単価 | 推奨 | 1件あたりの単価 | 「平均30万円」 |
| 各転換率（既知の場合） | 任意 | リード→MQL→SQL→商談→成約 | 「商談化70%、成約率20%」 |
| 営業組織体制 | 任意 | マーケ/IS/FS/CS の分業状況 | 「マーケ→IS→FS の3層体制」 |
| ターゲット属性 | 任意 | 業界／企業規模等 | 「従業員100名以上のSMB」 |

## 実行ステップ

### ステップ1：KGIから重要指標を分解する（KPIツリー）

- 入力：KGI、平均成約単価、ビジネスモデル
- 処理：以下のKPIツリーで分解
  - **MRR（売上）= 月額顧客単価 × 顧客数**
  - **顧客数 = 既存顧客 − 解約顧客 + 新規顧客**
  - **新規顧客 = 商談数 × 成約率**
  - **商談数 = リード数 × 商談化率**
  - **リード数 = 代理店 + 紹介 + 問合せ**
  - **問合せ = アクセス数 × CVR**
- **SaaSの典型KGI**：MRR（Monthly Recurring Revenue）／ARR（Annual Recurring Revenue）／ARPU（Average Revenue Per User）
- 出力：自社のKPIツリー（プロダクト特性に合わせて調整）

### ステップ2：ファネルに沿って中間KPIを決める

- 入力：ステップ1のKPIツリー、営業組織体制
- 処理：以下のファネル×組織×KPIマトリクスを作成
  - **プロセス**：リード創出 → 育成/選別 → 商談 → 成約 → 継続
  - **組織**：マーケティング → IS（インサイドセールス）→ FS（フィールドセールス）→ CS（カスタマーサクセス）
  - **KPI**：リード数 → MQL/SQL → 商談数 → 成約数 → MRR
- **設定例（新規MRR 月500万円・平均成約単価50万円・成約数月10件の場合）**：
  - リード355件/月（4,260件/年）→ MQL 177件（50%）→ SQL 71件（40%）→ 商談 50件（70%）→ 成約 10件（20%）
- **重要**：成約だけでなく**継続まで視野に入れた設計が必要**（CSの役割含む）
- 出力：ファネル×KPI×目標値表

### ステップ3：CPA目標を逆算する

- 入力：ステップ2のKPI、平均成約単価、粗利、回収期間
- 処理：以下の前提条件→CPA目標を算出
  - **前提条件**：成約平均単価／粗利／回収期間（例：成約30万円・粗利5万円・回収1年）
  - **コスト目標逆算**：
    - 成約CPA = 粗利 × 12か月（例：50,000 × 12 = 600,000円）
    - 商談CPA = 成約CPA × 成約率（例：600,000 × 20% = 120,000円）
    - SQL CPA = 商談CPA × 商談化率（例：120,000 × 70% = 84,000円）
    - MQL CPA = SQL CPA × MQL→SQL率（例：84,000 × 40% = 33,600円）
    - ML CPA（マーケリード）= MQL CPA × リード→MQL率（例：33,600 × 50% = 16,800円）
- 出力：階層別CPA目標表

### ステップ4：リードを定義する（ホット/コールド分類）

- 入力：ターゲット属性、ビジネス特性
- 処理：以下の分類で定義
  - **ホットリード**（インサイドセールスが商談打診）：
    - 新規リード：従業員数1,000名以上／500名以上かつ検討時期3か月以内
    - ハウスリスト：従業員数100名以上かつ①問合せ・トライアル申込あり ②ウェビナー参加・WPダウンロード等2回以上の接点あり
  - **コールドリード**（マーケのメルマガで定期コンタクト）：
    - 従業員数100名未満のリードすべて
- 出力：リードランク定義表（ランク／条件／アクション／担当）

### ステップ5：MQL／SQLの判断基準を決める（BANT＋業界別条件）

- 入力：ステップ4のリード定義、プロダクト特性
- 処理：BANT条件と業界別条件を設定
  - **BANT条件（一般）**：
    - **B**udget（予算）：予算の有無・規模
    - **A**uthority（決裁権）：決裁者・決裁プロセス
    - **N**eeds（ニーズ）：具体的な課題・要望
    - **T**ime frame（導入時期）：いつから使いたいか
  - **業界別条件例**：
    - 研修系：新卒採用実績
    - 法人向けバーチャルカード：月間利用額
    - クラウド経理：月間取引件数
    - ストレージサービス：NAS/ファイルサーバの利用実績
  - **MQL判断基準（3つ）**：
    1. **行動ベース指標**：ウェブ訪問／WPダウンロード／ウェビナー参加／メルマガ登録
    2. **リードスコアリング**：行動・属性ベースのスコア合計が一定値以上
    3. **興味の示し方**：製品情報リクエスト／過去購入履歴
  - **SQL判断基準（3つ）**：
    1. **具体的なニーズの明確化**：予算・導入時期・必要機能の具体希望
    2. **商談の進行状況**：決裁者と商談確定／具体提案を受けている
    3. **リードスコアリング高評価**：MQLからの移行時にスコアが高い／製品デモ・価格問合せ
- **アプローチ数 vs 商談化率**（営業リソースで使い分け）：
  - **リソース余裕**：アプローチ条件を低く設定し数を重視
  - **リソース限られている**：条件を高く設定し商談化率を重視
- 出力：MQL/SQLの判断基準＋業界別条件

### ステップ6：MQL/SQL設定のメリットを整理（4項目）

- 処理：以下4メリットを確認
  1. **効果的なリード管理**：マーケと営業の役割分担／購買意欲低はマーケで育成、高は営業対応
  2. **営業効率向上**：SQLのみ対象で商談成立率向上
  3. **部門間連携強化**：共通評価基準でスムーズな引き渡し
  4. **マーケ活動の効果測定**：MQL/SQL創出に寄与する施策が定量分析可能、ROI最大化
- 出力：自社における4メリットの当てはめ

### ステップ7：リード目標の評価ポイント

- 入力：チャネル別実績データ
- 処理：以下2軸で評価
  - **評価ポイント1：チャネル別獲得数とリフトアップで振り返る**
    - チャネル例：オウンドメディア／広告／ウェビナー／カンファレンス／展示会／SNS／記事広告／比較サイト／メール配信
    - 評価項目：リード獲得／Tier獲得／商談獲得／コスト／MROI（マーケROI）
    - **注意**：獲得入口だけで評価するとリスティング・比較サイト等のホットリード獲得チャネルばかり評価される。**態度変容・ハウスリスト接点としての中間コンテンツ貢献度も評価**する
  - **評価ポイント2：CPAだけで評価しない**
    - リード獲得（潜在ニーズ）と商談獲得（顕在ニーズ）の指標を分けて評価
    - **潜在ニーズ層×大量獲得**：オウンドメディア（コラム）／Facebook広告／カンファレンス／共催ウェビナー
    - **顕在ニーズ層×精度高**：リスティング広告／テレアポ（BDR）／自社ウェビナー／一括見積サイト／紹介
    - **複数接点を経て商談に至るケースがほとんど**。MAツール／アトリビューション計測ツールでタッチポイント全体を評価
  - **コンテンツ×タッチポイントの段階別マッピング**：
    - 初回接点（お役立ち資料）：FB広告／ディスプレイ広告／知人口コミ
    - 課題認知（スキルチェックシート）：リスティング／LP／ステップメール
    - 検討（活用事例集）：ステップメール／商談時
    - 決定（フリートライアル／体験コンテンツ）：商談時
- 出力：チャネル別評価表＋潜在/顕在ポジショニング図＋コンテンツマップ

### ステップ8：リードの質を上げ、増やす施策

- 処理：以下8施策（質×4＋量×4）を選定
  - **質を上げる施策**：
    1. 明確な基準の設定（MQL/SQLの文書化）
    2. リードスコアリング導入（行動×属性スコア）
    3. パーソナライズコンテンツ提供
    4. 定期的なフィードバックと改善（営業からマーケへ）
  - **数を増やす施策**：
    1. マルチチャネルマーケティング（広告／SNS／メール／イベント）
    2. コンテンツマーケティング強化
    3. リードジェネレーションキャンペーン（無料トライアル・割引）
    4. SEO対策強化（オーガニック流入）
- 出力：質×量それぞれ4施策の優先順位

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| マーケと営業でリード認識ズレ | MQL/SQL基準を文書化＋共通会議 | 引き渡しトラブルを防ぐ |
| 営業リソースが限られる | アプローチ条件を高く設定（質重視） | 商談化率を最大化 |
| 営業リソースに余裕 | アプローチ条件を低く（量重視） | 機会損失を防ぐ |
| BtoBコンプレックス商材 | BANTを厳格に運用 | 検討期間が長く決裁者複数 |
| BtoB SMB向けサブスク | 簡易BANT＋行動スコア重視 | 検討プロセスがシンプル |
| 新規参入で実績データなし | 業界平均の転換率（リード→MQL 50%・MQL→SQL 40%・SQL→商談 70%・商談→成約 20%）を仮置き | 仮説でも前進が必要 |
| アトリビューションが取れない | チャネル別CPAだけで評価せず、潜在/顕在マップで補完 | 中間貢献を見逃さない |
