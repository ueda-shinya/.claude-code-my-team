# ペルソナ設計スキル

ターゲット市場の中から特定のユーザーを象徴する架空のキャラクターを定義し、マーケ戦略・商品開発・営業戦略の意思決定基準にする。BtoBとBtoCで定義項目が異なるため、必ずモードを判別する。

## 適用条件

- このスキルを使う場面：
  - LP・広告・営業資料のターゲット明確化
  - 新規事業の顧客像確定
  - 商品開発の前段階
  - チーム内でターゲット認識を統一したい
- このスキルを使わない場面：
  - 市場全体のセグメント切り分け → `stp-analysis`
  - 顧客の課題と提供価値のマッチ → `value-proposition`
  - 顧客行動の時系列マップ → カスタマージャーニー関連スキル

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー語 | 「ペルソナ作って」 |
| BtoB / BtoC | 必須 | 商材タイプ | 「BtoB SaaS」「BtoCコスメ」 |
| 商材 | 必須 | 製品・サービス名 | 「クラウド会計SaaS」 |
| 既知の顧客情報 | 任意 | インタビュー結果・販売データ等 | 「アンケート結果あり」 |
| ペルソナ数 | 任意 | 必要な人数 | 「3人分」（複数の購買意思決定者がいる場合） |

## 実行ステップ

### ステップ1：目的を明確化（Purpose）

- 入力：商材、顧客像確定の用途
- 処理：
  1. 「製品やサービスがどんな課題を解決するか」を1文で定義
  2. 「どの顧客層に価値を届けるか」を1文で定義
  3. ペルソナを使う目的（マーケ戦略／商品開発／営業戦略のいずれか・複数可）を確定
- 出力：目的設定文

### ステップ2：データ収集（Data）

- 入力：既知の顧客情報、目的
- 処理：以下3手法のうち利用可能なものでデータ収集する
  1. **顧客インタビュー**：5〜10名対象の半構造化インタビュー
  2. **アンケート調査**：30〜100名対象の選択肢＋自由記述
  3. **ウェブ解析データ活用**：GA4・CRM・購買履歴
- 出力：基本情報（年齢／性別／職業／居住地）／行動（購入パターン／情報収集手段）／ニーズ・課題（不満／期待）

### ステップ3：ペルソナの定義（Definition）— BtoB / BtoC モード分岐

#### モード判定（BtoB / BtoC）

入力時点で確定する。複数モードが対象な場合は必ず両方作成する。

#### BtoBペルソナ定義シート

| 分類 | 項目 | 例 |
|---|---|---|
| **個人情報** | 氏名・性別・年齢 | 佐藤さん・男性・31歳 |
| | 部署・職業・役職 | 情シス・システムエンジニア・主任 |
| **会社情報** | 業種・従業員人数・売上・勤務地 | 建設コンサル・300人・100億円・東京 |
| **業務情報** | 主な業務内容 | 社内システムの開発・保守・ベンダー調達 |
| | 業務上のゴール・KGI・KPI | 社内システム環境の向上・コスト効率改善・全社的業務効率化 |
| | 購買プロセスにおける役割 | 情報収集・選定・決裁者への提言 |
| **情報収集傾向** | 関心事 | DX・AI・業務改善 |
| | よく使うデバイス | 社給パソコン・社給スマホ |
| | よく見ているメディア・サイト | ITmedia・IPA・ASCII・Qiita・日経新聞 |
| | よく使うアプリ | 社内ポータル・Outlook・Excel |

#### BtoCペルソナ定義シート

| 項目 | 例 |
|---|---|
| **ペルソナ名（年齢）** | ユリ（40歳） |
| **写真** | （イメージ写真） |
| **年齢／性別** | 40歳／女性 |
| **労働形態** | 専業主婦 |
| **家族構成** | 夫（50歳）、娘（大学生） |
| **世帯年収** | 700万円 |
| **住所** | 京都 |
| **行動範囲** | 日常買い物は周辺／情報収集は常時／ネットショッピング積極利用 |

**ペルソナの特性／行動／趣味嗜好（4面）**：
- **生活面**：時間配分・経済感覚・情報収集姿勢
- **仕事面**：労働状況・自分の役割認識
- **買い物／外出面**：購買行動・試行傾向
- **思考**：価値観・関心事・社会的承認欲求

#### BtoB vs BtoC 定義項目の違い

| 項目 | BtoB | BtoC |
|---|---|---|
| 性別 | △ | ◯ |
| 年齢 | △ | ◯ |
| 所属企業（業種・規模） | ◯ | △ |
| 役職・職種 | ◯ | △ |
| 年収 | × | ◯ |
| 世帯構成 | × | ◯ |
| 居住地／勤務地 | △ | ◯ |
| 情報収集チャネル | ◯ | ◯ |
| 新規／既存／購入履歴 | △ | ◯ |
| 趣味 | × | ◯ |
| ライフスタイル | × | ◯ |
| パーソナリティ・価値観 | × | ◯ |

凡例: ◯=必ず定義 / △=ケースによる / ×=定義不要

### ステップ4：共感マップの作成（Empathy）

ペルソナの視点から以下4要素を整理する：

| 要素 | 内容 |
|---|---|
| **考えていること** | 顧客が望む価値や課題 |
| **感じていること** | 喜び・不安・期待 |
| **言っていること** | 顧客が製品に関して口にする内容 |
| **していること** | 実際の行動や利用方法 |

### ステップ5：複数ペルソナ対応（必要時）

BtoBは購買プロセスに**複数の意思決定者**がいる場合が多いため、必要に応じて複数ペルソナを作成する：
- 起案者（現場担当者）
- 評価者（部門マネージャー）
- 決裁者（役員・経営層）
- 影響者（情シス・法務）

各ペルソナごとに上記シートを作成し、購買プロセスでの役割を明示する。

### ステップ6：ペルソナの検証と活用（Verification）

- 定期的にペルソナの有効性を見直す（推奨：四半期ごと）
- 実際の顧客データと乖離がないかチェック
- マーケティング戦略・商品開発・営業戦略への反映状況を確認

### ステップ7：完了報告

ステップ1〜6を以下フォーマットで一括出力する。

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| BtoBなのに「年収」「趣味」を埋めようとする | BtoB項目に絞る（業務情報重視） | 業務上の意思決定要因が重要 |
| BtoCなのに「会社情報」を埋めようとする | BtoC項目に絞る（個人ライフスタイル重視） | 個人購買決定の要因が重要 |
| ペルソナが「全国民」レベルで広い | 1ペルソナ＝1名前まで絞る | ターゲット広すぎは効果的訴求不可 |
| ペルソナを5人以上設定したい | 3人以下に絞る／優先順位付け | 全方位施策は失敗 |
| データなしでペルソナを作る | アンケート・インタビューを先行実施 | 想像ペルソナは現実乖離 |
| BtoBで意思決定者が1人と仮定 | 起案者・決裁者・影響者の3人ペルソナ作成 | 単独決裁は中堅以上では稀 |
| BtoCで写真を選ばない | フリー素材から選定して必ず添付 | 視覚化が共感を生む |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ペルソナを「自分たちの理想の顧客像」で作る | 顧客インタビュー・アンケートで実データ収集 | 想像ペルソナは現実乖離 |
| BtoBに趣味・価値観を埋めようとする | BtoBは業務KPI・購買プロセスを重視 | 個人趣味より業務目的が重要 |
| BtoCに会社情報を埋めようとする | BtoCは個人ライフスタイル重視 | 個人購買決定の要因が重要 |
| ペルソナを5人以上作る | 3人以下に絞る | 全方位施策は失敗 |
| 1回作って放置 | 四半期ごとに見直し | 顧客は変化する |
| BtoBで決裁者1人だけのペルソナ | 起案者・決裁者・影響者の3人ペルソナ | 単独決裁は中堅以上では稀 |
| 名前・写真を入れない | 必ず実在しそうな名前と写真を添付 | 共感不足で施策がブレる |

## 連携スキル

- **前段**：`stp-analysis`（市場の切り分け）、`market-competitor-research`（顧客理解）
- **同時併用**：`value-proposition`（価値提案マッチ）、`3c-analysis`（クロス3CのCustomer視点）
- **後段**：カスタマージャーニー設計、施策設計関連スキル、`/blog-post` 執筆時のターゲット明確化

## 5状態出力契約

外部API・サブプロセス・外部設定依存なし。運用上のマーカー：
- `[FAIL]`：必須入力（BtoB/BtoC・商材）が欠落
- `[OK:0件]`：データ不足でペルソナ確定不能（要追加調査）
