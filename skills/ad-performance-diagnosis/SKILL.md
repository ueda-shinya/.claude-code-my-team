# ad-performance-diagnosis

Web広告の運用データを分解式で診断し、課題発見→仮説立案→検証アクションまで一気通貫で導くスキル。

## 適用条件

- このスキルを使う場面：
  - 配信中のWeb広告でCPA上昇／CV減少が発生したとき
  - 配信2週間以上経過後の定期レビュー
  - クリエイティブ更新やターゲティング変更の判断材料が欲しいとき
  - Meta広告／Google広告／ディスプレイ広告等の効果が出ていないとき
- このスキルを使わない場面：
  - 配信開始から1週間未満（学習期間中はデータが不安定）
  - キャンペーン設計そのものから見直す段階 → `meta-ad-campaign-design` または `ad-mix-design`
  - 広告コピー作成 → `ad-copy-7-angles`
  - LPの改善診断 → `ga-gsc-diagnosis` または `lp-create`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 媒体名 | 必須 | Meta／Google／その他 | 「Meta広告」 |
| 配信期間 | 必須 | データ集計期間 | 「2026-04-01〜2026-04-28」 |
| IMP（インプレッション数） | 必須 | 表示回数 | 「150,000」 |
| Click数 | 必須 | クリック数 | 「1,200」 |
| CV数 | 必須 | コンバージョン数 | 「12」 |
| 広告費 | 必須 | 期間総額 | 「30万円」 |
| 目標CPA | 必須 | KPI目標値 | 「12,000円」 |
| ターゲティング設定 | 任意 | 現状の設定 | 「広島県・40-60代経営者・興味関心:ITサービス」 |
| 配信面 | 任意 | プラットフォーム/配置 | 「Facebookフィード自動配置」 |
| クリエイティブ | 任意 | 現状の素材 | 「画像広告3種・カルーセル1種」 |
| 過去同期間データ | 任意 | 比較用 | 「先月CPA 8,000円・CTR 1.2%」 |

## 実行ステップ

### ステップ1：結果を把握する（指標を計算する）

- 入力：IMP / Click / CV / 広告費
- 処理：以下5指標を計算する
  - CTR（クリック率）= Click ÷ IMP × 100（％）
  - CVR（コンバージョン率）= CV ÷ Click × 100（％）
  - CPC（クリック単価）= 広告費 ÷ Click（円）
  - CPA（コンバージョン単価）= 広告費 ÷ CV（円）
  - フリークエンシー（取得可能なら）= IMP ÷ リーチUU
- 検証式：
  - CPA = CPC ÷ CVR の式が成立するか確認（電卓検算）
  - CV = IMP × CTR × CVR の式が成立するか確認
- 過去比較／業界ベンチマークと比較：
  - 過去同期間のデータと差分を出す（CTRが0.5%下がった、CVRが0.2%上がった等）
  - 業界平均ベンチマーク（リスティングCTR 3-4%／Meta CTR 1%目安等）との乖離を確認
- 出力：5指標の現状値＋過去比較／ベンチマーク差分

### ステップ2：課題を発見する（事象ツリーで分類する）

- 入力：ステップ1の指標と目標値
- 処理：以下の事象ツリーで「目標未達の事象」を特定する
  ```
  課題
  ├─ CPAが高い ＜目標CPA超過＞
  │   ├─ CPCが高い（クリック単価が高すぎる）
  │   └─ CVRが低い（クリックは取れてもCVに繋がらない）
  └─ CVが少ない ＜目標CV数未達＞
      ├─ IMPが少ない（そもそも配信量が足りない）
      └─ CTRが低い（表示はされてもクリックされない）
  ```
- 判定ルール：
  - CPA > 目標CPA × 1.2 → 「CPAが高い」事象 ON
  - CV < 目標CV × 0.8 → 「CVが少ない」事象 ON
  - 両方ON → 重大課題、深堀必須
- どの指標がボトルネックかを特定する（複数該当もあり）：
  - CPCが業界ベンチマークより高い → 「CPCが高い」
  - CVRが業界ベンチマークより低い → 「CVRが低い」
  - IMPが日予算上限まで届いていない → 「IMPが少ない」
  - CTRが1%未満（Meta基準）→ 「CTRが低い」
- 出力：特定された課題（CPCが高い／CVRが低い／IMPが少ない／CTRが低いの組み合わせ）

### ステップ3：仮説立案する（4切り口×事象マトリクス）

- 入力：特定された課題
- 処理：以下のマトリクスで原因仮説を立てる
  | 原因＝仮説の切り口 | CPCが高い | CVRが低い | IMPが少ない | CTRが低い |
  |---|---|---|---|---|
  | ターゲティング | 競合の多い属性に絞っている | 課題を持たない層に届いている | オーディエンスが狭すぎ | 興味のない層に届いている |
  | 配信面 | 競合の多い配置（リール等）に偏っている | LP遷移後のミスマッチ（モバイルとPCの差）| 特定配置のみで配信枠不足 | 配信面とクリエイティブの不一致 |
  | クリエイティブ | 関連度スコア低で入札不利 | LPとの訴求ズレ／フォーム長すぎ | 関連度スコア低で配信抑制 | 画像/コピーが目を引かない |
  | 配信期間 | 学習期間中で最適化未完成 | 季節要因／曜日要因 | 短期間で最適化不足 | クリエイティブ疲れ／フリークエンシー高 |
- 各セルで仮説を立てる際のルール：
  - **データ裏付けのある仮説のみ採用**（推測のみは✗）
  - 例：「ターゲティングが狭い」と言うなら推定オーディエンスサイズを確認
  - 例：「クリエイティブ疲れ」と言うならフリークエンシー数値を確認
- 1つの事象に対して仮説は2-3個に絞る（多すぎると検証コストが増大）
- 出力：仮説リスト（事象 × 切り口で✓マークされたセル＋根拠データ）

### ステップ4：仮説検証アクションを設計する

- 入力：仮説リスト
- 処理：各仮説に対して具体的な検証アクションを設計する
  | 仮説 | 検証アクション | 期間 | 成功基準 |
  |---|---|---|---|
  | ターゲティングが狭い | オーディエンスを類似1%→3%に拡大 | 1週間 | IMPが2倍以上、CPC維持 |
  | クリエイティブ疲れ | 画像3パターンを新規追加し並行配信 | 2週間 | 新クリエイティブのCTRが1.5倍 |
  | LPとの訴求ズレ | LPファーストビューを広告コピーと統一 | 1週間 | CVRが1.5倍 |
  | フォーム長すぎ | フォーム項目を10→5に削減 | 1週間 | CVRが1.3倍 |
- 優先順位ルール：
  - **影響度大×実装容易**を最優先（クリエイティブ追加など）
  - **影響度大×実装難**を次（LP改修・フォーム改修）
  - 一度に複数仮説を同時検証しない（A/Bテストで1要素ずつ）
- 検証期間：最低1週間、できれば2週間（統計的有意性のため）
- 出力：仮説×検証アクション×期間×成功基準の表

### ステップ5：診断レポートとしてまとめる

- 入力：ステップ1〜4の全成果物
- 処理：出力フォーマットに沿って整形する
- 出力：広告診断レポート

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 配信1週間未満 | 診断は時期尚早。学習期間終了まで待つ | データが不安定で誤った仮説を立てるリスク |
| CV数が10件未満 | 統計的有意性なし。マイクロCVで代替評価 | 数値変動が大きすぎて判断できない |
| CPCが業界ベンチマークの2倍超 | ターゲティング絞りすぎ／競合過多が主因 | 入札競争が激しい属性に偏っている |
| CTR < 1%（Meta） | クリエイティブ要因を最優先で疑う | 配信されてもクリックされない＝素材問題 |
| CTR > 3%だがCVR < 0.5% | LP/フォーム要因を疑う | クリック後の離脱が多い |
| フリークエンシー > 5 | クリエイティブ疲れ確定。新素材追加 | 同じ広告の繰り返し表示で関心低下 |
| 関連度スコア低（Meta） | クリエイティブとターゲットの整合性問題 | 配信機会と入札効率の両方が悪化 |
| IMPが日予算上限到達せず | ターゲティング狭すぎ／入札額低すぎ | 配信機会が物理的に足りない |
| 月内のCV数が安定して取れている | CPA改善より「クリエイティブ拡張」が効果的 | 既存パフォーマンスを崩さない |
| 過去ピーク時のCPAから2倍以上悪化 | 媒体側のアルゴリズム変更／競合参入を疑う | 自社要因では説明できない劣化 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| データなしで仮説を量産する | 各仮説にデータ裏付けを添える | 推測のみだと検証で空振りになる |
| 一度に5個以上の仮説を同時検証 | 影響度大×実装容易の上位2-3個に絞る | A/Bテストで因果特定不能 |
| クリエイティブ・ターゲティングを同時に変える | 1要素ずつ変える | どちらが効いたか判別できない |
| 1日のデータで判断する | 最低1週間、できれば2週間で判断 | 曜日変動・統計的有意性なし |
| CV数 < 10件で精緻な分析を試みる | マイクロCVで代替評価 | 統計的に有意な判断ができない |
| 業界ベンチマーク無視で判断 | 業界平均と比較してから良し悪しを言う | 自社の絶対値だけでは良し悪しが見えない |
| 「広告がダメ」と一括結論 | 5指標で分解して特定する | どこを直すべきかわからない |
| LPの問題なのに広告側だけいじる | CVRボトルネック時はLP診断（`ga-gsc-diagnosis`）に振る | 広告改善しても無駄打ちになる |
| 学習期間中（1週目）に判断する | 学習完了（2週目以降）まで待つ | データが不安定 |
