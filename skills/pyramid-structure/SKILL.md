# ピラミッドストラクチャー（ロジックツリー）スキル

情報を「結論→理由→詳細」の3層構造で整理し、明確かつ論理的に伝える方法論。プレゼン・レポート・問題解決等で活用。

## 適用条件

- プレゼン・提案の論理構造が散らかっている
- 結論と根拠がうまく繋がっていない
- 問題の原因を構造的に特定したい
- KPI/KGIの分解構造を作りたい

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| 結論／結果／目的 | 必須 | トップに置きたいメッセージ |
| ツリー種類 | 任意 | ゴール／因果／因数分解 |
| 考え方 | 任意 | 演繹法／帰納法 |

## 実行ステップ

### ステップ1: ツリーの種類を選ぶ

| 種類 | 用途 | トップ要素 |
|---|---|---|
| **ゴールツリー** | 目的 → 手段の分解 | 目的（達成したいこと） |
| **因果ツリー** | 結果 → 原因の分解（問題分析） | 結果（起きている事象） |
| **因数分解ツリー** | 全体 → 構成要素 | 全体概念（例：経営＝人×モノ×金） |

### ステップ2: 考え方アプローチを選ぶ

| アプローチ | 方向 | 説明 | 例 |
|---|---|---|---|
| **演繹法**（トップダウン） | 上→下 | 結論を根拠づける事実を調べる | 「東大出身は頭がいい」← 偏差値67.2 / 国庫運営費の半数 / 政治家ランキング1位 |
| **帰納法**（ボトムアップ） | 下→上 | 複数事実から共通項を見出す | 弁護士A・医者B・コンサルC が東大出身 → 「東大出身は頭がいい」 |

### ステップ3: 構成要素（因果ツリー例）の3層

```
結論（トップ）：最重要メッセージ・主張
  例「新しいマーケティング戦略を採用すべき」
   │
   ├─ 理由（ミドル）：2〜4つの主な根拠
   │   例 1. ターゲット層の行動変化
   │      2. 競合他社が同様戦略で成功
   │      3. コスト効率改善の見込み
   │
   └─ 詳細（ボトム）：データ・事例で補強
      例 ターゲット層のSNS利用率／競合事例データ
```

### ステップ4: 5ステップ活用法

1. **全体の目的を明確化** — 誰に・何を・なぜ伝えたいか
2. **結論を最初に考える** — 最重要メッセージを1文に
3. **理由をブレストで洗い出す** — 結論を支える理由を複数挙げ重要なものを選ぶ
4. **詳細を追加** — データ・事例で補強
5. **論理的な流れを確認** — 結論・理由・詳細が一貫しているか

### ステップ5: メリット・注意点チェック

| メリット | 注意点 |
|---|---|
| 明確さ：聞き手が理解しやすい | 結論の明確化：曖昧だと論理崩壊 |
| 説得力：根拠で支える納得感 | 理由の一貫性：結論と矛盾していないか |
| 効率性：結論先で要点把握 | 詳細の適切な量：過不足なく |
| 柔軟性：プレゼン・レポート・日常で活用 | 対象者に応じた調整：知識レベル合わせる |

## 判断基準

| 状況 | 判断 |
|---|---|
| 結論が複数候補で迷う | まず1つ仮置きしてピラミッド作成→修正 |
| 理由が5つ以上 | 重要な2〜4つに絞る（MECE意識） |
| 詳細データが弱い | リク（fact-checker）に検証依頼 |
| 演繹法と帰納法どちらか不明 | プレゼンなら演繹法（結論先出）、分析なら帰納法 |
| 因果ツリーで「東大出身は頭がいい」のような曖昧主張 | 結論を測定可能な形に書き直す |

## 出力フォーマット

```markdown
# ピラミッドストラクチャー（〇〇ツリー・〇〇法）

**トップ**: 〇〇

## 構造
- 結論: 〇〇
  - 理由①: 〇〇
    - 詳細1-1: 〇〇
    - 詳細1-2: 〇〇
  - 理由②: 〇〇
    - 詳細2-1: 〇〇
  - 理由③: 〇〇
    - 詳細3-1: 〇〇

## 一貫性チェック
- [x] 結論明確
- [x] 理由が結論を支える
- [x] 詳細が理由を裏付ける
- [x] MECE（漏れなく重複なく）
```

## 連携スキル
- 提案資料 → `proposal-builder` `sales-deck-template`
- 問題分析 → `as-is-to-be-gap-solution`
- KPI分解 → `kgi-kpi-kai-design`
- 戦略フレームワーク全般

## 5状態出力契約
- `[FAIL]`: 結論／結果／目的が未指定
- `[OK:0件]`: 詳細データが0件で構造化不能
