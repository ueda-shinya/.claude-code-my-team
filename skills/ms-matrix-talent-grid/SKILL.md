# ms-matrix-talent-grid スキル

MSマトリクス（Mind × Skill Matrix） = マインド（意欲・姿勢）とスキル（能力・技術）の2軸で人材を4象限に分類し、最適な育成方法や配置戦略を決めるためのフレームワーク。事業戦略スクール「知足」のフレームワーク準拠。出典：吉田行宏『全員経営者マインドセット――MSマトリクスで実現する次世代組織』。

## 適用条件

- このスキルを使う場面：
  - チーム/部署の人材ポートフォリオを可視化したい
  - 人材ごとの育成方針を象限別に決めたい
  - 採用戦略（新卒／中途）でターゲット象限を明確化したい
  - 「育成すべき人」「リーダー候補」「配置転換検討」「専門職パス」を整理したい
- このスキルを使わない場面：
  - 組織全体の組織図設計 → `organization-planning`
  - 評価制度の策定 → `evaluation-system-design`
  - 採用ガイドラインの策定 → `recruitment-strategy`
  - キャリアパス設計 → `career-roadmap-development`
  - 給与レンジ決定 → `salary-range-design`
  - 関係性の改善（成功循環モデル） → `success-cycle-relationship-quality`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象組織情報 | 必須 | 業種・規模・対象部署 | 「Web制作会社・全社20名・営業部7名対象」 |
| 評価対象者リスト | 必須 | 名前または匿名ID（A〜Z等） | 「A〜Gの7名」 |
| 各人材の評価素材 | 任意 | 既存評価・1on1記録・成果実績等 | 「直近半期の評価面談メモ」 |
| 利用目的 | 必須 | 配置／採用／育成のどれを優先するか | 「育成枠の特定」「リーダー候補抽出」 |
| 自社の現状課題 | 任意 | 「リーダー不足」「即戦力欠如」「離職率高」等 | 「中堅リーダー不足」 |

## 実行ステップ

### ステップ1：2軸の定義を自社用に具体化する

- 入力：対象組織情報、自社の現状課題
- 処理：
  1. 標準定義をベースに、自社が重視する観点を追加する
     - **マインド（Mind）標準定義**：仕事への意欲、向上心、主体性、チーム貢献意識
     - **スキル（Skill）標準定義**：業務遂行能力、専門知識、実務経験、成果を出す力
  2. 自社特有の観点を追加（例：「クライアントへの当事者意識」「自走力」「稼働率」等）
  3. 各軸を「高／中／低」または「5段階」のスコアリング基準に落とし込む
- 出力：自社版の2軸定義書（マインド／スキルそれぞれの評価観点リストとスコア基準）

### ステップ2：対象者を4象限にプロット

- 入力：評価対象者リスト、評価素材、自社版2軸定義
- 処理：
  1. 各人材についてマインド軸・スキル軸を評価（高／低の2値、または点数）
  2. 4象限マップに配置（軸の中央が境界）
  3. 各人材の「現在地」を以下4象限に振り分け：

| 象限 | タイプ | マインド | スキル | 特徴 |
|---|---|---|---|---|
| ① | **伸びしろ大（成長枠）** | 高 | 低 | 強い向上心・学習意欲が高い／現状はスキル不足だが成長ポテンシャル高／適切な環境と指導があれば大きく成長 |
| ② | **理想型（リーダー・エース候補）** | 高 | 高 | 高い能力と意欲を持ち組織の成長をけん引／自発的に動けるリーダー資質／周囲に好影響、チームワークも強い |
| ③ | **要注意（意識改革が必要）** | 低 | 低 | 仕事への意欲が低くスキルも未熟／受け身でモチベーションが不安定／組織全体に悪影響を及ぼすリスク |
| ④ | **専門家タイプ（個別対応が必要）** | 低 | 高 | 高度な専門スキルを持つが意欲が低い／自分の領域以外に興味を持たない／コミュニケーションが苦手な場合もある |

- 出力：人材プロット表（名前 / マインドスコア / スキルスコア / 象限判定）と4象限マトリクス図

### ステップ3：象限別に育成・対応策を決める

- 入力：人材プロット表
- 処理：象限別の標準対応策をベースに、対象者ごとの個別具体策を作成

| 象限 | 育成・対応策の方向性 |
|---|---|
| **① 伸びしろ大** | ・OJT（実務経験）を通じてスキルを伸ばす<br>・研修・教育プログラムを充実させる<br>・成果を出す成功体験を積ませる |
| **② 理想型** | ・マネジメントスキルを磨かせる<br>・責任のあるポジションを与える<br>・権限を委譲し、意思決定を任せる |
| **③ 要注意** | ・本人のモチベーションの源泉を探る<br>・明確な目標を設定し、小さな成功体験を与える<br>・変化が見られない場合、配置転換 or 退職勧奨を検討 |
| **④ 専門家タイプ** | ・専門職としてのキャリアパスを用意する<br>・裁量権を与え、個人のスタイルを尊重する<br>・マインド向上のための報酬・評価制度を整える |

- 出力：対象者×個別対応策テーブル（誰に何を、いつまでに）

### ステップ4：3つの活用領域別に戦略を導出する

- 入力：人材プロット表、自社の現状課題
- 処理：以下3領域それぞれで戦略を導出

#### 4-1. 人材配置・役割設計（**個別の人に対する役割判断**）

※ 4-1は「対象者個人ごとに、どの役割を割り当てるか」の個別判断。4-3は「象限ごとの組織制度・規程の設計」を扱う（書き分け）。

| 象限 | 適切な役割 |
|---|---|
| ①伸びしろ大 | 育成対象として配置（OJT・研修で育てる） |
| ②理想型 | リーダー・マネージャーに登用（権限委譲） |
| ③要注意 | 個別の段階的処理：①モチベーション源泉ヒアリング → ②改善期間設定（3〜6ヶ月） → ③改善が見られない場合は配置転換 → ④最終手段として退職勧奨検討 |
| ④専門家 | エキスパート職に配置（裁量権を与え、個人のスタイルを尊重） |

#### 4-2. 採用・育成戦略

| 採用区分 | どの象限をターゲットにするか |
|---|---|
| **新卒採用** | ①伸びしろ大を重視（ポテンシャル採用） |
| **中途採用** | ②理想型・④専門家を狙う（即戦力採用） |
| **育成方針** | ①を②に育てる／③の改善が難しければ見切る |

#### 4-3. 組織文化・マネジメント施策（**象限ごとの組織制度・規程の設計**）

※ 4-3は「象限カテゴリに対して組織として用意する制度・仕組み」の設計。4-1の個別判断（人ごとの役割付け）を運用するための土台となる規程整備を扱う（書き分け）。

| 対象象限 | 組織制度・規程の設計 |
|---|---|
| ①伸びしろ大 | マインド・スキル両軸の評価制度を設計／成長機会（OJT枠・研修予算）の制度化 |
| ②理想型 | リーダー育成・幹部候補プログラムの整備／権限委譲ルールの明文化 |
| ③要注意 | 改善期間規程・面談プロセス手順書・配置転換／退職勧奨フローの**仕組み化**（個別ケースの実行は4-1で実施） |
| ④専門家 | 専門職等級制度の制定／マインド向上に依存しない処遇テーブルの設計 |

- 出力：3領域別の戦略提案書

### ステップ5：実行プランと再評価サイクルを設計

- 入力：ステップ3〜4の戦略
- 処理：
  1. **短期施策（3ヶ月以内）**：①の育成プログラム開始、②へのリーダー任命、③の1on1強化、④のキャリア面談
  2. **中期施策（半年〜1年）**：①の②昇格判定、③の処遇判断、④の専門職等級設計
  3. **再評価頻度**：四半期に1度、4象限マップを更新（メンバーの移動を可視化）
  4. **記録方法**：四半期ごとにスナップショット保存、移動軌跡を分析
- 出力：実行プラン（短期／中期／再評価サイクル）

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 評価対象者数が30人を超える | 部署単位で分割実施 | 1マップに30人以上は可読性が落ちる |
| 評価素材が不足 | ステップ1の前に1on1またはアンケートを実施 | 主観評価のブレを防ぐ |
| ③要注意が組織の30%以上を占める | 個別対応より組織文化／採用基準を見直す | 個人問題ではなく組織課題 |
| ②理想型が0人 | 即時にリーダー候補育成プログラム発動／中途採用検討 | 組織継続性のリスク |
| ①伸びしろ大が80%以上 | 中堅・即戦力枠（②④）の中途採用を検討 | 育成負荷が高すぎる |
| ④専門家のマインドが急低下 | 関係の質低下の可能性 → `success-cycle-relationship-quality` 併用 | 個人問題ではなく組織問題の可能性 |
| 評価が分かれる人材（マインドかスキルどちらか中間） | スキル偏重ではなくマインド優先で象限判定 | マインドは育てにくく、スキルは育てられる |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 評価素材なしで一発判定 | 1on1や直近評価を必ず参照する | 主観バイアスで誤判定する |
| ③要注意の人を即「退職勧奨」と決める | まず本人のモチベーションの源泉を探る／配置転換／改善期間設定 | 組織側の問題（配置ミス・関係の質）の可能性がある |
| ④専門家を「②に育てよう」とマインド改造を試みる | ④は専門職パスを用意して活かす | マインドは強制で変えられない／本人特性を尊重する |
| 全員を②理想型扱いして甘い育成計画 | 4象限を厳密に分け、対応を変える | フレームワークの意味がなくなる |
| 1回プロットして終わる | 四半期ごとに再評価し移動を見る | 静的分析では成長/劣化が見えない |
| マインド偏重で「やる気はあるけどスキルなし」を②扱い | ①伸びしろ大に正しく分類し育成する | 過大評価で本人の成長機会を奪う |
| ②理想型が0人なのに「現メンバーで頑張る」と決める | 即時に外部採用 or ①の育成加速で対処する | リーダー不在は組織継続リスク |
| ③が組織の3割超なのに個別対応で済ませる | 採用基準・組織文化・関係の質を見直す | 構造問題を個人問題にすり替えない |

## 連携スキル

- `organization-planning`：組織図設計・人員配置（本スキルの出力を組織図に反映）
- `evaluation-system-design`：評価制度策定（マインド評価とスキル評価をどう制度化するか）
- `recruitment-strategy`：採用戦略（中途で②④を狙う場合の具体ガイドライン）
- `career-roadmap-development`：キャリアパス設計（④専門家用のキャリア定義）
- `salary-range-design`：給与レンジ（②と④を別等級で扱う場合）
- `success-cycle-relationship-quality`：関係の質改善（③が多い場合・マインド低下時に併用）
- `teaching-coaching-leading`：①育成時のコーチング設計
- `katz-three-skill-approach`：3スキル階層と組み合わせて②のリーダー育成を深掘り

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。本契約は適用しないが、運用上の**任意**ベストプラクティスとして以下のマーカー使用を**推奨**する：

- `[OK:0件]`：対象者0名／全象限が空のとき
- `[PARTIAL:N成功/M失敗]`：複数部署を一括分析して一部の評価素材が揃わなかったとき
- `[FAIL]`：評価素材が全く取得できず判定不能のとき。エラー要約を必ず添える
- `[SKIP]`：評価素材未取得で評価対象外と判定した人材がいる場合（試みなかったケースのみ）
