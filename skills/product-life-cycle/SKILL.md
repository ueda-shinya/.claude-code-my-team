# プロダクトライフサイクル分析スキル

製品・サービスを4段階（導入期・成長期・成熟期・衰退期）のいずれかに位置付け、フェーズに応じた戦略・マーケティング重点・投資判断を導出するフレームワーク。複数製品を並行管理する場合のポートフォリオ判断にも活用可能。

## 適用条件

- このスキルを使う場面：
  - 自社製品が今どのフェーズか判定したい
  - フェーズに応じた戦略の方向性を決めたい
  - 複数製品をライフサイクル管理したい
  - 撤退判断（衰退期到達）の検討
- このスキルを使わない場面：
  - 顧客層別の浸透戦略 → `innovator-theory`（5層別プロモ設計）
  - PMF段階の診断 → `pmf-journey`
  - 市場規模そのものの予測 → 市場規模予測スキル

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー語 | 「PLC分析して」 |
| 分析対象 | 必須 | 製品・サービス名 | 「クラウド会計SaaS A」 |
| 売上推移 | 任意 | 直近1〜3年の推移 | 「前年比+50%」「横ばい」 |
| 競合状況 | 任意 | 競合数の増減 | 「競合5社→10社に増えた」 |
| 利益率 | 任意 | 直近の利益率 | 「30%→15%に低下」 |

## 実行ステップ

### ステップ1：4段階の特徴を提示する

導入期・成長期・成熟期・衰退期の4段階それぞれの「特徴4項目＋戦略3〜4項目」を出力する。判定に使うシグナルを明示する。

| フェーズ | 特徴 | シグナル |
|---|---|---|
| **導入期** | 市場認知低／売上少／競合少／投資コスト高／消費者ハードル高 | 売上水準が低い・競合がほぼいない・PRに重投資 |
| **成長期** | 売上急増／競合増／差別化重要／コスト低下／利益率向上 | 売上急成長・競合参入が増加・差別化が必須に |
| **成熟期** | 売上ピーク／競争激化／価格競争／差別化困難／コスト削減要請 | 売上横ばい・競合多数・価格競争・利益率低下 |
| **衰退期** | 需要減少／代替品登場／価格競争激化・利益率低下／在庫処分検討 | 売上減少・代替品の登場・販売終了検討 |

### ステップ2：自社製品の現フェーズを判定する

- 入力：分析対象、売上推移、競合状況、利益率
- 処理：
  1. 売上推移・競合数・利益率の3指標から現フェーズを判定する
     - 売上：低い／急増／横ばい／減少 → 導入／成長／成熟／衰退
     - 競合：少／増加中／多数／減少 → 導入／成長／成熟／衰退
     - 利益率：低い→上昇／高水準→低下／更に低下 → 導入→成長／成熟→衰退
  2. 3指標が一致する場合 → そのフェーズで確定
  3. 不一致の場合 → 売上推移を最優先指標として暫定判定し、他2指標との不一致点を備考に明示
  4. 判定根拠を1〜2文で言語化する
- 出力：現フェーズ判定 + 根拠 + 不一致点（あれば）

### ステップ3：フェーズ別マーケティング戦略テーブル

該当フェーズの行を強調して、以下のテーブルを出力する。

| 項目 | 導入期 | 成長期 | 成熟期 | 衰退期 |
|---|---|---|---|---|
| **顧客** | イノベーター／アーリーアダプター | アーリーアダプター／アーリーマジョリティ | レイトマジョリティ／ラガード | ラガード |
| **マーケ目標** | 市場拡大 | 市場浸透 | シェア維持 | 生産性確保 |
| **マーケ重点** | 製品認知 | ブランド | ブランドロイヤリティ | 選択的 |
| **売上** | 低い | 高い | 横ばい | 低い |
| **投資コスト** | 高い | 高い | 低い | 低い |
| **競合状況** | 少ない | 増加 | 多い | 減少 |
| **利益** | 少ない | 増加 | 多い | 減少 |

### ステップ4：フェーズ別の戦略アクション提案

判定したフェーズに応じて、3〜4項目の戦略アクションを提示する。

#### 導入期戦略（市場認知向上が最優先）
1. プロモーション強化（広告／PR／インフルエンサー活用）
2. 製品の差別化ポイントを明確にする
3. 早期アダプター（イノベーター層）へのアピール
- 関連スキル: `innovator-theory`（イノベーター層へのアプローチ設計）

#### 成長期戦略（市場シェア拡大／競争優位性確立）
1. マーケットリーダーとしてブランドを確立
2. 販売チャネル拡大（オンライン・オフライン両方）
3. 価格調整（プレミアム or 低価格戦略）
4. 機能拡張（アップグレード版投入）
- 関連スキル: `innovator-theory`（キャズム超え戦略）、`stp-analysis`

#### 成熟期戦略（ブランド維持・利益最大化）
1. 価格競争を避けるブランド力強化
2. 新機能・バリエーション追加
3. 新ターゲット層開拓（高級ライン or 低価格ライン）
4. リピート率向上のマーケティング強化
- 関連スキル: `value-proposition`（再差別化）、`vrio-analysis`

#### 衰退期戦略（製品整理／リソース再配分）
1. コスト削減・生産縮小で利益確保
2. 販路見直し・在庫処分セール
3. ブランド価値を守るための計画的撤退／後継商品開発
4. ニッチ市場特化（細く長く売る戦略）
- 関連スキル: `medium-term-business-plan`（事業ポートフォリオ再編）

### ステップ5：キャズム警告（導入期→成長期移行フェーズの場合）

導入期から成長期への移行を判定した場合、**キャズム理論**を併用する：
- 16%（イノベーター2.5% + アーリーアダプター13.5%）の壁を越える必要がある
- アーリーマジョリティ（34%）への訴求方法はアーリーアダプターと異なる
- アーリーアダプター向けの訴求のままだとキャズムで失速する
- 詳細戦略は `innovator-theory` スキルへ誘導する

### ステップ6：複数製品ポートフォリオ管理（複数製品時）

複数製品を保有している場合、ポートフォリオバランスを評価する：
- 全製品が同一フェーズ → リスク高（一斉に衰退する可能性）
- 成長期×成熟期×導入期がバランス → 健全
- 成熟期・衰退期に偏る → 新規製品開発を推奨（`medium-term-business-plan` 連携）

### ステップ7：完了報告

ステップ1〜6を統合して以下フォーマットで出力する。

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 売上推移・競合数・利益率の3指標が不一致 | 売上推移を最優先指標とし、不一致点を備考に明示 | 売上は最終アウトプット指標 |
| 売上が急増だが利益率が下がっている | 成長期確定（コスト低下が利益率向上に追いつかない時期） | 成長期初期によくあるパターン |
| 売上横ばい・競合減少 | 成熟期後期 or 衰退期初期 | 競合が減ったのは撤退組が出始めたサイン |
| 「我が社の製品は永遠に成長期」と主張 | 客観データを再収集する／ `competitive-absence-audit` 連携 | 楽観バイアス排除 |
| 衰退期と判定したが「まだ需要がある」 | ニッチ市場特化（細く長く売る）戦略を提案 | 撤退一択ではない |
| 単一製品しかない場合 | ポートフォリオ管理（ステップ6）省略 | 該当なし |

```markdown
# プロダクトライフサイクル分析

**分析対象**: 〇〇（製品名）
**分析日**: YYYY-MM-DD

## 現フェーズ判定: 【成長期】

### 判定根拠
- 売上：前年比+50%（→ 急成長）
- 競合：5社→10社（→ 増加中）
- 利益率：15%→25%（→ 向上）
- 不一致点：なし／あり（具体的に記述）

## フェーズ別マーケティング戦略テーブル
（4段階 × 7項目の表を出力。該当フェーズを強調）

## 該当フェーズの戦略アクション
1. マーケットリーダーとしてブランドを確立
2. 販売チャネル拡大
3. 価格調整（プレミアム戦略採用）
4. 機能拡張（v2.0リリース）

## キャズム警告（該当時のみ）
（適用される場合のみ出力）

## ポートフォリオ管理（複数製品時のみ）
| 製品 | フェーズ | 推奨アクション |
|---|---|---|
| 製品A | 成長期 | シェア拡大 |
| 製品B | 成熟期 | ブランド維持 |
| 製品C | 導入期 | 認知拡大 |
**バランス評価**: 健全 / 偏りあり（詳細）

## 連携スキル誘導
- イノベーター理論で詳細な層別プロモ設計 → `innovator-theory`
- フィットジャーニーで PMF 段階診断 → `pmf-journey`
- 中長期戦略への組込 → `medium-term-business-plan`
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 「我が社は永遠に成長期」と決めつける | 客観データ（売上・競合・利益率）で判定する | 主観判定は楽観バイアス |
| 衰退期＝撤退一択と思い込む | ニッチ市場特化（細く長く売る）の選択肢も提示 | 撤退以外の道もある |
| 導入期→成長期の移行でキャズムを無視 | キャズム理論を必ず併用 | 失速の主原因 |
| 単一指標（売上のみ）でフェーズ判定 | 売上＋競合＋利益率の3指標で判定 | 単一指標だと誤判定リスク |
| 複数製品を一括フェーズ判定 | 製品ごとに個別判定 | 製品ごとにフェーズが異なる |

## 連携スキル

- **同時併用**：`innovator-theory`（5層別プロモ／キャズム超え）、`pmf-journey`（PMF段階）、`stp-analysis`（フェーズ別ターゲット）
- **前段**：`pest-analysis` `five-forces-analysis`（市場環境）
- **後段**：`medium-term-business-plan`（フェーズに応じた中長期戦略）、`value-proposition`（再差別化）

## 5状態出力契約

外部API・サブプロセス・外部設定依存なし。運用上のマーカー：
- `[FAIL]`：必須入力（分析対象）が欠落 → 処理中断
- `[OK:0件]`：判定不能（売上推移・競合状況・利益率の全てが不明）
