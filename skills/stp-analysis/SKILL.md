# stp-analysis

STP分析（S/T/P）でマーケティング戦略の3層（市場切り口→狙う市場→立ち位置）を設計する。「セグメントだけ作って終わる」を防ぐため、必ずポジショニングマップ＋差別化メッセージまで進める。

## 適用条件

- 使う場面：
  - 新商品・新サービスの市場戦略を立てる
  - 既存商品でターゲットがブレている／曖昧な状態を整理する
  - 競合との差別化ポイントを見える化したい
  - 広告・LP・営業資料の訴求軸を決める前段階
- 使わない場面：
  - 顧客・競合・自社の3視点 → 3C分析（`3c-analysis`）
  - 内部リソースの評価 → VRIO分析（`vrio-analysis`）
  - 業界の競争構造 → 5フォース分析（`five-forces-analysis`）
  - 価値提案そのものの言語化 → バリュープロポジション（`value-proposition`）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 分析対象 | 必須 | 商品・サービス | 「自社の和菓子ブランド」「新規SaaS」 |
| 分析の目的 | 必須 | 新規開発／既存改善／施策設計 | 「新規LPの訴求軸を決めたい」 |
| 既知の競合 | 任意 | 競合一覧 | 「競合A・B・C」 |
| 顧客タイプ | 任意 | B2C／B2B | 「B2C小売／一般消費者」 |

## 実行ステップ

### ステップ1：S（Segmentation）市場を細分化する

- 入力：分析対象、顧客タイプ
- 処理：以下4つのセグメント軸を**最低3軸以上**使って市場を切り分ける
  - **地理的**: 国／地域／都市／首都圏／郊外／地方
  - **人口動態**: 年齢／性別／職業／家族構成／所得／企業規模・業種（B2B時）
  - **心理的**: 価値観／ライフスタイル／趣味／月額予算／インドア・アウトドア
  - **行動的**: 自社商品の購買経験（未認知／認知未購入／1回／リピート／休眠）／購買頻度／意思決定者
- マトリクス形式で「縦＝セグメント軸／横＝選択肢」を作る
- 各セグメントに「規模感」「成長性」「アクセスしやすさ」のメモを添える
- 出力：セグメント軸×選択肢のマトリクス（最低3軸×3選択肢）

### ステップ2：T（Targeting）狙う市場を2層で決める

- 入力：ステップ1のセグメントマトリクス
- 処理：以下2層でターゲットを設計する
  - **戦略ターゲット**: 範囲から外れていないマーケットを広く捉えたターゲット（例：20〜40代女性／首都圏在住）
  - **コアターゲット**: 中心点にいる最も狙うターゲット（例：30代女性／首都圏／DINKS／月3万円までの趣味費用／ギフト用購入）
- 戦略ターゲットの中に複数のセグメントが入り、コアターゲットはその中の1〜2つに絞る
- ターゲット選定の評価軸：
  - 市場規模（顧客数 × 客単価）
  - 成長性
  - 競合状況（既存プレイヤーの強さ）
  - 自社のリソース適合度
  - リピート性
- 出力：戦略ターゲット（範囲）＋コアターゲット（中心点）の2層定義

### ステップ3：P（Positioning）自社の立ち位置を決める

- 入力：ステップ2のターゲット
- 処理：以下を実施する
  - **2軸を選定**: ターゲットが意思決定で重視する2軸（例：価格軸×用途軸／品質軸×スピード軸）
  - **4象限マップを作成**: 縦軸×横軸で4象限を切り、各象限にセグメント名を配置
  - **競合をマッピング**: 主要競合3〜5社を4象限に配置
  - **自社をマッピング**: 自社がどの象限にいるか、どの象限に行きたいかを明記
  - **ホワイトスペースを発見**: 競合が薄い／不在の象限を特定
- 軸選定のルール：
  - 顧客が実際に意思決定する基準を選ぶ（価格、品質、用途、ブランド、利便性、信頼性等）
  - 「自社が良い」軸を選ばない（競合と差がついて見える軸を選ぶ）
- 出力：ポジショニングマップ（4象限＋競合・自社マッピング）

### ステップ4：差別化メッセージを言語化する

- 入力：ステップ3のポジショニング結果
- 処理：以下のフォーマットで差別化メッセージを作る
  - **誰に**: コアターゲット（30代女性／DINKS／首都圏）
  - **何を**: 自社のオファー（プレミアム手土産用スイーツ）
  - **どう違う**: 競合と異なる軸（贈り物利用＋3,000-5,000円帯×独自製法）
  - **理由**: なぜそれが提供できるか（自社の強み）
- 1文で書き切る：「○○な人に、××を、△△な競合と違って〜という理由で提供する」
- 出力：差別化メッセージ（1〜2文）

### ステップ5：マーケティング施策に接続する

- 入力：S/T/Pの結果＋差別化メッセージ
- 処理：以下を整理する
  - **想定広告チャネル**: ターゲットがどこにいるか（Web／SNS／オフライン／業界誌等）
  - **訴求軸**: ポジショニング軸を広告コピーに反映
  - **検証KPI**: ターゲット内のリーチ率／興味度／購入転換率
- 出力：施策接続シート

### ステップ6：STP分析シートを完成させる

- 入力：ステップ1〜5の全成果物
- 処理：以下のテンプレートに統合する
- 出力：STP分析完成シート

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| セグメント軸が1〜2軸のみ | 必ず3軸以上で切る | 単一軸のセグメントは粗すぎる |
| ターゲットが「全員」「一般消費者」 | コアターゲットを1セグメントに絞る | 全員ターゲットは誰にも届かない |
| 戦略ターゲットとコアターゲットを混同 | 2層で必ず分ける | コアに集中しないと施策がブレる |
| ポジショニング軸が「自社が良い軸」 | 顧客が意思決定する軸に置き換える | 自社視点の軸は差別化にならない |
| 4象限の全てに競合がいる | 軸を切り直して再度マッピング | ホワイトスペースが見つかる軸が真の差別化軸 |
| 自社と競合が同じ象限 | 別の差別化軸を探す or 撤退検討 | 同質競合は価格競争に巻き込まれる |
| ニッチ市場すぎてビジネスにならない | ターゲットを少し広げる／隣接セグメントを取り込む | 絞りすぎは規模が出ない |
| 1度作ったら見直さない | 半年〜1年で再評価 | 市場・競合は動く |
| SWOTや3Cと矛盾する | 他フレームワーク結果と整合性を確認 | 単独STPは偏る |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| セグメント軸が1〜2軸 | 3軸以上で切る | 粗いセグメントは精度が出ない |
| ターゲット「全員」「一般」 | コアを1セグメントに絞る | 全員ターゲットは誰にも届かない |
| 戦略ターゲットとコアの混同 | 2層で明確に分ける | コア集中しないと施策がブレる |
| ポジショニング軸が「品質高い」「サービス良い」（自社評価軸） | 顧客が意思決定する軸を選ぶ | 自社視点軸は差別化にならない |
| 競合をマッピングしない | 必ず競合3〜5社をプロット | 競合不在の検討は楽観バイアス |
| ホワイトスペース探しを省略 | 4象限の空白を必ず確認 | 差別化の宝庫を見逃す |
| 差別化メッセージを書かない | 1文で「誰に何をどう違って提供するか」を言語化 | 施策に繋がらない |
| 1度作って放置 | 半年〜1年で再作成 | 市場・競合は動く |
