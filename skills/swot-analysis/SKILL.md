# swot-analysis

SWOT分析（4象限）とクロスSWOT（4戦略）を組み合わせて、現状把握から実行可能な戦略までを一気通貫で出力する。「現状の整理止まり」を防ぐため、必ずクロスSWOTで戦略立案、KPI設定まで進める。

## 適用条件

- 使う場面：
  - 事業戦略・サービス戦略を立てる前の現状把握
  - 中期計画・年間計画の前提整理
  - 競合との差別化ポイントを構造的に確認したい
  - 撤退・縮小判断の前に客観的なリスク評価をしたい
- 使わない場面：
  - 顧客の「ジョブ」分析 → Jobs to Be Done
  - 競争構造の深掘り → 5フォース分析（`five-forces-analysis`）
  - 外部環境のマクロ分析のみ → PEST分析（`pest-analysis`）
  - 内部の経営資源評価のみ → VRIO分析（`vrio-analysis`）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 分析対象 | 必須 | 会社・事業・サービス単位 | 「自社のSEO支援事業」「新規プロダクトX」 |
| 分析の目的 | 必須 | 現状把握／差別化確認／撤退判断 | 「生成AI領域での差別化戦略を立てる」 |
| 既知の事実 | 任意 | 競合情報・市場データ・社内事情 | 「競合A社は300万円で内製化支援」 |
| 期間軸 | 任意 | 今期／中期／長期 | 「今期+3年」 |

## 実行ステップ

### ステップ1：外部環境（O / T）から分析する

- 入力：分析対象、分析の目的、既知の事実
- 処理：思考順序のセオリー通り、**外部から先に分析する**（内部から始めると主観に引きずられるため）
  - **O（機会）の収集質問**:
    - 業界・市場で伸びている領域は？（市場規模・成長率データを引用）
    - 顧客ニーズ・行動の変化は？（具体的な事象を1つ以上）
    - 技術革新・法規制の緩和で開かれる新市場は？
    - 競合が対応できていない領域は？
  - **T（脅威）の収集質問**:
    - 競合の新サービス・新参入者は？
    - 経済状況・規制強化のリスクは？
    - 顧客単価が下がる可能性のある外部要因は？（AI代替・代替品の出現など）
    - 自社が「乗り遅れている」と感じる領域は？
- 出力：外部環境の機会4〜6項目、脅威4〜6項目（数値・事実つき）

### ステップ2：内部環境（S / W）を客観的に整理する

- 入力：ステップ1の結果、既知の事実
- 処理：
  - **S（強み）の収集質問**:
    - 競合にない自社だけの強みは？（製品・人材・コスト・ブランド・財務）
    - 顧客が選んでくれる理由は？（受注時のヒアリング・営業現場の声を引用）
    - 過去の成功要因として再現可能なものは？
  - **W（弱み）の収集質問**:
    - 失注理由の上位3つは？
    - 社内で不満・離職理由として繰り返し出る項目は？
    - 競合と比較して劣る指標は？（コスト・スピード・品質・組織力）
- **客観性チェック**：各項目に「なぜそう言えるか」の根拠（数値・事実・第三者の声）を必ず1つ添える。根拠がない項目は△マークを付ける（後で再調査する印）
- 出力：強み3〜5項目、弱み3〜5項目（根拠つき）

### ステップ3：SWOTマトリクスを作成する

- 入力：ステップ1〜2の項目
- 処理：以下の2軸4象限マトリクスに整理する
  - 軸：内部環境（横）×影響（縦：好影響/悪影響）
  - 各象限に項目を配置
  - 項目間で重複や矛盾がないか確認
  - 抽象的な表現（「人材力が高い」等）は具体化（「○○の経験を持つマーケター3名在籍」）
- **3原則チェック**:
  - 客観的か（事実ベース／主観でないか）
  - 具体的か（抽象表現を残していないか）
  - 重複・矛盾がないか
- 出力：SWOTマトリクス（完成版）

### ステップ4：クロスSWOTで4戦略を立案する

- 入力：SWOTマトリクス
- 処理：4象限の組み合わせで戦略を考える
  - **S×O 戦略（攻めの戦略）**: 強みを活かして機会を最大限に活用する。**最優先**で立案する
    - 例：「生成AI対応の強み × 大手SEO会社が未対応の機会 → AI×コンサル統合サービスを業界先駆けで投入」
  - **W×O 戦略（克服戦略）**: 弱みを克服して機会を取りに行く
    - 例：「マーケ部門なし × 代理店経由案件の機会 → 代理店向けホワイトラベル提供で営業負荷を下げる」
  - **S×T 戦略（防衛戦略）**: 強みを活かして脅威を回避・対抗する
    - 例：「内製化支援の強み × AI による単価下落の脅威 → 成果報酬型プランを導入して単価依存度を下げる」
  - **W×T 戦略（撤退・縮小戦略）**: 弱みと脅威が重なる領域から撤退・縮小する
    - 例：「組織ロイヤリティ低い × 突発リソース逼迫 → 低単価案件を切り捨てて高単価伴走支援に集中」
- **戦略立案の制約**:
  - 各象限で戦略は1〜2個に絞る（多すぎると優先順位が消える）
  - 戦略名は短く（10字以内）、内容は1文で
  - 抽象論禁止（「強化する」ではなく「○○を△△する」と動詞＋目的語で書く）
- 出力：4戦略（合計4〜8個）

### ステップ5：優先順位とKPIを設定する

- 入力：4戦略
- 処理：
  - **優先順位ルール**: S×O 戦略 > W×O 戦略 ≧ S×T 戦略 > W×T 戦略（攻めから順に）
  - **時間軸の振り分け**:
    - 短期（〜3ヶ月）：S×O 戦略のうち実行容易性が高いもの
    - 中期（3〜12ヶ月）：W×O 戦略・S×T 戦略
    - 長期（1年以上）：W×T 戦略（撤退・縮小）
  - **KPI設定**: 各戦略に「達成判定可能な数値」を1つ以上付ける
    - 例：「新規顧客数を△件増加」「市場シェアを○%拡大」「単価を×円維持」
    - 抽象指標（「強化する」「向上させる」）は禁止
- **戦略のリソース見立て**:
  - 各戦略に必要な人員・予算・期間を概算で添える
  - リソース不足が見えた戦略は「実行容易性：低」マークを付ける
- 出力：優先順位＋時間軸＋KPI付きの戦略一覧

### ステップ6：SWOT分析シートを完成させる

- 入力：ステップ1〜5の全成果物
- 処理：以下の出力フォーマットに統合する
- 出力：SWOT分析完成シート（運用に乗せられる形）

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 内部環境から分析を始めたくなる | 必ず外部（O/T）から始める | 外部から始めないと内部の評価が主観的になる |
| 強み・弱みに根拠が無い項目 | △マークを付け、根拠調査タスクを別出し | 根拠なき分析は形骸化する |
| SWOT 4象限が項目ゼロの象限がある | その象限を埋めるための追加質問を投げる | 4象限揃わないとクロスSWOTが成立しない |
| クロスSWOT4戦略のうちどれを優先するか迷う | S×O > W×O ≧ S×T > W×T の順 | 攻めから優先するセオリー |
| W×T 戦略が「現状維持」になる | 撤退・縮小・回避の3択で具体化 | 「現状維持」はW×Tの本質と矛盾 |
| KPI が「強化する」「向上させる」 | 数値目標に書き換える | 達成判定不能な指標は機能しない |
| クロスSWOT4戦略が10個以上 | 各象限1〜2個に絞る | 多すぎると優先順位が消える |
| 「主観的な意見」「経験則」しか出ない | データ・第三者の声を必須化（受注理由・失注理由・市場データ等） | SWOTの最大の落とし穴は主観バイアス |

### 記入例（SEO支援会社の例）


## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 内部環境から分析を始める | 必ず外部（O/T）から始める | 内部から始めると主観バイアスで強みを過大評価する |
| 強み「人材力が高い」等の抽象表現 | 「○○の経験を持つマーケター3名在籍」等の具体表現 | 抽象表現はクロスSWOTで戦略に変換できない |
| SWOT 4象限を埋めて満足する | クロスSWOTで4戦略まで必ず立てる | SWOTだけだと現状整理止まり |
| クロスSWOT 戦略が「強化する」「がんばる」 | 動詞＋目的語＋数値で書く | 抽象戦略は実行不能 |
| W×T 戦略が「現状維持」 | 撤退・縮小・回避の3択で具体化 | W×Tの本質は引き算の戦略 |
| KPI が無い／「向上させる」 | 数値目標で書く（「○件増加」「△%拡大」） | 達成判定不能な目標は形骸化 |
| 全戦略を全部実行しようとする | S×O > W×O ≧ S×T > W×T の優先順位で絞る | リソースが分散すると全部中途半端 |
| 1度作ったら見直さない | 環境変化があるたびに再作成（最低半年に1回） | 外部環境は変わり続ける |
| 主観・経験則だけで項目を入れる | データ・第三者の声で根拠を添える | SWOTの最大の落とし穴は主観バイアス |
