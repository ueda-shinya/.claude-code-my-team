# yony-sales-simulation スキル

YonY（Year on Year＝前年同期比）で中長期売上シミュレーションを設計し、固定費・変動費・損益分岐点を組み合わせて、楽観・現実・悲観の3シナリオで定量予測する。事業戦略スクール「知足」フレームワーク準拠。

## 適用条件

- このスキルを使う場面：
  - 中長期（5〜10年）の売上をシミュレーションしたい
  - 楽観・現実・悲観の3シナリオで売上予測を作りたい
  - 損益分岐点を把握し、最低限必要な売上目標を設定したい
  - 価格改定や広告投資の効果を前年データと比較したい
  - 金融機関への融資申請に売上シミュレーションが必要
- このスキルを使わない場面：
  - 中長期事業計画の全体像 → `medium-term-business-plan` を使う
  - 財務三表（BS/PL/CF）の基礎 → `financial-statements-fundamentals` を使う
  - KGI/KPI/KAI 設計 → `kgi-kpi-kai-design` を使う
  - 個別プロジェクト収支 → `project-finance-contract` を使う

## 入力

| 項目 | 必須/任意 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガーフレーズのいずれか |
| 過去の売上データ | 必須 | 直近2〜3期の売上実績（細分化されているほど精度向上） |
| 業種・事業構成 | 必須 | 事業部別の売上構成（マーケ／制作／外食 等） |
| 固定費・変動費の内訳 | 任意 | 月次／年次の費用構造 |
| シミュレーション期間 | 任意 | 5年・10年（デフォルト10年） |

## 実行ステップ

### ステップ1：YonYの定義と4洞察を理解する

**YonY = Year on Year**：ビジネス計画や財務分析を簡潔かつ効果的に行うための概念

**4つの洞察**：
| # | 洞察 | 内容 |
|---|---|---|
| ① | 売上の成長予測 | 過去数年間のYonY成長率を基に、将来の売上を予測 |
| ② | 市場変化への対応 | 市場全体の成長率や消費者動向を取り入れて売上影響をシミュレート |
| ③ | 施策の効果検証 | 新規顧客獲得施策や単価引き上げの効果をYonYで評価（例：価格10%引き上げ時の売上影響） |
| ④ | リスクシナリオの検討 | 楽観・中立・悲観の3シナリオを設定し、それぞれのYonY成長率で売上予測 |

### ステップ2：5設計ポイントで売上シミュレーションを構築する

| # | ポイント | 内容 |
|---|---|---|
| 01 | 適切な期間を選定する | 比較対象として前年同月や同四半期のデータを用いる。季節性業種は同じ季節を比較 |
| 02 | 過去データを整備する | 地域別・商品別・顧客別の売上、キャンペーン期間中の売上に細分化 |
| 03 | 影響する要因の把握 | 外部要因（市場需要変化／経済状況）と内部要因（価格改定／新商品導入／広告投資）を特定 |
| 04 | 複数のシナリオを検討する | 例1：年間2%増／例2：広告投資で5%増／例3：市場縮小で1%減 |
| 05 | シミュレーション結果の検証と調整 | 実績データと比較し予測値の妥当性を検証。偏差が大きければモデル見直し |

### ステップ3：売上シミュレーションの3要素を理解する

※ 詳細な財務基礎（BS/PL/CFとの連携、財務体質改善）は `financial-statements-fundamentals` ステップ7と整合させること。本ステップは予測モデル構築の前提整理。

**売上シミュレーション** = 事業計画における将来の収入・支出予測モデル。資金繰り・収益性評価のために使用。

| 要素 | 内容 | 例 |
|---|---|---|
| 固定費 | 売上に関係なく一定期間ごとに必ず発生する費用 | 人件費／家賃／水道光熱費／販売管理費 |
| 変動費 | 売上に応じて変動する費用 | 商品仕入れ／消耗品費／広告宣伝費 |
| 損益分岐点 | 収入と支出が等しくなり、利益も損失もない売上高 | 固定費÷（1−変動費率） |

### ステップ4：6必要要素でシミュレーションを実施する

| # | 要素 | 内容 |
|---|---|---|
| 01 | 固定費の算定 | 人件費や家賃など、売上に関係なく発生する費用を明確化 |
| 02 | 変動費の算定 | 売上に応じて変動する費用を、売上高に対する割合（変動費率）で算出 |
| 03 | 粗利益の算定 | 売上総利益÷売上高（粗利率）を計算し、業界平均と比較して収益性評価 |
| 04 | 損益分岐点の把握 | 固定費と変動費から、損益ゼロ売上高を計算 → 最低限必要な売上目標 |
| 05 | 目標売上の設定 | 客単価×販売数量×営業日数等を基に、現実的な売上目標を具体数値で設定 |
| 06 | シミュレーション実施 | 楽観・現実・悲観の3シナリオを作成し、各ケースの収支予測 |

### ステップ5：Before/Afterで効果を確認する

**Before（YonYで売上シミュレーションをしない場合の3問題点）**：
| # | 問題点 | 内容 |
|---|---|---|
| 01 | 資金繰りに支障 | 収入と支出のタイミング・金額が不明確で、資金ショートリスク。新規事業や変動の多い業界で特に深刻 |
| 02 | 売上目標が現実的でなくなる | 達成困難な目標は従業員モチベーション低下、低すぎる目標は成長機会逸失 |
| 03 | 金融機関からの信用を得られない | 具体的収支計画なしでは融資審査が通らない、信用度低下、不利な条件での融資 |

**After（YonYで売上シミュレーションを行った場合のアウトプット例）**：
事業部別×期目別の見える化テーブル：
- 売上（マーケ／制作／外食 等）
- 外注・原価（原価率連動）
- 人件費（人件費/売上比率連動）
- 社員数イメージ（平均年収×人数）
- 営業利益

例（マーケティング会社・10年シミュレーション）：
- 売上合計：12期目700万 → 21期目7,052万（年率約25%成長想定）
- 外注・原価合計：380万 → 3,871万
- 人件費合計：130万 → 1,095万
- 社員数：32名 → 229名
- 営業利益合計：190万 → 2,085万

**重要な注記**：上記の数値例は**楽観シナリオ**の一例であり、自社で適用する際は必ず**現実シナリオと悲観シナリオも併記**すること。年率25%成長や人員229名等の前提は、市場環境・競合・自社実行能力の3要素を全て満たした場合のみ実現可能。「楽観だけで投資判断する」のは資金繰り破綻リスクの最大要因（CLAUDE.md「Competitive Absence Audit Rule」とも整合）。

### ステップ6：3STEP実践課題を実行する

| STEP | 内容 |
|---|---|
| STEP01 | 自社の固定費と変動費を洗い出し、正確な数値を算定する |
| STEP02 | 損益分岐点売上高を計算し、最低限必要な売上目標を設定する |
| STEP03 | 楽観的・現実的・悲観的な3シナリオで売上シミュレーションを作成し、各ケースでの収支予測を行う |

### ステップ7：3つの活用メリットを確認する

| # | メリット |
|---|---|
| 01 | 重要性：事業計画における将来収支予測・資金繰り評価の不可欠ツール |
| 02 | 活用：現実的な売上目標設定／リスク管理／資金調達時の信用獲得 |
| 03 | 実践：固定費・変動費の正確な算定／損益分岐点把握／複数シナリオでのシミュレーション |

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 過去2期未満のデータしかない | YonY成長率算定は保留、業界平均成長率で代替 | 信頼性のある成長率には最低2年データ必要 |
| 季節性が強い業種（小売・観光） | 月次／四半期単位でYonY比較 | 年間集計だと季節要因が見えなくなる |
| 急成長期（成長率20%超） | 楽観・悲観シナリオの幅を広めに（±10%以上） | 高成長期は変動が大きい |
| 安定成長期（成長率5%以下） | 楽観・悲観シナリオの幅を狭めに（±2-3%） | 安定期は変動小さく、過剰な幅は無意味 |
| 価格改定や新商品投入を計画中 | 内部要因として明示し、シナリオに反映 | 施策効果の事前シミュレーション |
| 「うちは成長率毎年2倍が確実」と楽観判断 | 楽観・現実・悲観の3シナリオを必ず作成 | 単一楽観予測は資金繰り破綻リスク最大 |
| 「うちは独自事業で市場変動の影響を受けない」 | 外部要因（市場需要・経済状況）を必ず織り込む | `competitive-absence-audit` で楽観判断検証 |
| 損益分岐点を計算していない | 最優先で計算（固定費÷限界利益率） | 最低必要売上目標が定まらない |
| 3シナリオで悲観シナリオでも黒字 | 比較的安全な事業構造 | 投資・拡大検討余地あり |
| 3シナリオで現実シナリオでも赤字 | 事業構造の見直し最優先 | 固定費削減／変動費率改善が必須 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 単一の楽観シナリオで売上予測 | 楽観・現実・悲観の3シナリオを必ず作成 | 資金繰り破綻リスク回避の基本 |
| 過去2期未満のデータで予測 | 業界平均成長率を補助データとして使用 | 信頼性に欠ける予測は意思決定の害 |
| 季節性業種で年間集計のみ比較 | 月次／四半期単位でYonY比較 | 季節要因が見えず、月別資金繰りが破綻 |
| 損益分岐点を計算していない | 最優先で計算 | 最低必要売上目標なき経営は方向喪失 |
| 固定費・変動費の区別が曖昧 | 売上比例性で機械的に分類 | 損益分岐点計算の前提崩壊 |
| 「成長率毎年2倍」等の根拠なき楽観 | 過去実績ベース＋業界平均で補正 | 楽観予測は人材投資・固定費膨張の主因 |
| 外部要因（市場・経済）を無視 | 外部・内部要因を必ず両方織り込む | 想定外の市場変動で破綻 |
| 「うちは独自事業で外部影響を受けない」 | `competitive-absence-audit` で検証 | 楽観判断の典型 |
| シミュレーション結果を実績と検証しない | 実績との偏差を毎期チェックしモデル更新 | 形骸化したシミュレーションは意思決定の害 |
| Excelテンプレを埋めるだけで終わる | 結果をBS/PL/CFと連動させ、財務戦略に反映 | シミュレーションは意思決定のための道具 |

## 連携スキル

- `medium-term-business-plan`：中長期事業計画の全体像と整合
- `financial-statements-fundamentals`：BS/PL/CF基礎で結果を解釈
- `financing-strategy`：シミュレーション結果に基づく資金調達戦略
- `kgi-kpi-kai-design`：シミュレーション結果をKGI/KPIに落とし込む
- `growth-phase-strategy`：成長フェーズに応じたシナリオ幅の設定
- `pmf-journey`：PMF段階に応じた成長率前提の調整
- `competitive-absence-audit`：「うちは特殊／市場影響受けない」等の楽観判断を検証
- `project-finance-contract`：個別プロジェクトの収支管理と整合

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし以下のケースで明示マーカーを使うこと：

- `[OK:0件]`：自社の過去実績データが不足し、フレームワーク説明のみで終了した場合
- `[SKIP]`：競合不在・独自事業等の楽観判断が出た場合は `competitive-absence-audit` への誘導でスキップ

外部API・サブプロセスを持たないため、`[FAIL]`（試行後の失敗）は通常発生しない。
