# project-level-definition スキル

クライアント案件を5指標×5レベルのマトリクスで判定し、適切な提案・体制・契約条件・期待値調整を導出する。

## 適用条件

- このスキルを使う場面：
  - 新規問い合わせ案件のレベル感を判定し、提案方針を決めたい
  - 既存案件のレベルを再評価し、契約条件・体制を見直したい
  - クライアントとの期待値ギャップ（工数期待 vs 成果期待）を診断したい
  - 営業段階で「この案件はうちに合うか」「どのレベルの提案を出すか」を判断したい
  - レベル別の標準提案テンプレートを整備したい
- このスキルを使わない場面：
  - 個別商談の提案書作成（ストーリー組み立て） → `proposal-builder`
  - 営業資料の構成テンプレ設計 → `sales-deck-template`
  - クライアント期待値調整の心理ハンドリング詳細 → `client-expectation-management`
  - PJ体制（人員・組織図）の具体設計 → `project-team-structure`
  - 自社の成長フェーズ別経営戦略 → `growth-phase-strategy`（自社向け／クライアント案件レベル判定とは別）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 案件名／企業名 | 必須 | 判定対象 | 「ABC社 マーケ支援案件」 |
| 問い合わせ内容／RFP | 必須 | クライアントからの依頼内容 | 「リスティング広告運用代行を月50万で」 |
| 想定期間 | 必須 | スポット／月額／長期 | 「月額12ヶ月」 |
| クライアント企業の状態 | 任意 | 既存事業／成長／新規／IPO・M&A準備 等 | 「既存事業の運用改善フェーズ」 |
| 想定予算規模 | 任意 | 月額／総額 | 「月額50万円」 |
| クライアント担当者の役職 | 任意 | 現場担当／部長／役員／代表 | 「マーケ担当者（部長）」 |

## 実行ステップ

### ステップ1：5指標で案件を診断する

各指標について、現在の案件がどの状態にあるかを観察する。

| 指標 | 観察観点 |
|---|---|
| ① スコープの広さ | 単一施策か／複数施策か／広範囲統合か／顧客開発まで含むか／組織設計含むか |
| ② 経営レベル | 経営戦略への影響なし／部分的マーケ戦略／事業戦略と密接／組織財務に影響／会社全体の方向性に関与 |
| ③ 事業の特性 | 既存事業の運用改善／成長フェーズ売上拡大／既存事業グロース／新規事業立ち上げ／IPO・M&A準備 |
| ④ クライアントのコミュニケーション | 指示型・受け身／質問や要望は投げるが最終判断は自社／意見を求めるが決定権はクライアント／共同作業型（週次MTG）／フルパートナー型（意思決定の一部を担う） |
| ⑤ クライアントの期待値 | 工数期待が高い／成果期待あり中期視点／成果期待が高く専門性も求められる／成果期待が大きく柔軟対応／高度な専門性と経営視点 |

### ステップ2：5レベルのマトリクスで判定する

下記マトリクスから、案件が最も近いレベルを特定する（複数該当する場合は、最も多くの指標が一致するレベルを採用）。

| 指標 | レベル1（スポット施策） | レベル2（小規模マーケ戦略） | レベル3（事業成長戦略） | レベル4（新規事業マーケ支援） | レベル5（経営戦略マーケ支援） |
|---|---|---|---|---|---|
| **スコープの広さ** | 単一施策のみ（広告運用・SEO対策など） | 複数施策の組み合わせ（広告＋LP最適化） | 広範囲のマーケ施策（広告・SEO・CRMなど統合） | マーケ全般＋顧客開発＋プロダクト改善提案も含む | マーケ×経営戦略×組織設計（M&A・資金調達にも影響） |
| **経営レベル** | 施策レベルのみ（経営戦略への影響なし） | 部分的なマーケ戦略（KPI設定・運用改善提案） | 事業戦略と密接に関わる（中期戦略・組織への影響） | 事業戦略と連携し、組織・財務への影響も考慮 | 経営陣と議論し、会社全体の方向性に関与 |
| **事業の特性** | 既存事業の運用改善フェーズ | 成長フェーズの支援（売上拡大施策） | 既存事業のグロースフェーズ | 新規事業の立ち上げフェーズ | 企業の成長戦略フェーズ（IPO・M&A準備など） |
| **クライアントのコミュニケーション** | 指示型・受け身（要望を伝え、対応を求める） | 質問や要望を投げるが、最終判断は自社 | 意見を求めるが、決定権はクライアント | 共同作業型（週次MTGで戦略相談） | フルパートナー型（意思決定の一部を担う） |
| **クライアントの期待値** | 工数期待が高い（「◯時間で何をしてくれるか」） | 成果期待あり、中期視点の期待 | 成果期待が高く、専門性も求められる | 成果期待が大きく、柔軟な対応を求められる | 高度な専門性と経営視点を求める |

### ステップ3：レベル別の提案・体制・契約条件を導出する

判定したレベルに応じて、提案方針・体制・契約条件のガイドラインを適用する。

| レベル | 提案の主軸 | 推奨体制 | 契約条件の傾向 | 単価帯（参考） |
|---|---|---|---|---|
| レベル1 | 工数ベースの実務代行 | 担当1名（実務オペレーター） | 月額固定／時間単価／成果連動少 | 低（月額数万〜数十万） |
| レベル2 | 複数施策パッケージ＋KPI改善 | 担当1〜2名（オペ＋ジュニア戦略） | 月額固定＋一部成果報酬 | 中（月額数十万〜100万） |
| レベル3 | 事業成長への貢献提案 | 担当2〜3名（戦略＋実務＋分析） | 月額顧問＋プロジェクト型 | 中〜高（月額100万〜数百万） |
| レベル4 | 新規事業の事業化支援 | 担当3〜5名（戦略＋プロダクト＋実務） | プロジェクト型＋成功報酬／エクイティ可 | 高（プロジェクト数百万〜千万） |
| レベル5 | 経営戦略パートナー | シニア＋専門家チーム | 顧問契約＋経営参画／取締役就任打診も視野（クライアント側の判断による・善管注意義務等の法務確認必須） | 最高（顧問+成功報酬+エクイティ） |

※単価帯は参考目安であり、業界・案件特性・自社のフェーズで変動する。確定値ではない。

### ステップ4：レベル間ギャップを点検する

クライアントの期待レベルと自社が提供できるレベルにギャップがあると、案件は失敗する。下記のギャップパターンに注意する。

| ギャップ | 症状 | 対応 |
|---|---|---|
| クライアント期待がレベル1なのに自社がレベル3で提案 | 「予算オーバー」「過剰提案」と感じられる | レベル1〜2に絞った提案に縮小／別案として段階的契約を提示 |
| クライアント期待がレベル3なのに自社がレベル1の提供 | 「物足りない」「経営に貢献しない」と感じられる | レベル2〜3にスコープ拡大／戦略レイヤーの担当者を投入 |
| 契約はレベル2なのに運用中にレベル4要求が出る | スコープクリープ／工数オーバー | 契約再交渉／別契約として切り出す |
| クライアントのコミュ形式がレベル1（指示型）なのに自社がレベル4（共同作業型）の進行を期待 | 議論にならず提案が空回り | コミュ形式をレベル1〜2に合わせる／リードする側の役割を明確化 |

### ステップ5：判定結果と次のアクションをまとめる

判定したレベル、ギャップ、対応方針を1枚のサマリーにまとめ、社内合意を取る。

## 判断基準

| 状況 | 判断 |
|---|---|
| 5指標すべてがレベル1に該当 | レベル1案件として工数ベース提案 |
| 5指標のうち過半数（3つ以上）が同レベルに該当 | そのレベルとして判定 |
| 指標がバラつき、特定レベルに収束しない | 「ハイブリッド案件」と判定し、各指標で最高レベルに合わせて体制を組む |
| クライアントの期待値（指標⑤）と他指標がズレる | 期待値調整を最優先（`client-expectation-management`に引き継ぐ） |
| レベル4以上と判定 | 自社の提供能力／リソースを再点検／プロジェクト型契約＋成功報酬を検討 |
| レベル5と判定 | 経営層の承認必須／顧問契約＋経営参画の枠組みを設計 |
| 判定後に「自社では提供できないレベル」と判明 | 案件辞退または下位レベルへの絞り込み提案を検討 |
| クライアント自身も自社のレベル感を把握していない | ヒアリングで5指標を聞き出してから判定 |
| 提案書ストーリー設計が必要 | 判定結果を入力として `proposal-builder` に引き継ぐ |
| 営業資料テンプレートが必要 | 判定レベル別に `sales-deck-template` を引き当てる |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 単価や予算規模だけでレベル判定する | 5指標全体で判定する | 予算が大きくてもクライアント期待がレベル1の場合あり／その逆もある |
| クライアント担当者の役職だけで「レベル5」と判定 | 5指標を観察し、コミュ形式・期待値も確認 | 役職と意思決定権はイコールではない |
| 一度判定したレベルを最後まで変えない | PJ進行中に指標が変化したら再判定する | スコープ追加／クライアント側の体制変更でレベルが変わることがある |
| ハイブリッド案件を無理に1レベルに分類 | ハイブリッドと明示し、各指標で最高レベルに合わせて体制を組む | 無理な分類は契約条件・体制ミスマッチの原因 |
| 自社が提供できないレベル4〜5案件を引き受ける | 提供能力を再点検し、辞退または下位レベル絞り込み提案 | 提供できない案件は失注より大きな損失（解約・評判悪化） |
| クライアント期待がレベル1なのにレベル3提案 | 期待レベルに合わせて提案を縮小／段階的契約として提示 | 過剰提案は「予算オーバー」で失注する |
| クライアント期待がレベル3なのにレベル1提供 | スコープ拡大／戦略レイヤー担当を投入してレベルアップ | 物足りないと判断され、契約解消につながる |

## 連携スキル

- **前段（ヒアリング）**：`hearing-questioning-skills`（5指標を引き出すための質問設計）
- **並走（営業）**：`sales-playbook`（営業相談オーケストレータ）
- **後段（提案資料）**：`proposal-builder`（提案書ストーリー）／ `sales-deck-template`（24pテンプレ）
- **後段（期待値調整）**：`client-expectation-management`
- **後段（PJ体制設計）**：`project-team-structure`（人員・役割・組織図）
- **後段（PJ収支・契約）**：`project-finance-contract`
- **後段（PJ進行リスク管理）**：`pm-risk-management`
- **後段（デリバリー標準化）**：`delivery-build`
- **関連（自社の成長フェーズ判定／自社向け）**：`growth-phase-strategy`（クライアント案件のレベル判定とは別、自社向けスキル）

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存を持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし運用上のベストプラクティスとして、5指標のうち情報不足のものは `[SKIP]` で「{{指標名}}が未確定のため判定保留」と明示し、推測で埋めない。3指標以上が `[SKIP]` の場合は判定全体を保留し、追加ヒアリングを実施する。
