# success-cycle-relationship-quality スキル

成功循環モデル = MITダニエル・キム教授提唱、組織のパフォーマンス向上に関するフレームワーク。「関係の質→思考の質→行動の質→結果の質」の好循環を生み出すことが目的。事業戦略スクール「知足」のフレームワーク準拠。

## 適用条件

- このスキルを使う場面：
  - 組織パフォーマンスが低下している原因を「関係の質」起点で診断したい
  - 心理的安全性／信頼関係／コミュニケーションの現状をスコアで測りたい
  - 離職率低下・エンゲージメント向上の打ち手を考えたい
  - 「結果が出ない → 責任の押し付け合い → 関係悪化」の悪循環を断ち切りたい
- このスキルを使わない場面：
  - 個人の能力／意欲評価（人材4象限） → `ms-matrix-talent-grid`
  - 評価制度の策定 → `evaluation-system-design`
  - 会議体の運営改善 → `meeting-cadence-design`
  - MVV策定 → `mvv-design`
  - クレド（文化）の言語化 → `credo-language-design`
  - リーダー指導アプローチ → `teaching-coaching-leading`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 対象組織 | 必須 | 全社／部署／チーム単位 | 「営業部7名」「全社20名」 |
| 現状の症状 | 必須 | 起きている問題 | 「離職率高い」「主体性なし」「責任押し付け」 |
| 既存データ | 任意 | エンゲージメントサーベイ結果、1on1記録 | 「ESサーベイ過去2回分」 |
| 評価実施方法 | 必須 | 自己診断／メンバーアンケート／第三者観察 | 「メンバーアンケート全員回答」 |
| 改善期間 | 任意 | 短期（1ヶ月）／中期（半期）／長期（1年） | 「半期で実感したい」 |

## 実行ステップ

### ステップ1：成功循環モデルの現在地を判定する

- 入力：現状の症状、既存データ
- 処理：
  1. 「好循環」と「悪循環」のどちらに入っているかを判定
  2. 4つの質それぞれの現状を5段階で自己評価

| 循環タイプ | 起点 | 進行順 | 結果 |
|---|---|---|---|
| **好循環（Good cycle）** | 関係の質 | 1.関係の質 → 2.思考の質 → 3.行動の質 → 4.結果の質 | さらに関係が良くなる |
| **悪循環（Bad cycle）** | 結果の質 | 1.結果の質 → 2.関係の質 → 3.思考の質 → 4.行動の質 | さらに結果が悪くなる |

3. **重要原則**：「結果を変えたいなら、関係の質から変える」
4. 現在のサイクル位置を可視化（どの質から悪化しているか）

- 出力：循環タイプ判定、4つの質の現状スコア、ボトルネック特定

### ステップ2：「関係の質」の現状を13プロパティ×5レベルで診断する

- 入力：対象組織のメンバー、評価実施方法
- 処理：以下のチェックシートで自社の段階を判定（ヒューマンバリュー社の調査ベース）

| レベル | プロパティ | 説明 | チェック |
|---|---|---|---|
| **1** | 挨拶 | お互いに自然な挨拶をしている度合い | □ |
| **1** | 声掛け | 誰とでも軽い世間話をしている度合い | □ |
| **2** | つながり | 普段から気兼ねなく話せる人の多さ | □ |
| **2** | 会話量 | 職場での会話量の多さ | □ |
| **3** | ありがとう | 感謝の言葉が自然に出ている度合い | □ |
| **3** | 活気 | 職場の明るさやみんなの表情の柔らかさ | □ |
| **3** | 尊重 | 役職や経験にかかわらず、お互いが一人の人間として関わり合っている度合い | □ |
| **4** | 背景理解 | 相手の仕事や状況を理解し合っている度合い | □ |
| **4** | 率直さ | 自分の考えや気持ちを素直に伝え合っている度合い | □ |
| **4** | 横断 | 違う仕事や他の部署の人たちとのつながりやコミュニケーションの度合い | □ |
| **5** | 一体感 | 仕事への想いやビジョンを共有している度合い | □ |
| **5** | 協働 | 既存の役割や立場を超えて協力し合っている度合い | □ |
| **5** | 信頼 | お互いを信じ、受容し合っている度合い | □ |
| **5** | 越境 | 今までの習慣や活動の領域を超えて社外の人々と関わり合い、ビジョンを語り合っている度合い | □ |

3. 評価方法（必ず1つ選ぶ）：
   - **自己診断**：管理者/リーダーが当てはまる項目をチェック（最も簡易、ただし主観バイアス強）
   - **メンバーアンケート**：各メンバーが匿名で5段階評価（1：当てはまらない〜5：強く当てはまる）。**推奨**
   - **第三者観察**：外部コンサル/HRが観察してチェック（最も客観だがコスト大）
4. 平均スコアと最低スコアの両方を算出（最低スコアの項目が「弱点」）
5. **判定ルール**：あるレベルの全プロパティの平均スコアが**4.0以上**のとき、そのレベルを「**到達済み**」と判定。1つでも4.0未満があるレベルは未到達。
6. **到達レベルの定義**：到達済みレベルのうち最高のレベル番号（連続して達成している最高レベル）

- 出力：13プロパティのスコアシート、組織が到達している最高レベル、改善すべき最低レベル

### ステップ3：到達レベルと弱点を踏まえた改善方針を決める

- 入力：13プロパティのスコアシート
- 処理：
  1. **段階別アプローチ**：レベル1〜5は順に積み上げるもの。レベル1（挨拶・声掛け）が崩れていれば、まずそこから着手する
  2. **下位レベル優先原則**：レベル4・5を高めようとする前に、レベル1〜3が「全項目で平均4以上」かを確認。土台が崩れていれば下から建て直す
  3. 弱点プロパティ別に、関係の質を高める4ポイントから処方箋を選ぶ

| 弱点プロパティ | 推奨処方箋 |
|---|---|
| 挨拶／声掛け／つながり／会話量 | POINT 01 信頼関係を築く＋POINT 02 心理的安全性 |
| ありがとう／活気／尊重 | POINT 02 心理的安全性＋POINT 04 適切なフィードバック |
| 背景理解／率直さ／横断 | POINT 02 心理的安全性＋POINT 04 適切なフィードバック |
| 一体感／協働／信頼／越境 | POINT 03 ビジョン・目的を共有＋POINT 02 心理的安全性 |

### ステップ4：関係の質を高める4ポイントを具体施策化する

- 入力：弱点プロパティと推奨処方箋
- 処理：以下4ポイントから自社で実行する施策を選定

| POINT | 内容 | 具体施策例 |
|---|---|---|
| **POINT 01 信頼関係を築く** | ・約束を守る<br>・相手を尊重する<br>・誠実なコミュニケーションを行う | ・週次1on1の徹底（約束遵守）<br>・「ありがとう」を口に出すルール化<br>・上長の発言を録音／メモして「言ったこと」を可視化 |
| **POINT 02 心理的安全性を確保する** | ・自由に意見を言える場を作る<br>・失敗を責めず、挑戦を評価する | ・ブレストで「批判禁止／質問は歓迎」ルール<br>・失敗事例を経営層から共有<br>・1on1の冒頭で雑談タイム5分必須 |
| **POINT 03 ビジョン・目的を共有する** | ・共通の目標を持つことで、一体感が生まれる | ・MVV／クレドの再共有会<br>・全社目標→部署目標→個人目標のカスケード可視化<br>・月次全社朝会でビジョン唱和 |
| **POINT 04 適切なフィードバックを行う** | ・相手の成長を促すポジティブなフィードバックを増やす | ・360度フィードバックの導入<br>・FB研修（SBI法／I message等）<br>・KPT 1on1の実施 |

- 出力：施策一覧（POINT × 具体施策 × 担当者 × 開始時期）

### ステップ5：関係の質改善が企業にもたらす4効果で目標KPIを設定

- 入力：自社の現状課題（離職率／生産性／イノベーション件数等）
- 処理：以下4効果のうち優先指標をKPIとして設定

| 効果 | 内容 | KPI例 |
|---|---|---|
| **1. エンゲージメント向上** | 社員が主体的に動くようになる | エンゲージメントサーベイスコア（半期比較）／自発提案件数 |
| **2. 生産性の向上** | チームワークが良くなる | 1人あたり売上／プロジェクト完了率／会議数削減 |
| **3. イノベーション創出** | 新しいアイデアが出やすくなる | 新規企画提案数／実験プロジェクト数 |
| **4. 離職率の低下** | 心理的安全性で人が辞めにくくなる | 半期離職率／勤続年数中央値 |

- 出力：優先KPI×目標数値×測定頻度

### ステップ6：再評価サイクルを設計する

- 入力：施策実行プラン
- 処理：
  1. 関係の質13プロパティの再評価頻度を決める：**3ヶ月ごと推奨**
  2. KPIの追跡頻度を決める：エンゲージメント半期、離職率四半期、生産性月次
  3. 次回診断日と振り返り会議をカレンダー登録
- 出力：再評価サイクル表

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 結果が出ていない／業績低迷 | まず関係の質から診断する（結果から手を付けない） | 悪循環の起点は結果ではなく結果起点の関係悪化 |
| レベル1（挨拶・声掛け）の項目スコア平均が4.0未満 | レベル4・5の施策は実行しない／レベル1から再構築 | 土台がないと積み上がらない |
| 着手レベル判定 | 到達済みレベル（全プロパティ平均4.0以上）+1 を着手レベルとする | 段階的な積み上げの基本 |
| 「率直さ」が極端に低い | 心理的安全性の問題が深刻／匿名サーベイで本音を抽出 | 顕在化していない不満が多い |
| 「越境」のみ低くてレベル1〜4が高い | 健全／無理に越境を作る必要はない（成熟組織でも越境は最終段階） | 越境は組織成熟の最終段階で、必須ではない |
| メンバーアンケートで評価がバラバラ | 部署別／チーム別に分割して再診断 | 全社平均では実態が見えない |
| 経営者だけが「関係性は良い」と評価 | 経営者の自己診断は除外し、メンバー評価を採用 | 経営者バイアスで実態より高く出やすい |
| 改善施策を実行しても3ヶ月でスコアが上がらない | POINTを変える／外部ファシリ導入を検討 | 同じ施策の継続は時間ロス |
| 離職者が連続で出ている | 緊急度高として最優先で診断する | 関係の質悪化のシグナル最強 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 結果（業績）から手を付けて関係の質を後回し | 関係の質から着手する | 悪循環の起点を断たないと意味がない |
| レベル5（一体感・越境）を最初に目指す | レベル1〜3の土台を先に固める | 段階を飛ばして上だけ作っても崩れる |
| 経営者の自己評価のみで診断 | メンバー匿名アンケートを必ず実施 | 経営者バイアスで実態と乖離 |
| 全社平均で診断して終わる | 部署別／チーム別に分けて診断 | 部署間の差を見逃すと打ち手がぼやける |
| 1回診断して施策を打ったら満足 | 3ヶ月ごとに再診断する | 関係の質は揺れ動くため継続観察必須 |
| 「率直さ」スコアが低いのに「率直に発言してください」と号令 | 心理的安全性を物理的に確保する仕組み（匿名チャネル等）を作る | 号令だけでは出ない |
| 「越境」が低いだけで組織を「未成熟」と判定 | 越境は最終段階、レベル1〜4が高ければ健全 | 全項目満点を目指す必要はない |
| 関係の質改善で「とにかくみんなで仲良く」と曖昧 | 13プロパティのどれを改善するか具体化する | 抽象指示では行動変容しない |

## 連携スキル

- `ms-matrix-talent-grid`：個人レベルの「マインド」を上げる文脈で関係の質改善が効く
- `evaluation-system-design`：360度フィードバック実装時に併用
- `meeting-cadence-design`：1on1や全社朝会の運営設計
- `mvv-design`：POINT 03 ビジョン共有の前提となるMVVを策定
- `credo-language-design`：クレドはPOINT 03 の具体ツールとなる
- `teaching-coaching-leading`：POINT 04 フィードバック技法を深掘り
- `onboarding-design`：新入社員の関係の質形成（レベル1〜2構築）
- `meeting-cadence-design`：1on1頻度の設計

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。本契約は適用しないが、運用上の**任意**ベストプラクティスとして以下のマーカー使用を**推奨**する：

- `[OK:0件]`：診断対象0名のとき
- `[PARTIAL:N成功/M失敗]`：複数部署診断で一部のアンケート回収率が低かったとき
- `[FAIL]`：アンケート回収率が30%未満で診断不能のとき。エラー要約を必ず添える
- `[SKIP]`：アンケート未回収のプロパティがある場合（試みなかったケースのみ）
