# プロジェクト体制管理スキル

組織やプロジェクトが目標達成のため、適切な人材・リソースを配置し効率運営する管理手法。リソース不足・タスク混乱・進捗遅延を防ぐ。

## 適用条件

- 新規プロジェクト立ち上げ時の体制設計
- 既存組織図の役割明確化・更新
- 複数社協業のプロジェクト実施体制図作成

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| プロジェクト目標 | 必須 | 達成すべきゴール |
| 既存メンバー | 任意 | バイネームで列挙 |
| 期間 | 任意 | 短期PJ／恒常組織 |

## 実行ステップ

### ステップ1: 体制管理の目的5項目チェック
1. 効率的なリソース活用
2. 役割と責任の明確化
3. 業務プロセスの最適化
4. リスクの軽減
5. チームの一体感向上

### ステップ2: 5つの基本構成要素

| 要素 | 内容 | 例 |
|---|---|---|
| **1. 人員管理** | 必要なスキル・経験を持つメンバーを適切配置 | デザイナー／エンジニア／テスター割振り |
| **2. 役割分担** | 各メンバーの責任を明確に定義 | PM／リーダー／サポートスタッフ明確化 |
| **3. コミュニケーション体制** | 情報共有のルールを定め円滑な連携を確保 | 定例会議／タスク管理ツール |
| **4. リソース管理** | 人的資源以外（予算・設備）も確保 | 必要ツール・予算リスト化 |
| **5. 進捗管理** | タスク進行状況を把握、計画通り進むよう調整 | ガントチャート／進捗ミーティング |

### ステップ3: 組織図設計（バイネーム＋機能別チーム）

**原則**：
- 機能別にチームを分け、責任者を置く
- バイネーム（実名）で担当者の名前を入れる

**典型構成例**：
```
代表（CEO）
├─ COO（業務統括）
│   ├─ 新規開拓 → 代理店
│   └─ マーケティング → YouTube運営／X運営
├─ CDO（デザイン統括）
│   └─ デザイン → コンテンツ・資料／制作・LP/HP
├─ CSリーダー
│   └─ CS → 経理財務／顧客対応（コミュニティ運営）
└─ コンテンツ
    └─ コンテンツ動画制作 → 動画編集
```

横並び（オブザーバー・パートナー）：補助金パートナー／生成AI／SEO／事業戦略広告／コンテンツオブサーバー／デザインオブサーバー／メルマガ／プレス／イベント管理／入金管理／ロゴ作成／バナー作成

### ステップ4: 役割分担表（6列構成）

| 名前 | 役職 | 管掌範囲 | 機能・役割分担（概要） | 評価指標（半期） | 機能・役割分担（詳細） |
|---|---|---|---|---|---|
| 〇〇 | CEO（社長）取締役 | 全て | ①中長期戦略意思決定 ②採用最終決定 ③資源配分 ④メンバーへのエンゲージメント ⑤資金調達 ⑥営業統括 | 全ての業績目標達成 | 事業成長方針・利益意思決定／採用最終意思決定／戦略的経営リスク洗い出し |
| 〇〇 | COO 取締役 | マーケ・採用等 | ①コンサル事業の目標達成 ②顧客対応・戦略構築管理 ③MVVやメンバーエンゲージメント | 事業部の業績目標達成・解約率5% | 1on1管理／戦略リスク洗い出し／採用・人材育成／解約率5% |
| 〇〇 | CDO 取締役 | 制作 | ①事業部の目標達成 ②目標達成管理 ③採用・管理業務 | 事業部業績達成 | 1on1管理／戦略策定実行／採用計画策定 |

（COO予定／CDO予定／執行役員候補等を追加）

### ステップ5: プロジェクト実施体制図（複数社協業時）

**3区分構成**：
- **PM（プロジェクト管理）**：品質管理／スケジュール・進捗管理／課題・リスク管理／部署間調整・窓口
- **サービス別分科会**：オウンドメディア（プロデューサー／メディアディレクター／セールスマネージャー）／システム開発（PM／SE／PG／営業責任者／案件責任者）
- **全体統括**：統括責任者／アプリ開発チーム／コンサルタント／サービスグランドデザイン／データマネタイズセールス／全体像TO BE戦略

各区分が「連携」矢印で相互接続。

### ステップ6: 体制管理の5プロセス

1. **目標設定**：チーム全体の目標を明確に（例：新商品3か月以内リリース）
2. **必要リソースの洗い出し**：スキル・人員・設備・予算をリスト化
3. **役割分担とチーム編成**：各メンバーに適切タスクを割振り
4. **コミュニケーションルールの設定**：定例頻度／重要事項の即時報告ルール
5. **進捗確認と調整**：定期確認、遅延時は追加メンバー投入等

### ステップ7: よくある課題4つと解決策

| 課題 | 解決策 |
|---|---|
| 人員不足やスキルミスマッチ | 外部リソース（派遣・フリーランス）の活用 |
| 役割の曖昧さ | 各役割を明確化し文書化して共有 |
| コミュニケーション不足 | 定例会議やツールを活用して情報共有を徹底 |
| 過剰な負荷の集中 | タスクを細分化しメンバー間で分散 |

## 判断基準

| 状況 | 判断 |
|---|---|
| 1人に役割が集中 | 過剰負荷／タスク細分化＋分散 |
| 役職が曖昧（「なんとなく担当」） | 役割分担表で6列記述を強制 |
| プロジェクトが大規模・複数社 | 実施体制図3区分（PM／分科会／統括）を採用 |
| メンバー＜ 5名の小規模PJ | 組織図簡略化、役割分担表のみで十分 |
