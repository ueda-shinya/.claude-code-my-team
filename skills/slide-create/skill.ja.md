# /slide-create スキル

スライド制作依頼を受けたとき、アスカがソラ（slide-designer）に正しく依頼し、ツールに貼り込める原稿を出力させるまでの手順を定義するスキル。

## トリガー条件

以下のいずれかに該当する場合にこのスキルを実行してください：

- ユーザーが「スライドを作って」「スライド作って」と発言した
- ユーザーが `/slide-create` と入力した

## 制約事項（全工程共通）

- **ソラへの依頼はアスカが行う。** シンヤさんが直接ソラに話しかける場合も、アスカが情報を整理してからソラに渡すこと
- **ソラに「指定された内容のみで構成すること」と必ず明示する。** 存在しない機能・数値・サービスを盛り込ませない
- **Genspark用フォーマットを使う場合、ソラにGenspark用注意書きを必ず含めるよう指示する**

## 実行手順

以下のステップを順番に実行してください。

---

### ステップ1：情報確認（アスカがシンヤさんに確認）

以下の項目を確認してください。既にわかっている情報はスキップして構いません。

| 項目 | 内容 |
|---|---|
| テーマ・目的 | 何を伝えたいか |
| 対象読者 | 初心者 / 社内向け / 経営層向け / クライアント向け など |
| トーン | ビジネス / カジュアル / 教育的 |
| 使用ツール | Genspark / Gamma / Canva / その他 |
| スライド枚数の目安 | 指定なければソラに一任 |
| 既存の参考資料 | あれば内容をソラに渡す |

すべての項目を一度にまとめて聞いてください（1往復で完了させる）。

---

### ステップ2：ソラへの依頼（アスカが実行）

アスカはソラ（`subagent_type: slide-designer`）に以下を渡して構成案の作成を依頼してください：

- ステップ1で確認した全情報
- 使用ツールに応じたフォーマット指定
- 以下の指示を必ず含める：
  - 「指定された内容のみで構成すること。存在しない機能・数値・サービス・連携先を独自に追加しないこと」
  - Gensparkを使う場合：「Genspark用プロンプトにGenspark用注意書きを必ず含めること」
  - 「スキル経由の依頼のため、ヒアリング（Step 1）はスキップして構成案の作成に直接入ること」

---

### ステップ2.5：構成案の確認（アスカがシンヤさんに提示）

ソラが構成案（スライド数・各スライドのタイトル）を返してきたら、アスカがシンヤさんに提示する。

- シンヤさんの承認を得てから、ソラに詳細原稿の作成を指示する
- 修正指示があれば、その内容をソラに伝えて構成案を修正させる

---

### ステップ3：原稿の出力（アスカがシンヤさんに渡す）

ソラが出力した原稿をシンヤさんに渡してください。

- **Gensparkの場合：**「このテキストをGensparkのプロンプト欄に貼り付けてください」と案内する
- **Gammaの場合：**「このMarkdownをGammaに貼り付けてください」と案内する
- **その他のツールの場合：** ツールに応じた貼り付け方法を案内する
