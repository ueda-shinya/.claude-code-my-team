# hearing-questioning-skills スキル

ヒアリング力・質問力を体系的に高め、相手のニーズや本音を引き出す。事業戦略スクール「知足」のフレームワーク準拠。

## なぜヒアリング・質問力が重要か

1. **仕事の成果を左右する情報収集力**：クライアントのニーズを正しく理解しないと、提案が的外れになる
2. **相手のニーズを的確に把握し、最適解を導く力**：部下の悩みを正確に理解することで、適切なアドバイスができる
3. **信頼関係を築くためのコミュニケーションスキル**：「この人は自分を理解してくれている」と思わせることで成約率が上がる

### 活用場面（3カテゴリ）
| 場面 | やること |
|---|---|
| 商談・営業 | 顧客の課題を引き出し、適切な提案をする |
| マネジメント・人材育成 | 部下の成長課題を見極め、適切な指導を行う |
| 交渉・トラブル対応 | 相手の本音を探り、合意点を見出す |

## 適用条件

- このスキルを使う場面：
  - 商談ヒアリングの設計・準備（初回商談・N=1インタビュー）
  - 部下面談・1on1の質問設計
  - 交渉・トラブル対応で相手の意図を探りたい
  - 自身のヒアリング力／質問力を点検・向上させたい
- このスキルを使わない場面：
  - 営業提案書の生成 → `proposal-builder`
  - 商談スクリプト全体の組立 → `sales-script-builder`
  - 顧客の声から失注分析・KSF抽出 → `loss-analysis-kbf-ksf`
  - コーチング全般 → `teaching-coaching-leading`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 場面 | 必須 | 営業／マネジメント／交渉／その他 | 「初回商談」 |
| 相手 | 任意 | 相手の役職・属性・状況 | 「中小企業の経営者・初対面」 |
| 目的 | 必須 | ヒアリングで何を得たいか | 「課題と予算感を把握」 |
| 制約時間 | 任意 | 想定の所要時間 | 「30分」 |
| アウトプット形式 | 任意 | 質問リスト／台本／改善提案 | 「10項目の質問リスト」 |

## 「聞く」と「聴く」の違い

| 観点 | 聞く（hear） | 聴く（listen） |
|---|---|---|
| 意味 | 単に音を耳に入れる | 意図を持って深く理解する |
| 例 | 会議でただ流す | 「この発言の背景には何があるのか？」と考えながら聞く |

ヒアリングの基本は「聴く」。

## 良いヒアリングの3要素

| 要素 | 内容 | 具体表現例 |
|---|---|---|
| **共感力** | 相手の立場に立って話を聞く | 「それは大変でしたね」と気持ちに寄り添う |
| **構造化力** | 話を整理しながら理解する | 「つまり○○ということですね？」と要約して確認する |
| **深堀り力** | 本質的な情報を引き出す | 「なぜその結論に至ったのですか？」と背景を探る |

## 質問の4種類（使い分け）

| # | 質問種別 | 用途 | 例 |
|---|---|---|---|
| 01 | **オープンクエスチョン** | 広く情報を引き出す | 「このプロジェクトの課題は何ですか？」 |
| 02 | **クローズドクエスチョン** | 具体的な確認をする | 「納期は○月○日でよろしいですか？」 |
| 03 | **なぜ質問（Why）** | 背景や本質を探る | 「なぜその方法を選んだのですか？」 |
| 04 | **仮説質問（If/Suppose）** | 相手の思考を広げる | 「もし予算が2倍になったら、どんな施策を考えますか？」 |

### 質問のフレーミング（聞き方の工夫）2型
1. **「どうすれば」型**：前向きな回答を引き出す（例：「どうすればこの問題を解決できますか？」）
2. **「何が」型**：具体性を増す（例：「何が一番の課題ですか？」）

## 実践テクニック

### 3つの「間」を意識する

| 間 | 内容 |
|---|---|
| **沈黙の間** | 相手が考える時間を与える（質問後3〜5秒待つ） |
| **相槌の間** | 適度なリアクションで安心感を与える |
| **要約の間** | 話をまとめて確認する（「つまり〜ですね？」） |

### 相手に気持ちよく話してもらうコツ
- **ミラーリング**：相手の言葉を繰り返す（例：「○○が課題なんですね」と復唱）
- **ペーシング**：話すスピードやテンションを合わせる
- **肯定的なリアクション**：「なるほど、それは興味深いですね」

## 実行ステップ

### ステップ1：場面とゴールの明確化
- 入力：場面・相手・目的
- 処理：
  1. ヒアリングの最終ゴールを1文で言語化（「30分で予算感とMA導入時期を把握する」等）
  2. 場面に応じた基本姿勢を選定（営業：課題引き出し／マネジメント：成長支援／交渉：本音と合意点）
  3. 相手の前提情報（業界・役職・既知情報）を整理
- 出力：ゴール文＋基本姿勢＋前提情報

### ステップ2：質問項目の設計（10項目フレームを基本に）
- 入力：ステップ1のゴール
- 処理：以下の標準テンプレから場面に合うものを選定し、相手に合わせてカスタマイズ

#### テンプレA：見込顧客の初回商談における課題ヒアリング（10項目）
| # | ヒアリング項目 | 質問内容 |
|---|---|---|
| 1 | 現状のビジネス課題 | 現在、事業を運営するうえで特に課題に感じていることは何ですか？ |
| 2 | 目標とKPI | 今後1年間で達成したい売上や成長目標はありますか？KPIは設定されていますか？ |
| 3 | 集客・マーケティングの現状 | 現在どのようなマーケ施策を実施していますか？効果はどう感じていますか？ |
| 4 | 既存の顧客獲得手法 | 新規顧客はどのような経路で獲得していますか？最も成果を上げている方法は？ |
| 5 | 販売・営業のフロー | 問合せから成約までの流れを教えてください。どこにボトルネックを感じていますか？ |
| 6 | 競合との差別化ポイント | 競合と比較したとき、貴社の強みや独自性はどこにありますか？ |
| 7 | 広告・プロモーションの予算感 | マーケや広告にかける予算はどの程度ですか？費用対効果の考え方を教えてください |
| 8 | 社内のリソース状況 | 社内体制はどうですか？専任の担当者はいますか？ |
| 9 | これまでの施策とその結果 | 過去の施策で成功・失敗したものはありますか？理由は何だと考えていますか？ |
| 10 | 最も解決したい優先課題 | もし1つだけ課題を解決できるとしたら、どれを優先しますか？なぜですか？ |

#### テンプレB：既存顧客のN=1インタビュー（事例ヒアリング・10項目）
| # | ヒアリング項目 | 質問内容 |
|---|---|---|
| 1 | 導入前の課題や悩み | サービス導入前、どのような課題や悩みを抱えていましたか？ |
| 2 | サービス導入の決め手 | 当社のサービスを選んだ理由や決め手は何でしたか？ |
| 3 | 導入前と導入後の変化 | 導入前と比べて、具体的にどのような変化がありましたか？ |
| 4 | 最も効果を感じたポイント | 特に良かったと感じるポイントはどこですか？ |
| 5 | 具体的な成果や数字 | 売上・業務効率など、数字で表せる変化があれば教えてください |
| 6 | 活用している機能や施策 | どのように活用していますか？特に役立っている機能は？ |
| 7 | 導入時のハードルや懸念点 | 導入を決める際に不安だったことは？ |
| 8 | 社内やチームの反応 | 社内メンバーの反応はどうでしたか？特にどんな人が喜んでいましたか？ |
| 9 | 他社サービスとの違い | 他のサービスも検討しましたか？比較してどう違いましたか？ |
| 10 | これから期待すること | 今後さらに活用するうえで、期待することや改善してほしいことは？ |

- 出力：カスタマイズ済み質問リスト（10項目を基本、状況に応じて増減）

### ステップ3：質問の種類と順序を最適化
- 入力：質問リスト
- 処理：
  1. 各質問に「オープン／クローズド／なぜ／仮説」のラベルを付ける
  2. 序盤はオープン中心（緊張をほぐし広く情報収集）
  3. 中盤はなぜ質問で深掘り
  4. 終盤はクローズドで確認＋仮説質問で次のアクションを引き出す
  5. 「3つの間」のタイミング（要約の間を入れる箇所）を3〜5箇所マーキング
- 出力：質問の種類・順序付きヒアリング台本

### ステップ4：実践テクニックの組込み
- 入力：ヒアリング台本
- 処理：以下のテクニックを台本に組込み
  - ミラーリング挿入箇所（質問3〜5回ごと）
  - ペーシング指示（相手のスピードに合わせるノート）
  - 肯定的リアクションのテンプレ（5パターン）
  - 沈黙の間の指示（質問後3〜5秒待つ）
- 出力：実践指示付き完成台本

### ステップ5：振り返りと改善計画
- 入力：実施後のヒアリング結果
- 処理：以下の3習慣で改善を回す
  1. **毎日の会話で意識**：「相手の話をしっかり聴く」を最優先／自分の話ばかりせず相手から引き出す
  2. **成功した質問・ヒアリングの振り返り**：その日の会話を振り返り、良かった点・改善点を分析（「相手が話しやすくなった瞬間はどこか？」「深堀りできた質問は何か？」）
  3. **フィードバックを受ける**：上司や同僚に「自分のヒアリング力・質問力」について率直な意見をもらう／ロールプレイで具体的な改善点を指摘してもらう
- 出力：振り返りログ＋次回改善ポイント

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 初回商談で30分しかない | 10項目から優先度上位5〜6項目に絞る | 全項目を消化しようとして表面的になるのを防ぐ |
| 相手が経営者で時間に厳しい | 結論先出しで質問の意図を伝える | 「○○を把握したいので3つ質問させてください」と前置き |
| 質問しても短い回答しか返ってこない | クローズド→オープンに切替＋仮説質問を投入 | 「もし予算が無制限なら？」で思考を広げる |
| 相手が話しすぎて要点が見えない | 「要約の間」を強めに入れる | 「つまり○○ですね？」で構造化 |
| 沈黙が気まずくて質問を重ねる | 沈黙の間を3〜5秒は待つ | 沈黙は相手が考えている時間 |
| 相手の本音が引き出せない | 「なぜ質問」を3回繰り返す＋立場をずらした仮説質問 | 表層回答の奥を探る（→ `critical-thinking` の3視点ずらし併用） |
| 顧客満足度を測りたい（既存顧客） | テンプレB（N=1インタビュー10項目）を使用 | 顧客の声を構造化して取得 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| クローズド質問ばかりで情報が集まらない | オープン主体で序盤を構成 | YES/NOしか返ってこない |
| 質問を矢継ぎ早に投げる | 沈黙の間（3〜5秒）を必ず待つ | 相手の思考時間を奪う |
| 自分の話を長くする | 自分の話ではなく、相手から引き出す | ヒアリングは相手主役 |
| 表層回答で満足する | 「なぜ」を3回繰り返す | 本質情報に到達できない |
| 要約の間を省く | 構造化のために要約を必ず入れる | 認識ズレを後で修正するコストが大きい |
| 全質問を消化しようとする | 時間厳守で優先順位付けして絞る | 表面的になり成約率が下がる |
| ロールプレイ・振り返りをしない | 3習慣（意識／振り返り／FB）で継続改善 | 質問力は実践で磨かれる |

## 連携スキル

- `proposal-builder`：ヒアリング結果を提案書ストーリーに組み立てる
- `loss-analysis-kbf-ksf`：失注顧客や受注顧客のN=1インタビューでKBF/KSFを抽出する
- `sales-script-builder`：商談全体の台本に組み込む
- `teaching-coaching-leading`：マネジメント場面でコーチング質問を併用
- `critical-thinking`：相手の発言を「本当に？／だから何？」で批判的に検証
- `persona-design`：ヒアリング結果からペルソナを構築

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：ヒアリング項目が0件で組み立てられない場合（要再ヒアリング）
- `[PARTIAL:N成功/M失敗]`：質問リスト10項目のうち一部しか作成できなかった場合
- `[SKIP]`：場面・目的が不明確で台本作成を保留した場合（前提不足）

※外部API依存なしのため `[FAIL]` は通常発生しない。前提不足は `[SKIP]` で扱う。
