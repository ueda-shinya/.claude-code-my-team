# agent-study スキル

エージェントにドキュメントを「読ませて」、その知識を永続的なスキルとして身に付けさせる。

## ナレッジ素材の保管場所

読み込んだドキュメントは `~/.claude/knowledge/<カテゴリ>/` 配下に保管する規約とする。

| カテゴリ例 | パス |
|---|---|
| マーケティング | `~/.claude/knowledge/marketing/` |
| ツール・MCP | `~/.claude/knowledge/freee-mcp/` 等 |
| 業務フロー | `~/.claude/knowledge/workflow/` |
| その他 | `~/.claude/knowledge/<任意>/` |

- `reports/` にある学習素材は `knowledge/` へ移動して整理する
- 同じファイルを再度読ませるときは `knowledge/` 配下のパスを指定する

## 実行手順

### ステップ1：対象を特定する

ユーザーの指示から以下を読み取る：

- **ドキュメント**：ファイル名・パスの断片があれば Glob で検索して特定する
- **対象エージェント**：名前・ニックネームからマッピングして解決する

どちらか一方でも特定できない場合のみユーザーに確認する。両方揃っていれば確認不要で進む。

### ステップ2：両ファイルを読み込む

Read ツールで以下を同時に読み込む：
1. 対象ドキュメント（全文）
2. 対象エージェントファイル（全文）

### ステップ3：知識を抽出・構造化する

エージェントの**役割と業務**に照らして、ドキュメントから価値ある知識を選び出す。
すべてを転記するのではなく「このエージェントが判断・実行する場面で実際に使える情報」に絞ること。

抽出する知識の種類（優先順）：

1. **定義・概念**：正確な言葉・用語の意味
2. **フレームワーク・プロセス**：手順・流れ・適用順序
3. **判断基準・優先ロジック**：何を先にするか・なぜそうするか
4. **ケーススタディ・具体例**：状況→施策→KPIの組み合わせ
5. **注意点・よくある誤解**：やりがちなミスと正しい対処

### ステップ4：エージェントファイルを更新する

エージェントファイルの末尾に以下のセクションを追記（または上書き）する：

```markdown
---

## 〇〇知識ベース（ドキュメントのタイトル より）

（構造化した知識をここに記載）
```

**ルール：**
- 既に `## 〇〇知識ベース` セクションがある場合は、そのセクション全体を今回の内容で**上書き**する（重複・追記はしない）
- エージェントのキャラクター設定・役割定義・既存の指示は一切変更しない
- 見出し・表・箇条書きを活用して「参照しやすい形」にまとめる
- 長大なドキュメントでも知識ベースは**500行以内**に収める（要約・圧縮する）

### ステップ5：完了報告

以下の形式で報告する：

```
## agent-study 完了

**対象エージェント**：〇〇（ファイルパス）
**読み込んだドキュメント**：〇〇

### 追加した知識（主要ポイント）
1. 〇〇
2. 〇〇
3. 〇〇
（3〜5項目。エージェントの業務に最も直結するものを選ぶ）

### 更新の概要
- （追加 or 上書き）した内容を一行で
```
