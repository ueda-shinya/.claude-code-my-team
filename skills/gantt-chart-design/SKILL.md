# ガントチャート設計スキル

タスクを横軸の時間スケール上に視覚化し、プロジェクトの進行状況を一目で把握できるよう管理するツール。プロジェクトの透明性向上、チーム連携、依存関係の見える化を実現。

## 適用条件

- 新規プロジェクトのスケジュール設計
- 中長期戦略のロードマップ可視化
- 複数タスク・複数担当者の進捗管理
- スケジュール遅延の早期検知

## 入力

| 項目 | 必須 | 説明 |
|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー |
| プロジェクト目標 | 必須 | 達成すべきゴール |
| 期間 | 必須 | プロジェクト全体の開始〜終了 |
| タスクリスト | 任意 | 既知のタスクがあれば |

## 実行ステップ

### ステップ1: ガントチャートの基本構成4要素

| 要素 | 内容 |
|---|---|
| **1. 縦軸（タスク一覧）** | プロジェクトの作業内容をリスト形式で表示。各タスクは具体的な作業を表す |
| **2. 横軸（時間スケール）** | 日／週／月／半期／年で設定。プロジェクト期間を細かく管理可能 |
| **3. バー（タスクの進捗状況）** | 各タスクの開始日と終了日を棒グラフで視覚化。進行中・完了済みを色分け |
| **4. 依存関係（線や矢印）** | タスク間の関係性を表現。1タスクの遅延が他に与える影響を把握 |

### ステップ2: 活用5ステップ

1. **プロジェクトの目標を明確化**：プロジェクトの目的・成果物を設定
2. **タスクの洗い出し**：必要な作業をリストアップし開始日・終了日を設定
3. **依存関係の特定**：タスク間の優先順位や依存関係を明確化
4. **ガントチャート作成**：ツール（Excel／Google Sheets／Asana／Notion等）でタスクを時間軸に配置
5. **進捗のモニタリング**：定期的に見直し、進捗状況やスケジュール遅れを確認

### ステップ4: WBS（Work Breakdown Structure）番号付与ルール

- 階層構造で番号付与：1, 1.1, 1.1.1...
- 大項目（カテゴリ）：1, 2, 3...
- 中項目（サブカテゴリ）：1.1, 1.2...
- 小項目（個別タスク）：1.1.1, 1.1.2...

### ステップ5: 運用ベストプラクティス

- **目標達成のために必要な作業は細かく分解しリストアップ**
- **定期的に進捗状況を確認し、必要に応じて計画を修正**
- **プロジェクト終了後、計画と実績を比較し改善点を洗い出す**

## 判断基準

| 状況 | 判断 |
|---|---|
| タスクが10個未満 | 簡易ガントチャート（時間軸＋色分けバーのみ） |
| タスクが100個以上 | WBS3階層で構造化、ツール（Asana／Microsoft Project）必須 |
| 依存関係が多数 | 線や矢印でクリティカルパス可視化 |
| 進捗未更新 | 週次レビューを必須化 |
| 1人作業 | ガントチャートよりToDoリストで十分 |
