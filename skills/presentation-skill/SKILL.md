# presentation-skill スキル

プレゼンテーション能力 = 自分の意見や情報を相手にわかりやすく伝え、共感や行動を引き出すスキル。事業戦略スクール「知足」のフレームワーク準拠。

## 必要性（活用場面）

| カテゴリ | 例 |
|---|---|
| ビジネスシーン | 商品提案・プロジェクト報告・交渉 |
| 教育・研究 | 学会発表・セミナー講義 |
| 日常生活 | 説明・交渉・自己PR |

## 適用条件

- このスキルを使う場面：
  - プレゼン本番に向けて構成を組み立てたい
  - PREP法で結論先出しの説明を作りたい
  - 自身のプレゼン能力を5観点で評価したい
  - スライドの論理構成を整えたい
- このスキルを使わない場面：
  - スライドのビジュアル設計（配色・余白・図版） → `lp-design-system`／デザイナー
  - 営業提案書のストーリー＋コピー一気通貫 → `proposal-builder`
  - 営業スライド骨子（Genspark／Gamma連携） → `sales-slide-builder`
  - 結論先出しトーク台本（営業向け） → `conclusion-first-talk`
  - スライドプロンプト for Claude Design → `claude-design-prompt`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| プレゼンテーマ | 必須 | 何を伝えたいか | 「新サービス導入提案」 |
| 聴衆 | 必須 | 誰に向けて | 「クライアント経営層3名」 |
| 制約時間 | 必須 | 持ち時間 | 「15分＋質疑5分」 |
| ゴール | 必須 | 終了後に相手に取ってほしいアクション | 「導入の社内検討開始の合意」 |
| 既存資料 | 任意 | 流用したいスライドや情報 | 「会社概要・実績データ」 |

## プレゼンテーション能力の4要素

| 要素 | 内容 | 例・コツ |
|---|---|---|
| **コンテンツ設計スキル** | 論理的構成やストーリーテリング | PREP法（Point→Reason→Example→Point） |
| **ビジュアルデザインスキル** | スライド作成・データ可視化 | 配色は3色以内／フォント24pt以上 |
| **言語・非言語スキル** | 声の抑揚・ボディランゲージ・アイコンタクト | 強調したい部分で声のトーンを上げる |
| **聴衆対応スキル** | 聴衆の期待を把握し共感を得る | リサーチによる聴衆ニーズ分析 |

## プレゼンテーションの基本構成（3パート）

| パート | 役割 | 方法 |
|---|---|---|
| **オープニング** | 聴衆の注意を引く工夫 | ストーリー／質問／衝撃データを活用 |
| **メインボディ** | 論理的な流れに沿った内容展開 | **PREP法**：結論→理由→具体例→結論で話す |
| **クロージング** | 明確な結論を示し次のアクションを促す | まとめスライド／感謝の言葉／次回ステップ提示 |

## PREP法の構成（論理性フレーム）

PREP法 = はじめに要点（結論・主張）を伝えてから、結論にいたった理由を説明し、理由に説得力を持たせる事例やデータを提示し、最後にもう一度要点を述べる構成。

| # | フェーズ | 内容 | 例 |
|---|---|---|---|
| 01 | **Point（要点）** | 結論・主張を冒頭で提示 | 「上司に報告するときはPREP法が必要だ」 |
| 02 | **Reason（理由）** | 結論に至った理由を説明 | 「なぜならPREP法を用いると分かりやすく聞き手にストレスのかからない説明になるからだ」 |
| 03 | **Example（具体例）** | 理由を裏付けるデータ・事例 | 「要点から話すと聞き手は趣旨を理解してから説明を聞ける。理由と具体例を述べれば主張に説得力が生まれる」 |
| 04 | **Point（要点・再確認）** | 結論を再度提示し締める | 「したがって上司に報告するときはPREP法を意識すべきだ」 |

## プレゼン能力を高める4つの方法

| # | 方法 | 内容 |
|---|---|---|
| 01 | **構成力を磨く** | フレームワーク（PREP法）を活用し、1スライド1メッセージを意識 |
| 02 | **スライド作成のコツ** | シンプルなデザインと視覚効果を重視。データはグラフで表現、余白を活用 |
| 03 | **言語・非言語スキルの向上** | 練習と録画で改善ポイントを確認。アイコンタクトやジェスチャーを意識的に活用 |
| 04 | **聴衆の反応を読む** | リアクションに合わせて内容やトーンを柔軟に調整 |

## 実行ステップ

### ステップ1：聴衆分析と最終ゴール定義
- 入力：プレゼンテーマ、聴衆、制約時間、ゴール
- 処理：
  1. 聴衆の属性・前提知識・関心事・反対意見を整理
  2. 終了直後に相手に取ってほしいアクション（CTA）を1文で言語化
  3. 制約時間から逆算してスライド枚数の目安を出す（1分1枚ペース）
- 出力：聴衆プロファイル＋CTA文＋スライド枚数目安

### ステップ2：ストーリー骨子の設計（3パート）
- 入力：ステップ1の出力
- 処理：
  1. **オープニング**：聴衆の注意を引く起点を選定
     - パターンA：衝撃データ（「業界の70%が...」）
     - パターンB：問いかけ（「皆さんはこういう経験ありませんか？」）
     - パターンC：ストーリー（「ある顧客が...」）
  2. **メインボディ**：PREP法で1〜3トピック分構成
     - 各トピックを Point→Reason→Example→Point で組む
     - 1トピック当たり3〜5分の目安
  3. **クロージング**：再びゴール（CTA）を明示し、次回ステップを提示
- 出力：3パート構成のストーリー骨子

### ステップ3：PREP法でメインボディを詳細化
- 入力：ストーリー骨子
- 処理：トピックごとに以下を埋める
  - **P (Point)**：結論を1文で
  - **R (Reason)**：理由を3つまで
  - **E (Example)**：理由を裏付けるデータ／事例／顧客の声
  - **P (Point)**：結論を再度1文で
- 出力：トピックごとのPREP表

### ステップ4：1スライド1メッセージ化
- 入力：PREP表
- 処理：
  1. 各PREP要素を1〜2スライドにマッピング
  2. 各スライドのメインメッセージ（タイトル）を1文で書く
  3. データはグラフ化、テキストは箇条書き3項目以内
  4. 配色3色以内・フォント24pt以上のルール遵守
- 出力：スライド一覧＋各スライドのメッセージ

### ステップ5：言語・非言語の演出指示
- 入力：スライド一覧
- 処理：各スライドに以下の演出メモを付加
  - **強調箇所**：声のトーンを上げる／間を取る位置
  - **アイコンタクト**：聴衆を見るタイミング
  - **ジェスチャー**：強調する身振り
  - **想定質疑**：聴衆から出そうな質問と回答案を3つ準備
- 出力：演出付きスライド一覧（プレゼンノート）

### ステップ6：自己評価と改善
- 入力：完成したプレゼン
- 処理：以下5観点で自己評価（各5段階）
  1. **コンテンツ設計**：プレゼン内容が論理的で簡潔か？
  2. **デザインスキル**：スライドが見やすく、データが効果的に視覚化されているか？
  3. **話し方**：声のトーンやジェスチャーが適切か？（録画チェック推奨）
  4. **聴衆対応**：聴衆のニーズや反応に対応できているか？
  5. **定期練習**：定期的に練習とフィードバックを実施しているか？
- 出力：自己評価スコア＋改善計画

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 持ち時間15分以下 | 1トピック＋PREP1セットに絞る | 詰め込むと結論が伝わらない |
| 持ち時間30分以上 | 3トピック＋各PREPで構成 | 中だるみを防ぐためトピック切替で緊張感を保つ |
| 聴衆が経営層 | オープニングは数字／ROI起点 | 抽象論より具体ROIに反応 |
| 聴衆が現場担当 | オープニングはストーリー／問いかけ起点 | 共感から入る |
| 1スライドに3要素以上載っている | 分割する／削る | 1スライド1メッセージ違反 |
| データなしで主張だけ | Example部分にデータを必ず添える | 説得力不足 |
| 質疑想定が3つ未満 | ステップ5に戻り想定を増やす | 本番の質疑で詰まると印象が下がる |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 結論を最後まで言わない | PREP法で冒頭に結論 | 聴衆が理解する前に時間切れ |
| 1スライドに情報を詰め込む | 1スライド1メッセージ | 印象が残らない |
| データなしで主張だけ | Example部分にデータ必須 | 説得力不足 |
| 配色がカラフル | 3色以内 | ノイズが多くて主役が消える |
| 聴衆の反応を見ない | アイコンタクトと反応観察 | 一方通行のプレゼンになる |
| 練習なしで本番 | 録画チェック＋FB | 言語・非言語スキルは練習で改善 |
| 想定質疑なし | 必ず3問以上準備 | 質疑で崩れると印象台無し |
| クロージングで突然終わる | CTA＋次回ステップを必ず提示 | 行動を促せない |

## 5状態出力契約

外部API・サブプロセス・外部設定依存を持たないため、CLAUDE.md「5状態出力契約」の対象判定基準には該当しない。ただし以下のマーカーを使うこと：

- `[OK:0件]`：聴衆ニーズが不明で構成が組めない場合（要再ヒアリング）
- `[PARTIAL:N成功/M失敗]`：3パートのうち一部のみ完成した場合
- `[SKIP]`：持ち時間・ゴールが不明確で設計を保留した場合（前提不足）

※外部API依存なしのため `[FAIL]` は通常発生しない。前提不足は `[SKIP]` で扱う。
