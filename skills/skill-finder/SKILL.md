# Skill Finder

外部で公開されているClaude Codeスキル、テンプレート、MCPサーバーを検索し、評価・導入するためのワークフロー。

全6ステップで構成される。各ステップの完了後、必ずシンヤさんの確認を取ってから次に進むこと（勝手に先へ進めない）。

---

## Step 1: ヒアリング

シンヤさんから「こういう機能がほしい」「こんなスキルないかな」という要望を受け取り、要件を明確にするため以下を確認してください。すべてを1回のやりとりで聞くこと。

**必須確認事項:**
1. 具体的にどんな機能が必要か（ユースケースを1-2個）
2. 制約条件（動作環境、既存スキルやMCPとの関係、使用言語など）
3. 優先度の整理（必須機能 vs あれば嬉しい機能）

**任意（精度向上のため）:**
4. 出力形式の指定があるか
5. 絶対にやってはいけないこと（制約）があるか

ヒアリング結果を整理し、「この内容でリサーチします」とシンヤさんに確認を取る。

---

## Step 2: リサーチ・候補リストアップ

リサーチはMio（researcher エージェント）に委任してください。

Mioへの依頼時に以下を伝えること:
- 目的: 「Solution（最短）」
- Step 1 で整理した要件
- 検索対象リソース（**必須・推奨は原則全件、任意は時間が許せば**。★印は必ず含める）

### 検索対象リソース

**必須検索対象**: 以下の ★ 印は、スキル検索時に必ず含めること（スキル探索時の取りこぼし防止のため 2026-04-18 シンヤさん指示で固定化）。

| リソース名 | URL | 種別 | 優先度 |
|---|---|---|---|
| ★ **skills.sh**（Vercel Labs運営の統合ディレクトリ） | https://skills.sh | 公式・統合マーケットプレイス（91K+スキル、インストール数順ランキング・リスク評価付き） | **最優先** |
| ★ **anthropics/skills**（Anthropic公式） | https://github.com/anthropics/skills | 公式スキル集（frontend-design, skill-creator, mcp-builder等） | **必須** |
| ★ **vercel-labs/agent-skills** | https://github.com/vercel-labs/agent-skills | Vercel Labs公式スキル集（`npx skills add vercel-labs/agent-skills -l` で一覧取得可） | **必須** |
| VoltAgent/awesome-agent-skills | https://github.com/VoltAgent/awesome-agent-skills | 1,234+スキルのキュレーションカタログ（22K stars） | 推奨 |
| awesome-claude-code (hesreallyhim) | https://github.com/hesreallyhim/awesome-claude-code | スキル・ツール集 | 推奨 |
| awesome-claude-skills (travisvn) | https://github.com/travisvn/awesome-claude-skills | スキル集 | 推奨 |

| claude-code-templates (davila7) | https://github.com/davila7/claude-code-templates | テンプレート集 | 任意 |
| MCP Registry（公式） | https://registry.modelcontextprotocol.io/ | MCPサーバー | 必須（MCP探索時） |
| PulseMCP | https://www.pulsemcp.com/servers | MCPサーバー | 推奨（MCP探索時） |

### 検索順序の推奨

1. **skills.sh** でまず検索（一覧性・ランキング・セキュリティ評価が揃うため）
2. anthropics/skills と vercel-labs/agent-skills を個別確認（公式提供の有無）
3. 見つからなければ awesome-* 系カタログで個人作スキルを探索
4. MCPサーバーの場合は MCP Registry → PulseMCP の順

### 候補リストの形式

3〜5件の候補をリストアップし、各候補に以下を含めること:

```
1. **名前**: XXX
   - URL: https://...
   - 概要: 何ができるか（1-2文）
   - ライセンス: MIT / Apache 2.0 / 不明 など
   - 備考: 特筆事項があれば
```

### 候補が0件の場合

Mioから候補0件の報告を受けた場合、シンヤさんに以下の選択肢を提示する:
- **検索条件を緩和して再リサーチ** → 緩和案を提示し、承認後にMioへ再依頼
- **Step 1 に戻って要件を再ヒアリング** → 要件自体を見直す

候補が見つかった場合は、候補リストをシンヤさんに提示し、Step 3 に進んでよいか確認を取る。

## Step 3: 比較・選択

Asukaが候補を比較表にまとめ、シンヤさんに提示する。

### 比較表の形式

| 項目 | 候補A | 候補B | 候補C |
|---|---|---|---|
| 機能の一致度 | ... | ... | ... |
| ライセンス | ... | ... | ... |
| メンテナンス状況 | ... | ... | ... |
| 導入の手軽さ | ... | ... | ... |
| 既存環境との相性 | ... | ... | ... |

比較表をもとにシンヤさんに選んでもらう。選択された候補を「第1候補」、次点を「第2候補」として控えておく（Step 5 で第1候補に問題があった場合に使用）。

---

## Step 4: 中身確認

選択されたスキル/テンプレート/MCPサーバーの実際のファイル内容を取得し、以下の観点でチェックする。

**セキュリティチェックはSakura（code-reviewer エージェント）に委任してください。** これは導入前の外部コードの事前監査であり、以下の観点を重点的に確認する。

### チェック観点

1. **セキュリティリスク**（Sakuraが重点チェック）
   - **悪意あるコード**: 難読化されたコード、不審な動的評価（eval等）、バックドアの有無
   - **過剰な権限要求**: 必要以上のファイル操作権限、システム設定の変更、管理者権限の要求
   - **依存先の信頼性**: 依存パッケージの評判・メンテナンス状況、既知の脆弱性（CVE）
   - **外部送信先**: どこに何を送っているか、認証情報やユーザーデータの外部送信がないか
   - 認証情報の扱い（APIキー、トークンなどがハードコードされていないか）
   - 任意コード実行のリスク

2. **既存環境との互換性**
   - 既存のスキル・MCPサーバーとの競合がないか
   - 必要なツール・依存関係が満たされているか
   - 動作環境の制約（Mac/Windows）

3. **依存関係**
   - 外部パッケージの依存
   - Node.js / Python などのランタイム要件

チェック結果をシンヤさんに報告し、次のステップに進むか判断を仰ぐ。

---

## Step 5: 修正判断・実施

Step 4 の結果に基づいて判断する。

### そのまま使える場合
→ シンヤさんに「修正不要です。Step 6（テスト・導入）に進みます」と伝え、確認後 Step 6 へ。

### 修正が必要な場合

1. 修正内容をシンヤさんに提示する（何を、なぜ修正するか）
2. シンヤさんの承認を得る
3. **修正の実装はShu（backend-engineer エージェント）に委任してください**
4. 修正後、再度 Sakura にセキュリティチェックを依頼する
5. **再チェックでもNGだった場合**、シンヤさんに以下の選択肢を提示する:
   - **次点候補に切り替え** → 第2候補で Step 4 から再開
   - **再修正** → 修正内容を再提示し、承認後にShuへ再委任
   - **中止** → ワークフロー終了
6. チェックOKの場合、結果をシンヤさんに報告し、Step 6 に進むか確認する

---

## Step 6: テスト・導入

### テスト実施

Shu（backend-engineer エージェント）に動作テストを委任してください。

1. 導入先にファイルを配置する（スキルなら `~/.claude/skills/`、MCPならsettingsに追加など）
2. Shuが実際に動作テストを行う
3. 結果をシンヤさんに報告する

### 問題があった場合

シンヤさんに以下の選択肢を提示する:
- **修正して再テスト** → Step 5 に戻る
- **次点の候補に切り替え** → 第2候補で Step 4 から再開
- **中止** → ワークフロー終了

### 問題がなかった場合

以下を報告して完了:
- 導入したスキル/MCP の名前と概要
- 配置先のファイルパス
- 起動方法（どう呼び出すか）
- 注意事項があれば

---

## 制約事項

- 各ステップの間で必ずシンヤさんの確認を挟むこと。勝手に次のステップに進まない
- リサーチはMio、セキュリティチェック（外部コードの事前監査）はSakura、実装・修正・動作テストはShuにそれぞれ委任する。Asukaが直接コードを書かない
- 比較表の作成・提示はAsukaが行う
- ライセンスが不明または制限的なものは、その旨を明示してシンヤさんの判断を仰ぐ
- MCPサーバーの導入時は、既存のMCP設定と競合しないか必ず確認する
