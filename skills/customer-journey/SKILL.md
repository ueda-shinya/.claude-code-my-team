# カスタマージャーニー設計スキル

顧客が製品やサービスを知り、購入し、利用するまでの一連の行動・感情・課題を時系列で可視化し、各タッチポイントの最適化ポイントを特定する。マーケ・営業・CSの全顧客接点を一気通貫で設計するツール。

## 適用条件

- このスキルを使う場面：
  - 新規LP・広告キャンペーンの設計前
  - 顧客接点（タッチポイント）の最適化検討
  - 営業フローの再設計
  - 離脱率の高い段階を特定したい
- このスキルを使わない場面：
  - ペルソナのみの設計 → `persona-design`
  - マーケ・営業のフロー整備 → マーケ営業フロー策定スキル
  - PDCA運用 → `pdca-cycle`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガー語 | 「カスタマージャーニー作って」 |
| 商材 | 必須 | 製品・サービス | 「BtoB SaaS」「BtoCコスメEC」 |
| ペルソナ | 必須 | 設計済みペルソナ | 「情シス担当者・100人規模スイッチング検討」 |
| BtoB/BtoC | 必須 | 商材タイプ | BtoB / BtoC |
| 既知のタッチポイント | 任意 | 把握しているチャネル | SNS／検索広告／LP／問合せフォーム |

## 実行ステップ

### ステップ1：ペルソナを定義（Step 1）

- 入力：商材、想定顧客像
- 処理：`persona-design` スキルで既にペルソナが作られていれば再利用。なければ `persona-design` を先行起動する
- 出力：ペルソナ（氏名・属性4項目以上）

### ステップ2：顧客のタッチポイントを洗い出し（Step 2）

- 入力：ペルソナ、商材
- 処理：
  1. 顧客がブランド・製品と接触する全場面を列挙する
  2. オンライン／オフラインを問わず網羅する
  3. 例：SNS広告→商品ページ→カスタマーサポート→購入
  4. チャネル別に整理（典型例：SNS／広告／LP・サービスサイト／アクション）
- 出力：タッチポイントリスト（最低3〜5チャネル）

### ステップ3：各タッチポイントでの顧客体験を分析（Step 3）

- 入力：タッチポイントリスト、ペルソナ
- 処理：各接点で以下2問に答える：
  - 顧客は何を考え、感じているか？
  - どのタッチポイントで課題や不満が発生しているか？
- 出力：接点別の感情・課題リスト

### ステップ4：ジャーニーマップを作成（Step 4）

- 入力：ペルソナ、タッチポイント、感情・課題
- 処理：以下5フェーズ × チャネル別の二次元表で可視化する：

#### フェーズ（横軸）

| 5フェーズ | 内容 |
|---|---|
| **① 流入／情報収集** | 健康関連・流行り・お買い得情報の収集（顧客固有の関心テーマ） |
| **② 認知／興味・比較検討** | サービスを知る・興味を持つ・他社と比較する |
| **③ 問合せ・トライアル利用** | 資料請求／トライアル申込／問合せ |
| **④ 商談～成約** | 営業接触／社内検討／見積／決裁 |
| **⑤ 成約・CS・アップセル** | 利用開始／継続／追加購入 |

#### チャネル（縦軸）

| チャネル | 配置内容 |
|---|---|
| **SNS等** | フェーズ別の顧客行動（例：「SNSの発信でお役立ち資料を見る」） |
| **広告** | フェーズ別の顧客行動（例：「Facebook広告でウェビナーをみかける」） |
| **LP/サービスサイト** | フェーズ別の顧客行動（例：「LP内で価格・特徴・他社比較を確認」） |
| **アクション** | 各フェーズの最終アクション（例：「ウェビナー参加／お役立ち資料DL」） |

各セルに以下3要素を記入：
- 行動（具体アクション）
- 感情（喜び・不安・期待）
- 課題（このフェーズで阻害される要因）

### ステップ5：改善ポイントを特定（Step 5）

- 入力：ジャーニーマップ
- 処理：
  1. 各タッチポイントの課題から優先順位を付ける（影響度×実現可能性）
  2. 上位3〜5項目に対し、具体的な改善策を提示する
  3. 改善策を「短期（1か月以内）／中期（3か月以内）／長期（半年以上）」に分類
- 出力：優先順位付き改善策リスト

### ステップ6：5つのよくある問題チェック

以下5点のいずれかに該当していないか確認する：

| 問題 | 注意点 |
|---|---|
| **ペルソナが曖昧** | データを基に詳細なペルソナを作成（→ `persona-design`） |
| **タッチポイントの不足** | 顧客の行動データ・インタビューで全接点を網羅 |
| **感情や課題の見落とし** | 各接点で感情・課題を必ず記載する |
| **実行可能な改善策がない** | 優先順位をつけ、実行可能な施策を考える |
| **見直し不足** | 定期的に更新し、データやトレンドを反映する |

### ステップ7：完了報告

ステップ1〜6を以下フォーマットで一括出力する。

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| ペルソナが未定義 | `persona-design` を先行起動 | ペルソナなしでは設計不能 |
| BtoBとBtoCで5フェーズが違う | BtoBは「商談～成約」を必ず追加。BtoCは「リピート」フェーズを追加 | 業態差を反映 |
| タッチポイントが2チャネルしかない | 最低3〜5チャネル探索 | 網羅性不足 |
| 改善策を5項目以上挙げる | 優先順位3段階で振り分け | 全方位施策は失敗 |
| 各セルに「行動」だけ書く | 必ず行動・感情・課題の3要素 | 行動だけでは改善ポイント不明 |
| 1回作って放置 | 四半期ごとに見直し | 顧客行動は変化する |

## 各タッチポイントの感情・課題

### フェーズ① 流入／情報収集
- **感情**: 課題感はあるが具体策が見えていない／情報収集モード
- **課題**: 自社課題を言語化できていない・どこで何を探すか不明

### フェーズ② 認知／興味・比較検討
- **感情**: サービスの良し悪しがわからない／不安
- **課題**: 比較情報が散在／自社規模の事例が見つからない

### フェーズ③ 問合せ・トライアル
- **感情**: 期待と懸念が混在
- **課題**: トライアル設定が複雑／使いこなしの見通し不明

### フェーズ④ 商談～成約
- **感情**: 決裁取得の負担／予算根拠不安
- **課題**: 決裁者向け資料不足／ROI算出根拠不足

### フェーズ⑤ 成約・CS・アップセル
- **感情**: 使いこなせるか／費用対効果が見合うか
- **課題**: オンボーディング情報不足／追加機能の認知不足

## 改善策（優先順位付き）
| 優先 | 課題 | 改善策 | 期間 |
|---|---|---|---|
| 最優先 | 比較情報が散在 | 業界比較ホワイトペーパー作成・LP掲載 | 短期（1か月） |
| 次 | 決裁者向け資料不足 | 経営層向け1枚ROI試算ツール提供 | 中期（3か月） |
| 後 | オンボーディング情報不足 | 導入後30日サクセスプラン整備 | 長期（半年） |

## 5つのよくある問題チェック
- [x] ペルソナの具体性
- [x] タッチポイントの網羅性
- [x] 感情・課題の記載
- [x] 実行可能な改善策
- [x] 定期見直しの計画（四半期ごと推奨）

- マーケ営業フロー整備 → マーケ営業フロー策定スキル
- 運用フェーズ → `pdca-cycle`

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ペルソナを曖昧なまま設計 | 必ず `persona-design` で具体化してから | ペルソナなしでは精度低下 |
| 1チャネルだけで設計 | 最低3〜5チャネル網羅 | 顧客は複数チャネルを使う |
| 各セルに「行動」だけ書く | 行動＋感情＋課題の3要素必須 | 改善ポイントが見えない |
| 改善策を全部「最優先」 | 3段階の優先順位 | 全方位は失敗 |
| 5フェーズすべてSNSで埋める | 各フェーズに合うチャネルを使う | 顧客行動と乖離 |
| 1回作って放置 | 四半期ごとに見直し | 顧客行動は変化する |

## 連携スキル

- **前段**：`persona-design`（ペルソナ）、`stp-analysis`（ターゲット）、`value-proposition`（価値提案）
- **後段**：マーケ・営業フロー策定スキル、施策設計スキル、`pdca-cycle`（運用）

## 5状態出力契約

外部API・サブプロセス・外部設定依存なし。運用上のマーカー：
- `[FAIL]`：必須入力（ペルソナ・商材）が欠落
- `[OK:0件]`：データ不足で課題特定不能（要顧客調査）
