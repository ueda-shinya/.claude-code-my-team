# /sns-post：SNS投稿制作スキル

SNS投稿をコト → humanizer → 品質チェック → レンのレビューの4段階で制作し、投稿可能な状態まで仕上げる統合スキル。

## 起動条件

`/sns-post` で起動する。以下の情報を受け取る（不足分はシンヤさんに確認する）。

### 必須入力

| 項目 | 説明 | 例 |
|---|---|---|
| テーマ | 投稿の主題 | 「DXの第一歩は紙とペン」 |
| プラットフォーム | `X` / `Threads` / `両方` | `両方` |

### 任意入力（省略時はデフォルト）

| 項目 | デフォルト | 説明 |
|---|---|---|
| バズの型 | コトが最適な型を選択 | 8つの型から指定（例：「逆説型」「失敗談→教訓型」） |
| フックの型 | コトが最適な型を選択 | 5つの型から指定（例：「体験告白型」「数字インパクト型」） |
| トーン | `memory/project-branding.md` のブランドトーンに従う | 「カジュアル」「プロフェッショナル」等 |
| パターン数 | 3 | 生成するバリエーション数 |
| CTA | 余韻型 | 「余韻型」「CTA型」「問いかけ型」 |
| ペルソナ情報 | ミナトから提供があればそれを使用 | ターゲット属性・課題TOP3 等 |

## 実行フロー

### Phase 1：コト（コピーライター）による原案制作

コト（`subagent_type: copywriter`）に以下を指示して投稿原案を制作させる。

**コトへの指示テンプレート：**

```
SNS投稿を制作してください。

テーマ：{テーマ}
プラットフォーム：{プラットフォーム}
バズの型：{バズの型 or 「最適な型を選んで」}
フックの型：{フックの型 or 「最適な型を選んで」}
トーン：{トーン}
締め方：{CTA}
パターン数：{パターン数}
ペルソナ情報：{ミナトからの情報があれば記載 / なければ「ブランド設定に従う」}

制作ルール：
- copywriter.ja.md のSNS投稿制作ルールに従うこと
- フックは3ステップ設計（Xは2ステップ圧縮可）で設計すること
- X版は140字以内を厳守（再カウント必須）
- Threads版は500字以内で改行多め・カジュアル会話調
- 各パターンの狙い（どのフックの型を使ったか、なぜその型を選んだか）を明記すること
```

**プラットフォームが「両方」の場合：** X版とThreads版をそれぞれ独立して制作する（X版をThreadsに引き伸ばすのではなく、各プラットフォームの特性に合わせて別々に書く）。

**コトの出力形式：** コトの標準SNS出力（パターンA/B/C + 推奨パターン）

### Phase 1.5：パターン選択

コトの出力を受けて、シンヤさんに推奨パターンを提示する。

- コトの推奨パターンと各パターンの狙いを簡潔に報告
- シンヤさんが選んだパターン（デフォルト：コトの推奨）を Phase 2 に進める
- シンヤさんが「全部通して」と言った場合は全パターンを Phase 2 に進める

### Phase 2：humanizer による人間化

Phase 1.5 で選択されたパターンに `/humanizer` を適用する。

**humanizer への指示：**

```
以下のSNS投稿テキストを人間らしく仕上げてください。

トーン：{トーン}（SNS投稿・{プラットフォーム}向け）
文字数制限：{X: 140字 / Threads: 500字}

{選択されたパターンのテキスト}
```

**humanizer の3パス処理：**
1. パターン検出 + 書き直し（JP-1〜JP-15）
2. anti-AIパス（残っている違和感の洗い出し）
3. 日本語ネイティブチェック（翻訳的構文・カタカナ和語混在・会話的自然さ）

### Phase 3：品質チェック

humanizer 通過後のテキストに対して、以下の品質チェックを実行する。アスカが直接チェックする。

#### チェックリスト

**構成チェック：**
- [ ] フックが3ステップ設計（Xは2ステップ可）に沿っているか
- [ ] 1行目はスタッカート（短く圧縮）になっているか
- [ ] テーマ提示ではなくベネフィット/ペインで始まっているか
- [ ] 指定された「バズの型」に沿っているか

**プラットフォームチェック：**
- [ ] X版は140字以内か（文字数カウント実施）
- [ ] Threads版は500字以内か
- [ ] X版とThreads版でトーンが適切に変わっているか

**ブランド・ペルソナチェック：**
- [ ] ブランドトーン（一人称・語尾・NG表現）が維持されているか
- [ ] ペルソナの課題TOP3のいずれかに触れているか

**NGパターンチェック：**
- [ ] テンプレ的まとめになっていないか
- [ ] 当たり前を偉そうに言っていないか
- [ ] 感情フックのない情報羅列になっていないか
- [ ] 説教調になっていないか
- [ ] 驚き屋トーン（「ヤバい」「革命」「知らないと損」）を使っていないか

**humanizer チェック：**
- [ ] AIっぽい表現が残っていないか（JP-1〜JP-15のパターンに該当しないか）
- [ ] 自然な日本語として読めるか

#### 不合格時の対応

- 構成・バズの型の問題 → コトに差し戻し
- 文字数超過 → 原因を特定（コト原案が長い / humanizerが膨らませた）し、原因側に差し戻し
- AI臭さの問題 → humanizer に再度通す
- ブランドトーンの崩れ → humanizerが変えた箇所を特定し、humanizer に「トーンを維持して」と再指示
- 軽微な修正（1〜2語の差し替え程度） → アスカが直接修正（投稿テキストの微調整はコーディングに該当しない）

### Phase 4：レンのマーケティング整合チェック

品質チェック通過後、レン（`subagent_type: marketing-planner`）にマーケティング整合チェックを依頼する。CLAUDE.md の品質ゲート（コピー/マーケティング → レンのレビュー必須）に準拠。

**レンへの依頼内容：**
- 投稿テキスト（X版・Threads版）
- テーマ・使用した型・ペルソナ情報
- ブランド設定（`memory/project-branding.md`）との整合性

**レンの確認観点：**
- ブランドメッセージとの一貫性
- ターゲットペルソナへの訴求力
- 競合との差別化ポイント
- SNS戦略（ミナトの月間計画）との整合性

**レンが差し戻した場合：** 指摘内容に応じてPhase 1（コト）またはPhase 2（humanizer）に戻す。

## 出力形式

レンのレビュー通過後、以下の形式でシンヤさんに報告する。

```
## SNS投稿 完成版

### テーマ
{テーマ}

### 使用した型
バズの型：{型名}
フックの型：{型名}

### X版（{文字数}字）
{投稿テキスト}

### Threads版（{文字数}字）
{投稿テキスト}

### 制作メモ
- フック設計：{どのステップをどう使ったか}
- humanizer 修正箇所：{主な修正点}
- レンのレビュー結果：{OK / 修正した点}
- 推奨投稿タイミング：{あれば}
```

## Notion 登録

シンヤさんが投稿内容を承認したら、`notion-sns.py --add` で投稿管理DBに登録する。承認前には登録しない。

## 注意事項

- 事業戦略・ブランドトーンは `memory/project-branding.md` と `memory/project-business-strategy-2026.md` を参照する
- ペルソナ・ターゲット情報がミナト（sns-director）から提供されている場合はそれを最優先する
- 投稿の方向性（テーマの選定基準・優先順位）はミナトの月間計画に従う
- シンヤさんが「レビュー不要」と明示した場合のみ Phase 4 をスキップ可能（その場合、出力に [UNVERIFIED] を付与）
