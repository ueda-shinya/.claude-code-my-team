# meta-ad-campaign-design

Meta広告の配信を決める3要素・目的・ターゲティング・クリエイティブフォーマット・指標を一気通貫で設計するスキル。

## 適用条件

- このスキルを使う場面：
  - Facebook / Instagram / Messenger / Audience Network への広告出稿時
  - Meta広告のキャンペーン目的・ターゲティング・フォーマット選定が必要なとき
  - リターゲティング／類似オーディエンスの活用設計
  - Meta広告の予算配分とKPI設計
- このスキルを使わない場面：
  - 4分類10種類の広告手段全体から選定する段階 → `ad-mix-design`
  - Google検索広告（リスティング広告）の設計 → 別途リスティング広告スキル
  - 広告コピーの作成 → `ad-copy-7-angles`
  - 広告改善サイクル → `ad-performance-diagnosis`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 商品/サービス名 | 必須 | 広告対象 | 「月額ITサポート」 |
| 広告の主目的 | 必須 | 認知/検討/コンバージョンのどこを狙うか | 「コンバージョン（リード獲得）」 |
| ターゲット属性 | 必須 | 年齢/性別/地域/興味関心など | 「広島県・40-60代経営者」 |
| 月間予算 | 必須 | 媒体費 | 「月50万円」 |
| ランディング先URL | 必須 | 遷移先のLPまたはサイトURL | 「example.com/lp」 |
| 既存顧客リスト | 任意 | 類似オーディエンス用（メール/電話番号 最低100件） | 「既存300名分のメールアドレス」 |
| ピクセル設置状況 | 任意 | コンバージョン計測の準備状況 | 「設置済み」 |

## 実行ステップ

### ステップ1：キャンペーン目的を決定する

- 入力：広告の主目的、ビジネス課題
- 処理：以下のMeta公式13目的から1つ選ぶ
  | カテゴリ | 目的 | 適する状況 |
  |---|---|---|
  | 認知 | ブランド認知度アップ | 新ブランド／リーチを最大化したい |
  | 認知 | リーチ | 特定エリア・属性へ最大リーチ |
  | 検討 | トラフィック | LP/サイトへのアクセス数増 |
  | 検討 | エンゲージメント | いいね！コメント・シェアの増加 |
  | 検討 | アプリインストール | アプリDL促進 |
  | 検討 | 動画再生数アップ | 動画視聴の最大化 |
  | 検討 | リード獲得 | フォーム入力でリード収集（BtoB定番）|
  | 検討 | メッセージ | Messenger経由の問い合わせ獲得 |
  | コンバージョン | コンバージョン | 購入・申込の最大化（要ピクセル）|
  | コンバージョン | カタログ販売 | EC商品リストとの連動 |
  | コンバージョン | 来店数の増加 | 実店舗送客 |
- 判断ルール：BtoB高単価 → リード獲得／EC即決 → コンバージョン／新ブランド → ブランド認知度アップ／実店舗 → 来店数の増加
- 出力：確定したキャンペーン目的

### ステップ2：入札方式と予算を設定する

- 入力：キャンペーン目的、月間予算
- 処理：
  - 課金方式を決める：CPM配信（インプレッション課金）／CPC配信（クリック課金）
    - 認知系目的 → CPM配信
    - 検討/コンバージョン系 → CPC配信またはCPA最適化
  - 日予算を計算する：月予算 ÷ 配信日数
  - **コンバージョン目的の場合は週50CV以上獲得できる予算かを検証**：
    - 想定CPA × 50 = 週予算下限
    - 例）CPA 5,000円なら週25万円・月100万円が下限
    - 50CV未達の場合 → マイクロコンバージョン（カート追加・閲覧30秒等）を中間CVに設定
- 出力：課金方式／日予算／週CV見通し

### ステップ3：ターゲティングを4軸で設計する

- 入力：ターゲット属性、既存顧客リスト
- 処理：以下4軸から該当するものを選択する
  | 軸 | 設定項目 | 用途 |
  |---|---|---|
  | ユーザー情報 | 年齢／性別／地域／言語／学歴／役職／交際ステータス | 基本デモグラ絞り込み |
  | ライフイベント | 誕生日／交際／結婚 | ライフタイミング訴求 |
  | 興味・関心 | ファッション／ビジネス／金融など | 関心軸での絞り込み |
  | カスタム | リターゲティング／類似オーディエンス | 既存顧客活用・潜在層拡大 |
- ターゲティングのルール：
  - **細かく絞りすぎない**（50万UU以上のオーディエンスサイズを担保）
  - 既存顧客リスト100件以上ある → 類似オーディエンス（1-3%類似）を活用
  - リターゲティング設定 → サイト訪問者・カート離脱者など
  - 個人情報保護の観点でリターゲティング機能は制限傾向のため、ファーストパーティデータ活用を優先
- 出力：ターゲティング設定（ユーザー情報＋興味関心＋類似オーディエンス等の組み合わせ）＋オーディエンスサイズ見込み

### ステップ4：配信面（プラットフォーム×配置）を選定する

- 入力：ターゲティング、クリエイティブ素材の有無
- 処理：以下から配信面を選ぶ
  - プラットフォーム：Facebook／Instagram／Messenger／Audience Network
  - 配置：フィード／ストーリーズ／リール／インストリーム
- 選定ルール：
  - 初期は「自動配置」で全配置に出して、データを見て絞る
  - BtoB → Facebookフィード優先
  - BtoC・若年層 → Instagramフィード／リール／ストーリーズ
  - 動画素材あり → リール／ストーリーズ／インストリーム
  - 静止画のみ → フィード中心
- 出力：配信面の選択（自動配置 or 手動指定）

### ステップ5：クリエイティブフォーマットを13種から選定する

- 入力：商品特性、訴求内容、保有素材
- 処理：以下から最適なフォーマットを選ぶ
  | フォーマット | 適する用途 |
  |---|---|
  | 画像広告 | ブランド認知・簡単なメッセージ伝達 |
  | 動画広告 | 感情訴求・ストーリーテリング・デモ |
  | スライドショー広告 | 動画制作リソースが乏しい場合の動的表現 |
  | カルーセル広告 | プロセス訴求／メリット訴求／カタログ表示（最大10枚・個別リンク可） |
  | インスタントエクスペリエンス広告 | モバイル没入体験・ブランドストーリー |
  | コレクション広告 | 商品ラインナップ＋ショッピング体験 |
  | リード獲得広告 | フォーム埋め込みでメールリスト/問い合わせリスト拡充 |
  | ダイナミック広告 | 閲覧履歴ベースのパーソナライズ商品表示 |
  | ページへのいいね！広告 | SNSページのフォロワー増 |
  | イベントへの参加広告 | イベント登録/参加促進 |
  | Messenger誘導広告 | カスタマーサポート・個別問い合わせ |
  | プレイアブル広告 | ゲーム等のデモ体験 |
  | クーポン広告 | 販売促進・リピート購入 |
- 画像広告の制作ルール：
  - 本文 90文字以内推奨（一目で内容がわかる）
  - 画像内テキストは画像面積の20%以下に抑える
- カルーセル広告の使い道3パターン：
  - ①プロセスを訴求する
  - ②メリットを訴求する
  - ③カタログとして見せる
- 出力：選定したフォーマット＋制作仕様（本文/画像/動画/CTA）

### ステップ6：配信設定の必須事項を確認する

- 入力：選定済みクリエイティブ
- 処理：以下を必ず実施する
  - **URLパラメーターを付与する**（utm_source/utm_medium/utm_campaign/utm_content）
  - **コンバージョン目的の場合はドメイン認証を完了する**
  - Metaピクセルが正しく設置されていることを確認する
  - イベント設定（購入・リード・カート追加など）
- 出力：URLパラメーター付きランディングURL／ドメイン認証ステータス

### ステップ7：監視するKPI指標を定義する

- 入力：キャンペーン目的、課金方式
- 処理：以下4指標を監視ダッシュボードに設定する
  | 指標 | 意味 | 目安 |
  |---|---|---|
  | CVR / CPA | コンバージョン率／1CV獲得コスト | 業界相場による（事前にベンチマーク取得）|
  | CTR | クリック率 | まず1%を目指す |
  | CPC / CPM | クリック単価／1000表示単価 | 入札方式で異なる |
  | フリークエンシー | 1人あたり広告表示回数 | 3-5回が目安、それ以上は広告疲れ |
- レポーティング頻度：日次（配信開始1週間）→ 週次（安定後）
- 出力：KPIダッシュボード設計

### ステップ8：Meta広告キャンペーン計画書としてまとめる

- 入力：ステップ1〜7の全成果物
- 処理：出力フォーマットに沿って整形する
- 出力：Meta広告キャンペーン計画書

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 月予算20万円未満 | 単一目的・単一広告セットに集中 | 細分化すると配信最適化に必要な50万UU/週50CVに届かない |
| 既存顧客リスト100件未満 | 類似オーディエンス使えず → 興味関心ターゲティング中心 | 類似オーディエンス作成不可 |
| ピクセル未設置 | コンバージョン目的選択不可 → トラフィック目的＋GAでCV計測 | 計測できない目的は最適化が効かない |
| BtoB商材 | リード獲得広告＋Facebookフィード中心 | ビジネスユーザー集まりやすい |
| BtoC・若年層 | Instagramリール／ストーリーズ＋動画フォーマット | 視覚訴求が強い配置 |
| 商品ラインナップ複数 | カルーセル or コレクション広告 | 1広告で複数商品表示できる |
| 50万UU未満のオーディエンス | ターゲティングを緩める／類似オーディエンス1-3%拡大 | 配信最適化エンジンが機能しない |
| 週50CV未達 | マイクロCV（30秒滞在・カート追加等）に切替 | 最適化エンジン学習に必要 |
| フリークエンシー5超 | クリエイティブ更新／オーディエンス拡大 | 広告疲れの兆候 |
| CTR 1%未満 | クリエイティブ刷新（画像/コピー） | 関連度スコア低下リスク |
| 関連度スコア低 | ターゲットとの整合性を見直す／広告品質改善 | 入札効率が悪化する |

### 記入例

```markdown
# Meta広告キャンペーン計画書 — 月額ITサポート（広島県中小企業向け）

## 1. キャンペーン目的
- 目的：リード獲得
- 主要KPI：リード獲得月30件／CPA 12,000円以下

## 2. 予算と入札
- 月予算：50万円（日予算 約1.7万円）
- 課金方式：CV最適化（リード獲得最大化）
- 週CV見通し：約7件 → 50CV未達のため、マイクロCV（資料DL）を中間KPIに設定

## 3. ターゲティング
- ユーザー情報：広島県／35-65歳／全性別
- 興味・関心：中小企業経営／ITサービス／業務効率化
- カスタム：自社サイト訪問者リターゲティング＋既存顧客類似オーディエンス2%
- 推定オーディエンスサイズ：約60万人

## 4. 配信面
- プラットフォーム：Facebook / Instagram
- 配置：自動配置（学習後にフィード優先に絞り込み予定）

## 5. クリエイティブ
- フォーマット：リード獲得広告（フォーム埋め込み）
- 本文：「ITトラブルで業務が止まっていませんか？月額固定で広島の中小企業のIT全般を支援します。資料を無料でお届け。」（85文字）
- 画像：オフィスでPC作業する経営者写真／画像内テキスト「資料無料DL」のみ（20%以下）
- CTA：「詳しくはこちら」→ リードフォーム

## 6. 配信設定チェック
- [x] URLパラメーター付与済み（utm_source=facebook&utm_campaign=it-support-202604）
- [x] ドメイン認証完了
- [x] ピクセル設置・「リード」イベント設定済み

## 7. 監視KPI
- CPA：12,000円以下
- CTR：1%以上
- CPM：800円以下
- フリークエンシー：3-5回

## 8. 配信スケジュール
- 開始：2026-05-01
- 学習期間：5/1-5/7
- 改善サイクル開始：5/8〜（週次レビュー）
```

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| ターゲティングを細かく絞りすぎる（5万UU以下） | 最低50万UU以上を確保 | 配信最適化エンジンが学習できない |
| 学習期間中に予算/ターゲティングを変える | 1週間は触らない | 学習が振り出しに戻る |
| 1広告セットに10以上のクリエイティブを入れる | 3-5本に絞る | 配信機会が分散して各クリエイティブの学習が進まない |
| コンバージョン目的なのにピクセル未設置で進める | ピクセル設置→イベント発火確認→配信開始 | CVが計測できないと最適化されない |
| 画像内テキストを画像面積の50%にする | 20%以下に抑える | リーチ制限される |
| 本文を200文字以上書く | 90文字以内に収める | フィード上で省略表示される |
| URLパラメーター無しで配信する | 必ずutm付与 | GAやデータ分析で広告流入が識別できない |
| フリークエンシー無視で配信し続ける | 5超えたらクリエイティブ刷新／オーディエンス拡大 | 広告疲れでCTR半減 |
| 関連度スコア低のまま放置する | クリエイティブとターゲットの整合性を見直す | 入札効率悪化・配信機会減 |
| 50CV/週未達なのに本CVで最適化を続ける | マイクロCVに切替 | 学習データ不足で配信が止まる |
