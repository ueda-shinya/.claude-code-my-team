# recruitment-strategy スキル

採用戦略・採用ガイドラインを策定し、必要人材確保と採用活動の一貫性・透明性を両立させる。事業戦略スクール「知足」が定義する3重要ポイント／4主要素／6媒体別の傾向／4ステップ策定フロー／コンピテンシー7項目を統合的に運用する。

## 適用条件

- このスキルを使う場面：
  - 自社の採用戦略を体系的に策定したい
  - 採用ガイドライン（一貫性のある選考基準ドキュメント）を作りたい
  - スキル要件・コンピテンシーを定義したい
  - 採用チャネル（媒体）を選定したい
  - 採用フロー（一次／インターン／二次／決定）を設計したい
- このスキルを使わない場面：
  - 給与レンジの決定 → `salary-range-design` スキルを使う
  - キャリアロードマップ・育成計画 → `career-roadmap-development` スキルを使う
  - 評価制度の策定 → `evaluation-system-design` スキルを使う
  - 組織計画・組織図設計 → `organization-planning` スキルを使う
  - 役割と責任範囲の明確化 → `roles-responsibilities` スキルを使う

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 起動フレーズ | 必須 | 上記トリガーフレーズのいずれか | 「採用戦略を策定して」 |
| 会社・事業の前提 | 必須 | 業種・規模・成長フェーズ・現組織図 | 「広島の地域密着Web制作会社、社員8名、急成長期」 |
| 採用人数・期間 | 必須 | 何人をいつまでに採用したいか | 「6か月で20名」「年間5名」 |
| 募集する職種 | 必須 | エンジニア／営業／デザイナー等 | 「フロントエンドエンジニア／カスタマーサクセス」 |
| 経験レベル | 任意 | 即戦力中途／新卒／業務委託／インターン等 | 「即戦力中途」 |
| 既存の採用ガイドライン有無 | 任意 | あれば現状把握、なければゼロから策定 | 「現状なし」「Wantedly活用中」 |

## 実行ステップ

### ステップ1：採用戦略の3重要ポイントを確認する

**前提として、採用戦略・ガイドラインがなぜ重要かを以下の3観点で言語化する：**

| ポイント | 内容 |
|---|---|
| 採用戦略の目的 | ・必要な人材をタイムリーに確保し、企業成長を支える<br>・人材確保を効率化し、採用コストを最適化する |
| 採用ガイドラインの役割 | ・採用活動の一貫性を確保し、選考の質を向上させる<br>・社内外で共有することで、採用プロセスの透明性を高める |
| 採用活動の影響範囲 | ・優れた人材の確保が、企業文化や業績向上につながる<br>・適切な採用が従業員満足度の向上や離職率の低下に寄与する |

ヒアリング項目：
- 自社にとって採用戦略の3ポイントのうち、最も重視するのはどれか？
- 過去の採用で「一貫性が無かった」「コストが無駄だった」等の反省点はあるか？

### ステップ2：採用戦略の4主要素を設計する

**採用戦略を以下の4要素に分解して具体化する：**

| 要素 | 内容 | 出力例 |
|---|---|---|
| Goal 採用の目的と目標 | 短期目標（直近の採用人数）／中長期目標（次世代リーダー候補・多様性確保） | 「6か月で20名採用」「3年後の管理職候補を10名育成」 |
| Target 採用ターゲットの設定 | 即戦力中途／新卒／外国籍人材／業務委託 等 | 「5年以上の経験を持つ30代のエンジニア」「新規事業に対応できるデジタルマーケター」 |
| Channel 採用チャネルの選定 | 新卒／中途／外国籍／業務委託で適切な媒体を選定（次ステップで詳細） | 「中途はリファラル＋Wantedly、業務委託は複業クラウド」 |
| Standard 選考基準の策定 | スキル・経験・カルチャーフィットの3観点で評価 | 「スキル：React 3年以上／経験：自社サービスPM経験／カルチャー：当事者意識★」 |

### ステップ3：採用チャネル（媒体）を選定する

**6媒体の傾向を比較し、自社の採用ターゲットと契約形態に合わせて選定する：**

| 媒体 | 人の傾向 | 向いている契約形態 |
|---|---|---|
| リファラル | 人柄は分かって採用できるのでカルチャーマッチしやすい。仕事でもマッチする確率は50%くらい | 正社員 |
| SNS（主にX） | 応募の数は多い。ただしプロフィールが読みづらいので、何ができる人かをキャッチアップするのにパワーがかかる | 正社員、業務委託、インターン |
| Wantedly | プロフィールがしっかり書かれていて顔が見えるので、どんな人かわかりやすい。カルチャーマッチした人が獲得できる。プラットフォームがコモディティ化しすぎて、やや人材のクオリティにムラがある | 正社員、業務委託、インターン |
| 複業クラウド | 基本的にフリーランスか副業人材が採用できる。BizDev・Ops・デザイン・コーダーはスキル高めの人。やや時給単価は高め（3,000後半～5,000円） | 業務委託 |
| YOUTRUST | SNS的に使える。基本はビジネスサイド（営業・マーケ・バックオフィス）の人材獲得にむいているが、コーダーで採用できたケースもある | 正社員、業務委託、インターン |
| ナビ媒体（リクナビ／マイナビ） | 基本は正社員採用の媒体として使う。1職種30万円～なのでコスト高め。ただし母集団形成はできる（1求人で800件ほど応募の実績あり） | 正社員 |

選定の判断基準：
- カルチャー重視 → リファラル
- 母集団形成優先 → ナビ媒体
- スキル特化の業務委託 → 複業クラウド
- 顔の見える採用 → Wantedly／YOUTRUST
- 数を集めたいスタートアップ → SNS（X）

### ステップ4：採用ガイドライン策定の4ステップを実行する

**以下のフローでガイドラインをドキュメント化する：**

#### 4-1. 目標設定（Goal）
- 年間採用目標や採用における優先事項を定める
- 例：「応募数を20%増加させる」「内定承諾率を15%向上させる」

#### 4-2. 人材要件の定義（Goal）
**スキル要件**を以下の表で整理する：

| ポジション | 仕事内容 | 必要なスキルセット | 管理責任範囲 | 持っていると望ましい経験 | イメージ |
|---|---|---|---|---|---|
| | | | | | |

**コンピテンシー（マインド要件）**を以下の7項目で整理する（★は must have、それ以外は nice to have）：

| Competency | 内容 | 確認するポイント |
|---|---|---|
| ★社交性・好印象欲 | 社内・社外とも良好な人間関係が築ける、お客様から信頼される。人に対して、好印象を得ようとする積極的なコミュニケーションの姿勢、人間関係への影響を重要視する。感情に配慮する | 会話はスムーズか？人物印象は良いか？チームワークにおいて何を重要視するか？どのようなコミュニケーションスタイルか？ |
| ★達成意欲／責任感・徹底性 | 限られたリソースで成果を達成しようとする意欲。改善に向けて徹底して、継続的に取り組もうとする姿勢 | 一定レベルの困難な課題について「なぜ成し遂げようと思ったか？」 |
| ★当事者意識 | 与えられた仕事だけでなくよりよくするための提案ができる | 現在の職務・職場において、より良く改善できることがあるとすれば、どんなことか？ |
| ストレス耐性 | ストレスをチャレンジと捉え、その状況を前向きに考えられる | ストレスフルな状況に遭ったか、その時にどのように感じたか？ |
| リスク志向 | 自律している、前例のないことや、整っていない環境を言い訳にしない | 前例の無いことにチャレンジした経験はあるか？既にあることの改善、無いことのチャレンジ、どちらが好きか？ |
| 柔軟性 | 柔軟性がある、状況の変化に応じて柔軟に施策を変更できる | 状況の変化に応じて、施策を変更した経験はあるか？計画通りにやるのが好きか、柔軟に変えていくのが好きか？ |
| 概念理解力 | 組織全体から俯瞰して現状を把握し、適切な行動をとれる | 抽象的な質問や、ハイコンテクストな質問に対して、的確に返答しているか？ |

**重要ポイント**：
- 業務において発揮されるスタンスを言語化する
- 面談の中でコンピテンシーをどのように判断するかを事前に決めておき、面談実施者でコンピテンシーに照らし合わせて合否の判断をする

#### 4-3. 採用ガイドラインの策定（Process）
**3要素を整える**：
- **応募受付**：応募者管理の仕組みを整える（人数が少ないうちは、エクセルやスプレッドシート等で十分）
- **書類選考**：書類選考での通過条件を決める
- **面接フロー**：面接官のトレーニングを実施し、評価基準を統一

**採用フロー（一例）**：

| ステージ | 担当 | 実施内容 |
|---|---|---|
| 一次面接 | 社員 | 候補者の最低限のスキルとCompetencyを見る。Competencyとやる気、稼働条件を見てインターン実施の可否を決定する |
| インターン期間＋カジュアル面談（3か月～6か月） | メンター | インターン期間中はメインのメンターを決定し、メンターは週次で案件依頼の確認や稼働状況のキャッチアップ、サポートを行う。インターン期間中、必ず代表と月1回程度の面談を設定する |
| 二次面接（最終） | 代表 | 正社員の見込みがあるインターン生について、代表と面談し、最終合意をとる。基本的に会社側で社員採用の合意がとれた人物が進むため、この面接で合否をひっくり返すことはない |
| 決定 | - | 採用決定 |

**採用ポリシー（一例）**：
- インターン～本採用：インターン期間中はオンボーディングであり、かつ選考プロセスである
- インターン参加者も、インターン中も含めて選考であることを認識したうえで参加してもらう
- 採用決定までに、役員全員が一度は面談しており、可能な限り社員全員と接触している状態をつくる
- 採用の観点：学歴、職歴、性別、出自を採用の基準としない
- 過去の経験や今あるスキルよりも、マインドフィット・カルチャーフィットを重要視する

**ベンチャー特有の注意点**：
- ベンチャーは人数が少ないため、その組織にいる人との相性がかなり重要
- だめだったときに、チーム組を変える選択肢が少ない
- 一緒に仕事をしてみないとわからないことも多いので、正社員雇用に至る前に業務委託期間があったり、3ヵ月は試用期間として双方即日退職を可能にしていたりするケースもある

#### 4-4. 共有／採用活動の振り返り（Share）
- 採用チームや面接官とガイドラインを共有し、一貫性を保つ
- 面談前に必ず読んでもらい、同じ認識を持ったうえで採用に臨む

### ステップ5：採用戦略を成功させる3ポイントを設計する

| ポイント | 内容 |
|---|---|
| EVP（従業員価値提案）の明確化 | 「成長機会が多い」「フラットな社風」等、自社が候補者に提供できる価値を言語化 |
| デジタル活用 | 採用管理システム（ATS）やオンライン面接ツールを活用 |
| 採用ブランディング | 社員インタビューや社内イベントの様子をSNSで発信 |

**成功事例（参考）**：
- Google：明確な評価基準で、スキル・カルチャーフィット・将来性を多面的に評価 → トップタレントの採用を実現
- 国内大手メーカー：リファラル採用とデジタルツールの導入で、コスト削減と離職率低下を達成

### ステップ6：4ステップで自社の採用戦略を分析・改善する

| STEP | 内容 |
|---|---|
| STEP01 採用目標の明確化 | 現状の採用人数や人材タイプの目標設定を確認 |
| STEP02 採用チャネルの分析 | どのチャネルが最も効率的かをデータで評価 |
| STEP03 選考プロセスの見直し | 面接フローや評価基準が一貫しているかを確認 |
| STEP04 内定後のオンボーディング改善 | 入社後のギャップを解消するための施策を検討 |

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 採用人数が少ない（年5名以下）でカルチャー重視 | リファラル＋Wantedly中心 | コスト効率と質のバランス |
| 急成長期で母集団形成優先（年20名以上） | ナビ媒体＋SNS＋リファラル併用 | 数の確保が最優先 |
| 業務委託・スキル特化人材 | 複業クラウド＋YOUTRUST | スキル特化人材が集まりやすい |
| ベンチャー・スタートアップ | インターン経由の正社員採用フロー | 相性確認の期間を設けるため |
| 「コンピテンシーで何を見るかわからない」 | 7項目（★3＋4項目）で言語化、面談実施者で統一 | 面接官による判断ブレを防ぐ |
| 「採用しても定着しない」 | カルチャーフィット・マインドフィット重視に転換、オンボーディング強化 | スキル偏重採用の典型的な失敗パターン |
| 「面接官によって評価がブレる」 | 採用ガイドラインを策定し面談前に必読化 | 一貫性確保の本質的対策 |
| 競合分析で「うちの地域は採用競合がいない」と楽観判断が出た | `competitive-absence-audit` スキルで6仮説検証 | 採用市場の競合不在は需要不在（採用市場が成立していない＝候補者がいない）の可能性が高いため、楽観判断禁止 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 採用ターゲットを決めず「スキルがある人なら誰でも」と幅広く募集する | 「5年以上の経験を持つ30代のエンジニア」のように具体化する | ターゲット曖昧だと媒体選定もメッセージングもブレる |
| 媒体を1つに固定して使い続ける | 採用ターゲットと契約形態に応じて6媒体を使い分ける | 媒体ごとに集まる人材傾向が大きく異なる |
| コンピテンシーを言語化せず「なんとなく良さそう」で採用する | 7項目（★3項目＋4項目）で評価基準を統一する | 面接官による判断ブレが採用ミスマッチの最大原因 |
| 面接官が個別判断でガイドラインを共有しない | 面談前に採用ガイドラインを必読、同じ認識で臨む | 一貫性なしの選考は質の低下と離職率増加の原因 |
| ベンチャーで正社員採用を最初から打ち、ミスマッチで即退職になる | インターン期間（3-6か月）＋試用期間で相性確認してから正社員化 | ベンチャーは少人数で相性が重要、ミスマッチコストが高い |
| EVP（従業員価値提案）が曖昧で候補者に魅力が伝わらない | 「成長機会」「フラットな社風」等、自社の価値を明確化 | EVPなき採用は給与競争に陥る |
| 学歴・職歴・性別・出自で足切りしてしまう | 採用の観点で「これらを基準としない」と明文化 | 多様性確保と優秀人材逃しの両方の問題を防ぐ |
| 採用市場で「この地域は競合採用が少ない」と楽観判断する | 競合不在＝候補者不在の可能性を `competitive-absence-audit` で検証 | 採用市場の競合不在は需要不在を示唆することが多い |

## 連携スキル

- `salary-range-design`：採用後の給与レンジ決定（採用ターゲットと連動して給与を設計）
- `career-roadmap-development`：採用後のキャリアロードマップと育成計画
- `evaluation-system-design`：採用後の評価制度（コンピテンシーと連動）
- `roles-responsibilities`：採用ターゲットの役割と責任範囲を明確化
- `organization-planning`：採用人数を組織計画に反映
- `mvv-design`：採用ガイドラインの「採用ポリシー」をMVVと整合させる
- `branding`：採用ブランディング（SNS発信内容と整合）
- `competitive-absence-audit`：採用市場の競合不在を楽観判断しないための仮説検証
- `hearing-questioning-skills`：面接時のヒアリング力（コンピテンシー判定に直結）
- `ms-matrix-talent-grid`：採用ターゲット象限の決定（新卒は①伸びしろ大／中途は②理想型・④専門家を狙う）
- `credo-language-design`：クレドを採用ブランディング素材として公開する設計
- `success-cycle-relationship-quality`：採用後のオンボーディング期に関係の質レベル1〜2を構築する前提として連携

## 5状態出力契約

このスキルは外部API・サブプロセス実行・外部設定依存のいずれも持たないため、CLAUDE.md「スキル出力の5状態契約」の対象判定基準には該当しない。ただし以下のケースで明示マーカーを使うこと：

- `[OK:0件]`：対象組織の情報が不足し、採用戦略を設計するための前提が揃わない場合
- `[SKIP]`：競合不在の楽観判断が出た場合は `competitive-absence-audit` スキルへの誘導でスキップ

外部API・サブプロセスを持たないため、`[FAIL]`（試行後の失敗）は通常発生しない。
