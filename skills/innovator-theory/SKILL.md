# innovator-theory

イノベーター理論の5層分類で自社事業のフェーズを判定し、各層別の問題事象・原因・課題・解決策とプロモーション戦略を整理する。「2.5%/13.5%/34%/34%/16%」の人数比率と「16%＝キャズム」を市場シェアKPIの根拠にも使う。

## 適用条件

- 使う場面：
  - 新規事業・新サービスの普及戦略を立てる
  - 自社事業がどのフェーズ（導入期／成長期／成熟期／衰退期）にあるか判定したい
  - 各顧客層別にプロモーション戦略を作り分けたい
  - 「市場シェアの目標値」の根拠が欲しい（キャズム16%の活用）
  - キャズム（アーリーアダプター→アーリーマジョリティ）を越えられない原因を分析したい
- 使わない場面：
  - セグメンテーション・ターゲティング → STP分析（`stp-analysis`）
  - 顧客の根本ジョブ分析 → Jobs to Be Done
  - 価値提案の言語化 → バリュープロポジション（`value-proposition`）
  - PMF達成への準備 → PMF/フィットジャーニー（`pmf-journey`）

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 分析対象 | 必須 | 商品・サービス | 「自社のSaaS」「新規BtoBサービス」 |
| 分析の目的 | 必須 | フェーズ判定／プロモ設計／KPI根拠 | 「キャズム超えのプロモを作る」 |
| 現在の市場シェア／顧客数 | 任意 | 現状把握用 | 「ターゲット市場の3%」 |
| 既知の顧客特性 | 任意 | 顧客層の傾向 | 「初期はテック好きが多い」 |

## 実行ステップ

### ステップ1：5層の特性を理解し自社事業のフェーズを判定する

- 入力：分析対象、現在の市場シェア
- 処理：以下5層の特性を踏まえて、自社事業がどのフェーズにいるか判定する
  - **イノベーター（2.5%、導入期）**: 先進的でリスクを厭わない先駆者
  - **アーリーアダプター（13.5%、導入期後半）**: トレンドに敏感で社会的影響力が強い層
  - **キャズム（16%）**: アーリーアダプター→アーリーマジョリティへの大きな溝
  - **アーリーマジョリティ（34%、成長期）**: 慎重に検討してから採用する層
  - **レイトマジョリティ（34%、成熟期）**: 周囲の影響を受けてから採用する保守的な層
  - **ラガード（16%、衰退期）**: 最も遅れて採用する層
- フェーズ判定基準：
  - 市場シェア 0〜2.5%：イノベーター層獲得期（導入期前半）
  - 2.5〜16%：アーリーアダプター獲得期（導入期後半・キャズム前）
  - 16%超え：キャズム突破→アーリーマジョリティ獲得期（成長期）
  - 50%超え：レイトマジョリティ層獲得期（成熟期）
  - 84%超え：ラガード層獲得期（衰退期）
- 出力：自社の現フェーズ（5層のいずれか）

### ステップ2：5層別に問題事象・原因・課題・解決策を整理する

- 入力：ステップ1のフェーズ判定
- 処理：層ごとに以下4項目を整理する（5層×4項目＝20マス）
  - **問題事象**: その層で起きがちな表面的な現象
  - **原因**: 問題事象の根本原因
  - **課題**: 解決すべき本質的な課題
  - **解決策**: 具体的なアクションプラン
  - **事例**: 同じ層で成功／失敗した他社事例
- 自社の現フェーズに該当する層を最も詳しく書く
- 隣接層（前フェーズと次フェーズ）も準備として整理しておく
- **キャズム列**（アーリーアダプターとアーリーマジョリティの間）を独立した6列目として扱い、「キャズムで何が起きるか」を明示する
- 出力：5層×4項目（＋キャズム列）の整理表

### ステップ3：層別プロモーション戦略を設計する

- 入力：ステップ2の整理結果
- 処理：層ごとのプロモーション軸を以下の通り設計する
  - **イノベーター層**: 新しい技術・機能を重視したプロモーション（技術詳細・革新性の訴求）
  - **アーリーアダプター層**: インフルエンサー・レビューを活用した信頼性構築（社会的証明）
  - **アーリーマジョリティ層（キャズム超え後）**: 実績の提示、大衆向けキャンペーン展開（事例・実績訴求）
  - **レイトマジョリティ層**: 手頃な価格・安心感を訴求（リスク低減・コスパ訴求）
  - **ラガード層**: 必要性を強調、切り替え理由の明確化（古い手段が使えなくなる訴求）
- 自社の現フェーズに対応するプロモーション軸を最重点で設計する
- 出力：層別プロモーション戦略一覧

### ステップ4：市場シェア目標とキャズム超え戦略を立てる

- 入力：ステップ1〜3の結果
- 処理：
  - **市場シェアKPI**: ターゲット市場サイズ × 16%（キャズム）を最低目標として設定
    - 例：ターゲット市場1万社 → キャズム超えの目安シェア = 1,600社獲得
  - **キャズム突破戦略**（アーリーマジョリティ獲得時の最重要施策）:
    - 業界特化（ホールプロダクト戦略）
    - 信頼できるリファレンス事例の構築
    - 標準化された営業プロセス（再現可能性）
    - 中小企業向け使いやすさ重視の機能開発
- 自社が現在キャズム前ならアーリーアダプター満足度＋次フェーズ準備、キャズム後ならスケール戦略に振る
- 出力：市場シェアKPI＋キャズム突破/スケール戦略

### ステップ5：イノベーター理論シートを完成させる

- 入力：ステップ1〜4の全成果物
- 処理：以下のテンプレートに統合する
- 出力：イノベーター理論完成シート

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 市場シェアが2.5%未満 | イノベーター層獲得期。技術・革新性で訴求 | 大衆訴求は早すぎる |
| 市場シェアが2.5〜16%（キャズム前） | アーリーアダプター獲得期＋キャズム準備 | 単純な拡大ではキャズムを越えられない |
| 市場シェアが16%前後で停滞 | キャズム発生中。マジョリティ向け施策に転換 | アーリーアダプター戦術ではマジョリティに届かない |
| 市場シェアが16%超え（成長期） | アーリーマジョリティ獲得＋スケール体制構築 | 実績訴求と再現性が鍵 |
| 市場シェアが50%超え（成熟期） | レイトマジョリティ獲得＋差別化維持 | 価格・安心感訴求にシフト |
| 衰退期（市場縮小） | ラガード獲得＋次世代事業への投資 | 既存事業の収益最大化 |
| イノベーター・アーリー向け施策で停滞 | キャズム超え戦略への切替を検討 | 同じ施策の延長では超えられない |
| 全層に同じプロモーションを打つ | 層別に作り分ける | メッセージが弱い／ターゲットがブレる |
| キャズム概念を使わない | 必ずキャズム超え戦略を別立てで設計 | キャズムは事業成長の最大の壁 |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 全層に同じプロモーションを打つ | 層別にプロモ軸を作り分ける | 層によって意思決定基準が違う |
| イノベーター戦術のままマジョリティを狙う | キャズム突破戦略に切り替える | 同じ戦術ではキャズムを越えられない |
| キャズム概念を使わない | 16%を市場シェアKPIの根拠として明示 | キャズムは事業成長の最大の壁 |
| 自社のフェーズを過大評価する | 客観的な市場シェアで判定 | 楽観バイアスは戦略を狂わせる |
| 解決策が「がんばる」「強化する」 | 動詞＋目的語＋数値で書く | 抽象戦略は実行不能 |
| 事例を載せない | 各層で同業他社の成功・失敗事例を必ず載せる | 事例なき戦略は説得力に欠ける |
| 次フェーズ準備を怠る | 現フェーズ＋次フェーズの準備を並行 | フェーズ移行は予測しないと遅れる |
| 1度作って放置 | 半年〜1年で再評価 | フェーズは動く |
