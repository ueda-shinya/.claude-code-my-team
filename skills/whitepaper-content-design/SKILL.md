# whitepaper-content-design

BtoBマーケティングのリード獲得・育成・購買誘導を担うホワイトペーパー戦略を、カスタマージャーニー×ファネル軸で体系設計するスキル。検討フェーズに応じた4タイプ（サービス紹介系／ターゲット別／シミュ・テンプレ系／比較一覧表系）と8種類のコンテンツタイプから優先順位を決め、顕在層から優先して制作する。

## 適用条件

- 使う場面：
  - BtoBマーケティングでリード獲得用のWP（ホワイトペーパー）を企画
  - サービス紹介資料／導入事例／比較資料／調査リリース等の優先順位を整理
  - カスタマージャーニーとファネルを掛け合わせたコンテンツ設計
  - 既存WPの内容を「ターゲットに合っていない」と感じて見直したい
  - リード獲得手段が広告のみで、コンテンツ資産がない
- 使わない場面：
  - LP制作 → `lp-create`
  - SEO記事の執筆 → `blog-post`
  - SEO戦略全般 → `seo-content-strategy`
  - 営業提案書（個社向け） → `proposal-builder`
  - リードナーチャリング（取得後の育成） → `lead-nurturing`
  - PR・プレスリリース執筆 → `press-release-builder`

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 事業ドメイン | 必須 | BtoB SaaS／コンサル／製造業など | 「BtoB SaaS：マーケDX支援」 |
| ターゲット | 必須 | 業種・役職・課題 | 「中堅企業マーケ部長／リード不足」 |
| 現状の保有コンテンツ | 推奨 | 既存WP・記事・資料 | 「サービス紹介資料1点のみ」 |
| 目的 | 必須 | リード獲得／信頼構築／商談誘導 | 「新規リード月100件獲得」 |
| 検討フェーズ優先度 | 推奨 | 初回接点／比較検討／商談 | 「比較検討フェーズ最優先」 |

## 実行ステップ

### ステップ1：ホワイトペーパーの定義と関連用語を確認する

- 入力：事業ドメイン、目的
- 処理：基本概念を整理
  - **ホワイトペーパー（定義）**：特定の課題に対する解決策や専門情報を提示する文書。主にBtoBマーケティングで使用され、見込み顧客の教育やリード獲得を目的とする
  - **関連用語**（正しい定義）：
    - **リードジェネレーション**：見込み顧客の情報を収集し、販売活動に活用するプロセス
    - **リードナーチャリング**：見込み顧客を教育・育成して購買意欲を高めるプロセス（コンテンツマーケの主要目的の1つ）
    - **インバウンドマーケティング**：有益な情報提供を通じて顧客を引きつけるマーケティング手法（広告で押し売りせずコンテンツで来てもらう）
    - **ROI（Return on Investment／投資収益率）**：投下資本に対する利益率。コンテンツマーケでは「コンテンツ制作費に対するリード獲得・商談化貢献度」で評価する効果測定指標
    - **ICEスコア**：Impact（影響度）×Confidence（確信度）×Ease（実行容易性）の3軸で施策優先順位を点数化する評価フレーム
- 出力：定義の共有

### ステップ2：3特徴×3目的でWP戦略の方向性を決める

- 入力：ターゲット、目的
- 処理：
- **3つの特徴**：
  1. **専門性**：深い知識や具体的なデータを含む内容
  2. **信頼性**：問題解決のための具体的かつ実践的な情報を提供
  3. **教育性**：見込み顧客が抱える課題を明確にし、解決策を提案
- **3つの目的**：
  1. **リードジェネレーション**：フォームを通じて見込み顧客の情報を収集
  2. **信頼構築**：自社の専門性や業界知識をアピール
  3. **購買プロセスへの誘導**：問題意識を高め、次のステップ（問い合わせ、商談）につなげる
- 出力：戦略方向性（特徴×目的のマトリクスで優先度設定）

### ステップ3：4タイプから制作対象を選択する

- 入力：ターゲット、保有コンテンツ
- 処理：以下4タイプから優先制作対象を選定
  - **①サービス紹介系コンテンツ**：会社紹介やサービス紹介資料。最も基本のコンテンツで、最優先で作成
  - **②ターゲット別のサービス説明資料**：「○○業界向け」と表現し、サービス説明の見せ方や悩み・ベネフィット訴求をターゲットにアレンジ。業界別ターゲティングの検証に有効
  - **③シミュレーション・テンプレ系コンテンツ**：エクセルやGoogleスプシに数字を入れると、運用表として活用できるコンテンツ。メルマガ等からの引き上げ率も高い傾向。休眠掘り起こしの検証に有効
  - **④比較一覧表系コンテンツ**：担当者が複数ツールを検討している場合、稟議にそのまま使ってもらえる資料
- 出力：制作対象タイプ＋ねらい（業界別検証／休眠掘り起こし／稟議材料等）

### ステップ4：コンテンツ設計の2軸（ジャーニー×ファネル）を理解する

- 入力：目的、ターゲット
- 処理：
- WPは検討段階を前に進める「興味付け」のお役立ち資料として活用される
- コンテンツ作り方によって目的とする検討フェーズが異なる
- **設計の2軸**：
  - **カスタマージャーニー**：検討フェーズの階段設計に沿ったコンテンツ設計
  - **ファネル**：ニーズの顕在〜潜在ファネルに沿ったコンテンツ設計
- 出力：2軸の理解共有

### ステップ5：カスタマージャーニーで階段設計する（観点①）

- 入力：ターゲットペルソナ
- 処理：以下3階段に応じてWPを設計
  - **初回接点・課題喚起**：
    - サービス紹介に必ずしも直接関係しないが、ターゲットが関心を持つテーマでのお役立ち情報コンテンツ
    - 例：業界トレンド情報／テンプレート資料
  - **比較・検討**：
    - サービス選定を検討し始めた層の、検討プロセスをサポートするコンテンツ
    - 例：比較コンテンツ
  - **商談**：
    - 自社サービス・商品に関心をもった人に、自社の情報を伝えるコンテンツ
    - 例：サービス紹介コンテンツ／導入事例／トライアル・体験コンテンツ
- 出力：ジャーニー段階別コンテンツ計画

### ステップ6：ファネルで顕在〜潜在を整理する（観点②）

- 入力：保有コンテンツ
- 処理：
- ニーズの顕在〜潜在のファネルにおいて適切なコンテンツがある
- **顕在層に近いコンテンツから優先して作る**
- **8種類のコンテンツタイプ**（顕在 → 潜在の順）：

| コンテンツの種類 | 内容 | ファネル位置 |
|---|---|---|
| サービス紹介資料 | サービスの内容、機能、価格を説明した資料 | 最も顕在 |
| 導入事例記事・資料 | お客様名を明記した導入事例コンテンツ | 顕在 |
| 成功事例紹介（匿名） | 成果になった事例を具体的に紹介するコンテンツ | 顕在 |
| 比較資料 | サービス・ツール選定をすすめている見込み顧客に役立つ比較資料 | 比較検討 |
| サービス・商品を紹介するセミナー | サービス紹介につなげるお役立ちセミナー | 比較検討 |
| SEO記事 | 検索流入を目的とするコンテンツ。基本情報とサービス紹介を兼ねる | 準顕在 |
| テンプレート資料・チェックリスト | パワポやエクセルで、テンプレートとして業務に活用できる資料 | 潜在 |
| 調査リリース・カオスマップ | 業界トレンドについて情報提供する資料 | 最も潜在 |

- 出力：ファネル別優先制作リスト

### ステップ7：ジャーニー×ファネルマトリクスで実行計画を立てる

- 入力：ステップ5・6
- 処理：2軸を掛け合わせて優先順位を確定
- **顕在層×商談フェーズが最優先**：サービス紹介資料／導入事例
- **準顕在層×比較検討フェーズ次点**：比較資料／成功事例
- **潜在層×初回接点フェーズ最後段**：テンプレート資料／調査リリース／カオスマップ
- 出力：実行計画＋制作タイムライン

### ステップ8：構成案を作る（Before/After）

- 入力：選定したWPタイプ
- 処理：以下フォーマットで構成案を作成
- **構成案テンプレートの要素**：
  - テーマ
  - 納品場所
  - タイトル
  - ユーザーイメージ：レイヤー（マーケ担当者等）／抱えるペイン／読後感
  - デザインイメージ：参考URL
  - 構成案（項目／内容／得られる感情／参考記事）
- **Before例（よくある課題）**：
  - 顧客の課題が分からない（ターゲット層の具体的なニーズが不明確）
  - リード獲得手段が不足（サイト訪問者を効果的にリードへ変換できない）
  - 競合との差別化が難しい（自社の専門性や価値を十分に伝えられていない）
  - 原因：コンテンツ不足／質の低い情報提供／ターゲットペルソナが曖昧で内容が的を射ていない
- **After例（改善後）**：
  - タイトル明確化（例：「BtoBマーケにおけるLPOについて」）
  - レイヤー：マーケ担当者
  - 抱えるペイン：自社でLPを作成し広告を回しているが成果が出ない／効果は出ているがなぜ効果がいいのか・さらに伸ばす方法が分からない／いくつものLPO施策を試したがあまり効果が出ていない
  - 構成案：導入（5分）＝セミナーゴール・講師紹介・会社紹介／1（5分）LPOとは／2（5分）LPOがなぜ重要なのか／3（5分）LPOをすることでのメリット
- 出力：構成案ドラフト

### ステップ9：成功の鍵を確認する

- 入力：ステップ1〜8の計画
- 処理：以下3点を最終確認
  1. **ターゲットの課題を深く理解する**
  2. **読者に価値ある情報を提供する**
  3. **定期的に内容を見直し、最新の情報を反映する**
- WPの役割：
  1. リード獲得とナーチャリングを強化するための効果的な手段
  2. カスタマージャーニーや検討ファネルに合わせてコンテンツを作成する。顕在層や、商談に近いコンテンツから優先して作る
- 出力：実行可否の最終確認

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| WPゼロ、サービス紹介資料すらない | ステップ3-①「サービス紹介系」最優先 | 最も基本のコンテンツ |
| サービス紹介資料はあるが業界別の差別化が必要 | ステップ3-②「ターゲット別資料」 | 業界別ターゲティング検証に有効 |
| メルマガでハウスリスト掘り起こししたい | ステップ3-③「シミュ・テンプレ系」 | 引き上げ率が高い |
| 担当者が複数ツール検討中 | ステップ3-④「比較一覧表」 | 稟議にそのまま使える |
| 顕在層リードが少ない | ステップ6で顕在側コンテンツ優先 | 商談直結のWP不足 |
| ハウスリスト育成が課題 | ステップ6で準顕在〜潜在側を順次整備 | ナーチャリング素材不足 |
| 業界全体での認知が低い | 調査リリース・カオスマップでブランド構築 | 露出機会の創出 |
| ターゲットペルソナ未定義 | ステップ8の構成案前にペルソナ設計（`persona-design`） | ペルソナ曖昧だと刺さらない |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 顕在層のWPなしで潜在層の調査リリースから着手 | ステップ6で顕在側を最優先 | 商談直結資産がない |
| ターゲット曖昧でWP制作 | `persona-design`でペルソナ確定後に制作 | 刺さらない量産になる |
| 全部一気に作る（4タイプ×8種類） | ステップ7のマトリクスで優先度1〜2タイプに集中 | リソース分散 |
| サービス紹介資料を一度作って放置 | ステップ9で定期的見直し | 古い情報は信頼性低下 |
| WP作っただけで活用導線なし | LP・メール・営業ツールへの連携を設計 | リード活用率低下 |
| 業界別資料を作らず汎用版だけ | ステップ3-②で業界別差別化 | コンバージョン率が低い |
| 比較資料で自社優位を強調しすぎ | 公平な比較で「使ってもらえる」資料に | 信頼を失う |
| 検討フェーズと無関係なテーマ | ステップ5でジャーニー×フェーズ整合 | 興味喚起できない |

## 連携スキル

- 前段：`persona-design`（ターゲット明確化）／`stp-analysis`（STP分析）／`customer-journey`（ジャーニー詳細設計）
- 並走：`funnel-design`（ファネル詳細設計）／`lead-definition-mql-sql`（リード定義）
- 並走（資産化の往復）：`webinar-design`（WPをセミナー化／セミナーをWP化、双方向の資産展開）／`lead-nurturing`（取得リード育成）
- 関連：`seo-content-strategy`（SEO記事連動）／`lp-create`（WPランディングページ）／`press-release-builder`（調査リリース執筆）
