# ブログ記事作成ガイド

## 構成ルール

### 基本構造（3部構成・総文字数 約6,000字 目安）

| セクション | タグ | 目安文字数 | 要点 |
|---|---|---|---|
| リード文 | `<p>` のみ | 約250字 | 記事全体の要約・検索意図への結論先出し |
| 本文 | `<h2>`/`<h3>`/`<p>`/リスト/表 | 約5,000字 | 下記の見出し階層ルールに従う |
| まとめ | `<h2>まとめ</h2>` + `<p>` | 約250字（2〜3段落） | 3〜5つのポイント整理＋CTA |

### 見出し階層（SEO/LLMO 共通の必須ルール）

- **H1**：ページ内に **1つのみ**。検索キーワード必須。
- **H2**：**3〜5個**、各 **800〜1,200字**、各H2にキーワード/関連語を含める。
- **H3**：各H2配下 **2〜4個**、各 **200〜300字**、関連語を含む。
- **H4**：必要な場合のみ利用（使いすぎない）。
- レベル飛ばし禁止（H1→H3 等はNG）。

### キーワード配置

- **タイトル・リード文・各H2** に主キーワードを必ず含める。
- **キーワード密度 1.5〜2.0%**（不自然にならない範囲）。
- メインKW／関連KWを事前に決め、執筆前にリストアップする。

## 文体ルール

- 一文は **50〜80字** 目安、冗長な接続詞を避ける。
- 「です・ます」で統一（記事トーンを混在させない）。
- 各セクションは **PREP構成**（結論→理由→例→結論）を原則とする。
- 不確かな情報には ※ を付記。
- 接続語（「まずは」「次に」「さらに」）で流れを明確化。

## SEOルール

- 専門用語は初出時に短い説明を添える。
- 具体例・数字・事例を最低1つ含める。
- 内部リンクを最低2つ提案する。
- 統計・データを引用する場合は、**出典URL＋調査年月** を本文中に明記する（例：「出典: example.com、2025年7月調査」）。
- 主要画像すべてに **意味のある alt** を付与（空alt・機械的ファイル名はNG）。

## 構造化・視覚要素ルール（読まれる記事の定量目安）

下記は「滞在時間が長い記事」の平均値（N=881の実データ分析に基づく）。これは **目標値（ターゲット）** であり、下限として扱うと分布の半数が未達になるため、**「目安」として参照** する。

| 要素 | 目標値（平均） | 参考解釈 |
|---|---|---|
| 画像（imgタグ） | **2個**（平均1.98） | 滞在0-30秒グループの約11倍 |
| リスト（liタグ合計） | **14〜15個**（平均14.50） | 構造化と読みやすさの指標 |
| 表（tableタグ） | **1個程度**（平均0.86） | 比較・整理が必要な情報がある記事で有効 |
| 箇条書き（ulタグ） | **2〜3個**（平均2.39） | セクションごとに1つ程度が目安 |
| 文字数 | **4,400〜4,800字**（平均4,411字） | 記事主題・検索意図に応じて調整 |

出典：TechSuite社「ブラインドテストから見つける『読まれるコンテンツ』の共通点_v4」p13（2025年7月調査）。

**注記**：
- 本数値は主に **SEO記事** を対象とした TechSuite 社調査に基づく。**LP・商品ページ・業種特化ページ**では別途調整が必要。
- 平均値のため、達成度ではなく「近似できているか」の観点で参照する。

## 5状態出力契約（本スキルの対象判定）

本スキルは外部ネットワーク呼び出し・サブプロセス実行・外部設定依存のいずれも含まない純粋な執筆ガイドのため、CLAUDE.md「スキル出力の5状態契約」の**対象外**。

## プロンプト設計（AI執筆を依頼するときの構造）

AIに本記事構成で執筆依頼する場合は、下記5セクションでプロンプトを組み立てる。

| セクション | 主要な要素 | 役割 |
|---|---|---|
| 1. 導入部 | 対象読者の定義／出力形式の指定／メタ指示 | 応答文除外・HTML記事形式などを指示 |
| 2. 変数設定部 | `{{keyword}}` / `{{title}}` / `{{information}}` / 文字数制約 | 記事個別のトピック・参考情報・量を与える |
| 3. 構造定義部 | リード文250字／本文5,000字／まとめ250字 | 3部構成と各部の文字数配分を固定 |
| 4. 最適化指示部 | H1/H2/H3構造／KW密度1.5-2%／文体統一 | SEO仕様とスタイルを規定 |
| 5. 技術仕様部 | 使用可能HTMLタグ／禁止事項／コード例 | HTMLマークアップ規則を規定 |

### プロンプト手法5分類（組み合わせて使用）

| 手法 | 実装例 | 効果 | 主な適用箇所 |
|---|---|---|---|
| Few-Shot Learning | tableサンプルコード提供（`<table><thead>...`） | 期待する出力形式の明示 | 技術仕様部 |
| Chain-of-Thought | 執筆手順を5段階プロセス化（「1.構造決定→2.トピック化→…」） | 段階的思考の誘導 | 導入部末尾／執筆手順 |
| 制約ベース設計 | 文字数6,000／KW密度1.5-2%／H2: 3-5個 | 出力品質の保証 | 全セクション |
| テンプレート化 | `{{keyword}}` `{{title}}` `{{information}}` | 再利用性の向上 | 変数設定部 |
| ネガティブ制約 | 「応答文を一切含めず」「`<br/>`使用禁止」 | エラー防止 | 技術仕様部・導入部 |

### システムプロンプト vs ユーザープロンプト

- **システムプロンプト**：役割（「あなたは10年以上の経験を持つSEO専門ライターです」）、執筆方針（E-E-A-T重視）、守るべきルールを固定で書く。
- **ユーザープロンプト**：記事テーマ、ターゲット読者、文字数、メインKW、関連KWなど、記事ごとに変わる指示を書く。
- 衝突時はシステムプロンプトが優先される。`{{variable}}` テンプレートはユーザープロンプト側に置く。

参考：TechSuite社「AI記事作成プロンプト術_v5」p6-19。

## 執筆手順（AI執筆時の推奨5ステップ）

1. 全体構造と見出しを設計（H1/H2/H3のツリー）
2. 各セクションの主要トピックをリスト化
3. 検索意図を満たす内容を前半に配置
4. 専門用語は初出時に説明
5. SEO要件（KW密度・見出し階層・禁止タグ）と文章規則を最終チェック

## 連携スキル

- LLMO（AI検索エンジン引用獲得）を最初から意識したい新規記事は、**`llmo-compliant-coding` スキルの遵守ルールを併用**する。
- 公開後3ヶ月時点で **`llmo-audit`** による事後監査を推奨。
