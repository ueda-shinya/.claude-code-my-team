# youtube-account-design

YouTubeを「視聴履歴・検索意図重視で長時間滞在を評価される深いファン化メディア」と捉え、目的→市場調査→ポジショニング→KPI→施策→台本→分析改善のサイクルを実行可能な形で設計する。「YouTubeは構成力が8割」を前提に、サムネ・タイトル・冒頭・本編・CTAの一貫性を担保する。

## 適用条件

- このスキルを使う場面：
  - YouTubeチャンネルを新規開設するときの戦略設計
  - 既存YouTube運用が伸び悩んでおり、企画・台本・分析からテコ入れしたいとき
  - 信頼醸成／興味関心層獲得／購買・問合せ／認知拡大のいずれかでYouTubeを使うとき
  - 4機能（Shorts／Long／Live／Community）の優先度を整理したいとき
  - 動画1本の台本・タイトル・サムネを設計したいとき
- このスキルを使わない場面：
  - SNS全体戦略（媒体選定）の設計 → `sns-strategy-overview`
  - Instagram／TikTok単体の設計 → `instagram-account-design` / `tiktok-account-design`
  - 動画編集の具体テクニック → 別スキル／外注へ

## 入力

| 項目 | 必須/任意 | 説明 | 例 |
|---|---|---|---|
| 運用目的 | 必須 | 信頼醸成／興味関心層獲得／購買・問合せ／認知拡大 のいずれか | 信頼醸成 |
| 業態タイプ | 必須 | BtoC／BtoB／採用 | BtoB（コンサル） |
| 主要ターゲット | 必須 | デモグラ＋サイコグラ＋悩み | SNSが苦手な中小企業の経営者 |
| 事業の強み（USP） | 必須 | 競合にない独自切り口 | 売上1億円までの再現性ある営業設計 |
| 競合チャンネル | 任意 | 同業界2〜3チャンネル | 未指定の場合はステップ4で調査依頼 |
| 既存運用状況 | 任意 | 登録者数・投稿頻度・主要数値 | 登録者950・週1投稿 |
| 撮影リソース | 任意 | 撮影機材・編集体制 | 専任編集者なし |

## 実行ステップ

### ステップ1：運用目的を4パターンから選定する

- 入力：事業ゴール、業態タイプ
- 処理：
  1. 以下4目的から主目的を1つ選ぶ：

  | 目的 | 概要 | 主要KPI | 補足指標 | 目安ライン |
  |---|---|---|---|---|
  | 信頼醸成（ファン育成） | 商品・サービスを深く理解させ信頼を高める | 平均視聴時間／視聴維持率 | 直帰率／離脱ポイント | 視聴維持率40%以上 |
  | 興味関心層獲得 | 「また見たい」と思わせフォロワー増 | チャンネル登録数 | 高評価率／再生回数の安定性 | 登録率1〜3%以上 |
  | 購買・問い合わせ | 商談や購入に繋げる | 概要欄クリック数／問合せ数／LINE誘導数 | リンクCTR | （目的により設定） |
  | 認知拡大 | アルゴリズムに好まれ再生を伸ばす | CTR（クリック率）／再生回数 | コメント率／いいね率 | CTR4〜6%以上 |

  2. 「全体マーケ戦略と同じ方向を向いているか」を必ず確認する（目的が曖昧だと企画・台本・サムネが全てブレる）
- 出力：運用目的選定書（主目的1＋副目的0〜1）

### ステップ2：他SNSとの役割分担を確定する

- 入力：運用目的選定書、業態タイプ
- 処理：
  1. YouTubeの位置づけを以下表で確認する：

  | 媒体 | 主役割 | ユーザー層 | アルゴリズム |
  |---|---|---|---|
  | Instagram | 認知〜検討層を幅広くカバー | 30〜40代女性多 | 世界観構築・関係性醸成 |
  | TikTok | リーチ拡大・短尺で認知 | 10〜20代 | 非フォロワーへ積極配信 |
  | YouTube | 検討層・ファン向け詳細情報・リピート施策 | 幅広い年齢層（60代以上も） | 視聴履歴・検索意図・長時間滞在重視 |

  2. YouTubeを「より深い情報提供で顧客のファン化に繋げるSNS」と位置づける
  3. 業態別の活用ポイントを確定する：
     - **BtoC**：商品を視覚的に訴求（使い方・ビフォーアフター・レビュー動画）
     - **BtoB**：導入事例・顧客インタビューの発信（成果や信頼を可視化）
     - **採用**：社内文化・価値観・社員ストーリーの発信
- 出力：役割分担書（YouTubeの主機能＋他SNSとの連携）

### ステップ3：ターゲットを「具体的な1人」まで絞る

- 入力：主要ターゲット、業態タイプ
- 処理：
  1. 以下3問を埋める：
     - **誰に届ける？**（属性＋悩み＋心理状態を1人分の具体ペルソナに落とす）
     - **何を解決する？**（その人の悩みを具体的に1つ言語化）
     - **どんな未来を与える？**（解決後にどうなるか）
  2. YouTubeは深い発信ができるからこそ、ターゲットは明確に絞る（「20〜30代女性」のような広い設定はNG）
- 出力：ペルソナ＋解決する悩み＋提供する未来

### ステップ4：トレンド・競合をリサーチする

- 入力：業態タイプ、主要ターゲット
- 処理：
  1. 同業種で伸びている動画投稿を確認し、何が注目されているかを抽出する
     - コメント欄から視聴者のリアルなニーズを抽出
  2. GoogleトレンドやSNSの「急上昇ワード」を確認する
  3. 1年前同時期の「季節性」もチェックする
  4. 登録者数と再生回数の相関を見て、登録者数が少なくても伸びている動画を分析する（ポテンシャル企画の発見）
- 出力：トレンド・競合リサーチ表＋伸びる切り口リスト

### ステップ5：ポジショニングで戦う場所を決める

- 入力：トレンド・競合リサーチ表、USP
- 処理：
  1. 以下3軸でYouTubeの立ち位置を決定する：
     - **競合と比較した自社の強み**
     - **伸びている投稿の要素を反映できるか**
     - **ユーザーに刺さる「視聴メリット」とは**
  2. 4象限マッピングで自社のゾーンを決める（縦軸：ストーリー型⇄コンテンツ型／横軸：業界情報⇄企業情報）
- 出力：ポジショニングマップ＋差別化ポイント

### ステップ6：コンセプトを一言にまとめる

- 入力：ペルソナ、ポジショニング
- 処理：
  1. 以下3要素を1文で表現する：
     - 誰に：具体ペルソナ
     - 何を：提供価値
     - 「こういう動画を出すチャンネルです」と言える一言
  2. 例：「中小企業の売上を改善するチャンネル」（誰に：SNSが苦手な中小企業の経営者／何を：売上アップにつながる実用的な施策を紹介）
  3. コンセプトに沿った企画タイトル例を3〜5個試作する
     - 例：「【再現性100%】売れる会社が必ず使う営業テンプレート公開」
- 出力：コンセプト一言＋企画タイトル試作3〜5個

### ステップ7：4機能をカスタマージャーニー別に配置する

- 入力：コンセプト一言、運用目的選定書
- 処理：
  1. 4機能の特性を以下表で確認する：

  | 機能 | 役割 | 特徴 | 成功ポイント |
  |---|---|---|---|
  | YouTube Shorts | 新規視聴者リーチ／長尺動画への導線／ブランド第一印象 | 縦型・スマホ前提・登録者外視聴中心 | 最初3秒で引きつける／10秒以内でテーマ明示／1テーマ1メッセージ／週3〜5本投稿 |
  | YouTube Long | ブランドストーリー・商品理解・信頼関係強化 | 5〜15分前後・検索/関連動画/登録者からの流入・視聴維持率重視 | 冒頭15秒でテーマと価値明示／導入→問題→解決→まとめ／字幕・テンポ最適化／サムネ・タイトル・概要欄で文脈明確 |
  | YouTube Live | リアルタイム交流・親近感・信頼感・ファン化 | コメント/質問/アンケート・滞在時間長／アーカイブ再利用可 | 目的明確化／構成台本／コメント拾い／事前告知／アーカイブ価値設計 |
  | YouTube Community | 視聴者との日常的接点・予告/振り返り/補足 | テキスト/画像/投票/GIF・更新頻度高く保ちやすい | 週1〜2回定期投稿／投票機能で参加感／コメント返信／舞台裏発信／動画連動 |

  2. カスタマージャーニー別に機能を配置する：

  | フェーズ | 推奨機能 |
  |---|---|
  | 認知 | Shorts／広告配信／インフルエンサー |
  | 教育 | Long |
  | 販売 | Live |
  | リピート | Community |

  3. リソース制約に応じて初期段階は1〜2機能に絞る（すべて同時稼働は失敗パターン）
- 出力：機能配置マトリクス＋優先順位

### ステップ8：目的別KPIを設計する

- 入力：運用目的選定書、機能配置マトリクス
- 処理：
  1. 主目的に対応するKPIをステップ1の表から抽出する
  2. 各KPIに以下4要素を必ず付与する：
     - 数値目標
     - 達成期限（月次／四半期）
     - 測定方法（YouTube Studio）
     - 検証頻度
  3. 副KPIを1〜2個追加する
- 出力：KPI設計書（主KPI1＋副KPI1〜2）

### ステップ9：施策3タイプから組み合わせる

- 入力：KPI設計書、コンセプト一言
- 処理：
  1. 以下3施策から組み合わせる：

  | 施策 | 目的 | 内容 | 主役割 |
  |---|---|---|---|
  | YouTubeショート | 認知拡大 | Tips動画／業界あるある豆知識／本編切り抜き／テンポ重視ショート解説 | 接触回数の最大化（アルゴリズムに乗せる） |
  | 本編動画（Long） | 興味関心・ファン化 | 事例紹介(Before→After)／Vlog型／検索ワード意識企画／シリーズ化（◯◯講座／◯◯解説） | 理解促進と信頼獲得 |
  | YouTube LIVE | ファン化 | Q&A配信／実演デモ／限定コンテンツ（裏側・制作工程）／ファン向けコミュニティ強化 | 関係構築と濃いファン形成 |

  2. 各施策の頻度・担当者・連動KPIを明示する
- 出力：施策プラン

### ステップ10：台本3ステップで構成を設計する

- 入力：企画タイトル、コンセプト一言
- 処理：
  1. 「YouTubeは構成力が8割」の前提で、以下3ステップで台本を作る：

#### ①オープニング（最初の1分・特に冒頭10秒）

   - **冒頭10秒に「引き込み要素」を必ず入れる**：意外性・強いユーザーメリットなど
   - **ロゴ・長い挨拶は禁止**：即離脱につながる
   - **タイトル＆サムネの内容を冒頭で回収**：視聴者は「期待通りか」を判断するため、結論の一部を最初に触れる
   - **オープニングは撮影後に作り直すと精度UP**：本編を撮った後にベスト導入を再構築する

#### ②本編（結論→理由→具体例→TTPS）

   - **最初に結論を伝える**：視聴者の「見る価値ある？」を最初に解消
   - **数字を使い説得力を上げる**：「3つの理由」「売上◯%改善」など
   - **具体例はできる限り「自社事例」に寄せる**：自分ごと化＋信頼感UP
   - **TTPSで「似てるのに新しい」動画にする**：トレンド企画の要素を抽出し、独自切り口を加えて刷新（TTPS = 徹底的にパクってさらに磨く）

#### ③エンディング

   - **CTAは1つだけに絞る**：「資料請求」「別動画へ」など1つに
   - **行動内容は具体的に指示する**：「概要欄の◯◯をクリック」など迷わせない
   - **視聴者の「温度感」に合わせる**：認知目的なら「別動画へ」、業務目的なら「問い合わせへ」

  2. **ユーザーメリットを全要素（サムネ／タイトル／3秒フック／本編／エンディング）に必ず入れる**：
     - 「数字」で示す（売上◯%UP、コスト△円削減）
     - 冒頭で提示する
     - メリット曖昧な企画は二番煎じになる
     - 「視聴者にとっての利益」に変換する（自社都合の話ではなく、視聴者の悩みがどう解決されるか）

  3. 台本は「原稿」ではなく「順番の設計図」のイメージで、全文を書く必要はない（話す順番だけ整理）
- 出力：台本（オープニング／本編／エンディング3パート構成）

### ステップ11：タイトル＆サムネを設計する

- 入力：台本、コンセプト一言
- 処理：「視聴者が動画を開くかは"タイトル＆サムネ"で9割決まる」前提で、以下を満たす：
  1. **メリットを最短ワードで提示**：1メッセージ勝負（「売上UP」「失敗回避」など1番の価値を短く）
  2. **数字・具体性・ターゲットを入れて訴求力UP**：「売上◎%UP」「30代経営者向け」など誰に何が起きるか明確化
  3. **企画・タイトル・サムネを一致させる**：サムネで使った言葉は動画内で必ず言う（ズレは離脱率上昇）
  4. **サムネのテキストは10文字以内に圧縮**
- 出力：タイトル案＋サムネテキスト案

### ステップ12：撮影3要素を整える

- 入力：台本、撮影リソース
- 処理：撮影品質を上げるための3要素を必ずチェックする：
  1. **光**：逆光を避ける／顔・被写体に光を当てる
  2. **音**：雑音を減らす（マイク使用推奨）
  3. **背景**：シンプルにする（情報過多は集中を散らす）
  4. **テロップ位置を想定して撮影する**：顔や重要情報が字と被らない構図に
  5. **カメラ角度と「間」を意識する**：視聴者目線の高さに合わせ／無駄な間はカット前提でテンポよく
- 出力：撮影チェックリスト

### ステップ13：YouTube Studioで5基本指標を分析する

- 入力：投稿実績データ
- 処理：以下5指標を確認する：

  | # | 指標 | 何を示すか |
  |---|---|---|
  | 1 | 再生数 | テーマの強さ |
  | 2 | CTR（クリック率） | サムネ・タイトルの良し悪し |
  | 3 | 平均視聴時間 | テーマ／台本の良し悪し |
  | 4 | 視聴維持率 | オープニング・構成の強さ |
  | 5 | チャンネル登録数 | 興味関心層の獲得力 |

- 出力：5指標サマリー

### ステップ14：課題別の改善策を5パターンから適用する

- 入力：5指標サマリー、目的別KPI
- 処理：以下表で課題を特定し改善策を1つだけ選ぶ：

  | 課題 | 主原因 | 解決策 |
  |---|---|---|
  | CTRが低い | タイトルメリット弱／サムネ抽象的／誰向け不明瞭 | メリットを最短ワードで／数字・具体性・ターゲット明示／サムネテキスト10文字以内 |
  | 再生数が低い | 企画テーマ弱／検索ワード未挿入／ニーズ調査不足 | 競合ヒット企画10本分析／検索キーワードから企画／今月伸びているワード挿入 |
  | 視聴維持率が低い | オープニング弱／結論提示遅い／話が冗長 | 冒頭3秒で結論／先に「今日の成果」宣言／間カットでテンポ改善 |
  | 登録者が増えない | 価値提供質弱／再現性低／学びが抽象的 | 「結論→理由→具体例→TTPS」で構成改善／サムネと内容完全一致／見返す価値ある情報量 |
  | コメントが少ない | CTA曖昧／質問投げかけなし／アクション選択肢少 | 動画最後に1つだけ質問／コメントしやすい2択質問／CTA毎回固定化 |

- 出力：改善プラン（課題1＋原因仮説＋次に試す改善1）

### ステップ15：5基本チェックで動画を品質管理する

- 入力：制作した動画台本
- 処理：投稿前に以下5項目を必ず確認する：
  - [ ] この動画は「誰の・どんな悩み」を解決する？
  - [ ] メリットは「数字で」表現されている？
  - [ ] 冒頭3秒に「フック」はある？
  - [ ] タイトル・サムネ・内容は完全に一致している？
  - [ ] 台本は「結論→理由→具体例→TTPS」になっている？
- 出力：品質チェック済みフラグ

### ステップ16：PDCA運用体制を構築する

- 入力：チームメンバー、投稿頻度
- 処理：以下4フェーズで運用サイクルを設計する：

  | PDCA | 内容 |
  |---|---|
  | PLAN（計画） | 週1本投稿前提でテーマ・台本準備／追いかけるKPI明確化（CTR・再生数・維持率）／チームで「目的」統一 |
  | DO（実行） | 台本に沿って撮影・編集・投稿／同じフォーマットで品質安定化／必ずコメント対応 |
  | CHECK（検証） | 週ごとに分析（CTR／再生数／維持率）／月1回のKPI振り返りミーティング／「数値の良し悪し＋原因仮説」を共有 |
  | ACTION（改善） | 改善点を次の動画に反映／「完璧より継続」をチームルール化／同じ指標を見ることで意思決定が早まる |

- 出力：運用体制設計書

## 判断基準

| 状況 | 判断 | 理由 |
|---|---|---|
| 目的が曖昧（「とりあえずやる」） | 目的1つに絞ってから始める | 企画・台本・サムネが全てブレる |
| 業態がBtoB | 信頼醸成・教育を主目的に | 導入事例・専門知識動画と相性◎ |
| 業態がBtoC（D2C等） | 認知拡大→興味関心層獲得を主目的に | 商品ビジュアル訴求とリーチ拡大の両立 |
| 業態が採用 | 信頼醸成（社内文化可視化） | 応募者の安心感に繋がる |
| ターゲットが「20〜40代女性」など広い | 「具体的な1人」まで絞る | YouTubeは深い発信媒体・ぼやけたターゲットでは届かない |
| 4機能を全て同時に始めたい | 初期は1〜2機能に絞る | リソース分散で全機能の質が下がる |
| 視聴維持率40%未満 | 冒頭3秒・構成（結論→理由→具体例）を改善 | 維持率はオープニング・構成の強さの指標 |
| CTR 4%未満 | サムネ・タイトル・誰向けかを改善 | CTRはサムネ・タイトルの良し悪し |
| 再生数が伸びない | 企画テーマ・検索ワード・トレンドを見直す | 再生数はテーマの強さ |
| 登録者増えない | 「再現性」「見返す価値」を見直す | 登録は興味関心層の獲得力 |
| コメント少ない | 動画最後に1つだけ質問を入れる | アクション選択肢を提示 |
| オープニングに長い挨拶やロゴ | 削除→冒頭10秒で結論・メリット | 即離脱の典型 |
| CTAを複数入れたい | 1つに絞る | 散らばると行動率が下がる |
| 「完璧な動画」を作ろうとして投稿が止まる | 「完璧より継続」をルール化 | 改善はPDCAで回す前提 |
| 改善で複数要素を同時に変える | 1つずつ変える | 効果要因が判別不能になる |

## よくある失敗パターン

| NG | OK | 理由 |
|---|---|---|
| 目的を決めずに「とりあえず投稿」 | 4目的から1つ選んでから始める | 企画・台本・サムネが全てブレる |
| ターゲットを「20〜40代」など広く設定 | 「具体的な1人」まで絞る | YouTubeは深い発信媒体・ぼやけたターゲットでは届かない |
| 4機能を全て同時に始める | 初期は1〜2機能に絞る | リソース分散で全機能の質が下がる |
| オープニングに長い挨拶やロゴ | 冒頭10秒で結論・メリット | 即離脱の典型 |
| 1動画に複数メッセージ詰め込み | 1メッセージに絞る | 何が大事か視聴者が理解できない |
| サムネと動画内容がズレる | サムネで使った言葉は動画内で必ず言う | 離脱率上昇＋アルゴリズム評価低下 |
| サムネテキストが長い（15文字以上） | 10文字以内に圧縮 | スマホで読めない |
| メリットが抽象的（「ためになる」「役に立つ」） | 数字で表現（売上◯%UP・工数△時間削減） | 価値が伝わらず二番煎じになる |
| CTAを複数入れる（登録・コメント・別動画・LP） | 1つに絞る | 散らばると行動率が落ちる |
| 改善で複数要素を同時に変える | 1要素ずつ変えて検証 | 効果要因が判別不能 |
| 「完璧な動画」を作ろうとして投稿が止まる | 「完璧より継続」をチームルール化 | YouTubeはPDCAで伸びる |
| 撮影で光・音・背景を軽視 | 3要素を整える | 視聴維持率が下がる典型 |
