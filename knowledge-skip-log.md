# ナレッジ化スキップログ

アスカが「ナレッジ化しますか？」と確認したが、シンヤさんが「しない」と答えた事例を記録する。
アスカはこのログを参照して、確認するか・しないかの判断精度を上げていく。

## フォーマット
```
### YYYY-MM-DD
- **内容**：何についての確認だったか
- **理由**：なぜナレッジ化しなかったか
- **学習**：次回から何を変えるか（アスカが記入）
```

## ログ

### 2026-04-15
- **内容**：Shopifyで単品商品をセット化した際、セット商品側の配送設定が有効化されておらず送料が個数連動しなかった事例（トリガー①問題解決）
- **理由**：まれなケース（単発の設定ミス）であり、汎用的な知見として残す価値が薄い
- **学習**：「単発の設定ミス」系は、同種の再発可能性が低ければ確認自体をスキップしてよい。再発性・汎用性のあるパターン（仕様上の落とし穴、複数クライアントで起きうるもの）に絞って確認する
