## 0. スワイプLP設計原則（再設計の核）

### 0-1. index.htmlから確定した設計制約

```
画面全体の3層固定レイアウト:
┌─────────────────────────────┐
│  上部ページネーションバー    │  固定 36px / 「1 / 7」表示
├─────────────────────────────┤
│  スワイプ領域                │  Swiper.js / 画像1枚フル表示
│  （残り高さすべて）           │  object-fit: cover
│  スワイプ方向: 水平（左右）   │
├─────────────────────────────┤
│  下部CTAバー                 │  固定 80px / 全スライド共通
│  （position: fixed, bottom）│  ボタン + クレジット表示
└─────────────────────────────┘
```

各スライドの実体は **`slide-01.webp`〜`slide-07.webp` の画像1枚**。Canva等のデザインツールで4:5キャンバス（推奨1080×1350px）として書き出す。

### 0-2. 設計原則

| 原則 | 内容 |
|---|---|
| 1スライド = 1画像 | 連番slide-01〜07.webp / 4:5アスペクト比 |
| CTAボタン | index.html下部固定バーが全スライド共通担当・各スライド画像内に描かない |
| スワイプ方向 | 水平（左右） / Swiper.jsデフォルト |
| 各スライドの情報量 | 4:5キャンバスに収まる量のみ |
| スワイプヒント | 初回3秒「← スワイプ →」自動表示 + 画像内最下部帯に補助誘導 |

### 0-3. 前回設計からの根本変更

| 項目 | 前回（誤り） | 今回（正しい） |
|---|---|---|
| スライド実体 | HTMLセクション（縦スクロール可能） | 画像1枚（スクロールなし） |
| CTAボタン | Slide 4/6/7に複数配置 | 下部固定バーに1つのみ |
| スワイプ方向 | 縦スワイプ前提 | 水平（左右）スワイプ |
| 情報量 | 縦伸び可能 | 4:5キャンバス制約 |

## 3. 各スライド設計詳細

### Slide 1: Hook
- 役割: 1秒での自己投影
- 構成: 上部55% slide-01.webp（窓辺女性後ろ姿・コーラルOR薄オーバーレイ10%） / 下部45% 白帯にコピー
- コピー: ラベル「コーダー上田から、デザイナーさんへ」/ 大見出し「コーディングが**苦手**なデザイナーさんへ」/ サブ「外注費、もっと減らせるかもしれません。」
- クリフハンガー: 「かもしれません」の余韻
- 4:5判定: ✅ OK

### Slide 2: Problem
- 役割: 課題3連打
- 構成: `#FDF8F5`背景にカード3枚縦列＋ブリッジ（コンパクト化）
- カード仕様: 高さ最小化（カード1枚あたり16〜18%）/ 左ボーダー2px コーラルOR / 余白8px最小
- ブリッジ: 「こういう場面、わたしも色んなデザイナーさんと仕事しながら、ずっと見てきたんです。」（14px・グレー斜体右寄せ）
- クリフハンガー: 「わたしも」が「誰？」を喚起
- 4:5判定: ✅ OK（タイト）

### Slide 3: Empathy×Authority
- 役割: 協業者ポジション確立 / 警戒解除
- 構成: 上部30% slide-06.webp（暗室キーボード・コーラルORオーバーレイ40%）/ 中下部70% `#FDF8F5`に接続コピー＋人物ブロック＋Empathy＋Authorityバッジ
- 削除: 「配布動機」コピーをSlide 3から省略（Empathyで温度感担保）
- Authorityバッジ: 横並び2個（「Web制作 4年」「制作サイト 30以上」・コーラルOR背景白文字）
- 顔写真: 80px丸型（**新規必要・シンヤさん素材提供**）
- クリフハンガー: 「これ」が目的語のまま → Slide 4で答え
- 4:5判定: ✅ OK

### Slide 4: Reveal
- 役割: 解決策開示 / 「欲しい」発火
- 構成: 全画面 slide-04.webp（コーラルORオーバーレイ15%）/ 下部30%白帯（透明度90%）にお客様の声
- 削除: CTAボタン（index.html固定バーが担当）/ 補足画像配置
- お客様の声: コーラルOR左ボーダー / 「── いつも一緒に仕事しているデザイナーさん（実体験・1名）」明示
- クリフハンガー: 「画像を差し替えるだけ」→「どうやって？」
- 4:5判定: ✅ OK

### Slide 5: How it works
- 役割: 「自分でも使える」の確信
- 構成: `#FDF8F5`背景にステップ4枚＋ライセンス帯＋(c)層一言（最下部極小）
- コンパクト化: ステップ番号左配置＋テキスト2行 / ライセンスをアイコン＋テキスト横並び1行 / (c)層一言を11px最下部に
- 4:5判定: ⚠️ タイト（フォント14px・余白8px最小で実装時調整必要）
- **要シンヤさん判断**: 「7枚維持コンパクト化」or「Slide 5-A/5-B分割で8枚化」

### Slide 6: Meta-Proof
- 役割: このLP自体が証拠
- 構成: `#FDF8F5`背景・コーラルOR左アクセントライン4px / 大見出し＋7枚ミニチュアサムネイル横並び（Slide 6位置を枠強調）＋サブコピー＋powered by＋(c)層一言（最下部極小）
- 採用: **案4（構造解剖図）単独採用**（案2コードビューは可読性低・案1は単独で弱い）
- クリフハンガー: 「見てもらえればわかります」→「見た。じゃあどうするの？」
- 4:5判定: ✅ OK

### Slide 7: CTA（全画面CTA画面）
- 役割: 行動ガイド（Chatwork ID + 合言葉例文）
- 構成: 全画面 slide-08.webp（チャットUI・コーラルORオーバーレイ35%）/ 全テキスト白系で「ゴール画面感」演出
- 配置: 大見出し（白26px）/ 手順説明（白16px）/ Chatwork IDブロック（白背景角丸）/ 合言葉例文（グレー破線枠・透過85%白背景）/ Zoom補足（白11px透過70%最下部）/ フッター（白10px透過50%）
- **CTAボタンは画像内に描かない**（index.html下部固定バーが担当）
- 4:5判定: ⚠️ タイト（合言葉例文を枠内コンパクト配置で対応）

## 4. 視線誘導とクリフハンガー設計

### 水平スワイプLPの視線誘導

縦スクロールLPのFパターンとは異なる。各スライドは「1枚の画像の額縁」内で完結し、次へ進むには意図的なスワイプが必要。

### スワイプ動線上の信頼蓄積

```
Slide 1（Hook）→ Slide 2（Pain）→ Slide 3（Auth・第1信頼点）
→ Slide 4（Rev・第2信頼点）→ Slide 5（How）→ Slide 6（Meta・第3信頼点）→ Slide 7（CTA）
```

各スライド最下部に統一スワイプ誘導を配置（Slide 1〜6）。Slide 7はゴールのため誘導不要。

| Slide | 最下部誘導テキスト |
|---|---|
| 1 | 「スワイプして読む →」 |
| 2 | 「→」（矢印のみ） |
| 3 | 「→」（矢印のみ） |
| 4 | 「使い方は？ →」 |
| 5 | 「このLP、実は →」 |
| 6 | 「受け取り方は →」 |
| 7 | なし（ゴール画面） |

## 6. CTA設計（Slide 7のみ・1点集中・index.html連動）

### 6-1. CTAの実装主体はindex.htmlの固定バー

- 高さ: 80px（`--cta-height: 80px`）
- 配置: `position: fixed; bottom: 0`（全スライド常時表示）
- ボタン: `#cta-button`（最大幅480px・角丸6px・コーラルオレンジ背景・白文字）
- クレジット: ボタン下「Template by オフィスウエダ」（HTML5 UPモデル / Meta-Proof補完）

### 6-2. ボタンラベル

「Chatworkでテンプレを受け取る」（15文字・copy.md確定）

### 6-3. Chatwork deep link

**推奨**: Web URL `https://www.chatwork.com/#!rid{シンヤさんID}`（アプリ未インストールでも動作）
**設定**: シンヤさんが index.html の `href` を直接編集（READMEに手順記載推奨）

### 6-4. 各スライド画像内のCTAボタン禁止

Slide 4補助CTA・Slide 6テキストリンクCTAは**全削除**。理由: index.html下部固定バーが常時表示しているため2つのCTAになり混乱を招く。スワイプLPは「次が見たい」動線を維持するのが原則。

### 6-5. シュウへのindex.html設定指示

```html
<meta name="lp-ratio" content="4:5">
<meta name="cta-bg-color" content="#E8694A">
<meta name="cta-text-color" content="#ffffff">

<a id="cta-button" href="https://www.chatwork.com/#!rid{シンヤさんID}" rel="noopener noreferrer">
  Chatworkでテンプレを受け取る
</a>
```

## 7. レスポンシブ・パフォーマンス

### 7-1. index.htmlのレスポンシブ実装

スマホ: 幅100%基準 / PC: 高さ基準で中央寄せ（`calcSliderSize()`関数）/ アスペクト比4:5自動維持。各スライド画像を4:5で書き出せばスマホ・PC両対応が自動完結。

### 7-2. 画像書き出し仕様

| 項目 | 仕様 |
|---|---|
| 形式 | WebP |
| アスペクト比 | 4:5（推奨1080×1350px） |
| ファイルサイズ | 1枚200KB以内（FV表示速度） |
| ファイル名 | `slide-01.webp`〜`slide-07.webp` |
| loading | `eager`（index.html設定済み） |

### 7-3. パフォーマンス目標

- LCP 2.5秒以内 / Slide 1画像 200KB以内 / 全体 1MB以内

## 8. スライドトランジション

- 方向: 水平（`direction: 'horizontal'`）/ index.htmlデフォルト
- タイプ: `effect: 'slide'`（フェードは指追従感が低いため不採用）
- 速度: **350ms**（デフォルト300msより若干ゆっくり・「ページをめくる」感）
- シュウへの指示: index.htmlのSwiper初期化に `speed: 350` 追加のみ

## 9. 進捗可視化・スワイプ誘導

### index.html既存実装
- ページネーションバー: 上部固定36px・「1 / 7」形式
- SwipeHint: 初回3秒「← スワイプ →」フェードアウト
- ループ: なし（`loop: false`）/ Slide 7で終了

### 推奨改修（任意）
ページネーションバーをコーラルオレンジ薄背景・チャコールテキストに調整:
