## 設計全体方針

### カラー設計（全スライド共通）

```
プライマリ        #E8694A（コーラルオレンジ）  CTAボタン・アクセント・見出し下線
バックグラウンド  白 #FFFFFF または ウォームオフホワイト #FDF8F5
テキスト          チャコール #2D2D2D
サブカラー        ライトコーラル #F5B09A  グラデーション・帯背景に使用
強調語            コーラルオレンジ #E8694A（太字と色で二重強調）
禁止              青・緑系の寒色を組み合わせない。コーディング=暗い=黒背景は回避
```

**なぜこの方針か**: コーラルオレンジは「暖かさ・親しみやすさ・エネルギー」を喚起する色相。コーダー(=技術者)という文脈に冷たいイメージを持つ受け手の先入観をカラーで最初から崩す。デザイナーのポートフォリオ審美眼に耐えられる洗練度が必要なため、彩度を少し落とした Dusty Coral トーンを基準とする。

### スライド共通レイアウト仕様（モバイルファースト）

```
画面サイズ想定:    390px幅（iPhone 14基準）
縦高さ:            100svh（セーフエリア込み）
左右パディング:    24px
上下パディング:    48px
コーナーラジアス:  16px（カード等）
フォントサイズ基準:
  大見出し:     28〜32px / font-weight 800
  中見出し:     20〜22px / font-weight 700
  本文:         15〜16px / font-weight 400 / line-height 1.8
  強調語:       コーラルオレンジ + font-weight 700
スワイプ誘導インジケーター: 各スライド最下部に「›」または進行ドット（3px × 7個）
```

### ビジュアル指針（Slide 1）

**既存画像 slide-01.webp の活用方針**: 「窓辺でぼんやり座る女性の後ろ姿」で、感情的孤立・行き詰まり感を表現。Slide 1 の「コーディングに手が止まっている」感情状態に合致。上部60%をクロップして画面上部に配置し、下部に白帯を重ねて文字を載せる。

- 「コーディングが苦手」= 立ち尽くす後ろ姿が無言で語る（セリフ不要）
- 「外注費を減らしたい」= テキストで理性に訴える

**スワイプ誘導の視覚仕掛け**: スライド最下部に「右スワイプ or 下スワイプ」のアイコン＋ドットインジケーターを配置。「続きがある」と直感させる。

### ビジュアル指針（Slide 3）: コーダー職能アイデンティティのビジュアル化 — 最低2案

**前提**: 「自分もデザイナー出身」「同じ壁を経験」等の捏造表現は絶対禁止（§8-1 厳守）

**案A: 「顔写真 + コードエディタ背景」構成**

背景: 薄くぼかしたコードエディタのスクリーン（実際のコード画面をモノクロ / 30%透明度で背景に敷く）
中央前景: シンヤさん顔写真 丸型クロップ
顔写真の真下: コーラルオレンジのタグバッジ「Coder」「4年 / 30+ sites」

ポイント: 「コーダー = コードを書く人」をビジュアルで証明。背景のコード画面がアイデンティティの証拠として機能する。顔写真が前景にあることで親しみやすさを担保しつつ、コーダー資格を背景で主張する「見せ方のプロ感」がデザイナーに刺さる。

**案B: 「手元タイピング写真 + 顔写真の組み合わせ」構成**

上半分: slide-06.webp（キーボードをタイプする男性の手元・暗室）にコーラルOR半透明オーバーレイ
下半分: 白背景エリア + 丸型顔写真（左）+ 名前（右）+ 「4年・30サイト」バッジ

ポイント: slide-06.webp（キーボードタイプ・コード画面・暗室）は「コーダーとはこういう仕事をする人だ」という職業ビジュアルとして非常に適切。既存画像を上半分に再利用し、コーラルオレンジのオーバーレイを重ねることでカラーを統一しながら「コーダーの現場感」を演出する。既存画像の最高評価を生かすベスト選択。

**カイ推奨**: 案B（既存 slide-06.webp の再利用 + 顔写真）

理由: slide-06.webp の「暗室・キーボード・コード画面」は「本物のコーダーが書いている」を最も直感的に伝える。既存の最良評価画像を Slide 3 でも活用することで、新規生成コストゼロでコーダーアイデンティティを確立できる。

**既存画像 slide-05.webp の活用**: ノートPC画面に手をかざしているプレゼンテーション・操作感の写真。「操作している感じ」「分かりやすく説明している感じ」として How it works に連動。背景装飾として上部に配置。

**ステップUIのデザイン**: 各ステップ左に丸番号アイコン（コーラルOR塗り・白数字）、右にステップ内容。左ボーダーの縦線でステップを繋ぐことで「流れ」を視覚化。

### メタ証明ビジュアル仕掛け — 全4案

**案1: 「powered by swipe-lp template」透過表記**
スライド右下の隅に、コーラルオレンジ / 透明度60% / 10px 極小テキストで「powered by swipe-lp template / コーダー上田 / officeueda.com」を常時表示。全スライドに同じ表記を入れることも可能（認知固定効果）。HTML5 UPモデル応用（§6-4）に直結。

**案2: 「コードビュー風の演出」**
スクリーン風の枠で囲む。背景: #1E1E1E（VSCode風）、文字: グリーン / ホワイト。
「<section class="slide-6"><!-- このLPはコーダー上田が作ったテンプレで動いています --><img src="images/slide-06.webp" alt="Meta-Proof"></section>」
「コーダーが作ったという証明」をコードの形式で見せる。デザイナーにとって「コードって、こういうものか」と覗き穴になる好奇心刺激の仕掛け。

**案3: 「Canva で作ったと種明かし」**
Before（Canvaの編集画面）→ After（このLPのスライド）並列表示。矢印で繋ぐ。
「このスライドの画像、全部Canvaで作って差し替えただけです。」
Canvaユーザーの(b)層に「自分でもできる」をリアルに証明する。

**案4: 「スワイプLP の構造解剖ビジュアル」（カイ追加）**
縦7枚スライドのミニチュア一覧図。「[1] Hook ←今見てる」「[6] ← ここ！ Meta-Proof」と現在地を表示。
「このLP、7枚構造で全部テンプレで作っています。同じ構造を、あなたも使えます。」
読者がこのLPをスワイプしてきた体験そのものが「テンプレを体験している」ことに気づかせるメタ構造。

**カイ推奨案**: 案2（コードビュー）+ 案4（構造解剖）のダブル採用

理由: 案2はコーダーアイデンティティの直接証明、案4は「読者の体験がすでに証拠」という知的快感。2案を上下に並べることで、コーダー視点とユーザー体験視点の両面からメタ証明が完成する。Slide 6 の高さが足りない場合は案2のみ採用し、案4は省略可。

### ビジュアル指針（Slide 6）

**既存画像の活用**: slide-06.webp（キーボードタイプ・コード画面）は案2のコードビュー演出の背景として最適。モノクロ化またはコーラルオレンジオーバーレイを重ねてアクセントに使用。案3を採用する場合は新規 Canva 編集画面スクリーンショットが必要。
