## 使い方

- 案件ごとに直下にサブフォルダーを切る（例: `workspaces/<site-name>/`）
- 既存クライアントに紐づくものは `clients/<client>/site/` 側に置く（ここには置かない）
- 一時的な修正・検証・素振り用。長期運用に育ったら `clients/` に昇格させる
- 上記「現在のプロジェクト一覧」にない新規プロジェクトを追加したら、**必ずこの表に追記する**

## ディレクトリ命名ルール

- 英小文字ハイフン区切り（例: `acme-corp-top`, `sample-lp-fix`）
- 判別可能な識別子を付ける（`tmp` や `test` 単体は避ける）

## 注意

- `.git` を個別に切るかはサイトごとに判断（親リポジトリの追跡対象にしない場合は `.gitignore` 側で対応）
- ここに置くプロジェクトは **CLAUDE.md「Web Project Directory Structure Rule」「CSS Coding Rule」の適用対象外**（素振り・検証・配布テンプレート用途のため）。本運用に昇格する際に準拠化する
