# chisoku スキル インデックス

## 用途

- アスカが依頼受領時に Read して、該当 chisoku スキルがあるかを照合する
- 委任先エージェントに依頼を渡す際、該当スキル名を依頼文に明記する
- 月次棚卸し（morning-briefing-weekly）で「使われなかった chisoku スキル」を検知する

## 凡例

- **主管**: そのスキルを「業務遂行のため」に呼び出す中心エージェント
- **副次**: 整合確認や副次的な参照のために Read するエージェント
- 「全エージェント」表記は思考・整理用途で誰でも使えるもの

## 利用ガイド（アスカ運用）

### 依頼受領時のチェックフロー

1. シンヤさんから依頼を受けた瞬間、本ファイルを Read（既に context 内なら不要）
2. 依頼内容のキーワードから該当カテゴリを特定（例：「広告のCPA改善」→ カテゴリ5）
3. 該当スキルが存在する場合、委任先エージェント（主管）に依頼を渡す際、依頼文に**該当スキル名を明記**する
   - 例：「ヒカル、Meta広告のCPA改善診断をお願いします。`ad-performance-diagnosis` スキルを使ってください」
4. 該当スキルが存在しない場合、委任先エージェントの判断に委ねる

### 月次棚卸し（morning-briefing-weekly 第1日曜）

1. 直近1ヶ月の依頼ログ（Notion 案件管理）から、各 chisoku スキルの利用回数を集計
2. 「使うべきだったのに使われなかった」ケースを検知
   - 判定基準：依頼内容のキーワードがスキルの適用シーンと一致するのに、委任時にスキル名が明記されていなかった
3. シンヤさんに月次レポートで報告

### 新規スキル追加時の自動更新（実装予定 / Phase 2）

- `chisoku-skillize` スキル末尾に「インデックス自動追記ステップ」を組込
- 新規スキル化完了時、本ファイルに自動で行追加（カテゴリは仮置き、アスカが後追いで本配置調整）
