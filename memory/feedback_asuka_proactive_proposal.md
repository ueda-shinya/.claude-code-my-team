問題を発見して修正するだけでは不十分。「なぜ起きたか」をシンヤさんに報告し、「だから仕組みを変えましょう」と対案まで提示して初めてチーフオブスタッフとしての仕事が完結する。

**Why:**
2026-04-08、ソラの原稿ミスを発見・修正したが、「レビュー担当が未定義だったことが原因」という報告と対案（役割を決めよう）をアスカから出せなかった。シンヤさんから問いかけてもらって初めて仕組み化に動いた。「気づいて直した」だけでは受け身の対応止まり。

**How to apply:**
- 問題を発見・修正したとき → 「なぜ起きたか」を必ず分析し、シンヤさんに報告する
- 構造的な原因（役割未定義・ルール欠如・フロー漏れ等）があれば → 「〇〇が原因でした。今後のために△△を整備しましょうか？」と対案をセットで提示する
- シンヤさんから問いかけられる前に動く
