# リナ検証のリスク評価付き運用（2026-04-18 合意）

## ルール
リナに検証依頼する際、**【要修正】判定にはリスクレベル（1-5）を必ず併記**させる。シンヤさん判断で以下の閾値を適用：
- **リスク3以上**: 対処必須
- **リスク2以下**: 許容（運用中に気づいたら対応）

**Why:** リナは完璧主義モードに入ると無限に詳細化指摘を出し続け、議論が収束しない。2026-04-18 の CLAUDE.md External Skill Guard Rules 審議でリナ7回レビューを要したことから導入。リスク評価を併記させることで、客観的閾値で終結判断が可能になった。

**How to apply:**
- リナ召集プロンプトに「要修正の場合はリスクレベル1-5を必ず併記」を明記
- 指摘が複数ある場合は各指摘ごとにリスクレベルを併記
- シンヤさん判断で閾値以下は「許容」扱いとし、ルール本文に「既知の限界（許容）」として記録

## 適用実績
- 2026-04-18 CLAUDE.md External Skill Guard Rules: リナ7回 → 閾値導入後に承認
- 2026-04-18 gsc-ga4-analyzer rev.2 → rev.2.1: リナ4回 → リスク3/4対処で承認
- 2026-04-18 feature-flow ステップ2.5追加: リナ2回 → リスク3/4対処で承認

## 注意
- リスク評価はリナの主観だが、根拠を併記させることで客観性を担保
- シンヤさんセッション中はエスカレーションリミットなし（CLAUDE.mdルール）
- シンヤさん不在時は初回+2回=3回までで差分提示（CLAUDE.mdルール）
