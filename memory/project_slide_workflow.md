## 確立したフロー（2026-04-07）

**ソラ（slide-designer）が原稿作成 → Gensparkで生成**

1. アスカがソラに依頼（テーマ・対象読者・内容・使用ツールを渡す）
2. ソラが3フォーマット（Gamma用・Genspark用・汎用）で原稿を出力
3. シンヤさんがGensparkにプロンプトを貼り付けて生成

**Why:** HTMLを手書きするよりGensparkの方がデザイン品質が高い。ソラが構成を設計することで情報の正確性が確保できる。

## Gensparkプロンプトに必ず入れる注意書き

```
注意：存在しない機能・サービス・時刻・数値をGensparkが独自に追加しないこと。
指定した内容のみで構成すること。
```

Why: Gensparkは内容を"盛る"傾向がある（過去に「メール通知」「毎朝9:00実行」などを勝手に追加）

## 比較検証結果（3パターン比較）

| | アスカのHTML | 最初のPDF（Genspark単独）| ソラ原稿＋Genspark |
|---|---|---|---|
| デザイン | ★★★★☆ | ★★★★★ | ★★★★★ |
| 正確性 | ★★★★★ | ★★★☆☆ | ★★★★☆ |
| 構成 | ★★★★☆ | ★★★☆☆ | ★★★★★ |

→ ソラ原稿＋Gensparkが現時点で最高品質

## How to apply
スライド制作依頼が来たら、必ずソラに原稿を作らせてからGensparkに投げる。
Gammaも同じフローで使える（月$15のコストを許容するタイミングで試す価値あり）。
