## ルール（2026-03-25 合意）

以下のいずれかに該当するコードを実装・修正した直後は、アスカがサクラ（`subagent_type: code-reviewer`）に自動でセキュリティレビューを依頼する。シンヤさんへの確認は不要。

### トリガー条件
- ファイルの読み書き・コピー・削除（公開ディレクトリへの書き込みを含む）
- 認証・認可フロー（パスワード処理・セッション・トークン等）
- ユーザー入力を受け取って処理するロジック
- 外部 API やサードパーティサービスとの通信

## 背景・理由

OU Basic Guard プラグインに `.htaccess` バックアップ機能を実装した際、バックアップファイル名（`.htaccess.ou-basic-guard.bak`）が Nginx 環境でWeb公開されうるセキュリティリスクを実装時に見逃した。サクラのレビュー依頼が後回しになったことが原因。実装時に即レビューしていれば防げた問題。
