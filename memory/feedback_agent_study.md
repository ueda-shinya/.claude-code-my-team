## ルール（2026-03-28 シンヤさんと合意）

`/agent-study` をシンヤさんに提案・実行する前に、アスカが必ず以下を判断する。

### 判断基準（2026-03-28 更新・精緻化）

「Claudeが知っているかどうか」だけで判断するのは不十分。
**「行動を縛る効果があるか」も合わせて判断すること。**

| 内容の種類 | 例 | 価値 |
|---|---|---|
| 知識の定義 | PESTとは・SWOTとは | ❌ Claudeが元々知っている |
| **行動指示・順序ルール** | **この順番で進めること・必ず〇〇する** | ✅ 知っていても指示がないと従わない |
| チェックリスト・テンプレート | 課題定義シート・アウトプット形式 | ✅ 一貫性が生まれる |
| 固有知識 | シンヤさんの成功パターン・客層 | ✅ Claudeが知らない情報 |

**学ばせても意味がない → 提案しない・止める**
- フレームワークの定義説明（SWOT・STP・4Pとは何か等）
- 業界標準のノウハウ・教科書的内容の説明

**学ばせる価値がある → 提案してOK**
- 「必ずこの順番で」「〇〇の前に△△をすること」等の行動指示
- 課題定義シート・チェックリスト・アウトプットテンプレート
- シンヤさん固有の判断基準・成功/失敗パターン
- officeueda 独自の価格設計・顧客層・商圏情報

### 背景

- マーケ教科書20章・即決営業マニュアルをレン・タクに学習させたが、Claudeが元々知っている内容だったため効果ゼロ
- 結果として無駄なトークン（25,000+）が積まれ、シンヤさんの作業コストも発生した
- アスカが「読ませよう」と提案したことが原因。事前チェックが甘かった
