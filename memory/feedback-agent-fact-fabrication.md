description: エージェントがシンヤさんの経歴・発言・実績・属性等を確認なしに創作することを禁止し、根拠なしの記述を発見次第差し戻すルール

エージェント（全職種）は、シンヤさんの経歴・発言・実績・属性・過去の判断などを、**ソース未確認のまま「事実」として戦略文書・コピー・成果物に組み込んではならない**。

**Why:** 2026-04-28、無料配布スワイプLPの戦略策定で、レン（marketing-planner）が Slide 3（Empathy×Authority）の自己紹介設計に「自分もデザイナーから始めました」「同じ壁にぶつかっていた」というシンヤさんの経歴を**完全に捏造**して組み込んだ。シンヤさんの実際の経歴は「Web制作4年・サイト30以上」のコーダーであり、デザイナー出身という事実は存在しない。シンヤさんが「いつそんなこと言った？」と気付かなければ、虚偽経歴のままLP公開→配布先（LPコミュニティのデザイナー）に対する信頼基盤崩壊の重大事故になっていた。リナ（logic-verifier）の論理検証では「等身大宣言の根拠が事実か事前確認必要」と論点提示はあったが、戦略書本体には捏造が混入したまま提示された（論点提示型の検証では本体混入を捕捉しきれなかった）。

## 関連ルール

- `Deliverable Quality Gate`（CLAUDE.md）: 本ルールは Quality Gate の**前段**で機能する（リク到達前に捏造を弾く）。リクは事実検証、本ルールは捏造防止という役割分担
- `Competitive Absence Audit Rule`（CLAUDE.md）: 楽観主張への監査。本ルールは「シンヤさん／クライアントに関する事実」、Competitive Absence は「市場・競合に関する楽観判断」と射程が異なる（補完関係）
- `feedback-agent-skill-gap.md`: 同エージェントが3回違反した場合のスキル化ルートに接続
- `feedback-rina-risk-threshold.md`: ルール5でリナ判定を「リスク5（致命的）」に固定する根拠

**How to apply:**

### ルール1: 事実は「ソース」がない限り書かない（全職種のエージェントに適用）

戦略書・コピー・提案書・LP・スライド・SNS投稿・営業資料など、対外/対内問わず**シンヤさんに関する記述**を組み込む際、以下の3条件のいずれかを満たさない情報は**書かない**：

- (a) シンヤさんが本会話または過去会話で**明示的に発言した内容**（記録があるもの）
- (b) `memory/` `clients/` `knowledge/` 等の正式記録、および**ミオ（researcher）の `reports/` 配下リサーチレポート**に記載済みの事実
- (c) シンヤさん本人に**事前確認して合意を取った内容**

上記3条件外の「自然な推論」「業界一般論からの類推」「ストーリー上望ましい設定」での補完は**禁止**。例：「Web制作4年なら過去にデザイナーと組んだ経験があるはず」「30サイト作ってるなら下積み時代があったはず」等の推論補完は不可。

**判定主体の明示**：「これは推論かどうか」「これは類推かどうか」の判定権限はエージェント自身にない。シンヤさんに関する記述が `(a)(b)(c)` のいずれにも該当しないと判明した時点で、**エージェントの主観的判断によらず**【要事実確認】マーカー付与必須（ルール2参照）。エージェントが「これは一般論／文脈補完」と自己分類して逃げる余地を認めない。

### ルール2: 推測した事実は「推測」と明示する

戦略上どうしても仮説的な属性付けが必要な場合（共感ストーリー設計等）、以下のフォーマットで**仮説と明示**する：

> 【要事実確認】Slide 3 では「コーダーとして30サイト作る中でデザイナーが公開で詰まる場面を見てきた」というスタンスで Empathy を組み立てている。**この経験の有無はシンヤさん未確認のため、戦略採用前に事実確認が必要**。

「【要事実確認】」マーカーの無い記述は「事実として確定済み」とみなされる。マーカー無しでの推測記述は本ルール違反。

### ルール3: 戦略書には「事実根拠の出典セクション」を必須化

シンヤさん／クライアント／市場に関する**検証可能な事実主張を3件以上含む**戦略書・提案書・LP原稿・営業資料には、末尾に以下を必ず付記する：

```
## 本戦略書で引用した事実の出典
- 「Web制作4年・サイト30以上」: memory/project-branding.md L●● / シンヤさん 2026-04-XX 発言
- 「Canva愛好家のデザイナー層」: シンヤさん 2026-04-28 発言（本セッション冒頭）
- 「【要事実確認】デザイナーと組んだ過去案件」: 未確認（戦略採用前にシンヤさん確認必須）
```

**事実主張3件未満の小規模成果物**（短いSNS投稿・短文コピー・メール文面等）の場合、出典セクション省略可。ただし**該当行直後にインライン形式で出典を括弧書き**することで代替する（例：`Web制作4年（出典: memory/project-branding.md）`）。

出典を書けない事実は、書かない or 【要事実確認】マーカー付きで提示する。

### ルール4: アスカの差し戻し義務

アスカは委任先エージェントの成果物を受領後、**シンヤさんに関する記述を全件**チェックし、出典セクションが無い／【要事実確認】マーカーが無い記述があれば、シンヤさんに提示する前に**エージェントへ差し戻す**。

差し戻しの観点：
- シンヤさんの経歴（学歴・職歴・出身・年齢・家族構成など）
- シンヤさんの発言として書かれた内容
- シンヤさんの実績数値・案件履歴
- シンヤさんの心情・動機・価値観として書かれた内容
- クライアント／市場／競合に関する事実

### ルール5: リナの検証強化（手順固定）

リナは戦略書・コピー・提案書の論理検証時、以下の手順を必須とする：

1. シンヤさん／クライアント／市場に関する**固有記述を全件抽出**（リスト化）
2. 各記述について、ルール1の (a)(b)(c) のいずれかのソースが提示されているか または 【要事実確認】マーカーが付いているかを判定
3. いずれも満たさない記述があれば「事実根拠不明」として**リスク5（致命的）判定**で報告
4. 「事前確認が必要」という論点提示のみで終わらせない（本体への混入を見逃すリスクあり）

事実根拠不明は一律リスク5判定で固定するため、`feedback-rina-risk-threshold.md` の無限ループ抑制方針とも両立する（指摘件数の独自判断を入れない）。

### ルール6: 違反検知時の処置

エージェントが本ルールに違反した（事実捏造を行った）場合：

1. アスカは**即座にシンヤさんに報告**（差し戻しと並行）
2. `memory/feedback-agent-fact-fabrication.md`（本ファイル）の末尾に**違反履歴**を以下のフォーマットで追記：
   - 日付 / エージェント / 捏造内容 / 発覚経緯 / 影響範囲（公開前検知 or 公開後検知）/ 是正措置
3. 違反したエージェントの定義ファイル（`agents/<name>.ja.md` / `<name>.md`）に**本ルール参照**を追記（カナタに依頼）
4. 同エージェントの違反が3回目に達した場合は**kaizen案件として恒久対策設計**（指示改善ではなくスキル化／ガード／プロセス変更）

### ルール7: コピー・営業資料への適用範囲

**全職種のエージェント**に適用。特にコピー生成系（コト・タク・ハル・レン）は事実盛りの誘惑が強いため要注意。

ただし対象は「**検証可能な事実主張**」（数値・期間・経歴・実績・第三者の発言）に限定する。

- **本ルール対象**: 「Web制作4年」「30サイト以上」「デザイナー出身」「広島在住」等の検証可能な事実主張
- **本ルール対象外**: 「圧倒的な品質」「業界最速」等の修辞的誇張・感情的訴求（ただし `Competitive Absence Audit Rule` の楽観主張は別ルールで規制）

修辞的誇張までを本ルール対象にするとコピーライティングが機能停止するため、明確に分離する。

「セールス上効果的だから」「ストーリー上必要だから」を理由とした事実盛り・経歴盛り・実績水増しは、修辞的誇張ではなく事実主張の捏造であり、厳禁。
