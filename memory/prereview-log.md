# Rina Pre-Review Gate 履歴

CLAUDE.md「Rina Pre-Review Gate (新規作成時の前段関門, Added 2026-05-02)」に基づく事前レビューの実施履歴を記録する。

## 記録フォーマット

各エントリは以下のフォーマットで記録する：

```
## YYYY-MM-DD: <案件名>

- **対象種別**: 新規スキル / 新規エージェント / 大幅改修（機能追加 / 外部依存追加 / I/F変更）
- **発動条件**: 該当カテゴリ番号（1: 新規スキル / 2: 新規エージェント / 3: 大幅改修）
- **起案者**: アスカ
- **起案文書**: <パス or 案件名>
- **検証観点**:
  - ① 実スキーマ/実仕様との照合: 通過 / 差戻し（理由）
  - ② 既存同種実装への参照パス: 通過 / 差戻し（Grep結果記載の有無）
  - ③ 検知メカニズム: 通過 / 差戻し（理由）
  - ④ セルフチェック項目: 通過 / 差戻し（理由）
- **判定**: 通過 / 条件付き通過 / 差戻し
- **再レビュー回数**: 0 / 1 / 2 / ...
- **備考**: 特記事項・指摘事項のサマリー
```

## 「対象外」判定の記録（保守的判定原則）

アスカが「対象外」と判断した案件は、以下のフォーマットで別途記録する（月1監査用）：

```
## YYYY-MM-DD: <案件名>（対象外判定）

- **判定**: 対象外
- **判断根拠**: <なぜ対象外と判断したか>
- **該当カテゴリ**: 既存ルール改訂 / knowledge-memory追記 / 文言修正 / バグ修正 / 内部リファクタ
- **判断者**: アスカ
- **監査結果**: （月1監査時に記入。判断妥当 / 要修正）
```

## 月1監査記録

毎月第1日曜日の morning-briefing-weekly 時に、アスカが以下を実施し本セクションに記録する：

- 「対象外」判定案件の妥当性確認
- 差戻し率の集計（該当月の事前レビュー総数 / 差戻し件数）
- 自己検証バイアス兆候の確認（リナが甘く通している傾向はないか）

```
## YYYY-MM 月次監査

- 事前レビュー総数: N件
- 差戻し件数: M件（差戻し率 X%）
- 「対象外」判定案件の妥当性: 全件妥当 / N件再判定要
- バイアス兆候: なし / あり（詳細）
- 次月への申し送り: <あれば記載>
```

## 3ヶ月後再評価（2026-08-02 目処）

CLAUDE.md「Rina Pre-Review Gate」に基づき、本ルール運用後3ヶ月時点で以下を評価する：

- セカンドレビュー導入の必要性判断
