# クライアント案件受付ワークフロー

2026-03-30 確立。リナ検証済み（2026-03-30）。

## 適用範囲

- **対象**：外部クライアントから新規相談・初回ヒアリングがあったとき
- **対象外**：officeueda 自社案件・進行中案件への追加相談（本フロー対象外。アスカがその場で判断する）
- CLAUDE.md の「クライアント案件の自動判断」ルール（`clients/` 配下を参照して推測）は**リサーチ依頼向け**。本フローは**案件受付向け**で目的が異なる

## 標準フロー

1. **顧客リスト確認**
   - `notion-crm.py search <名前>` で既存顧客かチェック（会社名・担当者名・メモを横断検索）
   - 見つかった場合 → 「〇〇様の案件として進めます」と一言添えて次へ
   - 見つからない場合 → 情報が揃えば新規顧客として追加、不足なら「追加に必要な情報を確認しますか？」とシンヤさんに確認

2. **Notion 案件リスト登録**
   - 案件名：`<顧客名>_<相談内容の短い名称>`（例: 吉野真由美様_メルマガ運用相談）
   - ステータス：提案中
   - 概要：相談内容を2〜3文で記載
   - 顧客リレーション：CRM と紐づける

3. **clients ディレクトリ確認・作成・更新**
   - `clients/<ディレクトリ名>/` が**存在しない**場合 → 新規作成
   - `clients/<ディレクトリ名>/` が**存在する**場合 → README.md の「支援案件」「支援履歴」に追記
   - 事業が複数になった場合は CLAUDE.md の「クライアントディレクトリの構成ルール」に従い `biz-` サブディレクトリ構成へ移行

### ディレクトリ名の命名規則

既存の `clients/` を参照して一貫性を保つ。迷ったらシンヤさんに確認：
- 会社名略称が明確な場合 → 会社名略称（例: `us-saijo`）
- 担当者名が識別しやすい場合 → 担当者名ローマ字（例: `yoshino-mayumi`）
- 同姓が出た場合 → `<名前>-<会社略称>` で区別

### README.md の最低限の記載項目

- 基本情報（担当者・会社名・ステータス）
- 事業内容（初回は簡潔でOK・随時追記）
- 現在の相談内容（背景・課題・要望）
- 支援案件テーブル
- 支援履歴テーブル
